# @keyframes

A at-rule **@keyframes** permite definir animações CSS em etapas, especificando como os estilos de um elemento devem evoluir ao longo do tempo. Com ela, é possível criar transições suaves e efeitos dinâmicos, sem a necessidade de JavaScript.

---

## 1. Introdução

As animações em CSS transformam a experiência do usuário, adicionando interatividade e dinamismo aos elementos de uma página. A at-rule **@keyframes** é a base para criar esses efeitos, pois permite definir diferentes "estados" da animação em pontos específicos da sua duração. Com ela, você pode controlar propriedades como opacidade, posição, escala, cor e muitas outras, criando efeitos complexos de forma declarativa.

---

## 2. Quando e Por Que Usar @keyframes

- **Criação de Animações Suaves:**
    
    Permite definir transições e efeitos que melhoram a usabilidade e a experiência visual do usuário.
    
- **Interatividade e Engajamento:**
    
    Animações podem guiar a atenção do usuário e tornar a interface mais envolvente.
    
- **Desempenho:**
    
    Animações CSS são geralmente mais eficientes que as implementadas via JavaScript, especialmente quando aceleradas por hardware.
    

---

## 3. Sintaxe e Estrutura

### Sintaxe Básica

A estrutura de uma animação com **@keyframes** envolve a definição de uma regra que especifica os estados da animação ao longo de sua duração. A sintaxe é a seguinte:

```css
@keyframes nome-da-animacao {
  /* Define o estado inicial */
  from {
    /* Propriedades iniciais */
  }
  /* Define o estado final */
  to {
    /* Propriedades finais */
  }
}

```

Além de `from` e `to`, você também pode utilizar porcentagens para definir estados intermediários:

```css
@keyframes slideIn {
  0% {
    transform: translateX(-100%);
    opacity: 0;
  }
  50% {
    transform: translateX(0);
    opacity: 0.5;
  }
  100% {
    transform: translateX(0);
    opacity: 1;
  }
}

```

## 4. Exemplos Práticos e Aplicações

### Exemplo 1: Animação de Fade In

```css
@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

.fade-in {
  animation: fadeIn 2s ease-in-out;
}

```

> Comentário:
> 
> 
> A animação `fadeIn` altera a opacidade de 0 a 1 em 2 segundos, criando um efeito de transição suave. A classe `.fade-in` aplica essa animação a um elemento.
> 

---

### Exemplo 2: Animação de Slide

```css
@keyframes slideIn {
  0% {
    transform: translateX(-100%);
  }
  100% {
    transform: translateX(0);
  }
}

.slide {
  animation: slideIn 1s forwards;
}

```

> Comentário:
> 
> 
> O exemplo `slideIn` move um elemento da esquerda para a posição original. O uso de `forwards` em `animation-fill-mode` garante que o elemento permaneça na posição final após a animação.
> 

---

### Exemplo 3: Animação com Múltiplos Estados

```css
@keyframes pulse {
  0% {
    transform: scale(1);
    opacity: 1;
  }
  50% {
    transform: scale(1.2);
    opacity: 0.7;
  }
  100% {
    transform: scale(1);
    opacity: 1;
  }
}

.pulse {
  animation: pulse 1.5s infinite;
}

```

> Comentário:
> 
> 
> A animação `pulse` cria um efeito de pulsação ao escalar o elemento até 1.2 vezes seu tamanho e, em seguida, retornar ao tamanho original, enquanto ajusta a opacidade. A propriedade `infinite` faz a animação se repetir indefinidamente.
>

## 5. Boas Práticas

- **Defina Estados Claros:**
    
    Utilize `from` e `to` ou porcentagens para definir com clareza os pontos de início, meio e fim da animação.
    
- **Utilize Propriedades Suportadas:**
    
    Verifique se as propriedades animadas são compatíveis com aceleração por hardware para melhorar o desempenho.
    
- **Controle da Duração e Delays:**
    
    Ajuste a duração (`animation-duration`), atraso (`animation-delay`) e o modo de preenchimento (`animation-fill-mode`) para um comportamento consistente.
    
- **Comentários e Organização:**
    
    Documente suas animações para facilitar a manutenção, especialmente em projetos maiores.
