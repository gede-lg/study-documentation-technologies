# @supports

A at-rule **@supports** permite condicionalmente aplicar estilos CSS com base no suporte do navegador a determinadas propriedades ou valores. Também conhecida como feature queries, essa regra possibilita que você escreva código que se adapte às capacidades do ambiente, garantindo uma experiência consistente mesmo em navegadores com diferentes níveis de suporte às tecnologias CSS.

---

## 1. Introdução

Em um cenário onde as especificações CSS evoluem rapidamente, o suporte a novas propriedades nem sempre é garantido em todos os navegadores. A at-rule **@supports** foi criada para resolver esse desafio, permitindo que desenvolvedores apliquem estilos somente se o navegador suportar as funcionalidades desejadas. Isso possibilita a implementação de recursos avançados com segurança, fornecendo fallbacks adequados para ambientes que não os suportam.

**Principais Vantagens:**

- **Compatibilidade:** Permite escrever código que se adapta dinamicamente ao suporte do navegador.
- **Progressive Enhancement:** Possibilita a aplicação de estilos avançados sem comprometer a experiência em navegadores mais antigos.
- **Manutenção:** Facilita a escrita de código condicional, deixando claro quais funcionalidades são opcionais.

---

## 2. Quando e Por Que Usar @supports

- **Aplicar Estilos Condicionalmente:**
Use `@supports` para aplicar estilos somente se o navegador oferecer suporte a determinadas propriedades ou valores.
- **Implementar Novas Funcionalidades:**
Ao utilizar recursos CSS mais recentes, você pode definir fallbacks para navegadores que não os suportam.
- **Teste de Suporte:**
Serve como uma ferramenta para detectar a existência de determinadas funcionalidades antes de utilizá-las, prevenindo comportamentos inesperados.

---

## 3. Sintaxe e Estrutura

### Sintaxe Básica

```css
@supports (condição) {
  /* Regras CSS que serão aplicadas se a condição for verdadeira */
}

```

- **Condição:**
Uma expressão que verifica se uma determinada propriedade ou valor é suportado pelo navegador. Você pode usar a sintaxe CSS para testar propriedades, por exemplo, `(display: grid)` ou condições mais complexas.

### Exemplo Simples

```css
/* Aplica estilos avançados se o navegador suportar Grid Layout */
@supports (display: grid) {
  .layout {
    display: grid;
    grid-template-columns: 1fr 2fr;
    gap: 20px;
  }
}

/* Fallback para navegadores que não suportam Grid Layout */
@supports not (display: grid) {
  .layout {
    display: block;
  }
}

```

> Comentário:
> 
> 
> No exemplo, os estilos dentro do primeiro bloco `@supports` serão aplicados apenas se o navegador suportar `display: grid`. Caso contrário, o bloco `@supports not` define um fallback que utiliza `display: block`.
>

## 4. Exemplos Práticos e Aplicações

### Exemplo 1: Estilos para Propriedades Avançadas

```css
/* Estilos para navegadores que suportam a propriedade "backdrop-filter" */
@supports (backdrop-filter: blur(5px)) {
  .modal {
    backdrop-filter: blur(5px);
    background-color: rgba(255, 255, 255, 0.8);
  }
}

/* Fallback para navegadores sem suporte */
@supports not (backdrop-filter: blur(5px)) {
  .modal {
    background-color: rgba(255, 255, 255, 0.95);
  }
}

```

> Comentário:
> 
> 
> Este exemplo aplica um efeito de desfoque no fundo da modal utilizando `backdrop-filter` se suportado, e utiliza um fundo sem desfoque como fallback.
> 

### Exemplo 2: Condições Compostas

```css
/* Aplica um conjunto de estilos se ambos os recursos forem suportados */
@supports ((display: grid) and (gap: 20px)) {
  .grid-container {
    display: grid;
    gap: 20px;
  }
}

/* Caso contrário, utiliza um layout alternativo */
@supports not ((display: grid) and (gap: 20px)) {
  .grid-container {
    display: flex;
    flex-wrap: wrap;
    margin: -10px;
  }
  .grid-container > div {
    padding: 10px;
  }
}

```

> Comentário:
> 
> 
> Aqui, a condição composta verifica se o navegador suporta tanto `display: grid` quanto a propriedade `gap`. Se ambas as funcionalidades estiverem disponíveis, um layout grid será aplicado; caso contrário, um layout flexível com margens negativas e padding é usado como fallback.
>

## 5. Boas Práticas e Considerações

- **Documentação e Comentários:**
    
    Ao usar `@supports`, comente o propósito da condição para facilitar a manutenção e compreensão do código.
    
- **Uso de Fallbacks:**
    
    Sempre defina um bloco `@supports not` para garantir que os navegadores sem suporte às novas funcionalidades ainda exibam um layout funcional.
    
- **Teste em Diferentes Navegadores:**
    
    Teste suas condições `@supports` em diversos navegadores para verificar que os fallbacks estão funcionando corretamente.
    
- **Combinação com Outras At-Rules:**
    
    `@supports` pode ser combinada com `@media` para aplicar condições ainda mais refinadas, como diferentes estilos baseados em recursos e tamanhos de tela.
