# @media

A at-rule **@media** é uma ferramenta essencial para o desenvolvimento de designs responsivos. Ela permite aplicar estilos CSS com base em condições específicas do ambiente, como largura, altura, orientação e resolução da tela. Com o uso de media queries, é possível criar layouts que se adaptam dinamicamente a diferentes dispositivos e tamanhos de tela.

---

## 1. Introdução

No mundo do desenvolvimento web, os usuários acessam os sites por meio de uma variedade de dispositivos – de smartphones a desktops. A at-rule **@media** possibilita que você escreva estilos que se ajustam a essas diferenças, garantindo uma experiência consistente e otimizada para cada contexto. Ao aplicar condições, você pode alterar tamanhos de fonte, layouts, cores e muito mais conforme o ambiente.

---

## 2. Quando e Por Que Usar @media

- **Design Responsivo:**
    
    Permite que o site se adapte a diferentes tamanhos de tela, melhorando a usabilidade em dispositivos móveis, tablets e desktops.
    
- **Otimização para Impressão:**
    
    Pode ser utilizada para definir estilos específicos para impressões, assegurando que o conteúdo seja apresentado de forma adequada no papel.
    
- **Adaptação a Recursos Específicos:**
    
    Possibilita a criação de layouts que respondem a condições como a orientação da tela (paisagem ou retrato) ou à resolução do dispositivo.
    

---

## 3. Sintaxe e Estrutura

### Sintaxe Básica

A sintaxe da at-rule **@media** permite definir uma ou mais condições para a aplicação de estilos:

```css
@media (condição) {
  /* Regras CSS aqui serão aplicadas somente se a condição for verdadeira */
}

```

- **Condição:** Pode ser uma ou mais expressões envolvendo propriedades como `max-width`, `min-width`, `orientation`, `resolution`, entre outras.

### Exemplo Simples

```css
/* Estilos padrão */
body {
  font-size: 16px;
  background-color: #fff;
}

/* Estilos aplicados quando a largura da tela for menor que 600px */
@media (max-width: 600px) {
  body {
    font-size: 14px;
    background-color: #f0f0f0;
  }
}

```

> Comentário:
> 
> 
> No exemplo, os estilos dentro do bloco `@media` são aplicados somente se a condição `(max-width: 600px)` for verdadeira, ou seja, em dispositivos com largura de tela menor que 600px.
> 

---

## 4. Exemplos Práticos

### Exemplo 1: Layout para Dispositivos Móveis

```css
/* Estilos para telas pequenas */
@media (max-width: 480px) {
  .container {
    padding: 10px;
  }

  nav ul {
    flex-direction: column;
  }
}

```

> Comentário:
> 
> 
> Aqui, quando a tela tem no máximo 480px, o contêiner recebe um padding reduzido e o menu de navegação é reorganizado em coluna para melhor se adaptar ao espaço disponível.
> 

### Exemplo 2: Estilos para Impressão

```css
/* Estilos para impressão */
@media print {
  body {
    font-size: 12pt;
    color: #000;
  }

  .no-print {
    display: none;
  }
}

```

> Comentário:
> 
> 
> Neste caso, as regras definidas dentro de `@media print` garantem que, ao imprimir a página, o conteúdo seja exibido de forma adequada, com tamanho de fonte ajustado e ocultando elementos que não são desejados na versão impressa.
> 

### Exemplo 3: Condições Compostas

```css
/* Estilos para dispositivos com largura entre 768px e 1024px */
@media (min-width: 768px) and (max-width: 1024px) {
  .sidebar {
    display: block;
  }

  .content {
    width: 70%;
  }
}

```

> Comentário:
> 
> 
> Este exemplo mostra como combinar múltiplas condições usando o operador `and`. Aqui, as regras se aplicam somente a dispositivos cuja largura está entre 768px e 1024px.
>

## 5. Boas Práticas

- **Organize suas Media Queries:**
    
    Agrupe e ordene as media queries de forma lógica, seja por ordem crescente ou decrescente de tamanho, para facilitar a manutenção.
    
- **Evite Repetição de Código:**
    
    Sempre que possível, escreva estilos básicos fora dos blocos `@media` e apenas sobreponha os ajustes específicos dentro das media queries.
    
- **Teste em Múltiplos Dispositivos:**
    
    Verifique como os estilos se comportam em diferentes tamanhos e orientações de tela para garantir uma experiência consistente.
    
- **Utilize Ferramentas de Build:**
    
    Em projetos complexos, ferramentas como SASS podem ajudar a modularizar e manter media queries de forma mais organizada.
