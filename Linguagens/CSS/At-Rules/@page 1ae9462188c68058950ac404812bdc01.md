# @page

A at-rule **@page** é utilizada para definir estilos específicos para a impressão de documentos. Com ela, é possível personalizar margens, cabeçalhos, rodapés e outras configurações que influenciam a formatação da página impressa, garantindo que o conteúdo seja exibido de maneira apropriada em papel.

---

## 1. Introdução

No design web, nem sempre os estilos aplicados na tela são os ideais para a versão impressa. A at-rule **@page** permite ajustar esses estilos para otimizar a apresentação em mídia impressa. Ao definir regras específicas para páginas, você pode controlar margens, tamanhos e outras características importantes para a impressão, proporcionando uma experiência de leitura mais adequada e profissional.

---

## 2. Quando e Por Que Usar @page

- **Otimização para Impressão:**
    
    Permite criar versões de páginas que sejam legíveis e bem formatadas quando impressas, ajustando margens, cabeçalhos e rodapés.
    
- **Consistência:**
    
    Garante que os documentos impressos mantenham um layout consistente, independentemente do navegador ou dispositivo de impressão.
    
- **Personalização:**
    
    Possibilita a aplicação de estilos específicos para a mídia impressa, como ocultar elementos interativos (ex.: menus, botões) e ajustar fontes e cores para melhor leitura em papel.
    

---

## 3. Sintaxe e Estrutura

### Sintaxe Básica

A at-rule **@page** é declarada da seguinte forma:

```css
@page {
  /* Regras específicas para a página impressa */
  margin: 1cm;
}

```

Você também pode aplicar regras a pseudo-classes, como `:first` para a primeira página:

```css
@page :first {
  margin-top: 2cm;
}

```

- **margin:** Define as margens da página impressa.
- **Pseudo-classes:** Permitem personalizar a primeira página ou páginas especiais (ex.: `:left`, `:right`) em impressões com múltiplas páginas.

---

## 4. Exemplos Práticos e Aplicações

### Exemplo 1: Definindo Margens Gerais

```css
@page {
  margin: 1cm;
}

```

> Comentário:
> 
> 
> Este exemplo define uma margem de 1cm para todas as páginas impressas, criando um espaçamento uniforme ao redor do conteúdo.
> 

---

### Exemplo 2: Personalizando a Primeira Página

```css
@page :first {
  margin-top: 2cm;
  margin-bottom: 2cm;
}

```

> Comentário:
> 
> 
> Aqui, a primeira página impressa terá margens superiores e inferiores maiores, o que pode ser útil para incluir cabeçalhos ou rodapés diferentes na capa ou página de rosto.
> 

---

### Exemplo 3: Páginas Laterais em Impressões Duplex

```css
@page :left {
  margin-left: 2cm;
  margin-right: 1cm;
}

@page :right {
  margin-left: 1cm;
  margin-right: 2cm;
}

```

> Comentário:
> 
> 
> Este exemplo ajusta as margens de páginas ímpares e pares (usando `:left` e `:right`), o que é útil em impressões duplex (frente e verso) para garantir uma melhor encadernação e apresentação.
> 

---

## 5. Boas Práticas

- **Testar em Diferentes Navegadores:**
    
    A renderização de páginas impressas pode variar entre navegadores. Teste seus estilos de impressão em diferentes ambientes para garantir consistência.
    
- **Separar Estilos de Impressão:**
    
    Considere colocar as regras de impressão em um arquivo CSS separado e importá-lo com uma media query `@media print` para facilitar a manutenção.
    
- **Utilizar Fallbacks:**
    
    Lembre-se de que nem todas as propriedades podem ser suportadas em todos os navegadores. Mantenha estilos básicos para assegurar que o conteúdo seja sempre legível.
    
- **Evitar Elementos Interativos:**
    
    Esconda ou ajuste elementos que não fazem sentido na versão impressa (por exemplo, menus de navegação, botões de interação, links externos) usando `display: none;` dentro de um bloco `@media print` ou `@page`.
