# @property

A at-rule **@property** faz parte da CSS Properties and Values API e permite declarar propriedades CSS customizadas com tipagem, valores iniciais e comportamentos específicos. Essa funcionalidade possibilita que variáveis CSS personalizadas sejam tratadas de forma semelhante às propriedades nativas, permitindo transições e herança controladas.

---

## 1. Introdução

Tradicionalmente, as variáveis CSS (custom properties) não têm um comportamento tipado e não suportam transições de forma nativa. Com o **@property**, você pode definir regras para propriedades customizadas, especificando o tipo de dado, se a propriedade herda ou não e seu valor inicial. Essa abordagem traz mais robustez e previsibilidade ao uso de variáveis, facilitando animações e transições suaves com propriedades personalizadas.

---

## 2. Quando e Por Que Usar @property

- **Tipagem de Propriedades Customizadas:**
    
    Permite definir o tipo de dado esperado (por exemplo, `<color>`, `<length>`, `<number>`) para uma variável CSS, ajudando a prevenir erros e melhorar a consistência.
    
- **Transições Suaves:**
    
    Com a declaração do tipo, é possível aplicar transições (como `transition`) de forma consistente nas propriedades customizadas, proporcionando efeitos animados mais fluidos.
    
- **Controle de Herança e Valores Iniciais:**
    
    Você pode especificar se a propriedade deve herdar valores e definir um valor inicial padrão, o que garante um comportamento previsível mesmo se a variável não for explicitamente definida em algum elemento.
    

---

## 3. Sintaxe e Exemplo

### Sintaxe Básica

```css
@property --nome-da-propriedade {
  syntax: '<tipo>';
  inherits: <true|false>;
  initial-value: <valor>;
}

```

- **-nome-da-propriedade:**
    
    Nome da propriedade customizada, que deve começar com dois traços.
    
- **syntax:**
    
    Especifica o tipo de dado que a propriedade aceitará (por exemplo, `<color>`, `<length>`, `<number>`, etc.).
    
- **inherits:**
    
    Define se a propriedade deve ser herdada pelos elementos filhos (`true` ou `false`).
    
- **initial-value:**
    
    Define o valor padrão que a propriedade terá se não for explicitamente definida.
    

### Exemplo Prático

```css
@property --main-color {
  syntax: '<color>';
  inherits: true;
  initial-value: blue;
}

.box {
  /* Utilizando a propriedade customizada com transição */
  background-color: var(--main-color);
  transition: background-color 0.5s ease;
}

.box:hover {
  /* Alterando o valor da propriedade para disparar a transição */
  --main-color: red;
}

```

> Comentário:
> 
> 
> No exemplo, a propriedade customizada `--main-color` é definida com tipo `<color>`, herdada pelos elementos e com valor inicial `blue`. Ao aplicar uma transição no `background-color`, a mudança para `red` ao passar o mouse sobre o elemento `.box` ocorrerá de forma suave.
>

## 4. Considerações Importantes

- **Status Experimental:**
    
    A at-rule **@property** está em fase experimental em alguns navegadores. É importante verificar a compatibilidade antes de usá-la em produção.
    
- **Definição de Tipos:**
    
    Especificar corretamente o tipo (syntax) é fundamental para que as transições e validações funcionem conforme o esperado.
    
- **Impacto no Desempenho:**
    
    Como a funcionalidade ainda está em evolução, seu impacto no desempenho pode variar. Realize testes e esteja atento às atualizações dos navegadores.
    

---

## 5. Boas Práticas

- **Utilize com Moderação:**
    
    Use **@property** para propriedades customizadas que realmente se beneficiam da tipagem e das transições suaves, evitando abusos que possam complicar a manutenção.
    
- **Documente o Código:**
    
    Comente as declarações de **@property** para explicar o propósito e o tipo de dados esperado, facilitando o trabalho em equipe e a manutenção futura.
    
- **Monitore a Compatibilidade:**
    
    Verifique regularmente as atualizações dos navegadores e as especificações da CSS Properties and Values API, pois melhorias e mudanças podem ocorrer.
