# visibility

## 1. Introdução

A propriedade **visibility** em CSS determina se um elemento é visível ou não, sem removê-lo do fluxo do layout. Diferente de **display: none**, um elemento com **visibility: hidden** continua a ocupar espaço na página, mas seu conteúdo não é exibido. Essa propriedade é útil para alternar a visibilidade de elementos mantendo a estrutura do layout intacta.

### Conceitos Fundamentais

- **Tema Principal (Propriedade visibility):**
    
    Controla se um elemento e seu conteúdo são visíveis ou ocultos, sem alterar o espaço que ele ocupa no layout.
    
- **Valores Comuns:**
    - `visible`: O elemento é renderizado normalmente (valor padrão).
    - `hidden`: O elemento não é exibido, mas ainda ocupa espaço no layout.
    - `collapse`: Utilizado principalmente para linhas de tabela e colunas; remove o espaço ocupado (comportamento similar a display: none em alguns casos) – seu suporte pode variar.

## 3. Conteúdo Detalhado

### 3.1. Sintaxe e Estrutura

A sintaxe básica para definir a propriedade **visibility** é:

```css
.seletor {
    visibility: valor;
}

```

### 3.2. Valores Comuns

- **visible:**
    
    O elemento é renderizado normalmente e é visível ao usuário.
    
    ```css
    .exemplo {
        visibility: visible;
    }
    
    ```
    
- **hidden:**
    
    O elemento não é exibido, mas seu espaço permanece reservado no layout.
    
    ```css
    .exemplo {
        visibility: hidden;
    }
    
    ```
    
- **collapse:**
    
    Aplicável em contextos de tabelas, pode fazer com que linhas e colunas desapareçam sem ocupar espaço. Seu suporte e comportamento podem variar.
    
    ```css
    tr {
        visibility: collapse;
    }
    
    ```
    

### 3.3. Componentes Principais

- **Manutenção do Espaço no Layout:**
    
    Ao utilizar `visibility: hidden`, o elemento continua a ocupar o espaço definido, evitando reflow ou mudanças na estrutura da página.
    
- **Diferença para display:**
    
    Diferente de `display: none`, que remove completamente o elemento do fluxo do documento, **visibility** permite que o elemento seja oculto sem afetar a disposição dos demais elementos.
    

## 4. Exemplos de Código Otimizados

### Exemplo Básico: Ocultar um Elemento sem Remover seu Espaço

```css
.oculto {
    width: 200px;
    height: 100px;
    background-color: #3498db;
    visibility: hidden;
}

```

```html
<div class="oculto">
    Este conteúdo está oculto, mas o espaço permanece reservado.
</div>

```

### Exemplo Comparativo: visibility vs display

```css
.exemplo-visibility {
    width: 200px;
    height: 100px;
    background-color: #e74c3c;
    visibility: hidden;
}

.exemplo-display {
    width: 200px;
    height: 100px;
    background-color: #2ecc71;
    display: none;
}

```

```html
<div class="exemplo-visibility">
    Visibility hidden: O elemento não é visto, mas o espaço é mantido.
</div>
<div class="exemplo-display">
    Display none: O elemento é removido do fluxo e não ocupa espaço.
</div>

```

### Exemplo com Tabela: Uso de collapse

```css
table.tabela-collapse {
    border-collapse: collapse;
}

table.tabela-collapse td {
    border: 1px solid #333;
}

table.tabela-collapse tr:nth-child(2) {
    visibility: collapse; /* Em algumas implementações, essa linha não ocupará espaço */
}

```

```html
<table class="tabela-collapse">
    <tr>
        <td>Item 1</td>
        <td>Item 2</td>
    </tr>
    <tr>
        <td>Item 3</td>
        <td>Item 4</td>
    </tr>
    <tr>
        <td>Item 5</td>
        <td>Item 6</td>
    </tr>
</table>

```

## 5. Informações Adicionais

- **Acessibilidade:**
    
    Elementos com `visibility: hidden` ainda podem ser lidos por leitores de tela dependendo do contexto, portanto, é importante usá-la conforme a intenção de ocultar visualmente sem remover completamente o conteúdo da página.
    
- **Impacto no Layout:**
    
    Como o espaço é mantido, a troca entre `visible` e `hidden` pode ser feita sem causar reflow, o que pode ser útil em animações e transições.
    
- **Uso de collapse:**
    
    O valor `collapse` é mais adequado para linhas ou colunas de tabelas e seu comportamento pode variar entre navegadores.

A propriedade **visibility** é uma ferramenta útil para controlar a exibição de elementos sem remover seu espaço no layout. Ao utilizar `visibility: hidden`, desenvolvedores podem ocultar elementos de forma discreta, mantendo a estrutura da página intacta. A compreensão dessa propriedade e de suas diferenças em relação a outras (como `display: none`) é fundamental para criar interfaces que atendam tanto aos requisitos estéticos quanto às necessidades funcionais e de acessibilidade.
