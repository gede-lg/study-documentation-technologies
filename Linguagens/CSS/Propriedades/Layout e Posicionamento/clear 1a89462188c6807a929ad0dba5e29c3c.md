# clear

## 1. Introdução

A propriedade **clear** é utilizada em CSS para controlar o comportamento de elementos que seguem itens flutuantes (float). Em outras palavras, ela determina se um elemento deve se posicionar abaixo de elementos que utilizam a propriedade **float**. Essa propriedade é especialmente útil para resolver problemas de colapso de contêineres, garantindo que o fluxo do documento não seja afetado por elementos flutuantes.

### Conceitos Fundamentais

- **Tema Principal (Propriedade clear):**
    
    Define como um elemento se comporta em relação aos elementos flutuantes que o precedem.
    
- **Subtemas:**
    - **Valores Comuns:**
    Os principais valores incluem `left`, `right`, `both`, e `none`.
    - **Contexto de Uso:**
    Utilizada para evitar que elementos se posicionem ao lado de elementos flutuantes, forçando-os a "limpar" esses floats e começar em uma nova linha.

## 3. Conteúdo Detalhado

### 3.1. Sintaxe e Estrutura

A sintaxe básica para utilizar a propriedade **clear** é:

```css
.seletor {
    clear: valor;
}

```

**Exemplos de declaração:**

- **Clear à esquerda:**
    
    ```css
    .elemento {
        clear: left;
    }
    
    ```
    
- **Clear à direita:**
    
    ```css
    .elemento {
        clear: right;
    }
    
    ```
    
- **Clear em ambos os lados:**
    
    ```css
    .elemento {
        clear: both;
    }
    
    ```
    
- **Nenhum clear (valor padrão):**
    
    ```css
    .elemento {
        clear: none;
    }
    
    ```
    

### 3.2. Componentes Principais

### Valores e Seus Comportamentos

- **none:**
    
    É o valor padrão. O elemento não irá limpar nenhum float e se posicionará normalmente em relação aos elementos flutuantes.
    
- **left:**
    
    O elemento não ficará ao lado de elementos flutuantes à esquerda. Ele se moverá para a linha seguinte se houver floats do lado esquerdo.
    
- **right:**
    
    O elemento não ficará ao lado de elementos flutuantes à direita. Ele se posicionará abaixo dos floats que estejam à direita.
    
- **both:**
    
    O elemento irá limpar os floats de ambos os lados, garantindo que ele comece em uma nova linha, longe de quaisquer elementos flutuantes à esquerda ou à direita.
    

### Interação com Elementos Flutuantes

- Quando elementos são flutuados utilizando a propriedade **float**, eles podem fazer com que o contêiner não "envolva" corretamente seu conteúdo, resultando em problemas como o colapso do contêiner.
- A propriedade **clear** é frequentemente aplicada a elementos subsequentes (ou a elementos de *clearfix*) para forçar a quebra do fluxo e garantir que os elementos flutuantes não interfiram no posicionamento do conteúdo que os segue.

### Restrições e Melhores Práticas

- **Uso específico:**
A propriedade **clear** deve ser utilizada apenas quando necessário, geralmente em elementos que precisam ser posicionados abaixo de floats.
- **Combinação com técnicas de clearfix:**
É comum usar **clear** em conjunto com outras técnicas (como a técnica de clearfix) para resolver problemas de layout envolvendo floats.
- **Cuidado com o fluxo do documento:**
O uso indevido pode resultar em espaçamentos indesejados. É importante testar o comportamento em diferentes navegadores e dispositivos.

## 4. Exemplos de Código Otimizados

### Exemplo Básico: Limpeza de Floats

Imagine um layout com elementos flutuantes onde o parágrafo seguinte precisa começar abaixo dos elementos flutuantes:

```css
/* Elementos flutuantes */
.img-flutuante {
    float: left;
    margin: 0 15px 15px 0;
    width: 150px;
}

/* Elemento que deve limpar os floats */
.limpar-floats {
    clear: both;
}

```

```html
<img src="exemplo.jpg" alt="Imagem Exemplo" class="img-flutuante">
<p>
    Este parágrafo acompanha o texto ao lado da imagem flutuante.
</p>
<div class="limpar-floats"></div>
<p>
    Este parágrafo iniciará abaixo dos elementos flutuantes, graças à propriedade clear.
</p>

```

### Exemplo Avançado: Clear com Elemento de Rodapé

Utilizando **clear** para garantir que um rodapé seja posicionado corretamente abaixo de elementos flutuantes:

```css
.container {
    overflow: hidden; /* Alternativa para limpar floats */
}

.rodape {
    clear: both; /* Garante que o rodapé não flutue ao lado dos elementos anteriores */
    background-color: #333;
    color: #fff;
    text-align: center;
    padding: 15px;
}

```

```html
<div class="container">
    <div class="conteudo">
        <img src="exemplo.jpg" alt="Imagem Exemplo" style="float: left; margin-right: 15px;">
        <p>Conteúdo com elementos flutuantes que podem afetar o fluxo.</p>
    </div>
    <footer class="rodape">
        Rodapé posicionado corretamente abaixo dos elementos flutuantes.
    </footer>
</div>

```

## 5. Informações Adicionais

- **Técnicas Complementares:**
    
    Além da propriedade **clear**, o uso de técnicas de *clearfix* (como a pseudo-classe `::after`) pode ser muito eficaz para evitar problemas de colapso de contêineres.
    
- **Desafios Comuns:**
    - Quando não se aplica a limpeza, elementos subsequentes podem se sobrepor ou iniciar ao lado dos floats, prejudicando o layout.
    - Pode haver variações de comportamento entre navegadores, exigindo testes abrangentes.
- **Dicas para Uso Eficiente:**
    - Utilize **clear: both** em situações onde há floats em ambos os lados.
    - Combine **clear** com outras propriedades de layout (como `overflow`) para obter um controle mais refinado.

A propriedade **clear** é uma ferramenta essencial para gerenciar o fluxo de elementos em layouts que envolvem floats. Ao permitir que elementos subsequentes "limpem" os efeitos de elementos flutuantes, ela assegura que o design mantenha a estrutura desejada e evita problemas comuns como o colapso de contêineres.
