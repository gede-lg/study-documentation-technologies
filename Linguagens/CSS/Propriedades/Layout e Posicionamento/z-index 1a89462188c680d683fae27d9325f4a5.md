# z-index

## 1. Introdução

A propriedade **z-index** controla a ordem de empilhamento de elementos posicionados em uma página. Em outras palavras, ela define quais elementos aparecem na frente ou atrás de outros quando eles se sobrepõem. Essa propriedade é especialmente relevante em interfaces com layouts complexos, sobreposições, modais e menus, permitindo um gerenciamento preciso da hierarquia visual dos elementos.

### Conceitos Fundamentais

- **Tema Principal (Propriedade z-index):**
    
    Determina a prioridade de exibição de elementos que se sobrepõem, permitindo definir quais elementos ficam "acima" ou "abaixo" dos outros.
    
- **Subtemas:**
    - **Posicionamento (position):**
    O **z-index** só funciona em elementos cujo **position** não seja o valor padrão (*static*). Geralmente, é utilizado em elementos com **relative**, **absolute**, **fixed** ou **sticky**.
    - **Valores Numéricos:**
    Aceita valores inteiros (positivos, negativos ou zero), onde valores mais altos indicam maior prioridade de empilhamento.

## 3. Conteúdo Detalhado

### 3.1. Sintaxe e Estrutura

A sintaxe básica para utilizar a propriedade **z-index** é:

```css
.seletor {
    z-index: valor;
}

```

**Exemplos:**

- **Valor Positivo:**
    
    ```css
    .elemento {
        position: relative;
        z-index: 10;
    }
    
    ```
    
- **Valor Negativo:**
    
    ```css
    .elemento {
        position: absolute;
        z-index: -1;
    }
    
    ```
    
- **Valor Zero:**
    
    ```css
    .elemento {
        position: fixed;
        z-index: 0;
    }
    
    ```
    

### 3.2. Componentes Principais

### Relação com a Propriedade **position**

- **Requisito de Posicionamento:**
Para que o **z-index** funcione, o elemento deve possuir uma propriedade **position** definida como *relative*, *absolute*, *fixed* ou *sticky*. Elementos com **position: static** (valor padrão) não serão afetados pelo **z-index**.

### Contexto de Empilhamento (Stacking Context)

- **Criação de Contextos de Empilhamento:**
    
    Um novo contexto de empilhamento é formado não apenas por elementos com **z-index**, mas também por alguns outros critérios (como elementos com opacidade menor que 1, transformações CSS, etc.). Cada contexto de empilhamento é gerenciado de forma independente, e os valores do **z-index** são comparados apenas entre elementos dentro do mesmo contexto.
    
- **Hierarquia Visual:**
    
    Em um mesmo contexto, elementos com valores de **z-index** mais altos serão exibidos sobre elementos com valores menores. Se dois elementos têm o mesmo **z-index**, a ordem de declaração no HTML pode influenciar a exibição.
    

### Valores e Comportamento

- **Valores Inteiros:**
Podem ser positivos, negativos ou zero. Valores positivos trazem o elemento para a frente, enquanto valores negativos o enviam para trás.
- **Herança e Escopo:**
O **z-index** não é herdado, mas o contexto de empilhamento pode ser influenciado pelos elementos pais que criam um novo contexto.

## 4. Exemplos de Código Otimizados

### Exemplo Básico: Sobreposição Simples

```css
/* Elemento inferior */
.elemento-inferior {
    position: relative;
    z-index: 1;
    background-color: #FFC107;
    padding: 20px;
}

/* Elemento superior */
.elemento-superior {
    position: relative;
    z-index: 5;
    background-color: #03A9F4;
    padding: 20px;
    margin-top: -30px; /* Exemplo para provocar sobreposição */
}

```

```html
<div class="elemento-inferior">
    Elemento Inferior
</div>
<div class="elemento-superior">
    Elemento Superior
</div>

```

### Exemplo Avançado: Contexto de Empilhamento com Elementos Aninhados

```css
/* Contêiner que cria um novo contexto de empilhamento */
.container {
    position: relative;
    z-index: 10;
    background-color: #E0E0E0;
    padding: 20px;
}

/* Elemento dentro do contêiner */
.elemento-dentro {
    position: absolute;
    z-index: 5; /* Relativo ao contexto do contêiner */
    background-color: #8BC34A;
    padding: 20px;
}

/* Elemento fora do contêiner */
.elemento-fora {
    position: relative;
    z-index: 15; /* Pertence a um contexto de empilhamento diferente */
    background-color: #F44336;
    padding: 20px;
}

```

```html
<div class="container">
    <div class="elemento-dentro">
        Elemento Dentro do Container
    </div>
</div>
<div class="elemento-fora">
    Elemento Fora do Container
</div>

```

## 5. Informações Adicionais

- **Contextos de Empilhamento Complexos:**
    
    É comum encontrar problemas de sobreposição inesperada quando múltiplos contextos de empilhamento são criados. Uma boa prática é planejar a hierarquia visual e testar em diferentes navegadores.
    
- **Melhores Práticas:**
    - Utilize **z-index** apenas quando necessário e mantenha os valores o mais simples possível.
    - Evite usar números muito altos ou negativos sem um motivo claro, pois isso pode dificultar a manutenção do código.
    - Lembre-se de que o **z-index** só afeta elementos posicionados (com **position** diferente de *static*).
- **Considerações sobre Acessibilidade:**
    
    A ordem de empilhamento pode impactar a navegação por teclado e leitores de tela, por isso, utilize o **z-index** com cautela para manter a usabilidade e acessibilidade do seu site.
