# touch-action

## 1. Introdução

A propriedade **touch-action** é fundamental para o controle de interações em dispositivos com tela sensível ao toque. Ela permite especificar quais comportamentos de toque (como rolagem, pinça para zoom ou gestos de arrasto) devem ser permitidos ou desabilitados em um elemento. Ao definir **touch-action**, você pode otimizar a experiência do usuário, impedir comportamentos indesejados e melhorar a performance de aplicações web que dependem de gestos touch.

---

## 2. Conceitos Fundamentais

- **Definição:**
    
    **touch-action** instrui o navegador sobre como responder aos gestos de toque realizados pelo usuário em um elemento. Isso inclui determinar se o navegador deve permitir rolagem, zoom ou outras interações padrão ao tocar no elemento.
    
- **Objetivos e Benefícios:**
    - **Controle de Interatividade:** Permite desabilitar comportamentos padrão do navegador (como rolagem e zoom) em elementos específicos, permitindo que os desenvolvedores implementem gestos customizados com JavaScript.
    - **Melhoria da Performance:** Ao desativar comportamentos não desejados, o navegador pode otimizar o processamento dos eventos de toque, evitando atrasos e melhorando a responsividade.
    - **Experiência de Usuário Personalizada:** Garante que os elementos respondam exatamente da forma planejada, sem interferência dos gestos padrão do navegador, especialmente em aplicações web interativas e jogos.
- **Contexto de Uso:**
    
    Essa propriedade é aplicada a elementos interativos que precisam de um comportamento de toque personalizado, como sliders, carrosséis, mapas interativos e outros componentes que dependem de gestos.
    

---

## 3. Sintaxe e Valores

A sintaxe básica para **touch-action** é:

```css
seletor {
  touch-action: valor;
}

```

### Valores Comuns

- **auto:**
    
    Valor padrão. Permite que o navegador determine o comportamento de toque com base na configuração e no dispositivo.
    
    ```css
    element {
      touch-action: auto;
    }
    
    ```
    
- **none:**
    
    Desativa todas as ações padrão de toque, como rolagem, zoom e arrasto. Usado quando o elemento precisa capturar todos os gestos para comportamentos customizados.
    
    ```css
    element {
      touch-action: none;
    }
    
    ```
    
- **pan-x:**
    
    Permite apenas gestos de rolagem horizontal (panning no eixo X). Gestos verticais serão bloqueados.
    
    ```css
    element {
      touch-action: pan-x;
    }
    
    ```
    
- **pan-y:**
    
    Permite apenas gestos de rolagem vertical (panning no eixo Y). Gestos horizontais serão bloqueados.
    
    ```css
    element {
      touch-action: pan-y;
    }
    
    ```
    
- **manipulation:**
    
    Permite ações que não envolvam rolagem ou zoom, como gestos de arrastar. Esse valor é uma forma abreviada que habilita comportamentos de manipulação de toque com foco em interatividade.
    
    ```css
    element {
      touch-action: manipulation;
    }
    
    ```
    
- **Combinações:**
    
    É possível combinar valores para definir comportamentos específicos. Por exemplo:
    
    ```css
    element {
      touch-action: pan-x pinch-zoom;
    }
    
    ```
    
    *Nota:* A combinação de valores pode variar conforme o suporte dos navegadores e a sintaxe da especificação.

## 4. Exemplos Práticos

### Exemplo 1: Desativando a Rolagem e o Zoom para um Elemento Interativo

Imagine um jogo ou um componente interativo que utiliza gestos personalizados. Para evitar que os gestos de toque padrão (como rolagem ou pinça para zoom) interfiram, você pode definir:

```css
.interactive {
  touch-action: none;
}

```

```html
<div class="interactive">
  <!-- Conteúdo interativo que gerencia gestos via JavaScript -->
  Jogue aqui!
</div>

```

*Explicação:*

Ao definir `touch-action: none;`, o elemento não responderá a gestos padrão de toque, permitindo que o JavaScript capture e processe esses eventos para criar interações personalizadas.

---

### Exemplo 2: Permitindo Apenas Rolagem Vertical

Em um contêiner que deve permitir apenas a rolagem vertical (por exemplo, uma lista ou um feed), você pode usar:

```css
.vertical-scroll {
  overflow-y: auto;
  touch-action: pan-y;
}

```

```html
<div class="vertical-scroll">
  <p>Item 1</p>
  <p>Item 2</p>
  <p>Item 3</p>
  <!-- Mais itens -->
</div>

```

*Explicação:*

Com `touch-action: pan-y;`, o contêiner permite a rolagem apenas na direção vertical, bloqueando gestos horizontais, o que pode ser importante para manter a integridade do layout.

---

### Exemplo 3: Habilitando Manipulação para Elementos de Arraste

Se você deseja permitir gestos de manipulação (como arrastar), mas bloquear outras ações de toque, pode utilizar:

```css
.draggable {
  touch-action: manipulation;
}

```

```html
<div class="draggable">
  Arraste-me!
</div>

```

*Explicação:*

O valor `manipulation` habilita apenas ações que envolvem a manipulação direta (como arrastar), enquanto bloqueia gestos que podem interferir, como rolagem ou zoom, permitindo um controle preciso sobre as interações.

## 5. Informações Adicionais

- **Compatibilidade:**
    
    A propriedade **touch-action** é suportada nos navegadores modernos, mas alguns valores e combinações podem variar conforme o dispositivo e a versão do navegador. Sempre teste os comportamentos em diferentes plataformas, especialmente em dispositivos móveis.
    
- **Interação com JavaScript:**
    
    Ao usar **touch-action**, você pode melhorar a precisão dos eventos de toque capturados pelo JavaScript, permitindo a criação de gestos personalizados sem a interferência dos comportamentos padrão do navegador.
    
- **Aplicações em Design Responsivo:**
    
    **touch-action** é particularmente útil em layouts responsivos e aplicativos web onde o comportamento de toque deve ser ajustado para oferecer a melhor experiência em diferentes dispositivos, desde smartphones até desktops com telas sensíveis ao toque.
    
- **Fallback e Gerenciamento:**
    
    Em casos onde **touch-action** não é suportada ou o comportamento padrão não é desejado, considere implementar fallbacks via JavaScript para gerenciar os eventos de toque de forma personalizada.

## 7. Conclusão

A propriedade **touch-action** é essencial para controlar a maneira como os dispositivos sensíveis ao toque interagem com os elementos de uma página. Ao especificar quais gestos devem ser permitidos ou bloqueados, os desenvolvedores podem evitar comportamentos indesejados, como rolagem ou zoom acidental, e criar interfaces mais responsivas e interativas. Seja para desabilitar todos os gestos com `none`, permitir apenas a rolagem vertical com `pan-y`, ou habilitar manipulações específicas com `manipulation`, dominar **touch-action** é crucial para otimizar a experiência do usuário em dispositivos móveis e outros dispositivos touch.
