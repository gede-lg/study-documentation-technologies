# resize

## 1. Introdução

A propriedade **resize** é utilizada para controlar se um elemento, geralmente um `<textarea>` ou outro bloco com `overflow` definido, pode ser redimensionado pelo usuário. Essa propriedade oferece uma maneira de melhorar a experiência do usuário ao permitir ou restringir alterações manuais no tamanho do elemento. Com **resize**, você pode especificar se o elemento pode ser redimensionado horizontalmente, verticalmente, ambos ou se o redimensionamento deve ser desabilitado completamente.

---

## 2. Conceitos Fundamentais

- **Definição:**
    
    A propriedade **resize** controla se e como um elemento pode ser redimensionado pelo usuário através da interação direta com a interface, geralmente utilizando o mouse ou outro dispositivo de entrada.
    
- **Aplicabilidade:**
    
    É comumente aplicada a `<textarea>`, mas pode ser utilizada em outros elementos que possuem `overflow` configurado (como `overflow: auto` ou `overflow: scroll`), permitindo o redimensionamento.
    
- **Usabilidade e Acessibilidade:**
    
    Permitir o redimensionamento pode melhorar a usabilidade, especialmente em formulários ou áreas de edição de texto. Por outro lado, em alguns casos, pode ser desejável restringir o redimensionamento para manter um layout consistente.
    

---

## 3. Sintaxe e Valores

A sintaxe básica é:

```css
seletor {
    resize: valor;
}

```

### Valores Possíveis:

- **none:**
    
    Impede que o elemento seja redimensionado.
    
    ```css
    textarea {
        resize: none;
    }
    
    ```
    
- **both:**
    
    Permite que o elemento seja redimensionado tanto horizontalmente quanto verticalmente.
    
    ```css
    textarea {
        resize: both;
    }
    
    ```
    
- **horizontal:**
    
    Permite apenas o redimensionamento horizontal (largura).
    
    ```css
    textarea {
        resize: horizontal;
    }
    
    ```
    
- **vertical:**
    
    Permite apenas o redimensionamento vertical (altura).
    
    ```css
    textarea {
        resize: vertical;
    }
    
    ```
    
- **auto:**
    
    O comportamento padrão do navegador, que normalmente permite o redimensionamento em ambas as direções para `<textarea>`.
    

---

## 4. Exemplos de Código Otimizados

### Exemplo 1: Desabilitar o Redimensionamento

```css
textarea.no-resize {
    resize: none;
    width: 300px;
    height: 150px;
    border: 1px solid #ccc;
    padding: 10px;
}

```

```html
<textarea class="no-resize" placeholder="Redimensionamento desabilitado"></textarea>

```

*Explicação:*

O `<textarea>` não poderá ser redimensionado pelo usuário, mantendo as dimensões especificadas.

### Exemplo 2: Redimensionamento Vertical Apenas

```css
textarea.vertical-resize {
    resize: vertical;
    width: 100%;
    height: 150px;
    border: 1px solid #2ecc71;
    padding: 10px;
}

```

```html
<textarea class="vertical-resize" placeholder="Redimensionamento apenas vertical"></textarea>

```

*Explicação:*

O `<textarea>` pode ser redimensionado verticalmente para ajustar sua altura, mas a largura permanecerá fixa.

### Exemplo 3: Redimensionamento em Ambas as Direções

```css
textarea.both-resize {
    resize: both;
    width: 300px;
    height: 150px;
    border: 1px solid #e74c3c;
    padding: 10px;
}

```

```html
<textarea class="both-resize" placeholder="Redimensionamento em ambas as direções"></textarea>

```

*Explicação:*

O `<textarea>` pode ser redimensionado tanto na largura quanto na altura, permitindo ao usuário ajustar as dimensões conforme necessário.

## 5. Informações Adicionais

- **Requisitos para Funcionar:**
    
    A propriedade **resize** só tem efeito se o elemento tiver um valor de `overflow` diferente de `visible` (por exemplo, `overflow: auto` ou `overflow: scroll`).
    
- **Aplicação em Outros Elementos:**
    
    Embora seja mais comum em `<textarea>`, você pode aplicar **resize** em outros elementos, como `<div>` ou `<img>`, desde que eles tenham o `overflow` configurado, mas o suporte e o comportamento podem variar.
    
- **Experiência do Usuário:**
    
    Permitir redimensionamento pode melhorar a acessibilidade e a usabilidade, especialmente em áreas de edição de conteúdo. No entanto, em layouts que exigem um controle rígido, pode ser preferível desabilitar essa funcionalidade.
