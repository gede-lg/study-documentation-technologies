## 1. Introdução

As cores em hexadecimal são uma das formas mais antigas e difundidas de especificar cores na web. No CSS 3, temos três variantes principais de notação hexadecimal: com 3, 6 ou 8 dígitos. Esses valores permitem representar, de maneira concisa e precisa, combinações de vermelho, verde e azul (RGB) — e, no caso de 8 dígitos, também o canal alfa (opacidade).

- **Relevância**: quase todos os designers e desenvolvedores front-end usam hexadecimais, pois são suportados desde as primeiras versões do CSS e facilitam a integração com ferramentas de design.
- **Definições e Conceitos Fundamentais**:
    - **Tema principal**: “Notação hexadecimal de cores no CSS3”.
    - **Subtemas**:
        1. Shorthand de 3 dígitos (`#RGB`)
        2. Notação padrão de 6 dígitos (`#RRGGBB`)
        3. Extensão de 8 dígitos com alfa (`#RRGGBBAA`)
        4. **Interpretação dos dígitos hexadecimais e construção de cores**
    - **Para que servem**: especificar cores de forma compacta e, no caso do canal alfa, controlar transparência diretamente na cor.

## 3. Conteúdo Detalhado

### 3.1. Sintaxe e Estrutura

- **3 dígitos**: `#RGB`
    
    Cada dígito é um nibble (4 bits), expandido para dois dígitos:
    
    ```css
    #3a7 → #33aa77
    
    ```
    
- **6 dígitos**: `#RRGGBB`
    
    Dois dígitos hex por canal (00–FF).
    
    ```css
    #ff6600 /* R=FF (255), G=66 (102), B=00 (0) */
    
    ```
    
- **8 dígitos**: `#RRGGBBAA`
    
    Os primeiros seis dígitos são RGB, os dois últimos representam alfa (00 transparente → FF opaco).
    
    ```css
    #0000ff80 /* azul com 50% de opacidade */
    
    ```
    

### 3.2. Componentes Principais

- **Canais de Cor**
    1. **Vermelho (Red)**: primeiro par de dígitos em 6/8 dígitos.
    2. **Verde (Green)**: segundo par.
    3. **Azul (Blue)**: terceiro par.
    4. **Alfa (Alpha)**: quarto par, somente em 8 dígitos.
- **Expansão do Shorthand**
    
    `#RGB` → `#R R G G B B` (navegador duplica cada nibble).
    
- **Interação com Variáveis CSS**
    
    ```css
    :root {
      --primary: #3498db;
      --primary-alpha: #3498db80; /* 50% de opacidade */
    }
    button {
      background: var(--primary);
      border-color: var(--primary-alpha);
    }
    
    ```
    

### 3.3. Restrições de Uso

- **Compatibilidade**:
    - 3 e 6 dígitos são suportados por **todos** os navegadores modernos e antigos.
    - 8 dígitos (incluindo alfa) têm suporte amplo (Chrome ≥ 62, Firefox ≥ 49, Edge ≥ 79, Safari ≥ 12).
- **Case-insensitive**: letras A–F podem ser maiúsculas ou minúsculas.
- **Precauções**: nem todas as ferramentas de design exportam o canal alfa em hex; muitas usam `rgba()` ou `hsla()`.

### 3.4. Interpretação dos Dígitos Hexadecimais e Construção de Cores

- **Sistema Hexadecimal**:
    - Base 16, dígitos de 0 a 9 (valores 0–9) e A a F (valores 10–15).
    - Cada dígito representa um nibble (4 bits); dois dígitos representam 8 bits (0–255).
- **Converter Hex → Decimal**:
    - Ex.: `7B` em hex = 7×16¹ + 11×16⁰ = 112 + 11 = 123.
- **Construir uma cor específica**:
    1. Defina o valor decimal desejado para R, G e B (0–255).
    2. Converta cada valor para um par hexadecimal (00–FF).
        - Se o resultado for um único dígito, prefixe com 0 (ex.: decimal 5 → `05`).
    3. Una as três partes: `#RRGGBB`.
- **Exemplo prático**: criar um tom médio de rosa, RGB(200, 100, 150):
    1. 200 → C8, 100 → 64, 150 → 96
    2. Cor hex = `#C86496`
- **Com canal alfa**: para 60% de opacidade
    - Alfa em decimal = 0.6 × 255 ≈ 153 → 99 em hex
    - Cor hex+alpha = `#C8649699`

## 4. Exemplos de Código Otimizados

```css
/* 1. Shorthand de 3 dígitos */
.badge {
  background-color: #3a7; /* → #33aa77 */
  color: #fff;
}

/* 2. Hex de 6 dígitos com comentários */
header {
  /* Azul Royal: #1e90ff */
  background-color: #1e90ff;
  color: #ffffff;
}

/* 3. Hex de 8 dígitos para transparência */
.overlay {
  /* Preto 70% opaco */
  background-color: #000000b3; /* b3 hex = 179/255 ≈ 70% */
  position: absolute;
  inset: 0;
}

/* 4. Uso avançado com custom properties */
:root {
  --brand: #ff6347;      /* tomato */
  --brand-60: #ff634799; /* 60% de opacidade */
}
.btn {
  background: var(--brand);
  box-shadow: 0 0 0 3px var(--brand-60);
}

```

## 5. Informações Adicionais

- Embora hex seja popular, funções como `rgba()`, `hsl()` e `hsla()` oferecem maior legibilidade quando se trabalha com variações de um mesmo tom.
- Para sistemas de design, muitas equipes escolhem `hsl()` + variáveis CSS pela facilidade de criar escalas tonais programaticamente.
- Ferramentas como Stylelint podem validar e padronizar notações (forçar sempre 6 dígitos, proibir 3 dígitos etc.).
