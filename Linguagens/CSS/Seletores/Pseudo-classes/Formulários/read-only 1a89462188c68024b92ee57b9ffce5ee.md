# :read-only

---

## **1. Introdução**

A pseudo-classe `:read-only` é usada para selecionar elementos que não podem ser editados pelo usuário. Sua relevância está na **estilização de interfaces**, especialmente em formulários, para indicar visualmente campos bloqueados ou informações não editáveis. Ela melhora a experiência do usuário (UX) ao diferenciar conteúdo interativo do estático.

### **Definição e Conceitos Fundamentais**

- **Tema principal**: `:read-only` aplica estilos a elementos não editáveis, sejam eles:
    - Elementos de formulário com o atributo `readonly`.
    - Elementos não editáveis por natureza (ex: `<p>`, `<div>` sem `contenteditable="true"`).
- **Subtemas**:
    - `:read-write`: pseudo-classe oposta, que seleciona elementos editáveis.
    - Diferença entre `:read-only` e `:disabled`.

## **3. Conteúdo Detalhado**

### **Sintaxe e Estrutura**

```css
seletor:read-only {
  propriedade: valor;
}

```

- **Aplica-se a**:
    - Elementos com `readonly` (ex: `<input>`, `<textarea>`).
    - Elementos não editáveis por padrão (ex: `<div>`, `<p>`).
    - Elementos com `contenteditable="false"`.

### **Componentes Principais**

- **Interação com Atributos**:
    - `readonly`: define que um campo de formulário é apenas leitura.
    - `contenteditable`: controla a editabilidade de elementos não-formulário.
- **Pseudo-classes Relacionadas**:
    - `:read-write`: seleciona elementos editáveis.
    - `:disabled`: seleciona elementos desabilitados (não editáveis e excluídos do envio de formulários).

### **Restrições de Uso**

- **Suporte de Navegadores**:
    - Não suportado no Internet Explorer.
    - Suporte completo em navegadores modernos (Chrome, Firefox, Edge, Safari).
- **Não se aplica a elementos `:disabled`**:
    - Use `:disabled` para elementos desabilitados.

## **4. Exemplos de Código**

### **Exemplo 1: Campo de Formulário Read-Only**

```html
<input type="text" value="Não pode ser editado" readonly>

```

```css
input:read-only {
  background-color: #f0f0f0;
  border: 1px solid #ccc;
  cursor: not-allowed;
}

```

### **Exemplo 2: Elemento Contenteditable**

```html
<div contenteditable="false">Este texto não pode ser editado.</div>

```

```css
div:read-only {
  opacity: 0.7;
  border: 1px dashed #999;
}

```

### **Exemplo Avançado: Estilização Dinâmica**

```css
/* Estilo padrão para elementos não editáveis */
:read-only {
  color: #666;
}

/* Exceção para parágrafos em uma área editável */
.editor p:read-only {
  font-style: italic;
}

```

---

## **5. Informações Adicionais**

- **Diferença entre `:read-only` e `[readonly]`**:
    - `[readonly]` seleciona apenas elementos com o atributo HTML.
    - `:read-only` seleciona **todos** os elementos não editáveis, incluindo os naturais.
- **Performance**:
    - Evite seletores complexos como `input[type="text"]:read-only`; prefira classes em casos críticos.
