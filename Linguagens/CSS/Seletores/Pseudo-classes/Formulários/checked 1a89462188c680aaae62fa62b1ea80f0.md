# :checked

---

## 1. Introdução

A pseudo-classe **`:checked`** é uma ferramenta poderosa no CSS que permite selecionar elementos de formulário (como inputs do tipo checkbox ou radio) quando estão marcados. Essa funcionalidade é essencial para criar interações visuais dinâmicas sem a necessidade de JavaScript, facilitando o desenvolvimento de interfaces responsivas e acessíveis.

**Relevância e Importância:**

- **Interatividade sem JavaScript:** Permite alterar estilos com base no estado dos elementos, melhorando a experiência do usuário.
- **Acessibilidade:** Auxilia na indicação visual de estados, contribuindo para interfaces mais intuitivas.
- **Manutenção e Performance:** Reduz a necessidade de scripts adicionais, tornando o código mais limpo e performático.

**Definição e Conceitos Fundamentais:**

- **Pseudo-classe:** Uma extensão do seletor que aplica estilos com base em um estado específico do elemento (no caso, marcado ou não).
- **Elementos Alvo:** Geralmente aplicada a `<input>` de tipo checkbox e radio, mas também pode afetar outros elementos customizados através de técnicas combinadas.
- **Utilidade:** Permite, por exemplo, exibir menus, alternar temas ou revelar conteúdo adicional quando um item está selecionado.

## 3. Conteúdo Detalhado

### Sintaxe e Estrutura

A pseudo-classe `:checked` é utilizada para selecionar elementos que estão em um estado marcado. Sua sintaxe básica é simples:

```css
input:checked {
  /* Estilos aplicados quando o input está marcado */
  background-color: #e0ffe0;
  border: 2px solid #00cc00;
}

```

**Exemplo Explicado:**

- **Seleção do Elemento:** `input:checked` seleciona todos os elementos `<input>` que estejam com o atributo marcado.
- **Aplicação de Estilos:** Quando o elemento estiver marcado, os estilos dentro do bloco serão aplicados.

### Componentes Principais

### Funções e Propriedades Relacionadas

- **Seletores Combinados:** A pseudo-classe pode ser combinada com outros seletores para atingir elementos específicos.
    
    ```css
    /* Exemplo: estilizando uma label associada a um input marcado */
    input:checked + label {
      color: #007700;
      font-weight: bold;
    }
    
    ```
    
- **Estados de Interação:** Útil em casos onde o estado do input altera a aparência de outros componentes da interface, permitindo uma interação visual mais rica.

### Interação com Outros Seletores

- **`:not(:checked)`:** Pode ser utilizada para aplicar estilos quando o elemento não está marcado.
    
    ```css
    input:not(:checked) + label {
      color: #777;
    }
    
    ```
    

### Restrições de Uso

- **Aplicabilidade:** A pseudo-classe `:checked` é específica para elementos que possuem um estado "checado", como checkboxes e radio buttons. Aplicá-la em elementos que não suportam esse estado não produzirá efeito.
- **Compatibilidade:** Embora seja suportada pela maioria dos navegadores modernos, sempre é recomendável testar em ambientes onde a compatibilidade seja uma preocupação crítica.

## 4. Exemplos de Código Otimizados

### Exemplo Básico: Alterando a Aparência de um Checkbox

```html
<!-- HTML -->
<input type="checkbox" id="chk1">
<label for="chk1">Aceito os termos</label>

```

```css
/* CSS */
input[type="checkbox"]:checked + label {
  color: green;
  font-weight: bold;
}

```

*Comentário:* Quando o checkbox estiver marcado, a label associada muda a cor e o peso da fonte para indicar a seleção.

### Exemplo Avançado: Menu de Navegação com Toggle

```html
<!-- HTML -->
<input type="checkbox" id="menu-toggle">
<label for="menu-toggle">Menu</label>
<nav>
  <ul>
    <li><a href="#home">Início</a></li>
    <li><a href="#about">Sobre</a></li>
    <li><a href="#services">Serviços</a></li>
    <li><a href="#contact">Contato</a></li>
  </ul>
</nav>

```

```css
/* CSS */
/* Esconde o menu por padrão */
nav {
  display: none;
}

/* Exibe o menu quando o checkbox está marcado */
input[type="checkbox"]:checked ~ nav {
  display: block;
}

/* Estiliza a label como um botão */
label[for="menu-toggle"] {
  background-color: #007acc;
  color: #fff;
  padding: 10px 20px;
  cursor: pointer;
}

```

*Comentário:* Nesse exemplo, o checkbox age como um botão toggle para exibir ou ocultar o menu de navegação. O seletor `~` permite que o CSS altere o estilo de um elemento irmão, facilitando a criação de interações sem JavaScript.

## 5. Informações Adicionais

- **Combinação com JavaScript:** Embora o uso da pseudo-classe `:checked` permita muitas interações visuais puramente com CSS, sua combinação com JavaScript pode ampliar a funcionalidade para casos mais complexos.
- **Acessibilidade:** É importante sempre associar inputs a labels para melhorar a usabilidade, especialmente para usuários que dependem de leitores de tela.
- **Performance:** O uso de seletores simples, como `:checked`, tende a ter um impacto mínimo na performance do CSS, tornando-o uma escolha ideal para interações dinâmicas.
- **Design Responsivo:** A pseudo-classe pode ser integrada a layouts responsivos para alterar a interface com base no estado dos elementos, promovendo uma melhor experiência em dispositivos móveis.
