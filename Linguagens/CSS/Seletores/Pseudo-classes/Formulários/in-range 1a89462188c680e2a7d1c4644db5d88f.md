# :in-range

A pseudo-classe `:in-range` é uma ferramenta poderosa do CSS que permite estilizar elementos de formulário, como inputs, que possuem valores válidos dentro de um intervalo definido. Esse recurso é especialmente útil para melhorar a experiência do usuário, proporcionando feedback visual imediato conforme o usuário interage com formulários e campos de entrada.

---

## 1. Introdução

### Visão Geral

A pseudo-classe `:in-range` faz parte das pseudo-classes de formulário no CSS e é aplicada a elementos de formulário cujo valor se encontra dentro dos limites especificados por atributos como `min` e `max`. Essa funcionalidade é crucial para a criação de interfaces interativas e responsivas, pois permite destacar visualmente quando um valor digitado atende aos critérios de validação definidos.

### Relevância e Importância

- **Usabilidade:** Melhora a experiência do usuário ao oferecer feedback visual imediato.
- **Validação Visual:** Facilita a identificação de dados corretos e incorretos.
- **Interatividade:** Permite a criação de formulários mais dinâmicos e intuitivos.

### Definições e Conceitos Fundamentais

- **Pseudo-classe:** Um seletor que permite aplicar estilos a um elemento em um estado específico (por exemplo, `:hover`, `:focus`).
- **`:in-range`:** Seleciona elementos cujo valor atual está dentro dos limites permitidos pelos atributos de restrição (como `min` e `max`).
- **Atributos Relacionados:** Geralmente utilizado em conjunto com atributos de validação, como `min`, `max` e `step`.

## 3. Sintaxe e Estrutura

### Sintaxe Básica

A sintaxe para utilizar a pseudo-classe `:in-range` é simples e segue a mesma estrutura dos seletores CSS convencionais. Ela pode ser aplicada diretamente a elementos de formulário que possuam restrições de valor.

```css
input:in-range {
  /* Estilos aplicados quando o valor está dentro do intervalo */
  border-color: green;
  background-color: #eaffea;
}

```

### Estrutura e Funcionamento

- **Elementos Aplicáveis:** A pseudo-classe `:in-range` é geralmente aplicada a `<input>` de tipo `number`, `range`, `date`, entre outros que utilizam atributos `min` e `max`.
- **Avaliação:** Quando o valor do campo estiver entre os limites definidos pelos atributos `min` e `max`, os estilos associados à pseudo-classe serão aplicados.
- **Combinações:** Pode ser combinada com outras pseudo-classes, como `:focus` ou `:valid`, para criar efeitos mais complexos e responsivos.

## 4. Componentes Principais

### Funções e Propriedades

- **Interação com Atributos:** A pseudo-classe depende dos atributos `min` e `max` para determinar se o valor está dentro do intervalo permitido.
- **Validação Visual:** Funciona em conjunto com outras pseudo-classes, como `:out-of-range` (para valores fora do intervalo), oferecendo uma forma visual de feedback para o usuário.

### Métodos e Elementos Relacionados

- **`:in-range`:** Seleciona os elementos cujo valor está dentro dos limites.
- **`:out-of-range`:** Seleciona os elementos cujo valor está fora dos limites especificados.
- **`:valid` e `:invalid`:** Podem ser usados para validação adicional, abrangendo cenários que não se restringem apenas a intervalos.

### Interação entre Componentes

Ao combinar `:in-range` com `:out-of-range`, é possível criar um sistema visual robusto para formularios. Por exemplo, quando um usuário digita um valor dentro do intervalo, o campo pode mudar de cor para indicar que o valor é aceitável. Se o valor for alterado para fora do intervalo, outro estilo pode ser aplicado para alertar o usuário.

### Restrições de Uso

- **Compatibilidade:** Embora suportado pela maioria dos navegadores modernos, é importante verificar a compatibilidade para garantir que a experiência do usuário seja consistente.
- **Atributos Necessários:** A pseudo-classe só funcionará corretamente se os atributos `min` e `max` estiverem presentes e corretamente definidos no elemento.

## 5. Exemplos de Código Otimizados

### Exemplo Básico

```html
<!-- Exemplo de um input numérico com validação de intervalo -->
<input type="number" min="1" max="10" id="numero" placeholder="Digite um número de 1 a 10">

```

```css
/* Estilizando o input quando o valor estiver dentro do intervalo */
input:in-range {
  border: 2px solid green;
  background-color: #eaffea;
}

/* Estilizando o input quando o valor estiver fora do intervalo */
input:out-of-range {
  border: 2px solid red;
  background-color: #ffeaea;
}

```

*Comentário:* Neste exemplo, o campo de entrada muda sua aparência com base na validade do valor digitado.

### Exemplo Avançado com Combinação de Pseudo-classes

```html
<!-- Exemplo de input com foco e validação -->
<input type="number" min="50" max="100" id="idade" placeholder="Digite sua idade (50-100)">

```

```css
/* Estilo quando o input está dentro do intervalo e em foco */
input:in-range:focus {
  border: 3px solid darkgreen;
  background-color: #d0f0d0;
}

/* Estilo padrão para quando está dentro do intervalo */
input:in-range {
  border: 2px solid green;
  background-color: #eaffea;
}

/* Estilo para quando o input está fora do intervalo */
input:out-of-range {
  border: 2px solid red;
  background-color: #ffeaea;
}

```

*Comentário:* Esse exemplo mostra como é possível combinar `:in-range` com `:focus` para fornecer um feedback visual adicional durante a interação.

## 6. Informações Adicionais

- **Melhores Práticas:**
    - Sempre defina os atributos `min` e `max` para inputs que utilizarão `:in-range` e `:out-of-range`.
    - Combine com outras pseudo-classes para criar feedbacks mais ricos e interativos.
    - Teste a compatibilidade com os navegadores que seu público utiliza.
- **Uso Real em Aplicações:**
    - **Formulários de Cadastro:** Indicar visualmente quando o usuário preenche um campo corretamente.
    - **Interfaces Interativas:** Usar em componentes de seleção de data ou faixa numérica para melhorar a experiência do usuário.
- **Considerações de Acessibilidade:**
    - Garanta que as alterações visuais sejam acompanhadas de feedbacks textuais ou auditivos quando necessário, para que usuários com deficiências visuais ou outras limitações possam compreender o feedback.
