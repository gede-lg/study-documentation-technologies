# :enabled

## 1. Introdução

A pseudo-classe `:enabled` é utilizada em CSS para selecionar elementos de formulário que estão habilitados e podem interagir com o usuário. Esses elementos incluem, por exemplo, `<input>`, `<select>`, `<textarea>` e `<button>` quando não possuem o atributo `disabled`. A utilização dessa pseudo-classe é fundamental para a criação de interfaces responsivas e acessíveis, permitindo que desenvolvedores estilizem dinamicamente os elementos com base no seu estado de ativação.

**Relevância e Importância:**

- **Usabilidade:** Melhora a experiência do usuário, destacando visualmente os elementos que podem ser interagidos.
- **Acessibilidade:** Auxilia na criação de interfaces que respondem de forma adequada às interações dos usuários, inclusive em cenários onde a ativação dos elementos pode ser controlada por scripts.
- **Feedback Visual:** Permite alterar a aparência dos elementos habilitados para indicar sua disponibilidade, facilitando a navegação e compreensão do formulário.

**Definição e Conceitos Fundamentais:**

- **Pseudo-classe `:enabled`:** Seleciona elementos que estão habilitados para interação.
- **Elementos de Formulário:** São componentes como `<input>`, `<button>`, `<select>` e `<textarea>`, que podem ter estados habilitado ou desabilitado.
- **Contraparte - `:disabled`:** Seleciona elementos que estão desabilitados, geralmente estilizados de forma a indicar que não podem ser interagidos.

## 3. Conteúdo Detalhado

### Sintaxe e Estrutura

A sintaxe básica para utilizar a pseudo-classe `:enabled` em CSS é simples e direta. Ela é anexada ao seletor de um elemento para determinar que os estilos serão aplicados somente se o elemento estiver habilitado.

```css
/* Seleciona todos os elementos de formulário habilitados */
input:enabled,
select:enabled,
textarea:enabled,
button:enabled {
  background-color: #e0ffe0; /* Exemplo: fundo verde claro */
  cursor: pointer;
}

```

### Componentes Principais

- **Seletores:** A pseudo-classe `:enabled` é geralmente combinada com seletores de elementos específicos de formulário.
- **Aplicação de Estilos:** Permite definir estilos específicos, como cores, bordas ou sombras, que indicam a disponibilidade para interação.
- **Interação com Scripts:** Em conjunto com JavaScript, a alteração do estado dos elementos (habilitando ou desabilitando) pode modificar automaticamente a apresentação da interface.

**Funcionamento:**

- Quando um elemento de formulário não possui o atributo `disabled`, ele é considerado habilitado e, portanto, os estilos definidos com `:enabled` serão aplicados.
- Se o elemento for desabilitado (por exemplo, com `<input disabled>`), os estilos não serão aplicados, possibilitando a distinção visual entre elementos interativos e não interativos.

### Restrições de Uso

- **Compatibilidade:** A pseudo-classe `:enabled` é suportada pela maioria dos navegadores modernos. Contudo, é sempre bom verificar a compatibilidade para versões mais antigas.
- **Aplicabilidade:** Só é aplicável a elementos que podem ser habilitados/desabilitados, ou seja, elementos interativos de formulários.
- **Especificidade:** Deve ser utilizada com cautela em conjunto com outras regras CSS para evitar conflitos na cascata de estilos.

## 4. Exemplos de Código Otimizados

### Exemplo Básico

```html
<!DOCTYPE html>
<html lang="pt-br">
<head>
  <meta charset="UTF-8">
  <title>Exemplo :enabled</title>
  <style>
    /* Estilo para elementos habilitados */
    input:enabled, button:enabled {
      background-color: #dff0d8; /* Fundo verde claro */
      border: 1px solid #3c763d;
      padding: 8px;
      font-size: 14px;
    }
    /* Estilo para elementos desabilitados */
    input:disabled, button:disabled {
      background-color: #f2dede; /* Fundo vermelho claro */
      border: 1px solid #a94442;
      padding: 8px;
      font-size: 14px;
      cursor: not-allowed;
    }
  </style>
</head>
<body>
  <form>
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome" placeholder="Digite seu nome">

    <br><br>

    <label for="email">Email:</label>
    <input type="email" id="email" name="email" placeholder="Digite seu email" disabled>

    <br><br>

    <button type="submit">Enviar</button>
    <button type="button" disabled>Cancelar</button>
  </form>
</body>
</html>

```

*Comentário:*

No exemplo acima, os elementos `<input>` e `<button>` aplicam estilos diferentes com base em seu estado de habilitação. Elementos habilitados recebem um fundo verde, enquanto os desabilitados ficam com um fundo vermelho, proporcionando feedback visual claro.

### Exemplo Avançado com Interação via JavaScript

```html
<!DOCTYPE html>
<html lang="pt-br">
<head>
  <meta charset="UTF-8">
  <title>Exemplo Dinâmico :enabled</title>
  <style>
    input:enabled {
      border-color: #5cb85c;
      transition: border-color 0.3s ease;
    }
    input:enabled:focus {
      border-color: #449d44;
    }
  </style>
</head>
<body>
  <form>
    <label for="usuario">Usuário:</label>
    <input type="text" id="usuario" name="usuario" placeholder="Digite seu usuário" disabled>
    <br><br>
    <button type="button" id="toggleBtn">Ativar/Desativar</button>
  </form>

  <script>
    const inputUsuario = document.getElementById('usuario');
    const toggleBtn = document.getElementById('toggleBtn');

    toggleBtn.addEventListener('click', () => {
      // Toggle para habilitar/desabilitar o input
      inputUsuario.disabled = !inputUsuario.disabled;
    });
  </script>
</body>
</html>

```

*Comentário:*

Neste exemplo, um botão permite alternar o estado do campo de entrada. Quando o campo está habilitado, os estilos definidos pela pseudo-classe `:enabled` são aplicados, demonstrando a interação dinâmica entre CSS e JavaScript.

## 5. Informações Adicionais

- **Diferença entre `:enabled` e `:disabled`:**
    
    Enquanto `:enabled` aplica estilos aos elementos que podem receber interações, `:disabled` serve para identificar aqueles que não podem ser utilizados, geralmente alterando o visual para indicar indisponibilidade.
    
- **Uso Responsivo e Acessibilidade:**
    
    A diferenciação visual através dessas pseudo-classes contribui para uma melhor experiência do usuário, principalmente em dispositivos móveis e para usuários que dependem de leitores de tela.
    
- **Boas Práticas:**
    - Sempre testar os formulários em diferentes navegadores para garantir a consistência dos estilos.
    - Utilizar transições suaves para estados de foco e ativação, melhorando a experiência do usuário.
    - Evitar sobreposições de regras CSS que possam conflitar e dificultar a manutenção do código.
