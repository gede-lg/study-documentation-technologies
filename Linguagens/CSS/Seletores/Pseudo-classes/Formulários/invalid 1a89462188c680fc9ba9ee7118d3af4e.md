# :invalid

A pseudo-classe `:invalid` é uma funcionalidade do CSS que permite selecionar e estilizar elementos de formulário que não cumprem os critérios de validação definidos pelo HTML. Essa ferramenta é essencial para melhorar a experiência do usuário, fornecendo feedback visual imediato sobre dados inseridos incorretamente.

---

## 1. Introdução

### Visão Geral

A pseudo-classe `:invalid` é utilizada para identificar elementos de formulário que possuem valores inválidos, de acordo com atributos de validação como `required`, `pattern`, `min`, `max`, entre outros. Ao aplicar estilos específicos a esses elementos, os desenvolvedores conseguem orientar o usuário para corrigir erros de entrada, melhorando a interação e a acessibilidade da interface.

### Relevância e Importância

- **Feedback Visual:** Proporciona uma forma imediata de indicar problemas na entrada de dados.
- **Acessibilidade:** Auxilia usuários a entenderem quando um valor precisa ser corrigido.
- **Manutenção e Legibilidade:** Facilita a manutenção do código ao separar a lógica de validação da estilização visual.

### Definições e Conceitos Fundamentais

- **Tema Principal:** `:invalid` - uma pseudo-classe CSS que identifica elementos com dados inválidos.
- **Subtemas:**
    - **Validação HTML:** Utilização de atributos como `required`, `pattern`, `min`, `max`, etc., que definem as regras de validação dos campos.
    - **Feedback Visual:** Estratégias para informar o usuário de forma visual sobre a validade dos dados inseridos.
- **Objetivo:** Permitir que os estilos sejam aplicados dinamicamente, conforme a conformidade dos dados inseridos com as regras definidas.

## 3. Conteúdo Detalhado

### 3.1 Sintaxe e Estrutura

A pseudo-classe `:invalid` é aplicada diretamente em seletores CSS. Sua sintaxe básica é:

```css
selector:invalid {
  /* Estilos aplicados a elementos com dados inválidos */
}

```

**Exemplo:**

```css
input:invalid {
  border: 2px solid red;
}

```

Neste exemplo, qualquer `<input>` que não atenda aos critérios de validação terá uma borda vermelha.

### 3.2 Componentes Principais

### Atributos de Validação no HTML

A funcionalidade da pseudo-classe `:invalid` depende dos atributos de validação definidos nos elementos HTML:

- **`required`:** Indica que o campo deve ser preenchido.
- **`pattern`:** Define uma expressão regular que o valor deve seguir.
- **`min` e `max`:** Especificam os limites mínimo e máximo para números ou datas.
- **`type`:** Tipos específicos, como `email`, `url`, etc., possuem validação própria.

### Interação entre CSS e HTML

- **Feedback Imediato:** Ao digitar um valor que não cumpre as regras definidas, o navegador automaticamente aplica a pseudo-classe `:invalid` ao elemento, permitindo a alteração de estilos em tempo real.
- **Complementaridade com `:valid`:** Pode ser combinado com a pseudo-classe `:valid` para fornecer um contraste visual entre dados corretos e incorretos.

### 3.3 Restrições de Uso

- **Dependência dos Atributos HTML:** Se nenhum critério de validação for definido, o estado `:invalid` não será acionado.
- **Comportamento do Navegador:** Alguns navegadores podem aplicar a validação de forma ligeiramente diferente, embora os padrões estejam bem estabelecidos.

## 4. Exemplos de Código Otimizados

### Exemplo Básico

```html
<!-- HTML -->
<form>
  <label for="email">Email:</label>
  <input id="email" type="email" required placeholder="Digite seu email">
  <span class="error">Email inválido</span>
</form>

```

```css
/* CSS */
input:invalid {
  border: 2px solid red;
}

input:valid {
  border: 2px solid green;
}

/* Opcional: esconder mensagem de erro inicialmente */
.error {
  display: none;
}

/* Exibir mensagem de erro quando o campo estiver inválido */
input:invalid + .error {
  display: inline;
}

```

*Comentário:* Neste exemplo, o campo de email apresenta uma borda vermelha se o valor for inválido e uma borda verde se for válido, com uma mensagem de erro aparecendo em caso de dados incorretos.

### Exemplo Avançado

```html
<!-- HTML -->
<form>
  <label for="username">Usuário (mínimo 4 caracteres):</label>
  <input id="username" type="text" pattern=".{4,}" required placeholder="Digite seu usuário">
</form>

```

```css
/* CSS */
input:invalid {
  background-color: #ffe6e6;
  border: 2px dashed red;
  transition: background-color 0.3s ease;
}

input:focus:invalid {
  background-color: #ffcccc;
}

input:valid {
  background-color: #e6ffe6;
  border: 2px solid green;
}

```

*Comentário:* Aqui, o campo de usuário aplica estilos diferenciados ao ser focado e inválido, melhorando a experiência visual do usuário durante a interação com o formulário.

## 5. Informações Adicionais

- **Integração com JavaScript:** Embora o CSS possa fornecer feedback visual, a validação completa dos formulários geralmente é reforçada com JavaScript para uma experiência robusta.
- **Acessibilidade:** É recomendado complementar as mudanças visuais com feedback textual ou sonoro para usuários que dependem de leitores de tela.
- **Compatibilidade:** A pseudo-classe `:invalid` é amplamente suportada nos navegadores modernos, mas sempre é uma boa prática testar a experiência em diferentes ambientes.
- **Combinação com Outras Pseudo-classes:** Pode ser usada em conjunto com `:focus`, `:hover` e `:valid` para criar interações ricas e responsivas.
