# :default

---

# 1. Introdução

A pseudo-classe **`:default`** é uma ferramenta poderosa no CSS que permite selecionar o elemento padrão em um conjunto de elementos de formulário. Em contextos onde vários botões ou campos podem estar presentes, definir um padrão melhora a experiência do usuário e a semântica da interface.

- **Relevância:**
    
    Em interfaces com múltiplas opções (como formulários com botões de envio e reset), identificar e estilizar o elemento padrão pode facilitar a navegação e a interação do usuário. Essa pseudo-classe melhora a acessibilidade e a clareza da interface.
    
- **Definição e Conceitos Fundamentais:**
    - **Tema Principal:** Pseudo-classe `:default`
    Serve para selecionar elementos que são considerados padrão dentro de um grupo, como o botão que deve ser acionado quando o usuário pressiona "Enter".
    - **Subtemas (se aplicáveis):**
    Embora não haja subdivisões explícitas para a pseudo-classe `:default`, é interessante entender sua aplicação em conjunto com outras pseudo-classes de formulário (como `:focus` e `:hover`) para um design mais interativo e acessível.

# 3. Conteúdo Detalhado

## 3.1. Sintaxe e Estrutura

A pseudo-classe `:default` é aplicada diretamente a elementos de formulário, como `<button>` e `<input>`, para identificar o item padrão. Sua sintaxe é simples:

```css
/* Seleciona o elemento padrão */
button:default {
  background-color: #007BFF;
  color: #FFF;
  border: 1px solid #0056b3;
}

```

Neste exemplo, o botão que for considerado o padrão receberá um estilo diferenciado.

## 3.2. Componentes Principais e Interação

- **Elementos Aplicáveis:**
    
    A pseudo-classe `:default` é utilizada principalmente em elementos de formulário:
    
    - `<button>`
    - `<input type="submit">`
    - `<input type="reset">`
    
    Normalmente, o elemento padrão é aquele que, ao pressionar a tecla Enter em um formulário, é ativado automaticamente.
    
- **Função Principal:**
    
    Permitir que os desenvolvedores estilizem especificamente o botão ou campo que atua como a ação padrão, melhorando a usabilidade e a experiência do usuário.
    
- **Interação com Outras Pseudo-classes:**
    
    Pode ser combinada com outras pseudo-classes, como `:focus` e `:hover`, para criar um feedback visual robusto. Por exemplo:
    
    ```css
    button:default:hover {
      background-color: #0056b3;
    }
    
    ```
    
- **Restrição de Uso:**
    
    É importante notar que a pseudo-classe `:default` funciona em contextos de formulários, onde há um elemento designado como padrão pelo navegador ou explicitamente definido pelo desenvolvedor. Em situações onde não há definição de um padrão, a pseudo-classe não terá efeito.

# 4. Exemplos de Código Otimizados

## Exemplo Básico

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <title>Exemplo de :default</title>
  <style>
    /* Estilizando o botão padrão */
    button:default {
      background-color: #28a745; /* Verde para o padrão */
      color: #fff;
      padding: 10px 20px;
      border: none;
      border-radius: 4px;
      cursor: pointer;
    }
    /* Feedback visual ao passar o mouse */
    button:default:hover {
      background-color: #218838;
    }
  </style>
</head>
<body>
  <form>
    <button type="submit">Enviar</button>
    <button type="button">Cancelar</button>
  </form>
</body>
</html>

```

*Comentário:*

Neste exemplo, o botão de envio é estilizado como o padrão, utilizando `:default` para diferenciá-lo.

## Exemplo Avançado

```html
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <title>Exemplo Avançado com :default e Outras Pseudo-classes</title>
  <style>
    /* Estilização base para todos os botões */
    button {
      padding: 10px 20px;
      border: 1px solid #ccc;
      border-radius: 4px;
      cursor: pointer;
      background-color: #f8f9fa;
      transition: background-color 0.3s;
    }
    /* Estilização para o botão padrão */
    button:default {
      background-color: #007bff;
      color: #fff;
      border-color: #007bff;
    }
    /* Feedback ao passar o mouse */
    button:hover {
      background-color: #e2e6ea;
    }
    /* Feedback específico para o botão padrão */
    button:default:hover {
      background-color: #0056b3;
    }
    /* Foco nos botões para acessibilidade */
    button:focus {
      outline: 2px solid #0056b3;
    }
  </style>
</head>
<body>
  <form>
    <button type="submit">Confirmar</button>
    <button type="reset">Redefinir</button>
  </form>
</body>
</html>

```

*Comentário:*

Este exemplo demonstra como a pseudo-classe `:default` pode ser combinada com outras pseudo-classes para criar um design interativo e acessível.

# 5. Informações Adicionais

- **Compatibilidade:**
    
    A pseudo-classe `:default` é suportada pela maioria dos navegadores modernos, mas sempre é recomendável testar a compatibilidade caso se pretenda oferecer suporte a versões mais antigas.
    
- **Acessibilidade:**
    
    Ao definir um elemento como padrão, os desenvolvedores ajudam os usuários que navegam com teclado, pois facilita a ação de envio ou confirmação de um formulário.
    
- **Melhores Práticas:**
    - Use `:default` de forma complementar a outras pseudo-classes para garantir uma experiência visual consistente.
    - Teste a interação com diferentes navegadores e dispositivos.
    - Combine com atributos ARIA quando necessário para melhorar a acessibilidade sem comprometer a semântica.
