# :focus

---

## **1. Introdução**

A pseudo-classe `:focus` é usada para estilizar elementos HTML quando eles recebem foco, geralmente por meio de navegação via teclado (Tab) ou interações com o mouse (clique em campos de formulário). Ela é essencial para melhorar a acessibilidade e a experiência do usuário, indicando visualmente qual elemento está ativo ou pronto para interação.

**Relevância**:

- **Acessibilidade**: Auxilia usuários que dependem de navegação por teclado.
- **UX (Experiência do Usuário)**: Fornece feedback visual imediato.
- **Validação de Formulários**: Destaque campos inválidos ou obrigatórios.

**Conceitos Fundamentais**:

- **Elementos Focáveis**: Inputs, botões, links, elementos com `tabindex`.
- **Estado Temporário**: O estilo é aplicado apenas enquanto o elemento mantém o foco.

## **3. Conteúdo Detalhado**

### **Sintaxe e Estrutura**

```css
seletor:focus {
  propriedade: valor;
}

```

**Exemplo**:

```css
input:focus {
  outline: 2px solid blue;
  background-color: #f0f8ff;
}

```

### **Componentes Principais**

- **Propriedades Comuns**:
    - `outline`: Define uma borda externa sem afetar o layout.
    - `border`: Altera a borda do elemento.
    - `background-color`: Muda a cor de fundo.
    - `box-shadow`: Adiciona efeito de sombra.
- **Interações**:
    - Combine com `transition` para animações suaves.
    - Use `:focus-within` para estilizar o contêiner pai quando um filho tem foco.

### **Restrições**

- **Compatibilidade**: Funciona em todos os navegadores modernos, mas estilos personalizados podem ser inconsistentes em versões antigas.
- **Elementos Não Focáveis**: Não aplicável a `<div>` ou `<span>` sem `tabindex`.
- **Override do Navegador**: Alguns navegadores aplicam estilos padrão (ex.: `outline` azul em inputs), que podem precisar de reset.

## **4. Exemplos de Código**

### **Exemplo Básico**

**Estilizar um Input ao Receber Foco**:

```css
input:focus {
  outline: none; /* Remove a borda padrão */
  border: 2px solid #4CAF50;
  box-shadow: 0 0 5px rgba(76, 175, 80, 0.5);
}

```

### **Exemplo Avançado**

**Botão com Transição e Acessibilidade**:

```css
button {
  padding: 10px 20px;
  transition: all 0.3s ease;
}

button:focus {
  outline: 3px solid #ff5722;
  transform: scale(1.05);
  background-color: #ff5722;
  color: white;
}

```

**Uso em Formulários com `:focus-visible`**:

```css
/* Estilo apenas para foco via teclado */
input:focus-visible {
  outline: 2px solid #2196F3;
}

```

## **5. Informações Adicionais**

### **Boas Práticas**

- **Não Remova o `outline` Completamente**: Substitua por um estilo acessível.
- **Use `:focus-visible`**: Para diferenciar foco via teclado e mouse (evita estilos desnecessários).
- **Teste em Múltiplos Dispositivos**: Garanta consistência em navegadores e dispositivos móveis.

### **Nuances**

- **Ordem de Especificação**: Defina `:focus` após `:hover` na folha de estilo para evitar conflitos.
- **ARIA e `:focus`**: Elementos com `aria-disabled="true"` não devem receber foco.
