# :visited

A pseudo-classe **`:visited`** é utilizada em CSS para selecionar e aplicar estilos a links que já foram visitados pelo usuário. Essa funcionalidade melhora a experiência de navegação, permitindo que os usuários identifiquem quais páginas já exploraram, contribuindo para a usabilidade e a organização visual de sites e aplicações web.

### Relevância e Importância

- **Usabilidade:** Facilita a distinção entre links visitados e não visitados.
- **Acessibilidade:** Melhora a navegação e a orientação do usuário dentro de um site.
- **Estética:** Permite personalizar a aparência dos links de acordo com o design do site.

### Conceitos Fundamentais

- **Tema Principal:** Pseudo-classe `:visited` no CSS.
- **Subtemas:**
    - **Sintaxe e Estrutura:** Como escrever e aplicar a pseudo-classe.
    - **Componentes e Funções:** Métodos de estilização e interação com outros seletores.
    - **Restrições de Uso:** Limitações de segurança e privacidade na manipulação dos estilos.

## 3. Conteúdo Detalhado

### Sintaxe e Estrutura

A pseudo-classe `:visited` é aplicada diretamente aos seletores de link (`<a>`) para diferenciar os estilos dos links que já foram clicados.

**Sintaxe Básica:**

```css
a:visited {
  color: purple;
}

```

Neste exemplo, todos os links visitados serão exibidos com a cor roxa.

### Componentes Principais

- **Seleção de Links:** Aplica-se a elementos `<a>` que possuem um atributo `href`.
- **Propriedades Permitidas:** Devido a preocupações com a privacidade, apenas algumas propriedades CSS podem ser alteradas com `:visited`. Entre elas estão:
    - `color`
    - `background-color`
    - `border-color` (quando todas as bordas são alteradas igualmente)
    - `outline-color`
    - `column-rule-color`
    - Propriedades de SVG que correspondem a cores

### Restrições de Uso

Devido a questões de segurança e privacidade, os navegadores impõem restrições sobre quais propriedades podem ser modificadas com a pseudo-classe `:visited`. Isso impede que sites maliciosos determinem o histórico de navegação dos usuários. Algumas propriedades, como `background-image` ou `box-shadow`, não podem ser alteradas quando aplicadas a links visitados.

## 4. Exemplos de Código Otimizados

### Exemplo Básico

```css
/* Estilo para links não visitados */
a:link {
  color: blue;
  text-decoration: none;
}

/* Estilo para links visitados */
a:visited {
  color: purple;
}

```

*Comentário:* Neste exemplo, links não visitados serão exibidos em azul e, após serem clicados, mudarão para roxo.

### Exemplo Avançado com Hover e Foco

```css
/* Estilo para links não visitados */
a:link {
  color: blue;
  text-decoration: none;
}

/* Estilo para links visitados */
a:visited {
  color: purple;
}

/* Estilo para links ao passar o mouse */
a:hover {
  text-decoration: underline;
}

/* Estilo para links com foco (teclado) */
a:focus {
  outline: 2px solid orange;
}

```

*Comentário:* Este exemplo mostra a combinação de pseudo-classes para melhorar a experiência do usuário, integrando `:hover` e `:focus` com `:visited`.

## 5. Informações Adicionais

- **Privacidade e Segurança:** Os navegadores limitam as propriedades que podem ser modificadas usando `:visited` para evitar ataques de rastreamento do histórico de navegação.
- **Ordem de Aplicação:** É importante definir os estilos na ordem correta, respeitando a regra de cascata. Uma ordem comum é: `:link`, `:visited`, `:hover`, `:active`.
- **Compatibilidade:** Apesar das restrições, o uso de `:visited` é amplamente suportado nos navegadores modernos. É sempre recomendável testar em diferentes navegadores para garantir a consistência visual.
