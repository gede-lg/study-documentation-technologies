# <data>

A seguir, apresento uma explicação detalhada sobre a tag `<data>` do HTML5, abordando desde conceitos básicos até usos avançados, com exemplos práticos e referências para aprofundamento.

---

## 1. Introdução

A tag `<data>` é um elemento semântico do HTML5 que permite associar uma representação **legível por humanos** a um **valor legível por máquina**. Essa dualidade facilita o processamento automatizado dos dados por scripts, motores de busca e outras ferramentas, sem comprometer a apresentação visual para o usuário. Seu uso adequado contribui para a melhoria da acessibilidade, da indexação e da interoperabilidade dos conteúdos web, reforçando a importância de se adotar práticas semânticas na construção de páginas.

## 3. Conteúdo Detalhado

### 3.1. Definição e Conceitos Fundamentais

- **O que é a tag `<data>`?**
    
    A tag `<data>` é utilizada para associar um conteúdo exibido (legível por humanos) a um valor que pode ser interpretado por sistemas (legível por máquina). Isso é especialmente útil quando o mesmo dado precisa ser apresentado de forma amigável, mas também precisa ser extraído ou manipulado programaticamente.
    
- **Conceitos Básicos vs. Avançados:**
    - **Básicos:**
        - Utilização para representar dados com valor duplo: visual e computacional.
        - Melhora a semântica do documento HTML, facilitando a extração de informações.
    - **Avançados:**
        - Integração com APIs e frameworks que realizam extração ou atualização de dados.
        - Combinação com outras técnicas semânticas (como microdados) para enriquecer o conteúdo.
        - Uso dinâmico através de JavaScript para atualizar ou manipular o valor associado conforme mudanças na aplicação.

### 3.2. Sintaxe e Estrutura

A estrutura básica do `<data>` é simples, consistindo de um conteúdo (texto ou outro elemento inline) e de um atributo obrigatório `value` que contém a representação legível por máquina:

```html
<data value="valor_machine_readable">Conteúdo legível</data>

```

- **Exemplo Simples:**
    
    ```html
    <p>Preço do produto: <data value="19.99">R$19,99</data></p>
    
    ```
    
    Neste exemplo, o usuário vê "R$19,99", enquanto scripts podem utilizar o valor "19.99" para cálculos ou validações.
    

### 3.3. Componentes Principais

- **Elemento `<data>`:**
    
    O próprio elemento que encapsula os dados.
    
- **Atributo `value`:**
    
    É o atributo central do `<data>`, que armazena o valor de máquina associado ao conteúdo apresentado. Ele deve ser definido sempre que o `<data>` for utilizado.
    
- **Conteúdo Interno:**
    
    O conteúdo entre a abertura e o fechamento da tag é destinado à visualização pelo usuário.
    

### 3.4. Propriedades/Atributos Específicos

- **Atributo Obrigatório:**
    - `value`:
        - **Descrição:** Define o valor legível por máquina que está associado ao conteúdo apresentado.
        - **Possíveis valores:**
        Pode receber qualquer string que represente o dado de forma consistente com o que se deseja manipular. Por exemplo, números, datas, identificadores, etc.
- **Atributos Globais:**
    
    Como qualquer elemento HTML, `<data>` pode receber atributos globais (como `id`, `class`, `title`, etc.), mas não possui outros atributos específicos além do `value`.
    

### 3.5. Uso Avançado

- **Integração com JavaScript:**
    
    É possível utilizar o `<data>` para armazenar informações que podem ser lidas e atualizadas dinamicamente via JavaScript. Isso pode ser útil em aplicações que precisam sincronizar dados visíveis com valores utilizados em cálculos ou validações.
    
- **Enriquecimento Semântico:**
    
    Quando combinado com outros padrões semânticos (como microdados, RDFa ou JSON-LD), o `<data>` pode ajudar motores de busca e outras ferramentas a compreender melhor a estrutura dos dados apresentados.
    
- **Casos de Uso Reais:**
    - **E-commerce:** Exibir preços ou identificadores de produtos de forma amigável ao usuário, mas com valores precisos para processamento.
    - **Aplicações Interativas:** Representar dados que podem ser atualizados em tempo real (por exemplo, estatísticas, pontuações, datas) onde o valor precisa ser facilmente extraído e manipulado.
    - **Acessibilidade e SEO:** Ao associar dados estruturados aos elementos visuais, melhoram-se as informações semânticas disponíveis para leitores de tela e motores de busca.

## 4. Exemplos Práticos

### 4.1. Exemplo Básico

```html
<!DOCTYPE html>
<html lang="pt">
<head>
  <meta charset="UTF-8">
  <title>Exemplo Básico de &lt;data&gt;</title>
</head>
<body>
  <!-- Exibe o preço de um produto com valor machine-readable -->
  <p>Preço do produto: <data value="19.99">R$19,99</data></p>
</body>
</html>

```

*Comentário:*

Neste exemplo, o usuário vê "R$19,99", mas o valor "19.99" pode ser utilizado por scripts para operações matemáticas, conversões ou validações.

### 4.2. Exemplo Avançado com JavaScript

```html
<!DOCTYPE html>
<html lang="pt">
<head>
  <meta charset="UTF-8">
  <title>Exemplo Avançado de &lt;data&gt;</title>
  <script>
    // Função que obtém e exibe o valor machine-readable do elemento <data>
    document.addEventListener("DOMContentLoaded", function(){
      const dataElement = document.querySelector('data');
      const machineValue = dataElement.getAttribute('value');
      console.log("Valor machine-readable:", machineValue);

      // Exemplo de atualização dinâmica (simulando uma mudança de valor)
      const novoValor = "24.99";
      dataElement.setAttribute('value', novoValor);
      dataElement.textContent = "R$24,99";
      console.log("Novo valor atualizado:", dataElement.getAttribute('value'));
    });
  </script>
</head>
<body>
  <!-- Exibe a data de lançamento com valor machine-readable -->
  <p>Data de lançamento: <data value="2025-02-07">07 de Fevereiro de 2025</data></p>
</body>
</html>

```

*Comentário:*

Este exemplo demonstra como o valor do `<data>` pode ser lido e atualizado via JavaScript, possibilitando que a interface responda a mudanças dinâmicas sem comprometer a semântica dos dados.

## 5. Informações Adicionais

- **Semântica vs. Apresentação:**
    
    A tag `<data>` não altera o estilo visual dos dados. Ela é puramente semântica, o que significa que os navegadores não aplicam um estilo especial por padrão. A estilização pode ser feita com CSS, se necessário.
    
- **Diferenças em Relação a Outras Tags:**
    - **`<time>`:** Utilizada para representar datas e horas, mas possui atributos e comportamentos específicos para formatação temporal (por exemplo, o atributo `datetime`).
    - **`<span>`:** Um elemento genérico para agrupamento, que não fornece significado semântico ao conteúdo. O `<data>`, por outro lado, carrega uma semântica específica de dados legíveis por máquina.
- **Adoção e Suporte:**
    
    Embora o `<data>` seja parte do padrão HTML5, seu uso ainda é menos difundido em comparação com outras tags semânticas. No entanto, seu emprego pode agregar valor semântico e funcionalidade para aplicações que se beneficiam da separação entre apresentação e valor processável.
    
- **Boas Práticas:**
    - Sempre defina o atributo `value` quando utilizar `<data>`.
    - Utilize-o apenas quando houver uma necessidade real de associar um valor legível por máquina ao conteúdo visual.
    - Considere a integração com frameworks ou bibliotecas que possam extrair esses dados para melhorar a interatividade e a usabilidade da aplicação.

## 7. Considerações Finais

A utilização da tag `<data>` é uma prática recomendada quando há a necessidade de manter uma conexão entre a forma como os dados são apresentados ao usuário e a forma como eles serão processados por sistemas. Ao adotar essa tag, você torna seu código mais semântico, acessível e preparado para interações dinâmicas, contribuindo para a manutenção e evolução de aplicações web modernas.
