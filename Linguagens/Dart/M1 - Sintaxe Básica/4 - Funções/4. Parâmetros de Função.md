
## Parâmetros Posicionais Obrigatórios

### O que é e para que serve?
Parâmetros posicionais obrigatórios são aqueles que devem ser fornecidos na ordem correta quando a função é chamada. Eles são essenciais para a execução da função e não podem ser omitidos.

### Sintaxe de uso
```dart
void minhaFuncao(int a, String b) {
  print('a: $a, b: $b');
}

minhaFuncao(5, 'hello'); // Chamada válida
// minhaFuncao(5); // Erro: argumentos insuficientes
```

### Restrições de uso
- Devem ser fornecidos na ordem exata definida na assinatura da função.
- Não podem ser omitidos.

### Quando utilizar?
Use parâmetros posicionais obrigatórios quando a função não pode operar corretamente sem esses valores e a ordem dos argumentos é intuitivamente clara.

---

## Parâmetros Posicionais Opcionais

### O que é e para que serve?
Parâmetros posicionais opcionais são aqueles que podem ser omitidos na chamada da função. Eles são definidos entre colchetes na assinatura da função.

### Sintaxe de uso
```dart
void minhaFuncao(int a, [String? b]) {
  print('a: $a, b: $b');
}

minhaFuncao(5, 'hello'); // a: 5, b: hello
minhaFuncao(5);          // a: 5, b: null
```

### Restrições de uso
- Podem ser omitidos na chamada da função.
- Devem ser colocados após os parâmetros posicionais obrigatórios.
- Podem ter um valor `null` se não fornecidos e se o tipo permitir.

### Quando utilizar?
Use parâmetros posicionais opcionais quando você quer fornecer flexibilidade na chamada da função, permitindo que alguns argumentos sejam opcionais.

---

## Parâmetros Nomeados Opcionais

### O que é e para que serve?
Parâmetros nomeados opcionais são aqueles que podem ser fornecidos em qualquer ordem e são identificados por seus nomes na chamada da função. Eles são definidos entre chaves na assinatura da função.

### Sintaxe de uso
```dart
void minhaFuncao({int? a, String? b}) {
  print('a: $a, b: $b');
}

minhaFuncao(a: 5, b: 'hello'); // a: 5, b: hello
minhaFuncao(b: 'hello');       // a: null, b: hello
minhaFuncao();                 // a: null, b: null
```

### Restrições de uso
- Podem ser omitidos na chamada da função.
- Podem ser fornecidos em qualquer ordem.
- Podem ter um valor `null` se não fornecidos e se o tipo permitir.

### Quando utilizar?
Use parâmetros nomeados opcionais quando a função tem muitos parâmetros e quer garantir a clareza, permitindo que os chamadores especifiquem apenas os argumentos relevantes.

---

## Parâmetros Nomeados Obrigatórios

### O que é e para que serve?
Parâmetros nomeados obrigatórios são aqueles que devem ser fornecidos, mas podem ser especificados em qualquer ordem. Eles são marcados com `required` na assinatura da função.

### Sintaxe de uso
```dart
void minhaFuncao({required int a, required String b}) {
  print('a: $a, b: $b');
}

minhaFuncao(a: 5, b: 'hello'); // Chamada válida
// minhaFuncao(a: 5); // Erro: argumento b ausente
```

### Restrições de uso
- Devem ser fornecidos na chamada da função.
- Podem ser fornecidos em qualquer ordem.
- Devem ser marcados com `required`.

### Quando utilizar?
Use parâmetros nomeados obrigatórios quando você quer a clareza dos parâmetros nomeados mas precisa garantir que certos argumentos sempre sejam fornecidos.

---

## Parâmetros com Valor Padrão

### O que é e para que serve?
Parâmetros com valor padrão são aqueles que têm um valor definido na assinatura da função, que será usado se o parâmetro não for fornecido na chamada.

### Sintaxe de uso
```dart
void minhaFuncao([int a = 0, String b = 'default']) {
  print('a: $a, b: $b');
}

minhaFuncao(5, 'hello'); // a: 5, b: hello
minhaFuncao(5);          // a: 5, b: default
minhaFuncao();           // a: 0, b: default
```

### Restrições de uso
- Valores padrão devem ser constantes.
- Parâmetros com valor padrão são geralmente opcionais e podem ser posicionais ou nomeados.

### Quando utilizar?
Use parâmetros com valor padrão para fornecer valores sensatos quando um argumento não é fornecido, simplificando a chamada da função e evitando `null`.

## Exemplo Completo

```dart
void exemploCompleto(int posObrigatorio, 
                     [String? posOpcional], 
                     {required double nomeObrigatorio, String nomeOpcional = 'padrão'}) {
  print('posObrigatorio: $posObrigatorio');
  print('posOpcional: $posOpcional');
  print('nomeObrigatorio: $nomeObrigatorio');
  print('nomeOpcional: $nomeOpcional');
}

exemploCompleto(10, 'opcional', nomeObrigatorio: 5.5, nomeOpcional: 'especificado');
// posObrigatorio: 10
// posOpcional: opcional
// nomeObrigatorio: 5.5
// nomeOpcional: especificado

exemploCompleto(10, nomeObrigatorio: 5.5);
// posObrigatorio: 10
// posOpcional: null
// nomeObrigatorio: 5.5
// nomeOpcional: padrão
```

Este exemplo mostra uma função utilizando todos os tipos de parâmetros, ilustrando como cada um funciona e como eles podem ser combinados.

---

## Considerações Finais

- **Escolha com base na clareza e na necessidade:** A escolha do tipo de parâmetro deve facilitar a leitura e o uso da função.
- **Valores padrão evitam `null`:** Usar valores padrão pode ajudar a evitar problemas com valores `null` e tornar o código mais robusto.
- **Parâmetros nomeados aumentam a clareza:** Em funções com muitos parâmetros, usar parâmetros nomeados pode tornar as chamadas mais claras e menos propensas a erros.

Ao compreender e utilizar esses diferentes tipos de parâmetros, você pode escrever funções mais flexíveis, claras e robustas no Dart.
