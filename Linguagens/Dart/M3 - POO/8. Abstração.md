## O que é e para que serve?

#### Classes Abstratas
Uma **classe abstrata** é uma classe que não pode ser instanciada diretamente, ou seja, você não pode criar um objeto dessa classe usando o operador `new`. Em vez disso, as classes abstratas são destinadas a serem subclasses (ou seja, estendidas) por outras classes. Elas podem conter uma mistura de implementações concretas (com código funcional) e abstrações (sem implementação). O propósito de uma classe abstrata é fornecer uma definição base de uma classe que outras classes podem herdar, forçando as classes filhas a implementar métodos específicos.

#### Métodos Abstratos
Um **método abstrato** é um método que é declarado em uma classe abstrata, mas não tem implementação naquela classe. O método deve ser implementado em todas as subclasses concretas da classe abstrata. Isso é útil quando a classe base precisa definir um padrão ou um contrato para as subclasses, mas a implementação real do método depende de cada subclasse específica.

---
## Sintaxe de uso

#### Declaração de uma Classe Abstrata
Para declarar uma classe abstrata em Dart, você usa a palavra-chave `abstract` antes da palavra-chave `class`. Por exemplo:

```dart
abstract class Veiculo {
  void mover();
}
```

Neste exemplo, `Veiculo` é uma classe abstrata que contém um método abstrato `mover()`.

#### Declaração de Métodos Abstratos
Dentro de uma classe abstrata, você pode definir métodos abstratos simplesmente omitindo suas implementações (corpo do método), como mostrado acima com o método `mover()`. Note que métodos abstratos só podem existir dentro de classes abstratas.

#### Implementação em Subclasses
Para usar uma classe abstrata, você deve criar uma subclasse que estenda a classe abstrata e implemente todos os seus métodos abstratos. Por exemplo:

```dart
class Carro extends Veiculo {
  @override
  void mover() {
    print('O carro está se movendo');
  }
}
```

Neste exemplo, `Carro` é uma subclasse concreta de `Veiculo` e fornece uma implementação específica para o método abstrato `mover()`.

### Exemplo Completo

```dart
abstract class Animal {
  void comunicar(); // Método abstrato
}

class Cachorro extends Animal {
  @override
  void comunicar() {
    print('O cachorro late');
  }
}

class Gato extends Animal {
  @override
  void comunicar() {
    print('O gato mia');
  }
}

void main() {
  Cachorro cachorro = Cachorro();
  Gato gato = Gato();

  cachorro.comunicar(); // Saída: O cachorro late
  gato.comunicar(); // Saída: O gato mia
}
```

---
### Considerações Importantes

- **Flexibilidade e Contrato**: As classes e métodos abstratos fornecem um contrato que todas as subclasses devem seguir, garantindo uma certa consistência enquanto ainda permitem flexibilidade nas implementações específicas.
- **Polimorfismo**: Classes abstratas são uma forma de alcançar polimorfismo em Dart, onde uma referência de classe base pode apontar para um objeto de qualquer subclasse, permitindo a execução de diferentes implementações de métodos através da mesma interface.
- **Organização do Código**: Usar classes abstratas pode ajudar a organizar o código e reduzir a duplicação, definindo uma estrutura comum para um conjunto de subclasses.
