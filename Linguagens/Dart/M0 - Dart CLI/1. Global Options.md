# Guia Completo das Opções Globais do Dart CLI

O Dart é uma linguagem de programação otimizada para a construção de aplicativos móveis, web e de servidor. O Dart CLI (Command-Line Interface) é a interface de linha de comando que permite aos desenvolvedores interagir com o SDK do Dart, fornecendo comandos para criar, compilar, analisar e executar aplicativos Dart.

Neste guia, exploraremos detalhadamente as **Opções Globais** do Dart CLI, explicando o que são, para que servem, quando e como usá-las, além de fornecer exemplos e esclarecer possíveis restrições.

## Introdução às Opções Globais

As **Opções Globais** são parâmetros que podem ser aplicados a qualquer comando do Dart CLI. Elas modificam o comportamento geral da ferramenta ou fornecem informações adicionais. Essas opções são colocadas **antes** do comando específico que você deseja executar.

**Sintaxe Geral:**

```bash
dart [global options] <command> [arguments]
```

**Exemplo:**

```bash
dart --verbose run main.dart
```

Neste exemplo, a opção global `--verbose` é aplicada ao comando `run`.

---

## Opção `--verbose` (`-v`)

### O que é e para que serve?

A opção `--verbose` (ou `-v`) aumenta a verbosidade da saída do comando, exibindo informações adicionais que podem ser úteis para depuração ou entendimento detalhado do que o comando está fazendo.

### Quando utilizar?

- **Depuração:** Quando você deseja diagnosticar problemas ou entender em detalhes o que está acontecendo durante a execução de um comando.
- **Informações Detalhadas:** Ao precisar de informações mais aprofundadas sobre processos internos.

### Sintaxe de uso

**Forma longa:**

```bash
dart --verbose <command> [arguments]
```

**Forma curta:**

```bash
dart -v <command> [arguments]
```

**Exemplo:**

```bash
dart --verbose run main.dart
```

### Restrições de uso

- Pode aumentar significativamente a quantidade de informações na saída, o que pode ser confuso se não for filtrado adequadamente.
- Deve ser usado com comandos que suportam saída detalhada.

## Opção `--version`

### O que é e para que serve?

A opção `--version` exibe a versão atual do SDK do Dart instalado no seu sistema.

### Quando utilizar?

- **Verificação de Versão:** Ao precisar confirmar qual versão do Dart está instalada.
- **Compatibilidade:** Antes de iniciar um projeto ou depurar problemas relacionados à versão.

### Sintaxe de uso

```bash
dart --version
```

**Exemplo:**

```bash
C:\Users\lgust>dart --version
Dart SDK version: 2.18.0 (stable) on "windows_x64"
```

### Restrições de uso

- Não deve ser combinada com outros comandos; ela ignora qualquer comando fornecido após ela.
- Se usada com outros parâmetros, apenas a versão será exibida.

## Opção `--enable-analytics`

### O que é e para que serve?

Ativa o envio de dados de telemetria (analytics) para o Google, ajudando a melhorar o Dart e ferramentas relacionadas.

### Quando utilizar?

- **Opt-in de Telemetria:** Quando você deseja contribuir com dados de uso para o time de desenvolvimento do Dart.
- **Após desabilitar a telemetria e querer reabilitá-la.**

### Sintaxe de uso

```bash
dart --enable-analytics <command> [arguments]
```

**Exemplo:**

```bash
dart --enable-analytics run main.dart
```

### Restrições de uso

- **Privacidade:** Ativa o envio de dados de uso e diagnósticos, conforme a [Política de Privacidade do Google](https://policies.google.com/privacy).
- Deve ser usado conscientemente, entendendo que informações serão enviadas para o Google.

---

## Opção `--disable-analytics`

### O que é e para que serve?

Desativa o envio de dados de telemetria (analytics) para o Google, respeitando sua preferência de privacidade.

### Quando utilizar?

- **Opt-out de Telemetria:** Quando você não deseja que seus dados de uso sejam coletados.
- **Configuração de Ambiente:** Em ambientes onde a coleta de dados é restrita ou indesejada.

### Sintaxe de uso

```bash
dart --disable-analytics
```

**Exemplo:**

```bash
dart --disable-analytics
```

Após executar, uma mensagem confirmando a desativação será exibida.

### Restrições de uso

- Não combina com outros comandos; a ação principal é desativar a telemetria.
- Deve ser executada isoladamente para efetivar a mudança na configuração.

---

## Opção `--suppress-analytics`

### O que é e para que serve?

Impede o envio de dados de telemetria **apenas** para a execução atual, sem alterar a configuração global de analytics.

### Quando utilizar?

- **Execução Pontual:** Quando você deseja que uma execução específica não envie dados, mas mantenha a telemetria ativa para outras execuções.
- **Scripts Automatizados:** Em scripts ou pipelines onde a coleta de dados não é desejada para aquela execução.

### Sintaxe de uso

```bash
dart --suppress-analytics <command> [arguments]
```

**Exemplo:**

```bash
dart --suppress-analytics run main.dart
```

### Restrições de uso

- **Temporário:** Não altera a configuração global; apenas suprime a telemetria para aquela execução.
- Deve ser especificado a cada execução em que não se deseja enviar dados.

---

## Opção `--help` (`-h`)

### O que é e para que serve?

Exibe informações de ajuda sobre o Dart CLI, incluindo os comandos disponíveis e suas descrições.

### Quando utilizar?

- **Ajuda Rápida:** Quando precisar de um lembrete dos comandos disponíveis ou das opções globais.
- **Aprendizado:** Ao aprender sobre novos comandos ou explorar funcionalidades.

### Sintaxe de uso

**Forma longa:**

```bash
dart --help
```

**Forma curta:**

```bash
dart -h
```

**Exemplo:**

```bash
dart --help
```

### Restrições de uso

- Quando usada com um comando específico, exibe a ajuda relacionada àquele comando.

**Exemplo:**

```bash
dart run --help
```

Neste caso, exibirá a ajuda específica para o comando `run`.

## Informações Adicionais

### Combinação de Opções

As opções globais podem ser combinadas para ajustar o comportamento do Dart CLI conforme necessário.

**Exemplo:**

```bash
dart --verbose --suppress-analytics run main.dart
```

Neste exemplo, estamos aumentando a verbosidade e suprimindo a telemetria para a execução atual do comando `run`.

### Ordem das Opções e Comandos

As opções globais devem ser especificadas **antes** do comando.

**Correto:**

```bash
dart --verbose run main.dart
```

**Incorreto:**

```bash
dart run --verbose main.dart
```

No exemplo incorreto, `--verbose` seria interpretado como um argumento para o comando `run`, não como uma opção global.

### Telemetria e Privacidade

- **Telemetria Ativada por Padrão:** Por padrão, o Dart CLI coleta dados de uso para ajudar a melhorar a ferramenta.
- **Opt-out Disponível:** Você tem o controle total para desabilitar essa coleta a qualquer momento usando `--disable-analytics`.
- **Política de Privacidade:** Todos os dados são coletados de acordo com a [Política de Privacidade do Google](https://policies.google.com/privacy).

### Ambientes Automatizados

Em ambientes CI/CD ou scripts automatizados, é comum suprimir a telemetria para evitar interferências.

**Exemplo:**

```bash
dart --suppress-analytics test
```
