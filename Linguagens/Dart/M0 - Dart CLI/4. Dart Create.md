## O que é o comando `create` e para que serve?

O comando `create` do Dart CLI é utilizado para **criar um novo projeto Dart** com uma estrutura inicial pré-definida. Ele automatiza a geração dos arquivos e pastas necessários, permitindo que o desenvolvedor comece a codificar imediatamente, sem se preocupar com a configuração inicial do projeto.

### Principais finalidades:

- **Configuração Rápida:** Evita a necessidade de criar manualmente a estrutura de pastas e arquivos.
- **Padronização:** Garante que o projeto siga as convenções recomendadas pela comunidade Dart.
- **Flexibilidade:** Permite escolher entre diferentes templates (modelos) de projetos, adequando-se às necessidades específicas.

---

## Opções de Parâmetros

O comando `create` possui várias opções que permitem personalizar a criação do projeto. Abaixo, detalharemos cada uma delas, explicando quando e como utilizá-las.

### `--template` (`-t`)

#### O que é e para que serve?

Define o **template** (modelo) a ser usado na criação do projeto. Os templates fornecem estruturas pré-definidas adequadas para diferentes tipos de aplicativos ou pacotes.

#### Templates Disponíveis:

- `console`: Cria um aplicativo de console simples.
- `package`: Cria um pacote de Dart, ideal para bibliotecas.
- `web`: Cria um aplicativo web básico usando Dart.
- `server`: Cria um aplicativo de servidor.
- `web-server`: Combina o aplicativo web com um servidor básico.
- `flutter`: (Se o Flutter estiver instalado) Cria um aplicativo Flutter.
- `module`: Para criar módulos de integração com outras plataformas.

#### Quando utilizar?

- **Projeto de Linha de Comando:** Use o template `console` para aplicativos que serão executados no terminal.
- **Bibliotecas:** Use o template `package` quando estiver criando pacotes reutilizáveis.
- **Aplicativos Web:** Use o template `web` para projetos que serão executados no navegador.
- **Servidores:** Use o template `server` para aplicativos que rodarão no backend.
- **Aplicativos Flutter:** Se estiver desenvolvendo para mobile ou desktop com Flutter, use o template `flutter`.

#### Sintaxe de Uso

```bash
dart create --template <template_name> <project_name>
```

**Forma abreviada:**

```bash
dart create -t <template_name> <project_name>
```

#### Exemplo:

Criar um aplicativo de console chamado `meu_app`:

```bash
dart create --template console meu_app
```

### `--list-templates`

#### O que é e para que serve?

Exibe uma lista de todos os templates disponíveis para a criação de projetos.

#### Quando utilizar?

- **Exploração:** Quando você não tem certeza de quais templates estão disponíveis.
- **Atualizações:** Para verificar se novos templates foram adicionados em atualizações recentes.

#### Sintaxe de Uso

```bash
dart create --list-templates
```

#### Exemplo:

```bash
C:\Users\lgust>dart create --list-templates
```

A saída será uma lista dos templates disponíveis.

### `--output` (`-o`)

#### O que é e para que serve?

Especifica o diretório de saída onde o projeto será criado.

#### Quando utilizar?

- **Localização Personalizada:** Quando você deseja criar o projeto em um diretório específico diferente do atual.
- **Automação de Scripts:** Em scripts que gerenciam múltiplos projetos ou estruturas complexas.

#### Sintaxe de Uso

```bash
dart create --output <directory_path> <project_name>
```

**Forma abreviada:**

```bash
dart create -o <directory_path> <project_name>
```

#### Exemplo:

Criar o projeto `meu_app` no diretório `C:\Projetos`:

```bash
dart create -o C:\Projetos meu_app
```

### `--pub`

#### O que é e para que serve?

Força a execução do `pub get` após a criação do projeto, que baixa as dependências especificadas no `pubspec.yaml`.

#### Quando utilizar?

- **Atualização Forçada:** Quando você deseja garantir que as dependências sejam baixadas, mesmo se o comportamento padrão for não executá-lo.

#### Sintaxe de Uso

```bash
dart create --pub <project_name>
```

#### Exemplo:

```bash
dart create --pub meu_app
```

### `--no-pub`

#### O que é e para que serve?

Impede a execução do `pub get` após a criação do projeto. Útil para acelerar o processo quando você planeja executar `pub get` manualmente mais tarde ou em um ambiente sem conexão à internet.

#### Quando utilizar?

- **Criação Rápida:** Quando você quer criar a estrutura do projeto sem baixar as dependências imediatamente.
- **Ambientes Offline:** Em situações onde a conexão com a internet não está disponível.

#### Sintaxe de Uso

```bash
dart create --no-pub <project_name>
```

#### Exemplo:

```bash
dart create --no-pub meu_app
```

### `--force`

#### O que é e para que serve?

Força a criação do projeto, mesmo se o diretório de destino já existir e não estiver vazio.

#### Quando utilizar?

- **Sobrescrever Conteúdo:** Quando você deseja sobrescrever um projeto existente ou tem certeza de que o conteúdo no diretório pode ser substituído.
- **Automação:** Em scripts que precisam garantir a criação do projeto sem interrupções.

#### Sintaxe de Uso

```bash
dart create --force <project_name>
```

#### Exemplo:

```bash
dart create --force meu_app
```

### Combinando Parâmetros

Você pode combinar várias opções para personalizar a criação do projeto.

**Exemplo:**

Criar um aplicativo web chamado `site` no diretório `C:\WebProjects`, sem executar `pub get` e forçando a criação:

```bash
dart create -t web -o C:\WebProjects --no-pub --force site
```

## Sintaxe de Uso

A sintaxe geral do comando `create` é:

```bash
dart create [opções] <nome_do_projeto>
```

Onde:

- `[opções]` são as opções detalhadas anteriormente.
- `<nome_do_projeto>` é o nome que você deseja dar ao seu projeto.

**Exemplo Simples:**

Criar um projeto padrão:

```bash
dart create meu_app
```

**Exemplo Completo:**

Criar um pacote chamado `minha_biblioteca` usando o template `package` e especificando o diretório de saída:

```bash
dart create -t package -o C:\Pacotes minha_biblioteca
```

---

## Restrições de Uso

- **Nome do Projeto:**

  - Deve ser em **letras minúsculas** e pode conter underscores (`_`).
  - Não pode conter espaços ou caracteres especiais.
  - Não pode começar com números.

  **Exemplos de nomes válidos:**

  - `meu_app`
  - `minha_biblioteca`
  - `app_v2`

  **Exemplos de nomes inválidos:**

  - `MeuApp` (contém letras maiúsculas)
  - `app-v2` (contém hífen)
  - `123app` (começa com números)

- **Diretório de Destino:**

  - Se o diretório já existir e não estiver vazio, o comando falhará, a menos que a opção `--force` seja usada.
  - É necessário ter permissões de escrita no diretório de destino.

- **Conexão com a Internet:**

  - Por padrão, o `pub get` é executado após a criação do projeto para baixar as dependências.
  - Se não houver conexão com a internet, utilize `--no-pub` para evitar erros.

- **Templates Disponíveis:**

  - Nem todos os templates podem estar disponíveis se você não tiver o SDK ou ferramentas necessárias instaladas (por exemplo, `flutter` sem o SDK do Flutter).

## Exemplos Práticos

### Criar um Aplicativo de Console Simples

```bash
dart create -t console hello_world
```

Este comando cria um aplicativo de console chamado `hello_world`. A estrutura básica incluirá um arquivo `bin/hello_world.dart` com um código inicial.

**Conteúdo de `bin/hello_world.dart`:**

```dart
void main() {
  print('Hello, World!');
}
```

### Criar um Pacote de Biblioteca

```bash
dart create -t package minha_biblioteca
```

Cria um pacote de Dart adequado para reutilização e publicação no pub.dev.

**Estrutura do Projeto:**

- `lib/`: Contém o código-fonte da biblioteca.
- `test/`: Para escrever testes unitários.
- `pubspec.yaml`: Arquivo de configuração do pacote.

### Criar um Aplicativo Web

```bash
dart create -t web meu_site
```

Gera um aplicativo web básico com suporte para compilar para JavaScript usando `dart2js`.

### Forçar a Criação em um Diretório Existente

```bash
dart create --force projeto_existente
```

Utilize com cautela, pois pode sobrescrever arquivos existentes.
