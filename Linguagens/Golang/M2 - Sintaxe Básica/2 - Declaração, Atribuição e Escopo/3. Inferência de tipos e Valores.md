
Go é uma linguagem de programação fortemente tipada que suporta inferência de tipos e valores padrão. Esta capacidade permite ao compilador determinar automaticamente o tipo de uma variável com base no valor que lhe é atribuído. Neste guia, vamos explorar como a inferência de tipos e valores funciona em Go, quando usá-la e exemplos práticos para ilustrar seu uso.

## Inferência de Tipos em Variáveis

### Conceito

A inferência de tipos em Go permite ao compilador determinar o tipo de uma variável sem a necessidade de declarar explicitamente esse tipo. Isso é feito com base no valor inicial atribuído à variável.

### Sintaxe

A inferência de tipos em Go é usada principalmente com a declaração curta de variáveis, que utiliza o operador `:=`.

```go
variableName := value
```

### Para Que Serve e Quando Utilizar

A inferência de tipos serve para simplificar o código, tornando-o mais conciso e legível, especialmente quando o tipo do valor atribuído é evidente. Utiliza-se a inferência de tipos quando se deseja que o compilador deduza automaticamente o tipo de uma variável a partir do seu valor inicial.

### Exemplos

#### Inferência de Tipo Básica

Neste exemplo, o tipo de cada variável é inferido automaticamente com base no valor atribuído a ela.

```go
package main

import "fmt"

func main() {
    // Inferência de tipo
    x := 10           // int
    y := 3.14         // float64
    z := "Hello, Go!" // string
    b := true         // bool

    fmt.Printf("x: %d, y: %.2f, z: %s, b: %t\n", x, y, z, b)
}
```

#### Inferência em Funções

A inferência de tipos também ocorre dentro de funções, permitindo atribuir o retorno de uma função a uma variável sem declarar explicitamente seu tipo.

```go
package main

import "fmt"

func add(a, b int) int {
    return a + b
}

func main() {
    // Inferência do tipo do retorno da função add
    result := add(3, 4) // int

    fmt.Println("Result:", result)
}
```

### Restrições de Uso

- A inferência de tipos só funciona com a declaração curta `:=` e não pode ser usada fora de funções.
- Para variáveis globais, o tipo deve ser explicitamente declarado.

## Inferência de Valores Padrões em Variáveis

### Conceito

Em Go, quando uma variável é declarada sem um valor inicial, ela é atribuída a um valor padrão (zero value) de acordo com seu tipo. Este comportamento garante que todas as variáveis tenham um valor definido, evitando comportamentos indeterminados.

### Valores Padrões por Tipo

- **int**: 0
- **float64**: 0.0
- **string**: ""
- **bool**: false
- **pointers**: nil
- **slices, maps, channels**: nil

### Para Que Serve e Quando Utilizar

A inferência de valores padrão serve para garantir a segurança e a previsibilidade do código, evitando a utilização de variáveis não inicializadas. Utiliza-se a inferência de valores padrão ao declarar variáveis que serão inicializadas posteriormente ou que representam estados iniciais específicos.

### Exemplos

#### Valores Padrões

Aqui está um exemplo de declaração de variáveis sem inicialização explícita, demonstrando os valores padrão atribuídos.

```go
package main

import "fmt"

func main() {
    var i int       // 0
    var f float64   // 0.0
    var s string    // ""
    var b bool      // false
    var p *int      // nil
    var arr []int   // nil
    var m map[string]int // nil

    fmt.Printf("int: %d, float64: %.2f, string: '%s', bool: %t, pointer: %v, slice: %v, map: %v\n", i, f, s, b, p, arr, m)
}
```

#### Utilização em Estruturas

Os valores padrão também se aplicam a campos de structs que não são inicializados explicitamente.

```go
package main

import "fmt"

type Person struct {
    Name string
    Age  int
}

func main() {
    var p Person // Name: "", Age: 0

    fmt.Printf("Person: %+v\n", p)
}
```

### Restrições de Uso

- Valores padrão são atribuídos automaticamente e não podem ser alterados no momento da declaração. Para inicializar variáveis com valores diferentes dos padrões, uma atribuição explícita deve ser feita.

## Conclusão

A inferência de tipos e valores padrão em Go são características poderosas que ajudam a manter o código limpo e seguro. A inferência de tipos permite que o compilador deduza o tipo de uma variável com base no valor atribuído a ela, enquanto a inferência de valores padrão assegura que todas as variáveis tenham um valor inicial definido, evitando comportamentos indeterminados.

Utilizar a inferência de tipos é especialmente útil para reduzir a verbosidade do código quando o tipo é evidente, e compreender os valores padrão ajuda a evitar erros ao trabalhar com variáveis não inicializadas. Estes conceitos são fundamentais para escrever código Go eficiente e robusto.
