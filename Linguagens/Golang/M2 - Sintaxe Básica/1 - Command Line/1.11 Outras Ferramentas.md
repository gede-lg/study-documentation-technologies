
#### `go fmt`

##### Sintaxe do Comando
```sh
go fmt [pacote]
```

##### Para Que Serve e Quando Utilizar
O comando `go fmt` é usado para formatar o código fonte Go de acordo com os padrões de formatação da linguagem. Ele ajuda a manter a consistência e a legibilidade do código, automatizando a formatação.

##### Exemplos de Uso
Formatando todos os arquivos na pasta atual:
```sh
go fmt
```

Formatando um pacote específico:
```sh
go fmt ./mypackage
```

##### Restrições de Uso
- `go fmt` apenas formata o código; ele não corrige erros lógicos ou de sintaxe.

#### `go get`

##### Sintaxe do Comando
```sh
go get [opções] [pacote]
```

##### Opções de Uso do Comando
- `-u`: Atualiza os pacotes e suas dependências.
- `-d`: Apenas baixa os pacotes sem instalá-los.
- `-t`: Inclui dependências de testes.
- `-v`: Mostra detalhes sobre o que está sendo baixado e instalado.

##### Para Que Serve e Quando Utilizar
O comando `go get` é usado para baixar e instalar pacotes e suas dependências a partir de repositórios remotos. É uma ferramenta essencial para gerenciar dependências em projetos Go.

##### Exemplos de Uso
Baixando e instalando um pacote:
```sh
go get github.com/user/repo
```

Atualizando um pacote:
```sh
go get -u github.com/user/repo
```

Baixando dependências de testes:
```sh
go get -t github.com/user/repo
```

##### Restrições de Uso
- `go get` pode modificar o arquivo go.mod, adicionando ou atualizando dependências.
- Pode requerer permissões de rede para acessar repositórios remotos.

#### `go mod`

##### Sintaxe do Comando
```sh
go mod [subcomando] [argumentos]
```

##### Subcomandos Importantes
- `go mod init`: Cria um novo módulo e inicializa um arquivo go.mod.
- `go mod tidy`: Remove dependências não utilizadas do go.mod.
- `go mod vendor`: Cria um diretório vendor contendo todas as dependências.
- `go mod download`: Baixa todas as dependências especificadas no go.mod.

##### Para Que Serve e Quando Utilizar
O comando `go mod` é usado para gerenciar módulos em Go. Ele ajuda a definir, atualizar e verificar dependências de pacotes, garantindo a consistência do ambiente de desenvolvimento.

##### Exemplos de Uso
Inicializando um novo módulo:
```sh
go mod init mymodule
```

Limpando dependências não utilizadas:
```sh
go mod tidy
```

Criando um diretório vendor:
```sh
go mod vendor
```

Baixando todas as dependências:
```sh
go mod download
```

##### Restrições de Uso
- `go mod` deve ser usado em um projeto que esteja utilizando módulos (Go 1.11+).
- Certos comandos podem requerer acesso à internet para baixar dependências.

#### `go test`

##### Sintaxe do Comando
```sh
go test [opções] [pacote]
```

##### Opções de Uso do Comando
- `-v`: Modo verboso, exibe detalhes dos testes executados.
- `-run regex`: Executa apenas os testes que correspondem à expressão regular.
- `-cover`: Exibe a cobertura de código dos testes.
- `-race`: Detecta race conditions.
- `-bench regex`: Executa benchmarks correspondentes à expressão regular.

##### Para Que Serve e Quando Utilizar
O comando `go test` é usado para executar testes unitários e de integração. Ele é essencial para garantir a qualidade e a funcionalidade do código, permitindo a identificação de erros e regressões.

##### Exemplos de Uso
Executando todos os testes na pasta atual:
```sh
go test
```

Executando testes de um pacote específico com detalhes:
```sh
go test -v ./mypackage
```

Executando testes específicos usando uma expressão regular:
```sh
go test -run TestFunction
```

Verificando a cobertura de código:
```sh
go test -cover
```

Executando benchmarks:
```sh
go test -bench .
```

##### Restrições de Uso
- `go test` requer que os arquivos de teste sigam a convenção de nomenclatura (terminem com _test.go).
- Certos testes podem precisar de permissões ou configurações específicas do ambiente.

#### `go doc`

##### Sintaxe do Comando
```sh
go doc [pacote|identificador]
```

##### Para Que Serve e Quando Utilizar
O comando `go doc` exibe a documentação de pacotes, tipos, funções e outros identificadores. Ele é útil para entender a funcionalidade de pacotes e suas APIs, proporcionando uma referência rápida sem precisar sair do terminal.

##### Exemplos de Uso
Exibindo a documentação de um pacote:
```sh
go doc fmt
```

Exibindo a documentação de uma função específica:
```sh
go doc fmt.Printf
```

##### Restrições de Uso
- `go doc` exibe apenas a documentação disponível no código fonte, que deve estar adequadamente comentada.

---

#### `go vet`

##### Sintaxe do Comando
```sh
go vet [pacote]
```

##### Para Que Serve e Quando Utilizar
O comando `go vet` é uma ferramenta de análise estática que verifica o código fonte em busca de construções suspeitas ou erros comuns. Ele ajuda a identificar problemas que podem não ser capturados pelo compilador.

##### Exemplos de Uso
Verificando todos os pacotes na pasta atual:
```sh
go vet
```

Verificando um pacote específico:
```sh
go vet ./mypackage
```

##### Restrições de Uso
- `go vet` não substitui testes de unidade ou revisão de código. Ele deve ser usado como uma ferramenta adicional de verificação.

### Considerações Finais

As ferramentas de linha de comando do Go descritas acima são essenciais para o desenvolvimento eficiente e profissional. Aqui estão alguns pontos importantes a considerar:
- **Automatização e Integração**: Use essas ferramentas em scripts de automação e pipelines de CI/CD para garantir a consistência e a qualidade do código.
- **Boas Práticas**: Combine várias ferramentas (`go fmt`, `go vet`, `go test`) para manter o código limpo, funcional e bem documentado.
- **Documentação e Ajuda**: Use `go help [comando]` para obter mais informações sobre qualquer comando.

Ao dominar essas ferramentas, você poderá desenvolver, testar, documentar e manter aplicações Go de forma eficiente e profissional.
