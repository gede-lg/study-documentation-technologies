
#### `go fmt`

##### Sintaxe do Comando
```sh
go fmt [opções] [pacote]
```

##### Opções de Uso do Comando
- `go fmt` não possui opções específicas; ele formata o código-fonte em conformidade com as convenções de estilo do Go.

##### Para Que Serve e Quando Utilizar
O comando `go fmt` é usado para formatar o código Go de acordo com as convenções padrão da linguagem. Ele ajusta a indentação, espaçamento e outros aspectos de formatação para melhorar a legibilidade e consistência do código. Deve ser utilizado regularmente durante o desenvolvimento para manter o código limpo e padronizado.

##### Exemplos de Uso
Formatando o código na pasta atual:
```sh
go fmt
```

Formatando um pacote específico:
```sh
go fmt ./mypackage
```

##### Restrições de Uso
- `go fmt` só formata o código e não corrige erros de sintaxe ou lógica.
- Apenas formata arquivos `.go`; não afeta outros tipos de arquivos.

#### `gofmt`

##### Sintaxe do Comando
```sh
gofmt [opções] [arquivos]
```

##### Opções de Uso do Comando
- `-w`: Escreve o resultado no arquivo (substitui o original).
- `-l`: Lista os arquivos que precisam de formatação.
- `-s`: Simplifica o código.
- `-d`: Exibe as diferenças entre a versão original e a formatada.
- `-r`: Aplica reescritas específicas.
- `-e`: Exibe todos os erros encontrados.

##### Para Que Serve e Quando Utilizar
O `gofmt` é uma ferramenta de formatação de código mais flexível que `go fmt`, oferecendo opções adicionais para simplificação e reescrita de código. Pode ser usado para formatação avançada e automação de tarefas de formatação em scripts e pipelines de CI/CD.

##### Exemplos de Uso
Listando arquivos que precisam de formatação:
```sh
gofmt -l .
```

Formatando e escrevendo mudanças nos arquivos:
```sh
gofmt -w main.go
```

Simplificando o código:
```sh
gofmt -s -w main.go
```

##### Restrições de Uso
- `gofmt` pode sobrescrever arquivos se usado com a opção `-w`, então deve ser usado com cautela.

#### `go vet`

##### Sintaxe do Comando
```sh
go vet [opções] [pacote]
```

##### Opções de Uso do Comando
- `-all`: Habilita todas as verificações.
- `-asmdecl`: Verifica declarações de assembly.
- `-assign`: Verifica atribuições.
- `-atomic`: Verifica uso de funções atomics.
- `-bool`: Verifica expressões booleanas.
- `-buildtag`: Verifica tags de build.
- `-cgocall`: Verifica chamadas Cgo.
- `-composites`: Verifica inicializadores composite.
- `-copylocks`: Verifica cópias de valores com bloqueios.
- `-httpresponse`: Verifica resposta HTTP não fechada.
- `-lostcancel`: Verifica contextos cancelados perdidos.
- `-nilfunc`: Verifica chamadas de funções nulas.
- `-printf`: Verifica chamadas de funções printf-like.
- `-shift`: Verifica operações de shift.
- `-shadow`: Verifica variáveis sombreadas.
- `-stringintconv`: Verifica conversões string/int.
- `-structtag`: Verifica tags de struct.
- `-unreachable`: Verifica código inalcançável.
- `-unsafeptr`: Verifica uso inseguro de ponteiros.
- `-unmarshal`: Verifica unmarshaling.
- `-unusedresult`: Verifica resultados de função não usados.

##### Para Que Serve e Quando Utilizar
O `go vet` é uma ferramenta estática que analisa o código em busca de construções suspeitas ou erros comuns, como uso inadequado de funções, erros em tags de struct, e contextos perdidos. Deve ser usado durante o desenvolvimento para detectar problemas que podem não ser capturados pelo compilador.

##### Exemplos de Uso
Analisando o código na pasta atual:
```sh
go vet
```

Analisando um pacote específico:
```sh
go vet ./mypackage
```

Habilitando todas as verificações:
```sh
go vet -all ./mypackage
```

##### Restrições de Uso
- `go vet` não detecta todos os erros possíveis e não substitui testes de unidade ou revisões de código.
- Alguns falsos positivos podem ocorrer, especialmente em código complexo.

#### `golint`

##### Sintaxe do Comando
```sh
golint [opções] [pacote]
```

##### Opções de Uso do Comando
- `-set_exit_status`: Define o status de saída para 1 se houver problemas.

##### Para Que Serve e Quando Utilizar
O `golint` verifica o código Go em busca de problemas de estilo, como convenções de nomenclatura, documentação e práticas recomendadas. É útil para garantir que o código siga as convenções de estilo da comunidade Go.

##### Exemplos de Uso
Verificando o código na pasta atual:
```sh
golint
```

Verificando um pacote específico:
```sh
golint ./mypackage
```

Definindo o status de saída se houver problemas:
```sh
golint -set_exit_status ./mypackage
```

##### Restrições de Uso
- `golint` não corrige problemas automaticamente; apenas reporta.
- Não detecta problemas de lógica ou erros de execução.

#### `staticcheck`

##### Sintaxe do Comando
```sh
staticcheck [opções] [pacote]
```

##### Opções de Uso do Comando
- `-checks`: Especifica quais verificações realizar.
- `-ignore`: Ignora certos problemas.
- `-tags`: Especifica tags de build para incluir/excluir código.

##### Para Que Serve e Quando Utilizar
`staticcheck` é uma ferramenta de análise estática avançada que realiza uma ampla gama de verificações em busca de erros, problemas de estilo, e práticas de código inseguras. É mais abrangente que `go vet` e `golint`, fornecendo análises detalhadas.

##### Exemplos de Uso
Analisando o código na pasta atual:
```sh
staticcheck
```

Analisando um pacote específico:
```sh
staticcheck ./mypackage
```

Ignorando certos problemas:
```sh
staticcheck -ignore "SA9001" ./mypackage
```

##### Restrições de Uso
- `staticcheck` pode gerar muitos avisos em projetos grandes, requerendo configuração cuidadosa para evitar falsos positivos.

### Considerações Finais

Os comandos de análise de código do Go são essenciais para manter a qualidade e consistência do código. Aqui estão alguns pontos importantes a considerar:
- **Integração Contínua**: Use essas ferramentas em pipelines de CI/CD para garantir que problemas sejam detectados cedo.
- **Automatização**: Combine essas ferramentas com scripts e Makefiles para automatizar a análise de código.
- **Revisões de Código**: Use os relatórios gerados por essas ferramentas como parte das revisões de código para melhorar a qualidade geral.

Ao dominar essas ferramentas, você poderá garantir que seu código Go seja de alta qualidade, seguro e conforme com as melhores práticas da comunidade.
