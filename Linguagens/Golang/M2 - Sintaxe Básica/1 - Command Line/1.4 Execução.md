
#### `go run`

##### Sintaxe do Comando
```sh
go run [arquivos] [argumentos...]
```

##### Opções de Uso do Comando
- `-race`: Habilita a detecção de race conditions (somente em arquiteturas que suportam).
- `-gcflags string`: Passa argumentos para o compilador de Go (por exemplo, `-gcflags="-m"` para imprimir otimizações de memória).
- `-exec string`: Usa o comando especificado para executar o binário gerado.

##### Para Que Serve e Quando Utilizar
O comando `go run` compila e executa um ou mais arquivos Go em uma única etapa. É especialmente útil para desenvolvimento rápido, testes e execução de scripts, pois permite que você execute o código sem a necessidade de gerar um binário separado.

##### Exemplos de Uso
Executando um único arquivo Go:
```sh
go run main.go
```

Executando múltiplos arquivos Go:
```sh
go run main.go util.go
```

Passando argumentos para o programa:
```sh
go run main.go arg1 arg2
```

Habilitando a detecção de race conditions:
```sh
go run -race main.go
```

##### Restrições de Uso
- `go run` é principalmente usado para desenvolvimento e não para deploy em produção.
- Não gera um binário separado para distribuição.
- Pode ser mais lento do que executar um binário compilado, pois precisa compilar os arquivos antes da execução.

#### `go exec`

##### Sintaxe do Comando
```sh
go exec [-exec comando] arquivo_executável [argumentos...]
```

##### Para Que Serve e Quando Utilizar
Embora `go exec` não seja um comando padrão no conjunto de ferramentas Go, ele é frequentemente usado em scripts e ambientes de desenvolvimento para executar binários Go específicos com determinadas configurações ou em contextos controlados. É especialmente útil quando você precisa passar configurações específicas de execução.

##### Exemplos de Uso
Executando um binário Go com uma ferramenta específica:
```sh
go exec -exec='/usr/local/bin/custom_runner' mybinary
```

##### Restrições de Uso
- `go exec` depende da disponibilidade do comando `-exec` especificado no sistema.
- Usado principalmente em contextos de desenvolvimento ou scripts personalizados.

---

### Considerações Finais

Os comandos de linha de comando do Go para execução são ferramentas essenciais no ciclo de desenvolvimento, permitindo a execução rápida de programas e a detecção de problemas. Aqui estão alguns pontos importantes a considerar:
- **Execução Rápida**: Use `go run` para desenvolvimento e testes rápidos.
- **Configuração Específica**: Utilize `-race` para detectar condições de corrida durante a execução.
- **Automatização e Scripts**: Scripts personalizados podem utilizar `go exec` para adicionar configurações específicas durante a execução.

Ao dominar esses comandos, você pode melhorar sua eficiência durante o desenvolvimento e depuração de aplicações Go.
