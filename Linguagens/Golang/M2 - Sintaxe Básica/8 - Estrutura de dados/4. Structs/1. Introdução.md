
## O Que É e Para Que Serve?

No Go, uma `struct` (abreviação de "structure") é um tipo de dado composto que agrupa variáveis sob um mesmo nome. Essas variáveis, chamadas de "campos", podem ser de diferentes tipos. As structs são usadas para representar objetos ou entidades complexas com várias propriedades.

### Principais Usos das Structs:
- **Modelagem de Dados**: Utilizar structs para representar entidades do mundo real em um sistema (e.g., um "Carro", um "Cliente", um "Pedido").
- **Organização de Código**: Agrupar dados relacionados para facilitar a manutenção e a leitura do código.
- **Passagem de Dados**: Passar conjuntos de dados de forma organizada entre funções e métodos.

## Quando Utilizar?

Utilize structs quando precisar agrupar várias informações relacionadas em uma única entidade. Elas são especialmente úteis quando:
- Você tem um conjunto de dados que sempre viaja junto.
- Você quer definir um tipo que possui diversas propriedades.
- Você precisa de métodos associados a um conjunto específico de dados.

## Sintaxe de Uso

### Declaração de uma Struct
Para declarar uma struct, você usa a palavra-chave `type` seguida pelo nome da struct e a palavra-chave `struct`. Dentro das chaves `{}`, você define os campos da struct.

```go
type NomeEstrutura struct {
	Nome   string    //nome do campo - tipo do campo
}
```


### Acesso aos Campos de uma Struct
Você acessa e modifica os campos de uma struct usando a notação de ponto (`.`).

```go
fmt.Println(carro.Marca)  // Acessar o campo Marca
carro.Preco = 90000.00    // Modificar o campo Preco
```

### Métodos em Structs
No Go, você pode definir métodos para structs. Um método é uma função com um "receiver" especial, que é a instância da struct.

```go
func (c Carro) Descricao() string {
    return fmt.Sprintf("Marca: %s, Modelo: %s, Ano: %d, Preço: %.2f", c.Marca, c.Modelo, c.Ano, c.Preco)
}
```

Para chamar o método:

```go
fmt.Println(carro.Descricao())
```

### Embedded Structs (Composição)
Go não suporta herança, mas suporta composição. Você pode incluir uma struct dentro de outra struct.

```go
type Motor struct {
    Potencia int
    Tipo     string
}

type Carro struct {
    Marca   string
    Modelo  string
    Ano     int
    Preco   float64
    Motor   Motor
}
```

Você pode acessar os campos da struct embutida diretamente:

```go
carro := Carro{
    Marca:  "Toyota",
    Modelo: "Corolla",
    Ano:    2021,
    Preco:  85000.00,
    Motor:  Motor{Potencia: 150, Tipo: "Gasolina"},
}

fmt.Println(carro.Motor.Potencia)  // Acessar o campo Potencia do Motor
```

## Restrições de Uso

### Exportação de Campos
No Go, a visibilidade dos campos de uma struct (pública ou privada) é controlada pela primeira letra do nome do campo. Se a primeira letra for maiúscula, o campo é exportado (público) e pode ser acessado fora do pacote. Se for minúscula, o campo é não exportado (privado).

```go
type Carro struct {
    Marca   string  // Público
    modelo  string  // Privado
    Ano     int
    preco   float64
}
```

### Métodos e Pointers
Quando você define métodos para uma struct, pode escolher entre usar um receiver de valor ou um receiver de ponteiro. Use um receiver de ponteiro se o método precisa modificar a instância da struct ou se a struct é grande e você deseja evitar cópia de dados.

```go
func (c *Carro) AtualizarPreco(novoPreco float64) {
    c.Preco = novoPreco
}
```

### Literais de Struct
Os literais de struct exigem que todos os campos sejam inicializados, a menos que você utilize a inicialização nomeada. Não inicializar campos resultará em valores zero para esses campos.

```go
carro := Carro{
    Marca:  "Toyota",
    Modelo: "Corolla",  // Erro de compilação se omitir este campo
}
```

## Exemplos Completos

### Exemplo 1: Struct Básica
```go
package main

import (
    "fmt"
)

type Pessoa struct {
    Nome   string
    Idade  int
    Cidade string
}

func main() {
    p := Pessoa{
        Nome:   "João",
        Idade:  30,
        Cidade: "São Paulo",
    }

    fmt.Println(p.Nome)  // João
    p.Idade = 31
    fmt.Println(p.Idade)  // 31
}
```

### Exemplo 2: Métodos em Struct
```go
package main

import (
    "fmt"
)

type Retangulo struct {
    Largura  float64
    Altura   float64
}

func (r Retangulo) Area() float64 {
    return r.Largura * r.Altura
}

func (r Retangulo) Perimetro() float64 {
    return 2 * (r.Largura + r.Altura)
}

func main() {
    ret := Retangulo{Largura: 5, Altura: 3}
    fmt.Println("Área:", ret.Area())         // Área: 15
    fmt.Println("Perímetro:", ret.Perimetro()) // Perímetro: 16
}
```

### Exemplo 3: Struct Embutida
```go
package main

import (
    "fmt"
)

type Motor struct {
    Potencia int
    Tipo     string
}

type Carro struct {
    Marca  string
    Modelo string
    Motor  Motor
}

func main() {
    carro := Carro{
        Marca:  "Toyota",
        Modelo: "Corolla",
        Motor:  Motor{Potencia: 150, Tipo: "Gasolina"},
    }

    fmt.Println(carro.Marca)        // Toyota
    fmt.Println(carro.Motor.Potencia) // 150
}
```

Esses exemplos ilustram como utilizar structs em Go para modelar dados de forma organizada e eficiente, além de como adicionar métodos e utilizar composição para criar tipos complexos. As structs são uma ferramenta poderosa no Go e são fundamentais para escrever código limpo e bem estruturado.
