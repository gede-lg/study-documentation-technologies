# Passagem por Valor no Go

## O que é e para que serve?

A passagem por valor é uma técnica em programação onde uma cópia do valor da variável é passada para uma função ou método. No contexto de Go, isso significa que quando você passa um argumento para uma função, o Go cria uma cópia desse valor e a função trabalha com essa cópia, não com o valor original. Esse comportamento garante que alterações feitas no argumento dentro da função não afetem o valor original fora da função.

### Para que serve?

A passagem por valor é útil para garantir a imutabilidade do valor original, permitindo que a função trabalhe com uma cópia segura do dado. Isso pode ser benéfico em situações onde você deseja evitar efeitos colaterais indesejados e manter a integridade dos dados originais.

## Sintaxe de Uso

Em Go, a passagem por valor é o comportamento padrão quando você passa variáveis para funções. Aqui está a sintaxe básica:

```go
package main

import "fmt"

// Função que utiliza passagem por valor
func incrementar(x int) {
    x = x + 1
    fmt.Println("Dentro da função:", x)
}

func main() {
    var a int = 5
    incrementar(a)
    fmt.Println("Fora da função:", a)
}
```

### Explicação:

- A variável `a` é passada para a função `incrementar`.
- Dentro da função `incrementar`, `x` é uma cópia de `a`.
- Alterações feitas em `x` não afetam `a`.

## Restrições de Uso

A principal restrição da passagem por valor é que ela pode ser ineficiente para tipos de dados grandes, como grandes arrays ou estruturas (structs), pois envolve a cópia de dados. Além disso, se você precisar modificar o valor original dentro da função, a passagem por valor não é adequada. Para esses casos, a passagem por referência (usando ponteiros) é mais apropriada.

## Quando Utilizar?

A passagem por valor deve ser utilizada quando:

1. **Imutabilidade é Desejada:** Você deseja garantir que a função não altere o valor original.
2. **Tipos de Dados Pequenos:** É eficiente para tipos de dados pequenos como `int`, `float`, `bool`, `string`, etc.
3. **Segurança de Dados:** Para evitar efeitos colaterais e garantir que os dados originais permaneçam intactos.

### Exemplo Prático

Aqui está um exemplo prático para ilustrar a passagem por valor em uma função que calcula a área de um retângulo:

```go
package main

import "fmt"

// Estrutura para representar um retângulo
type Retangulo struct {
    largura  float64
    altura   float64
}

// Função para calcular a área, utilizando passagem por valor
func calcularArea(r Retangulo) float64 {
    return r.largura * r.altura
}

func main() {
    r := Retangulo{largura: 10, altura: 5}
    area := calcularArea(r)
    fmt.Println("Área do retângulo:", area)
    fmt.Println("Retângulo original:", r)
}
```

### Explicação:

- A estrutura `Retangulo` é passada para a função `calcularArea` por valor.
- A função trabalha com uma cópia de `Retangulo`.
- O valor original de `Retangulo` permanece inalterado após a função ser chamada.
## Conclusão

A passagem por valor em Go é uma técnica fundamental que garante a imutabilidade dos dados passados para funções. É eficiente e segura para tipos de dados pequenos e situações onde a integridade dos dados é crucial. No entanto, para tipos de dados grandes ou quando modificações no valor original são necessárias, a passagem por referência com ponteiros deve ser considerada.
