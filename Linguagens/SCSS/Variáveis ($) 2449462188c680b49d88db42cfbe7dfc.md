# Variáveis no Sass: Organizando e Escalando seu CSS

### Introdução

No mundo do desenvolvimento web, a manutenção e a escalabilidade de grandes folhas de estilo CSS podem se tornar um desafio. Cores, tamanhos de fontes e espaçamentos, quando repetidos em diversos lugares, dificultam atualizações e tornam o código propenso a erros. O Sass (Syntactically Awesome Style Sheets) surge como uma solução, introduzindo recursos como as **variáveis**, que transformam a maneira como escrevemos CSS. Elas nos permitem armazenar valores e reutilizá-los, centralizando informações e tornando nosso código mais limpo, DRY (Don't Repeat Yourself) e fácil de gerenciar.

### Sumário

Esta explicação abordará as variáveis do Sass de forma detalhada, cobrindo desde seus conceitos fundamentais até as melhores práticas. Exploraremos a sintaxe, escopo, restrições e, por fim, apresentaremos exemplos completos para ilustrar seu poder na prática.

## Conceitos Fundamentais

As variáveis no Sass são o equivalente a variáveis em linguagens de programação. Elas permitem que você nomeie um valor e o utilize em múltiplos lugares. A principal motivação para seu uso é a **manutenção e consistência**. Em vez de procurar e substituir um valor hexadecimal de cor em dezenas de arquivos, você simplesmente altera o valor da variável em um único lugar, e a mudança é refletida em todo o código.

**Propósito:**

- **Reutilização:** Evita a repetição de valores.
- **Manutenção:** Simplifica a atualização de valores globais (como cores da marca, fontes).
- **Consistência:** Garante que os mesmos valores sejam usados consistentemente em todo o projeto.
- **Legibilidade:** Torna o código mais fácil de ler e entender, pois nomes de variáveis autoexplicativos são mais claros que valores brutos.

## Sintaxe e Uso

A sintaxe para declarar e usar variáveis no Sass é simples e direta.

- **Declaração:** Uma variável é declarada usando o sinal de dólar (`$`), seguido por um nome e, em seguida, o valor.
- **Nomenclatura:** Os nomes das variáveis podem conter letras, números, hífens () e underscores (`_`). É uma boa prática usar hífens, seguindo o padrão de nomenclatura de propriedades do CSS.
- **Uso:** Para usar uma variável, basta chamá-la pelo nome onde o valor seria inserido.

**Exemplo de código:**

```sass
// Variáveis.scss

// Declaração de variáveis
$cor-primaria: #3498db; // Cor azul para o tema
$fonte-padrao: "Helvetica Neue", Arial, sans-serif; // Fonte principal do site
$tamanho-fonte-base: 16px; // Tamanho de fonte padrão para o corpo do texto
$espacamento-padrao: 20px; // Espaçamento para seções e elementos

// Uso das variáveis
body {
  font-family: $fonte-padrao; // Aplica a fonte definida
  font-size: $tamanho-fonte-base; // Aplica o tamanho da fonte
  color: #333;
}

.botao {
  background-color: $cor-primaria; // Usa a cor primária como background
  color: white; // Cor do texto do botão
  padding: $espacamento-padrao / 2 $espacamento-padrao; // Usa a variável de espaçamento
  border: none;
  border-radius: 4px;
}

```

Quando o Sass compila este código, o CSS gerado será:

```css
body {
  font-family: "Helvetica Neue", Arial, sans-serif;
  font-size: 16px;
  color: #333;
}

.botao {
  background-color: #3498db;
  color: white;
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
}

```

### O `!default` Flag

Uma variável no Sass pode ser definida com a flag `!default`. Essa flag atribui um valor à variável **apenas se ela ainda não tiver sido definida**. Isso é extremamente útil para a criação de bibliotecas, onde você pode fornecer valores padrão que o usuário pode sobrescrever facilmente.

**Exemplo:**

```sass
// _variaveis-biblioteca.scss
$cor-texto: #333 !default;
$tamanho-borda: 1px !default;

// _meu-projeto.scss
// Aqui eu posso sobrescrever o valor padrão
$cor-texto: #1c1c1c;

@import "variaveis-biblioteca";

body {
  color: $cor-texto; // A cor usada será #1c1c1c
  border: $tamanho-borda solid black; // A borda usada será 1px
}

```

No exemplo acima, `$cor-texto` foi sobrescrita, mas `$tamanho-borda` manteve seu valor padrão.

## Escopo e Hierarquia de Variáveis

O escopo de uma variável define onde ela pode ser acessada. No Sass, o escopo é **limitado ao bloco de código onde a variável é declarada**.

- **Variáveis Globais:** Declaradas no nível raiz do arquivo, fora de qualquer seletor. São acessíveis em qualquer lugar no código.
- **Variáveis Locais:** Declaradas dentro de um seletor ou bloco de código. São acessíveis apenas dentro desse bloco e de seus blocos aninhados.

**Exemplo de Escopo:**

```sass
$cor-global: #f39c12; // Variável global

.componente {
  $cor-local: #2ecc71; // Variável local

  background-color: $cor-global; // Funciona
  color: $cor-local; // Funciona

  .elemento-aninhado {
    border-color: $cor-global; // Funciona
    border-top-color: $cor-local; // Funciona
  }
}

.outro-componente {
  // color: $cor-local; // ERRO! A variável $cor-local não existe neste escopo.
  background-color: $cor-global; // Funciona
}

```

### O `!global` Flag

Para forçar uma variável a ser global, mesmo se ela for declarada dentro de um escopo local, você pode usar a flag `!global`. Isso pode ser útil em mixins ou funções para modificar o estado global.

**Exemplo:**

```sass
$valor: 10;

.container {
  $valor: 20 !global; // Força a variável a se tornar global
  width: $valor; // `width` será 20
}

.outro-container {
  // $valor agora é 20, não mais 10
  width: $valor; // `width` será 20
}

```

## Restrições de Uso

Embora as variáveis do Sass sejam incrivelmente úteis, há algumas restrições e cenários onde seu uso pode não ser ideal ou gerar erros.

1. **Declaração em seletor de classe/id:** As variáveis devem ser declaradas com o cifrão (`$`) para serem reconhecidas pelo Sass. Declará-las com `var(--)` como no CSS nativo não funciona no Sass.
2. **Variáveis dinâmicas no CSS nativo:** As variáveis do Sass são compiladas para valores estáticos no CSS. Elas não podem ser alteradas dinamicamente via JavaScript no navegador, como as variáveis CSS custom properties (`-`). Se você precisa de valores que possam ser modificados em tempo de execução, use as variáveis CSS custom properties.
3. **Restrição de Escopo:** Como vimos, uma variável local não pode ser acessada fora do seu escopo. Tentar fazer isso resultará em um erro de compilação.
4. **Nomenclatura:** Evite nomes de variáveis que sejam idênticos a palavras-chave do CSS ou Sass, para evitar confusão.

## Melhores Práticas e Casos de Uso

A utilização correta das variáveis do Sass pode elevar a qualidade do seu código.

### Casos de Uso Comuns

- **Paleta de Cores:** Armazenar todas as cores da sua marca ou paleta de cores.
    
    ```sass
    $cor-primaria: #007bff;
    $cor-secundaria: #6c757d;
    $cor-sucesso: #28a745;
    $cor-erro: #dc3545;
    
    ```
    
- **Tipografia:** Centralizar as configurações de fonte.
    
    ```sass
    $fonte-titulo: "Roboto", sans-serif;
    $fonte-texto: "Open Sans", sans-serif;
    $tamanho-fonte-grande: 2.5rem;
    $tamanho-fonte-pequena: 0.8rem;
    
    ```
    
- **Layout e Espaçamento:** Padronizar margens, paddings, larguras.
    
    ```sass
    $espacamento-base: 16px;
    $largura-conteudo: 1140px;
    $borda-arredondada: 5px;
    
    ```
    

### Melhores Práticas

- **Organização:** Crie um arquivo dedicado (ex: `_variables.scss`) para armazenar todas as suas variáveis e importe-o no início do seu arquivo principal.
- **Nomes Descritivos:** Use nomes de variáveis que sejam claros e autoexplicativos (ex: `$cor-primaria` em vez de `$azul`).
- **Evite Variações Desnecessárias:** Não crie variáveis para cada pequena variação. Use variáveis para valores que são reutilizados em todo o projeto.
- **Pense em Escopo:** Use o escopo global para variáveis que são usadas em todo o projeto, e o escopo local para variáveis que são exclusivas de um componente.

## Exemplo Completo: Cartão de Produto

A seguir, um exemplo completo que ilustra a aplicação de variáveis do Sass em um contexto prático.

```sass
// _variables.scss
$cor-fundo-cartao: #ffffff;
$cor-sombra: rgba(0, 0, 0, 0.1);
$cor-titulo: #333333;
$cor-descricao: #666666;
$cor-botao-primario: #007bff;
$cor-texto-botao: #ffffff;

$tamanho-fonte-titulo: 1.25rem;
$tamanho-fonte-descricao: 1rem;
$tamanho-fonte-botao: 0.875rem;

$espacamento-card: 20px;
$arredondamento-borda: 8px;
$transicao-padrao: 0.3s ease;

// main.scss
@import "variables";

.card {
  display: flex;
  flex-direction: column;
  width: 300px;
  background-color: $cor-fundo-cartao;
  border-radius: $arredondamento-borda;
  box-shadow: 0 4px 6px $cor-sombra;
  transition: transform $transicao-padrao;

  &:hover {
    transform: translateY(-5px);
  }

  .card-imagem {
    width: 100%;
    height: 200px;
    object-fit: cover;
    border-top-left-radius: $arredondamento-borda;
    border-top-right-radius: $arredondamento-borda;
  }

  .card-conteudo {
    padding: $espacamento-card;

    .card-titulo {
      margin-top: 0;
      font-size: $tamanho-fonte-titulo;
      color: $cor-titulo;
    }

    .card-descricao {
      font-size: $tamanho-fonte-descricao;
      color: $cor-descricao;
    }

    .card-botao {
      display: inline-block;
      padding: 10px 15px;
      font-size: $tamanho-fonte-botao;
      color: $cor-texto-botao;
      background-color: $cor-botao-primario;
      text-decoration: none;
      border-radius: $arredondamento-borda;
      transition: background-color $transicao-padrao;

      &:hover {
        background-color: darken($cor-botao-primario, 10%); // Exemplo de uso em funções
      }
    }
  }
}

```

Neste exemplo, todas as características visuais do cartão, como cores, tamanhos e espaçamentos, são controladas por variáveis. Se a cor primária da marca mudar, ou se o espaçamento padrão for ajustado, basta uma única alteração no arquivo `_variables.scss` para que o cartão e qualquer outro componente que use essas variáveis seja atualizado automaticamente.
