# 2 - Tags com anotação

### Definição completa da estrutura Tags por meio de anotação própria do swagger

Além da configuração global, é possível definir tags em níveis de operação específicos usando anotações nas classes ou métodos de um controlador Spring. Aqui está um exemplo:

```java
import io.swagger.v3.oas.annotations.Operation;import io.swagger.v3.oas.annotations.tags.Tag;import org.springframework.web.bind.annotation.GetMapping;import org.springframework.web.bind.annotation.RestController;@RestController@Tag(name = "Usuário", description = "Controlador de Usuários")public class UserController {    @GetMapping("/users")    @Operation(summary = "Obtém lista de usuários", tags = {"Usuário"})    public List<User> getAllUsers() {        // implementação do método    }    @GetMapping("/users/{id}")    @Operation(summary = "Obtém usuário por ID", tags = {"Usuário"})    public User getUserById(@PathVariable String id) {        // implementação do método    }}
```

Neste exemplo, o controlador `UserController` está anotado com `@Tag`, especificando que todos os métodos dentro dele pertencem à tag “Usuário”. Individualmente, cada método também pode especificar suas tags correspondentes através da anotação `@Operation`.

### Informações adicionais importantes

- **Boas práticas:** Utilizar tags de maneira consistente e clara é crucial para a manutenção da documentação da API. Evite usar muitas tags para a mesma categoria de operações, e mantenha a descrição das tags o mais informativa possível.
- **Swagger UI:** As tags definidas na sua documentação da API ajudam a organizar visualmente a interface do Swagger UI, o que facilita muito para os desenvolvedores entenderem e testarem a API.

Espero que essas informações ajudem você a entender melhor como trabalhar com tags no Swagger/OpenAPI em seus projetos Java!
