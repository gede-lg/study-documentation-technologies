## O que é e para que serve?

O comando `flutter create` é utilizado para **criar novos projetos Flutter**. Ele gera a estrutura de arquivos e pastas necessária para iniciar um novo aplicativo Flutter. Isso inclui a configuração de diretórios e arquivos para Android, iOS, web, desktop (Windows, macOS, Linux), além do código Dart base que inicializa o aplicativo.

**Para que serve:**

- **Inicialização de Projetos:** Cria a estrutura básica para novos projetos Flutter.
- **Personalização:** O comando permite a personalização de templates, nome do projeto, linguagens de código nativo (Kotlin/Java para Android e Swift/Objective-C para iOS), entre outros.
- **Criação Multi-Plataforma:** Facilita a criação de aplicativos prontos para múltiplas plataformas.

---

## Opções de Parâmetros do Comando `flutter create`

O `flutter create` possui diversas opções que permitem personalizar o projeto conforme as suas necessidades. Vamos explorar cada uma em detalhes.

---

### `--template`

**Descrição:** Define o **template** (modelo) que será utilizado ao criar o projeto. O Flutter oferece diferentes templates para criar projetos de diferentes naturezas.

**Opções Internas:**

- `app`: (Padrão) Cria um aplicativo completo.
- `package`: Cria um pacote Dart que pode ser utilizado em outros projetos.
- `plugin`: Cria um plugin Flutter, permitindo integração com plataformas nativas.
- `module`: Cria um módulo Flutter que pode ser adicionado a um aplicativo existente (nativo).

**Quando utilizar?**

- **app:** Quando o objetivo for criar um aplicativo Flutter completo.
- **package:** Ao querer criar uma biblioteca Dart que pode ser compartilhada ou publicada no pub.dev.
- **plugin:** Quando se deseja criar um plugin que usa código nativo para Android e iOS.
- **module:** Para adicionar funcionalidades Flutter a um app nativo já existente.

**Sintaxe:**

```bash
flutter create --template=<template> <project-name>
```

**Exemplo:**

```bash
flutter create --template=plugin my_flutter_plugin
```

---

### `--org`

**Descrição:** Define o **domínio da organização** do projeto. Isso é importante para a geração dos pacotes para Android e iOS, pois afeta a estrutura do pacote gerado.

**Quando utilizar?**

- **Customização de Pacotes:** Utilizado para definir o identificador da organização de forma que o pacote não conflite com outros pacotes no Google Play ou App Store.

**Sintaxe:**

```bash
flutter create --org=<com.example> <project-name>
```

**Exemplo:**

```bash
flutter create --org=com.mycompany my_flutter_app
```

---

### `--platforms`

**Descrição:** Define para quais **plataformas** o projeto será criado. O Flutter oferece suporte para várias plataformas como Android, iOS, Web, Windows, macOS, e Linux.

**Opções Internas:**

- `android`: Gera suporte para Android.
- `ios`: Gera suporte para iOS.
- `web`: Gera suporte para Web.
- `windows`: Gera suporte para Windows.
- `macos`: Gera suporte para macOS.
- `linux`: Gera suporte para Linux.

**Quando utilizar?**

- Quando você deseja restringir ou especificar as plataformas que o projeto suportará.
- Por exemplo, em um ambiente de desenvolvimento restrito, você pode desativar plataformas não utilizadas.

**Sintaxe:**

```bash
flutter create --platforms=<platforms> <project-name>
```

**Exemplo:**

```bash
flutter create --platforms=android,ios,web my_flutter_app
```

---

### `--project-name`

**Descrição:** Define o **nome do projeto**. O nome do projeto será usado para nomear pastas e arquivos internos.

**Quando utilizar?**

- Quando o nome do diretório e do projeto precisa seguir um padrão específico.

**Restrições:**

- O nome deve seguir as regras de nomes de pacotes do Dart: não pode conter espaços, letras maiúsculas ou caracteres especiais.

**Sintaxe:**

```bash
flutter create --project-name=<name> <directory>
```

**Exemplo:**

```bash
flutter create --project-name=my_flutter_app app_directory
```

---

### `--ios-language`

**Descrição:** Especifica a linguagem nativa a ser usada para o código iOS.

**Opções Internas:**

- `objc`: Usa Objective-C.
- `swift`: Usa Swift (padrão).

**Quando utilizar?**

- **Objective-C:** Se você já está trabalhando com um código-base antigo ou prefere Objective-C.
- **Swift:** Para novos projetos, ou quando se deseja utilizar a linguagem mais moderna para iOS.

**Sintaxe:**

```bash
flutter create --ios-language=<swift|objc> <project-name>
```

**Exemplo:**

```bash
flutter create --ios-language=objc my_flutter_app
```

---

### `--android-language`

**Descrição:** Especifica a linguagem nativa a ser usada para o código Android.

**Opções Internas:**

- `java`: Usa Java.
- `kotlin`: Usa Kotlin (padrão).

**Quando utilizar?**

- **Java:** Quando estiver integrando com código Java existente ou precisar de compatibilidade.
- **Kotlin:** Para novos projetos ou quando quiser aproveitar os recursos modernos da linguagem.

**Sintaxe:**

```bash
flutter create --android-language=<kotlin|java> <project-name>
```

**Exemplo:**

```bash
flutter create --android-language=java my_flutter_app
```

---

### `--description`

**Descrição:** Define uma **descrição** para o projeto. Esta descrição será armazenada no arquivo `pubspec.yaml` e pode ser usada para explicar o propósito do projeto.

**Quando utilizar?**

- **Documentação:** Para criar uma descrição clara do propósito do projeto, o que é importante para a colaboração e publicação em repositórios como o pub.dev.

**Sintaxe:**

```bash
flutter create --description="<description>" <project-name>
```

**Exemplo:**

```bash
flutter create --description="Um aplicativo de exemplo Flutter" my_flutter_app
```

---

### `--no-pub`

**Descrição:** Cria o projeto sem rodar o comando `flutter pub get`, que baixa as dependências do projeto.

**Quando utilizar?**

- **Ambientes Offline:** Quando você não tem acesso à internet ou deseja adiar o download de pacotes até um momento posterior.

**Sintaxe:**

```bash
flutter create --no-pub <project-name>
```

---

### `--offline`

**Descrição:** Executa o comando `flutter pub get` em **modo offline**. Útil quando você já tem as dependências baixadas localmente.

**Quando utilizar?**

- **Ambientes sem internet:** Quando não há conexão com a internet e as dependências já estão no cache local.

**Sintaxe:**

```bash
flutter create --offline <project-name>
```

---

### `--package`

**Descrição:** Especifica o nome do pacote para o projeto gerado, afetando o identificador do projeto em Android e iOS.

**Quando utilizar?**

- **Personalização do Pacote:** Quando você quer que o nome do pacote seja diferente do nome do projeto.

**Sintaxe:**

```bash
flutter create --package=<package-name> <project-name>
```

**Exemplo:**

```bash
flutter create --package=com.example.myapp my_flutter_app
```

---

### `--overwrite`

**Descrição:** Permite sobrescrever arquivos existentes na criação do projeto.

**Quando utilizar?**

- **Atualização de Projetos Existentes:** Quando você quer recriar a estrutura do projeto sem perder tempo apagando arquivos manualmente.

**Sintaxe:**

```bash
flutter create --overwrite <project-name>
```

---

## Sintaxe de Uso

A sintaxe básica para criar um projeto Flutter é:

```bash
flutter create [options] <project-name>
```

Exemplo básico:

```bash
flutter create my_flutter_app
```

Exemplo com parâmetros:

```bash
flutter create --org=com.mycompany --platforms=android,ios --ios-language=swift --android-language=kotlin my_flutter_app
```

---

## Restrições de Uso

- **Nome do projeto:** Deve seguir as regras de nomenclatura de pacotes do Dart (letras minúsculas, sem espaços ou caracteres especiais).
- **Compatibilidade de plataformas:** Nem todas

 as plataformas são suportadas por todos os templates (ex.: o template de plugin pode não ser usado com plataformas web).
- **Opções conflitantes:** Certos parâmetros, como `--no-pub` e `--offline`, podem resultar em erros se usados de forma incorreta ou desnecessária.

---

## Informações Adicionais

- **Pubspec.yaml:** O arquivo `pubspec.yaml` é gerado com as dependências e a configuração inicial do projeto.
- **Evolução do Projeto:** O Flutter CLI facilita a adição de novas plataformas no futuro, usando o comando `flutter create .` dentro do projeto existente para gerar os diretórios faltantes.
- **Boas Práticas:** Nomear o projeto e o pacote de forma consistente e clara é importante, principalmente para evitar conflitos de pacotes e facilitar o gerenciamento em lojas de aplicativos.

---

## Conclusão

O comando `flutter create` é o ponto de partida para qualquer desenvolvedor Flutter. Ele fornece uma maneira flexível de criar projetos com uma variedade de configurações e personalizações. Ao utilizar as diversas opções disponíveis, você pode ajustar a estrutura do projeto de acordo com as necessidades específicas, garantindo que esteja pronto para começar o desenvolvimento com as plataformas e linguagens corretas desde o início.

Com esse conhecimento, você pode iniciar novos projetos Flutter com confiança, adaptando a criação do projeto às suas necessidades específicas de desenvolvimento.
