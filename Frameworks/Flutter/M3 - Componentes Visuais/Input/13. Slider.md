## O que é e para que serve?

### O que é Slider?

O **Slider** é um widget do Flutter que permite aos usuários selecionar um valor dentro de um intervalo contínuo deslizando um "ponto de controle" (thumb) ao longo de uma linha horizontal ou vertical. Ele é altamente personalizável e pode ser integrado em diversas interfaces para proporcionar uma interação intuitiva e responsiva.

### Para que serve?

Os sliders são utilizados para:

- **Ajuste de configurações**: Como volume, brilho ou velocidade.
- **Seleção de intervalos**: Como faixas de preços ou tempo.
- **Feedback visual**: Mostrando progresso ou níveis.
- **Interatividade**: Permitir que os usuários modifiquem valores em tempo real.

## Como funciona?

O **Slider** funciona monitorando a interação do usuário através de gestos de arrastar. Conforme o usuário move o thumb, o valor associado ao slider é atualizado e pode ser usado para modificar estados ou executar ações dentro do aplicativo. O widget também oferece feedback visual através da linha track, que pode indicar progresso ou o intervalo de seleção.

### Componentes Principais

- **Thumb**: O ponto de controle que o usuário arrasta.
- **Track**: A linha que representa o intervalo de valores.
- **Dividers**: Linhas secundárias que podem indicar divisões no intervalo.

## Sintaxe de uso

A utilização do **Slider** no Flutter envolve a criação de um widget com uma série de parâmetros que definem seu comportamento e aparência. Abaixo, apresentamos a sintaxe básica seguida de uma descrição detalhada de cada parâmetro.

### Sintaxe Básica

```dart
Slider({
  Key? key,
  required double value,
  required ValueChanged<double> onChanged,
  double min = 0.0,
  double max = 100.0,
  int? divisions,
  String? label,
  Color? activeColor,
  Color? inactiveColor,
  SemanticFormatterCallback? semanticFormatterCallback,
  MouseCursor? mouseCursor,
  bool autofocus = false,
  FocusNode? focusNode,
})
```

### Descrição dos Parâmetros

#### Parâmetros Obrigatórios

- **`value`** (`double`): O valor atual do slider. Deve estar dentro do intervalo definido por `min` e `max`.
    
    - **Tipo Esperado**: `double`
    - **Obrigatório**: Sim
    - **Descrição**: Representa a posição atual do thumb no slider.
- **`onChanged`** (`ValueChanged<double>`): Callback que é chamado quando o valor do slider muda.
    
    - **Tipo Esperado**: `ValueChanged<double>`
    - **Obrigatório**: Sim
    - **Descrição**: Função que recebe o novo valor do slider sempre que o usuário interage com ele.

#### Parâmetros Opcionais

- **`key`** (`Key?`): Chave para identificar o widget na árvore de widgets.
    
    - **Tipo Esperado**: `Key?`
    - **Obrigatório**: Não
    - **Descrição**: Útil para otimizações e gerenciamento de estado.
- **`min`** (`double`): Valor mínimo do slider.
    
    - **Tipo Esperado**: `double`
    - **Obrigatório**: Não
    - **Padrão**: `0.0`
    - **Descrição**: Define o limite inferior do intervalo de valores.
- **`max`** (`double`): Valor máximo do slider.
    
    - **Tipo Esperado**: `double`
    - **Obrigatório**: Não
    - **Padrão**: `100.0`
    - **Descrição**: Define o limite superior do intervalo de valores.
- **`divisions`** (`int?`): Número de divisões discretas no slider.
    
    - **Tipo Esperado**: `int?`
    - **Obrigatório**: Não
    - **Descrição**: Se definido, o slider terá um número fixo de posições discretas.
- **`label`** (`String?`): Rótulo que aparece quando o thumb é arrastado.
    
    - **Tipo Esperado**: `String?`
    - **Obrigatório**: Não
    - **Descrição**: Pode ser usado para exibir o valor atual ou outra informação relevante.
- **`activeColor`** (`Color?`): Cor da parte ativa do track (até o thumb).
    
    - **Tipo Esperado**: `Color?`
    - **Obrigatório**: Não
    - **Descrição**: Personaliza a cor da parte do track que representa o valor atual.
- **`inactiveColor`** (`Color?`): Cor da parte inativa do track (após o thumb).
    
    - **Tipo Esperado**: `Color?`
    - **Obrigatório**: Não
    - **Descrição**: Personaliza a cor da parte do track que não representa o valor atual.
- **`semanticFormatterCallback`** (`SemanticFormatterCallback?`): Callback para formatar a mensagem semântica.
    
    - **Tipo Esperado**: `SemanticFormatterCallback?`
    - **Obrigatório**: Não
    - **Descrição**: Utilizado para acessibilidade, fornecendo uma descrição personalizada do valor atual.
- **`mouseCursor`** (`MouseCursor?`): Cursor do mouse quando pairado sobre o slider.
    
    - **Tipo Esperado**: `MouseCursor?`
    - **Obrigatório**: Não
    - **Descrição**: Personaliza o cursor exibido sobre o widget em dispositivos que suportam mouse.
- **`autofocus`** (`bool`): Se o slider deve receber foco automaticamente.
    
    - **Tipo Esperado**: `bool`
    - **Obrigatório**: Não
    - **Padrão**: `false`
    - **Descrição**: Define se o slider deve ganhar foco automaticamente ao ser exibido.
- **`focusNode`** (`FocusNode?`): Controla o foco do widget.
    
    - **Tipo Esperado**: `FocusNode?`
    - **Obrigatório**: Não
    - **Descrição**: Permite o controle manual do foco, útil para navegação por teclado.

### Exemplo de Uso

```dart
double _currentSliderValue = 20;

Slider(
  value: _currentSliderValue,
  min: 0,
  max: 100,
  divisions: 5,
  label: _currentSliderValue.round().toString(),
  onChanged: (double value) {
    setState(() {
      _currentSliderValue = value;
    });
  },
)
```

## Restrições de uso

- **Valores Fora do Intervalo**: O valor passado para o `value` deve estar dentro do intervalo definido por `min` e `max`. Valores fora desse intervalo podem causar comportamentos inesperados.
- **Interação Limitada**: Em contextos onde a precisão é crítica, como seleção de valores monetários exatos, sliders podem não ser a melhor escolha devido à natureza contínua da seleção.
- **Acessibilidade**: Deve-se garantir que o slider seja acessível, utilizando rótulos apropriados e callbacks semânticos para suportar leitores de tela.
- **Desempenho**: Sliders com muitas divisões ou animações complexas podem impactar o desempenho em dispositivos menos potentes.

## Quando utilizar?

Os sliders são ideais em situações onde:

- **Seleção Contínua**: Quando o usuário precisa selecionar um valor dentro de um intervalo contínuo, como volume ou brilho.
- **Ajustes Dinâmicos**: Permite ajustes em tempo real que afetam outros componentes da interface.
- **Feedback Visual**: Quando se deseja mostrar visualmente o progresso ou nível atual de uma métrica.
- **Estética e Interatividade**: Quando se busca uma interface interativa e esteticamente agradável.

### Exemplos de Uso

- **Configurações de Áudio**: Ajustar o volume do aplicativo.
- **Edição de Imagens**: Controlar o brilho ou contraste.
- **Jogos**: Selecionar a dificuldade ou velocidade.
- **Formulários**: Definir a faixa de preço em uma pesquisa.

## Exemplos de Código

### Exemplo 1: Slider Básico

```dart
import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatefulWidget {
  @override
  _MyAppState createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  double _currentSliderValue = 20;

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Exemplo de Slider',
      home: Scaffold(
        appBar: AppBar(
          title: Text('Slider Básico'),
        ),
        body: Center(
          child: Slider(
            value: _currentSliderValue,
            min: 0,
            max: 100,
            divisions: 5,
            label: _currentSliderValue.round().toString(),
            onChanged: (double value) {
              setState(() {
                _currentSliderValue = value;
              });
            },
          ),
        ),
      ),
    );
  }
}
```

### Exemplo 3: Slider com Semântica para Acessibilidade

```dart
import 'package:flutter/material.dart';

void main() => runApp(MyApp());

class MyApp extends StatefulWidget {
  @override
  _MyAppState createState() => _MyAppState();
}

class _MyAppState extends State<MyApp> {
  double _volume = 30;

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Slider com Acessibilidade',
      home: Scaffold(
        appBar: AppBar(
          title: Text('Slider com Acessibilidade'),
        ),
        body: Center(
          child: Slider(
            value: _volume,
            min: 0,
            max: 100,
            divisions: 10,
            label: 'Volume: ${_volume.round()}%',
            activeColor: Colors.blue,
            inactiveColor: Colors.blue[100],
            semanticFormatterCallback: (double newValue) {
              return 'Volume ajustado para ${newValue.round()}%';
            },
            onChanged: (double value) {
              setState(() {
                _volume = value;
              });
            },
          ),
        ),
      ),
    );
  }
}
```

## Considerações Finais

O **Slider** é um widget poderoso e versátil no Flutter, oferecendo uma maneira intuitiva para os usuários interagirem com valores contínuos. Sua flexibilidade permite uma ampla gama de personalizações, tornando-o adequado para diversas aplicações. Ao implementar sliders, é essencial considerar a usabilidade e acessibilidade para garantir que todos os usuários possam interagir eficazmente com a interface.

### Boas Práticas

- **Limitar o Número de Divisões**: Evite sliders com muitas divisões, que podem dificultar a seleção precisa.
- **Fornecer Feedback Visual**: Utilize rótulos e cores para indicar claramente o valor atual.
- **Acessibilidade**: Utilize callbacks semânticos e rótulos claros para suportar leitores de tela.
- **Consistência de Design**: Mantenha a aparência dos sliders consistente com o restante da interface do aplicativo.
