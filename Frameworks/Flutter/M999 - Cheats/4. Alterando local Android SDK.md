## Passo 1: Verifique a Instalação do Android SDK

Antes de alterar a pasta do SDK, é importante garantir que o Android SDK esteja instalado corretamente no seu sistema.

- **Android Studio:** O Android SDK é geralmente instalado junto com o Android Studio. Para verificar o caminho do SDK, abra o Android Studio, vá para:

  **File > Settings > Appearance & Behavior > System Settings > Android SDK** (em Windows e Linux)  
  **Android Studio > Preferences > Appearance & Behavior > System Settings > Android SDK** (em macOS)

- **Verifique o caminho do SDK atual:** Anote o caminho do SDK que será necessário para ajustar a variável de ambiente no próximo passo.

---

## Passo 2: Altere o Caminho do SDK no Sistema Operacional

Dependendo do seu sistema operacional, você precisará alterar as variáveis de ambiente para apontar para o novo caminho do SDK do Android.

### Windows

1. **Abra o Painel de Controle** e vá para **Sistema e Segurança > Sistema**.
2. Clique em **Configurações Avançadas do Sistema** e depois em **Variáveis de Ambiente**.
3. Na seção **Variáveis do Sistema**, encontre as variáveis `ANDROID_HOME` e `ANDROID_SDK_ROOT` (se não existirem, crie-as).
4. Edite ou adicione essas variáveis apontando para o novo caminho do Android SDK:

    - Nome: `ANDROID_SDK_HOME`
    - Valor: `C:\caminho\para\o\android-sdk`

    - Nome: `ANDROID_SDK_ROOT`
    - Valor: `C:\caminho\para\o\android-sdk`

5. Adicione também o caminho para a pasta `platform-tools` e `tools` do SDK à variável `Path`:

    - `%ANDROID_HOME%\platform-tools`
    - `%ANDROID_HOME%\tools`

6. Clique em **OK** para salvar as alterações.

### macOS e Linux

1. Abra o arquivo de configuração do terminal:

    - **macOS:** Edite o arquivo `~/.bash_profile` (ou `~/.zshrc` se estiver usando `zsh`).
    - **Linux:** Edite o arquivo `~/.bashrc` ou `~/.bash_profile`.

2. Adicione ou edite as seguintes linhas para apontar para o novo caminho do Android SDK:

    ```bash
    export ANDROID_HOME=/caminho/para/o/android-sdk
    export ANDROID_SDK_ROOT=/caminho/para/o/android-sdk
    export PATH=$PATH:$ANDROID_HOME/platform-tools
    export PATH=$PATH:$ANDROID_HOME/tools
    ```

3. Salve o arquivo e recarregue o terminal com o comando:

    ```bash
    source ~/.bash_profile
    ```

## Passo 3: Referenciar o SDK no Flutter

Depois de definir o caminho do SDK do Android no sistema, o próximo passo é garantir que o Flutter esteja ciente da nova localização.

1. **Verificar o Caminho do Android SDK no Flutter:**

    Use o comando:

    ```bash
    flutter config --android-sdk /caminho/para/o/android-sdk
    ```

    Esse comando irá configurar o Flutter para usar o novo caminho do Android SDK. Por exemplo:

    ```bash
    flutter config --android-sdk C:\caminho\para\o\android-sdk
    ```

    ou

    ```bash
    flutter config --android-sdk /caminho/para/o/android-sdk
    ```

2. **Verificar se a configuração foi aplicada corretamente:**

    Após alterar o caminho do SDK, execute o comando:

    ```bash
    flutter doctor
    ```

    Isso exibirá as informações do SDK Android configurado e confirmará se o Flutter reconhece o novo caminho.

---

## Passo 4: Verificar a Configuração no Flutter

Para garantir que a configuração foi realizada corretamente, execute o comando `flutter doctor` novamente:

```bash
flutter doctor
```

- Verifique a seção **[✓] Android toolchain**.
- Se estiver tudo correto, você verá a mensagem de que o SDK do Android foi encontrado no novo caminho.
- Caso o Flutter não consiga localizar o SDK, reveja as etapas de configuração do caminho.

## Restrições de Uso

- **Compatibilidade de Versão:** Certifique-se de que o SDK do Android é compatível com a versão do Flutter que você está utilizando.
- **Permissões de Pasta:** O Flutter precisa ter permissões adequadas para acessar o caminho do SDK do Android. Verifique as permissões de leitura e execução, especialmente em sistemas Unix-like (Linux/macOS).
- **Atualizações de SDK:** Quando o Android Studio for atualizado ou quando você instalar novas versões do SDK, pode ser necessário ajustar o caminho ou baixar novos componentes no SDK Manager.
