## Introdução

Ao desenvolver aplicativos móveis com Flutter, uma das primeiras experiências visuais que o usuário tem ao abrir o aplicativo é a SplashScreen. A SplashScreen é uma tela de carregamento que aparece enquanto o aplicativo está inicializando. No Flutter, a SplashScreen nativa do Android pode ser personalizada para melhorar a experiência do usuário e alinhar o design do aplicativo com a identidade visual desejada.

Neste guia, exploraremos detalhadamente como alterar a SplashScreen nativa do Android em um projeto Flutter, abordando desde os conceitos básicos até a implementação prática com exemplos de código.

## O que é a SplashScreen Nativa do Android no Flutter?

A **SplashScreen Nativa** no contexto do Flutter refere-se à tela de carregamento que é exibida imediatamente após o lançamento do aplicativo, antes que o framework Flutter seja inicializado. Essa tela é implementada diretamente no código nativo do Android, garantindo que o usuário veja uma interface visualmente agradável enquanto o aplicativo está carregando os recursos necessários.

## Para que Serve a SplashScreen?

A SplashScreen tem várias finalidades importantes:

- **Melhorar a Experiência do Usuário**: Proporciona uma transição suave entre o lançamento do aplicativo e a interface principal.
- **Ocultar o Carregamento do App**: Esconde o processo de inicialização, que pode levar alguns segundos, evitando que o usuário veja uma tela em branco.
- **Refletir a Identidade Visual**: Exibe o logotipo ou elementos de marca do aplicativo, reforçando a identidade visual da aplicação.
- **Preparar Recursos**: Permite que o aplicativo carregue recursos essenciais em segundo plano durante a exibição da SplashScreen.

## Como Alterar a SplashScreen Nativa no Android no Flutter

Alterar a SplashScreen nativa do Android em um projeto Flutter envolve a modificação de arquivos específicos no diretório Android do seu projeto Flutter. A seguir, apresentamos um passo a passo detalhado para realizar essa alteração.

### Passo a Passo Detalhado

1. **Abrir o Projeto Flutter no Editor de Código**: Utilize um editor como VS Code ou Android Studio.

2. **Navegar até o Diretório Android**: No seu projeto Flutter, localize e abra a pasta `android`.

3. **Modificar o Arquivo `styles.xml`**:
    - Caminho: `android/app/src/main/res/values/styles.xml`
    - Este arquivo define os estilos utilizados na aplicação Android, incluindo a SplashScreen.

4. **Definir o Tema da SplashScreen**:
    - Crie ou modifique um estilo específico para a SplashScreen.
    - Geralmente, um tema baseado em `Theme.AppCompat.NoActionBar` ou `Theme.MaterialComponents` é usado.

5. **Adicionar um Drawable para a SplashScreen**:
    - Crie um arquivo drawable que define a aparência da SplashScreen, como cor de fundo e logotipo.
    - Caminho sugerido: `android/app/src/main/res/drawable/splash_screen.xml`

6. **Atualizar o `AndroidManifest.xml`**:
    - Caminho: `android/app/src/main/AndroidManifest.xml`
    - Defina o tema da SplashScreen para a `MainActivity`.

7. **Adicionar o Logotipo ou Imagem**:
    - Coloque a imagem que será exibida na SplashScreen no diretório de recursos adequado, como `drawable`.

### Modificação de Arquivos Android

#### 1. Modificando o `styles.xml`

```xml
<!-- android/app/src/main/res/values/styles.xml -->
<resources>
    <!-- Tema padrão do aplicativo -->
    <style name="AppTheme" parent="Theme.MaterialComponents.Light.NoActionBar">
        <!-- Customize your theme here -->
        <item name="colorPrimary">#6200EE</item>
        <item name="colorPrimaryVariant">#3700B3</item>
        <item name="colorOnPrimary">#FFFFFF</item>
    </style>

    <!-- Tema para a SplashScreen -->
    <style name="SplashTheme" parent="Theme.MaterialComponents.Light.NoActionBar">
        <item name="android:windowBackground">@drawable/splash_screen</item>
    </style>
</resources>
```

#### 2. Criando o `splash_screen.xml`

```xml
<!-- android/app/src/main/res/drawable/splash_screen.xml -->
<layer-list xmlns:android="http://schemas.android.com/apk/res/android">
    <!-- Fundo da SplashScreen -->
    <item android:drawable="@color/white" />

    <!-- Logotipo do aplicativo -->
    <item>
        <bitmap
            android:gravity="center"
            android:src="@drawable/logo" />
    </item>
</layer-list>
```

> **Observação**: Substitua `@drawable/logo` pelo caminho da sua imagem de logotipo.

#### 3. Atualizando o `AndroidManifest.xml`

```xml
<!-- android/app/src/main/AndroidManifest.xml -->
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.exemplo.app">

    <application
        android:name="${applicationName}"
        android:label="App"
        android:icon="@mipmap/ic_launcher"
        android:theme="@style/SplashTheme">
        
        <activity
            android:name=".MainActivity"
            android:exported="true"
            android:theme="@style/SplashTheme"
            android:launchMode="singleTop"
            android:configChanges="orientation|keyboardHidden|keyboard|screenSize|locale|layoutDirection|fontScale|screenLayout|density|uiMode">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
    </application>

</manifest>
```

#### 4. Alterando o `MainActivity.java` ou `MainActivity.kt`

Para garantir que o tema principal seja aplicado após a SplashScreen, é necessário atualizar o tema na `MainActivity`.

**Em Java:**

```java
// android/app/src/main/java/com/exemplo/app/MainActivity.java
package com.exemplo.app;

import android.os.Bundle;
import io.flutter.embedding.android.FlutterActivity;

public class MainActivity extends FlutterActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        // Define o tema principal após a SplashScreen
        setTheme(R.style.AppTheme);
        super.onCreate(savedInstanceState);
    }
}
```

**Em Kotlin:**

```kotlin
// android/app/src/main/java/com/exemplo/app/MainActivity.kt
package com.exemplo.app

import android.os.Bundle
import io.flutter.embedding.android.FlutterActivity

class MainActivity: FlutterActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        // Define o tema principal após a SplashScreen
        setTheme(R.style.AppTheme)
        super.onCreate(savedInstanceState)
    }
}
```

## Sintaxe de Uso

A personalização da SplashScreen envolve a definição de estilos e recursos drawable no Android. Vamos detalhar os principais parâmetros e subparâmetros utilizados.

### Parâmetros e Subparâmetros

1. **`styles.xml`**:
    - **`<style>`**: Define um estilo ou tema.
        - **`name`**: Nome do estilo (ex.: `SplashTheme`).
        - **`parent`**: Tema base (ex.: `Theme.MaterialComponents.Light.NoActionBar`).
        - **`<item>`**: Define propriedades específicas do tema.
            - **`android:windowBackground`**: Define o background da janela, que pode ser um drawable.

2. **`splash_screen.xml`**:
    - **`<layer-list>`**: Define uma lista de camadas de drawables.
        - **`<item>`**: Cada item representa uma camada.
            - **`android:drawable`**: Define o drawable da camada (ex.: cor de fundo).
            - **`<bitmap>`**: Exibe uma imagem bitmap.
                - **`android:gravity`**: Define a posição da imagem (ex.: `center`).
                - **`android:src`**: Define a fonte da imagem (ex.: `@drawable/logo`).

3. **`AndroidManifest.xml`**:
    - **`<application>`**:
        - **`android:theme`**: Define o tema da aplicação (ex.: `@style/SplashTheme`).
    - **`<activity>`**:
        - **`android:theme`**: Define o tema específico da atividade.
        - **`android:launchMode`**: Define o modo de lançamento da atividade.
        - **`android:configChanges`**: Define as mudanças de configuração que a atividade pode manejar.

4. **`MainActivity.java` ou `MainActivity.kt`**:
    - **`setTheme(R.style.AppTheme)`**: Define o tema principal após a SplashScreen.

## Restrições de Uso

Ao personalizar a SplashScreen nativa do Android no Flutter, é importante considerar as seguintes restrições:

- **Consistência de Temas**: Alterações nos temas podem afetar outras partes da aplicação. Garanta que o tema da SplashScreen seja compatível com o tema principal.
- **Desempenho**: Evite imagens de alta resolução que possam aumentar o tempo de carregamento da SplashScreen.
- **Compatibilidade de Versão**: Certifique-se de que as alterações são compatíveis com as versões mínimas e alvo do Android especificadas no projeto Flutter.
- **Manutenção de Recursos**: Mantenha os recursos (como imagens) organizados e otimizados para diferentes densidades de tela (`drawable-mdpi`, `drawable-hdpi`, etc.).

## Quando Utilizar a SplashScreen Nativa?

A SplashScreen nativa deve ser utilizada nas seguintes situações:

- **Carregamento Inicial**: Quando o aplicativo requer tempo para inicializar, carregar recursos ou conectar-se a serviços.
- **Identidade Visual**: Quando se deseja exibir o logotipo ou elementos de marca imediatamente após o lançamento do aplicativo.
- **Melhoria da Experiência do Usuário**: Para evitar que o usuário veja uma tela em branco durante o carregamento.
- **Implementação de Transições Personalizadas**: Quando se quer criar uma transição suave entre a SplashScreen e a interface principal do aplicativo.

## Tópicos Adicionais Importantes

### Utilizando o Plugin `flutter_native_splash`

Embora este guia foque na personalização manual da SplashScreen, o plugin [`flutter_native_splash`](https://pub.dev/packages/flutter_native_splash) pode simplificar o processo, automatizando a geração e configuração da SplashScreen para Android e iOS.

**Exemplo de Configuração no `pubspec.yaml`:**

```yaml
flutter_native_splash:
  color: "#FFFFFF"
  image: assets/images/logo_app.png
  android: true
  ios: true
```

**Comandos para Aplicar:**

```bash
flutter pub get
flutter pub run flutter_native_splash:create
```

### Dicas de Design para SplashScreen

- **Simplicidade**: Mantenha a SplashScreen simples, evitando elementos excessivos que possam aumentar o tempo de carregamento.
- **Consistência de Marca**: Utilize cores e logotipos que estejam alinhados com a identidade visual do aplicativo.
- **Transições Suaves**: Implemente animações leves para transições entre a SplashScreen e a interface principal.
- **Compatibilidade de Tamanho**: Garanta que os elementos visuais estejam adequadamente dimensionados para diferentes tamanhos e densidades de tela.

### Otimização de Recursos

- **Compressão de Imagens**: Utilize formatos otimizados (como PNG ou WebP) e comprima as imagens para reduzir o tamanho do APK.
- **Diferentes Densidades de Tela**: Forneça versões da imagem de logotipo para diferentes densidades (`mdpi`, `hdpi`, `xhdpi`, etc.) para garantir uma aparência nítida em todos os dispositivos.

### Compatibilidade com iOS

Se o seu aplicativo Flutter também é destinado a dispositivos iOS, considere personalizar a SplashScreen nativa do iOS para manter a consistência visual entre plataformas. O processo é semelhante, mas envolve a modificação de arquivos no diretório `ios`.

## Conclusão

A personalização da SplashScreen nativa do Android em projetos Flutter é uma etapa crucial para oferecer uma experiência de usuário aprimorada e refletir a identidade visual do aplicativo. Embora o processo envolva a modificação de arquivos nativos do Android, seguindo este guia detalhado você poderá implementar uma SplashScreen personalizada de forma eficiente.

Independentemente da abordagem escolhida, uma SplashScreen bem implementada contribui significativamente para a percepção inicial do usuário sobre a qualidade e profissionalismo do aplicativo.

## O que é e para que serve a SplashScreenNativa no iOS?

A **SplashScreenNativa** no iOS refere-se à tela de lançamento que é exibida nativamente pelo sistema operacional quando o aplicativo está iniciando. Ela serve para:

- **Melhorar a experiência do usuário**: Oferece uma transição visualmente agradável enquanto o aplicativo carrega.
- **Ocultar processos de inicialização**: Disfarça o tempo necessário para carregar recursos e inicializar componentes.
- **Branding**: Exibe a identidade visual da marca, fortalecendo o reconhecimento da aplicação.

No Flutter, embora seja possível criar uma Splash Screen personalizada usando widgets, configurar a SplashScreenNativa garante que a tela de lançamento apareça imediatamente, sem depender da renderização do Flutter, proporcionando uma experiência mais fluida.

## Como alterar a SplashScreenNativa do iOS no Flutter

Alterar a SplashScreenNativa do iOS em um projeto Flutter envolve várias etapas, desde a configuração de arquivos de storyboard até a utilização de pacotes que automatizam parte do processo. A seguir, detalharemos cada passo.

### Preparação do Projeto

Antes de iniciar, certifique-se de que seu ambiente de desenvolvimento Flutter está corretamente configurado e que você possui um projeto Flutter válido.

1. **Atualize o Flutter**: Garanta que você está usando a versão mais recente do Flutter.
    ```bash
    flutter upgrade
    ```

2. **Abra o Projeto no Xcode**: Algumas alterações requerem o uso do Xcode.
    ```bash
    open ios/Runner.xcworkspace
    ```

### Configuração dos Arquivos de Storyboard

A Splash Screen no iOS é geralmente configurada utilizando um arquivo **Storyboard**. No Flutter, o arquivo padrão é `LaunchScreen.storyboard`.

1. **Localização do Storyboard**:
    - Navegue até `ios/Runner/Base.lproj/LaunchScreen.storyboard`.

2. **Editar o Storyboard**:
    - Abra o arquivo no Xcode.
    - Utilize o Interface Builder para adicionar ou modificar elementos visuais, como imagens, labels ou cores de fundo.

### Modificação das Imagens e Recursos Gráficos

Para personalizar a Splash Screen, você pode adicionar imagens personalizadas.

1. **Adicionar Imagens ao Projeto**:
    - Coloque as imagens desejadas na pasta `ios/Runner/Assets.xcassets`.

2. **Referenciar as Imagens no Storyboard**:
    - No Interface Builder, adicione um **Image View** e selecione a imagem adicionada aos assets.

### Alteração do Arquivo Info.plist

O arquivo `Info.plist` contém configurações do aplicativo. Embora não seja sempre necessário, em alguns casos, ajustes podem ser feitos para personalizar a Splash Screen.

1. **Localização do Info.plist**:
    - Navegue até `ios/Runner/Info.plist`.

2. **Modificar Propriedades (se necessário)**:
    - Por exemplo, alterar o nome da aplicação ou ícones pode afetar a Splash Screen.
    - Exemplo de entrada:
        ```xml
        <key>UILaunchStoryboardName</key>
        <string>LaunchScreen</string>
        ```

### Uso de Pacotes: `flutter_native_splash`

O pacote [`flutter_native_splash`](https://pub.dev/packages/flutter_native_splash) simplifica o processo de configuração das Splash Screens para iOS e Android. A seguir, como utilizá-lo.

1. **Adicionar o Pacote ao Projeto**:
    - No arquivo `pubspec.yaml`, adicione:
        ```yaml
        dependencies:
          flutter_native_splash: ^2.2.16
        ```

    - Execute:
        ```bash
        flutter pub get
        ```

2. **Configurar o Pacote**:
    - No mesmo `pubspec.yaml`, configure a seção `flutter_native_splash`:
        ```yaml
        flutter_native_splash:
          color: "#FFFFFF" # Cor de fundo
          image: assets/splash.png # Caminho da imagem de splash
          ios: true
          android: true
          # Configurações específicas para iOS
          ios_content_mode: scaleAspectFill
          # Outros parâmetros opcionais
        ```

3. **Gerar as Splash Screens**:
    - Execute o comando:
        ```bash
        flutter pub run flutter_native_splash:create
        ```

    - Isso irá gerar e configurar automaticamente as Splash Screens para as plataformas especificadas.

### Exemplo Prático de Implementação

Vamos realizar um exemplo completo utilizando o pacote `flutter_native_splash`.

1. **Adicionar a Imagem de Splash**:
    - Coloque a imagem de splash (`splash.png`) na pasta `assets/`.

2. **Configurar o `pubspec.yaml`**:
    ```yaml
    flutter:
      assets:
        - assets/splash.png

    flutter_native_splash:
      color: "#42a5f5" # Azul como cor de fundo
      image: assets/splash.png
      ios_content_mode: scaleAspectFit
      android_gravity: center
      # Opcional: definir a duração da splash screen no Android
      android_delay: 2000
    ```

3. **Gerar a Splash Screen**:
    - Execute:
        ```bash
        flutter pub run flutter_native_splash:create
        ```

4. **Verificar as Alterações**:
    - Abra o Xcode e verifique se o `LaunchScreen.storyboard` foi atualizado com a nova imagem e cor de fundo.

## Sintaxe de Uso: Parâmetros e Subparâmetros

Ao utilizar o pacote `flutter_native_splash`, você pode configurar diversas opções para personalizar sua Splash Screen. Abaixo estão os principais parâmetros e seus subparâmetros:

```yaml
flutter_native_splash:
  color: "#FFFFFF" # Cor de fundo da splash screen
  image: "assets/splash.png" # Caminho da imagem de splash
  image_dark: "assets/splash_dark.png" # Imagem para modo escuro (opcional)
  web: false # Habilita a splash screen para web
  android: true # Habilita a splash screen para Android
  ios: true # Habilita a splash screen para iOS
  ios_content_mode: scaleAspectFill # Modo de conteúdo para iOS (scaleAspectFit, scaleAspectFill, center)
  android_gravity: center # Posição da imagem no Android (center, top, bottom, etc.)
  android_scale_type: fitXY # Tipo de escala para Android (fitXY, centerCrop, etc.)
  android_delay: 2000 # Tempo em milissegundos para a splash screen no Android
  # Outros parâmetros opcionais
```

### Descrição dos Parâmetros

- **color**: Define a cor de fundo da Splash Screen. Deve ser uma string hexadecimal.
  
- **image**: Caminho para a imagem utilizada na Splash Screen. Deve estar na pasta de assets do projeto.

- **image_dark**: Define uma imagem específica para o modo escuro do iOS. Opcional.

- **web**: Habilita ou desabilita a Splash Screen para a plataforma web. Valor booleano.

- **android**: Habilita ou desabilita a Splash Screen para Android. Valor booleano.

- **ios**: Habilita ou desabilita a Splash Screen para iOS. Valor booleano.

- **ios_content_mode**: Define como a imagem será exibida na tela. Opções comuns:
  - `scaleAspectFit`: Ajusta a imagem para caber na tela mantendo a proporção.
  - `scaleAspectFill`: Preenche a tela com a imagem, podendo cortar partes.
  - `center`: Centraliza a imagem sem redimensionamento.

- **android_gravity**: Define a posição da imagem na tela do Android. Exemplos:
  - `center`
  - `top`
  - `bottom`
  - `fill`

- **android_scale_type**: Define como a imagem será escalada no Android. Opções:
  - `fitXY`: Escala a imagem para preencher exatamente as dimensões.
  - `centerCrop`: Escala a imagem mantendo a proporção e cortando o excesso.

- **android_delay**: Tempo, em milissegundos, que a Splash Screen permanecerá visível no Android.

## Restrições de Uso

Embora a Splash Screen seja uma ferramenta poderosa para melhorar a experiência do usuário, existem algumas restrições e boas práticas a serem observadas:

1. **Tempo de Exibição**: A Splash Screen deve ser exibida apenas enquanto o aplicativo está carregando. Evite prolongar desnecessariamente a exibição para não frustrar o usuário.

2. **Consistência Visual**: A aparência da Splash Screen deve estar alinhada com o design geral do aplicativo e da marca.

3. **Resolução das Imagens**: Utilize imagens com resolução adequada para evitar pixelização em diferentes dispositivos.

4. **Compatibilidade com Modo Escuro**: Considere fornecer variações da Splash Screen para modos claros e escuros, garantindo boa visibilidade em ambos.

5. **Performance**: Evite adicionar elementos complexos na Splash Screen que possam impactar o tempo de carregamento do aplicativo.

6. **Testes em Diferentes Dispositivos**: Verifique a aparência da Splash Screen em diferentes tamanhos de tela e versões do iOS para garantir compatibilidade.

## Quando Utilizar a SplashScreenNativa?

A SplashScreenNativa deve ser utilizada nas seguintes situações:

1. **Melhoria da Experiência do Usuário**: Proporciona uma transição visualmente agradável durante o carregamento do aplicativo.

2. **Branding**: Exibe elementos da identidade visual da marca, reforçando o reconhecimento.

3. **Ocultação de Processos de Inicialização**: Disfarça o tempo necessário para carregar recursos, evitando que o usuário veja telas em branco ou carregamento abrupto.

4. **Primeira Impressão**: Contribui para a impressão inicial do usuário sobre a qualidade e profissionalismo do aplicativo.
