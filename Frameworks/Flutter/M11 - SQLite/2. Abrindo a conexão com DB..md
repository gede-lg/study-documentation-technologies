# Introdução ao SQLite no Flutter

O Flutter é um framework de desenvolvimento multiplataforma que permite criar aplicativos nativos para iOS e Android a partir de um único código-fonte. Para aplicativos que exigem armazenamento de dados local, o SQLite é uma escolha popular devido à sua leveza e facilidade de uso.

O SQLite é um banco de dados relacional que não requer um servidor separado para operar. Isso o torna ideal para aplicativos móveis que precisam armazenar dados localmente no dispositivo do usuário.

# Configuração do Ambiente

Antes de começar a trabalhar com SQLite no Flutter, é necessário configurar o ambiente:

1. **Adicionar Dependência**: No arquivo `pubspec.yaml` do seu projeto Flutter, adicione a dependência do `sqflite` e `path`.

```yaml
dependencies:
  flutter:
    sdk: flutter
  sqflite: ^2.0.0+4
  path: ^1.8.0
```

2. **Atualizar Dependências**: Execute o comando abaixo no terminal para atualizar as dependências:

```bash
flutter pub get
```

# Importação do Pacote sqflite

No arquivo Dart onde você pretende usar o SQLite, importe os pacotes necessários:

```dart
import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';
```

- `sqflite`: Pacote que fornece a interface para trabalhar com SQLite no Flutter.
- `path`: Pacote que auxilia na manipulação de caminhos de arquivos de forma segura entre diferentes plataformas.

# Abrindo uma Conexão com o Banco de Dados

Para interagir com o banco de dados SQLite, é necessário abrir uma conexão. Isso é feito utilizando o método `openDatabase`.

## Método openDatabase

```dart
Future<Database> openDatabase(
  String path, {
  int? version,
  OnDatabaseConfigureFn? onConfigure,
  OnDatabaseCreateFn? onCreate,
  OnDatabaseVersionChangeFn? onUpgrade,
  OnDatabaseVersionChangeFn? onDowngrade,
  OnDatabaseOpenFn? onOpen,
  bool readOnly = false,
  bool singleInstance = true,
})
```

## Parâmetros do openDatabase

- **path**: (*String*) Caminho completo onde o banco de dados será armazenado.

- **version**: (*int?*) Versão do banco de dados. Usado para gerenciar migrações.

- **onConfigure**: (*OnDatabaseConfigureFn?*) Função chamada antes de abrir o banco de dados. Útil para configurações iniciais.

- **onCreate**: (*OnDatabaseCreateFn?*) Função chamada na criação do banco de dados. Onde você deve criar as tabelas iniciais.

- **onUpgrade**: (*OnDatabaseVersionChangeFn?*) Função chamada quando a versão do banco de dados é incrementada. Útil para migrações.

- **onDowngrade**: (*OnDatabaseVersionChangeFn?*) Função chamada quando a versão do banco de dados é decrementada.

- **onOpen**: (*OnDatabaseOpenFn?*) Função chamada toda vez que o banco de dados é aberto.

- **readOnly**: (*bool*) Indica se o banco de dados será aberto apenas para leitura.

- **singleInstance**: (*bool*) Indica se o banco de dados será uma única instância através da aplicação.

# Exemplo Completo

Abaixo está um exemplo completo de como abrir uma conexão com o banco de dados SQLite no Flutter:

```dart
import 'package:sqflite/sqflite.dart';
import 'package:path/path.dart';

class DatabaseHelper {
  // Singleton: uma única instância do DatabaseHelper
  static final DatabaseHelper _instance = DatabaseHelper._internal();
  factory DatabaseHelper() => _instance;
  DatabaseHelper._internal();

  // Instância do banco de dados
  static Database? _database;

  // Getter para o banco de dados
  Future<Database> get database async {
    if (_database != null) return _database!;
    // Inicializa o banco de dados
    _database = await _initDatabase();
    return _database!;
  }

  // Método para inicializar o banco de dados
  Future<Database> _initDatabase() async {
    // Obtém o caminho do diretório de documentos
    String databasesPath = await getDatabasesPath();
    String path = join(databasesPath, 'meu_banco.db');

    // Abre o banco de dados
    return await openDatabase(
      path,
      version: 1,
      onConfigure: _onConfigure,
      onCreate: _onCreate,
      onUpgrade: _onUpgrade,
    );
  }

  // Método chamado cada vez que a conexão com banco de dados é aberto
  Future<void> _onConfigure(Database db) async {
    await db.execute('PRAGMA foreign_keys = ON'); //Configura o banco para trabalhar com foreign_keys
  }

  // Método chamado na criação do banco de dados
  Future<void> _onCreate(Database db, int version) async {
    await db.execute('''
      CREATE TABLE usuario (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        nome TEXT NOT NULL,
        email TEXT NOT NULL UNIQUE
      )
    ''');
  }

  // Método chamado na atualização do banco de dados
  Future<void> _onUpgrade(Database db, int oldVersion, int newVersion) async {
    // Implementar lógica de migração se necessário
  }

  // Exemplo de inserção de dados
  Future<int> inserirUsuario(Map<String, dynamic> usuario) async {
    Database db = await database;
    return await db.insert('usuario', usuario);
  }

  // Exemplo de consulta de dados
  Future<List<Map<String, dynamic>>> obterUsuarios() async {
    Database db = await database;
    return await db.query('usuario');
  }
}
```

**Explicação do Código:**

- **Singleton Pattern**: Usamos o padrão Singleton para garantir que apenas uma instância do `DatabaseHelper` seja criada.

- **Método `_initDatabase()`**: Responsável por inicializar o banco de dados. Obtém o caminho adequado e chama o `openDatabase`.

- **Função `onCreate`**: Executa comandos SQL para criar as tabelas necessárias quando o banco de dados é criado pela primeira vez.

- **Inserção e Consulta**: Métodos `inserirUsuario` e `obterUsuarios` mostram como interagir com o banco de dados.

# Melhores Práticas

- **Gerenciamento de Recursos**: Sempre feche a conexão com o banco de dados quando não for mais necessária usando o método `db.close()`.

- **Tratamento de Erros**: Implemente blocos `try-catch` para capturar e lidar com possíveis exceções.

- **Migrations**: Utilize os métodos `onUpgrade` e `onDowngrade` para gerenciar mudanças na estrutura do banco de dados entre versões.

- **Injeção de Dependência**: Considere usar injeção de dependência para facilitar os testes e a manutenção.
