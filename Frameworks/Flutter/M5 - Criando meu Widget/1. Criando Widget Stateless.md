## Como criar um Widget Stateless no Flutter

### O que é e para que serve?

No Flutter, um `StatelessWidget` é um tipo de widget que não possui estado mutável. Isso significa que uma vez criado, ele não será alterado durante a execução do aplicativo. Ele é ideal para componentes de UI que não precisam de mudanças dinâmicas ou de interação do usuário para atualizar seu conteúdo.

### Sintaxe de uso

A criação de um `StatelessWidget` envolve a extensão da classe `StatelessWidget` e a implementação do método `build`, que retorna a árvore de widgets que compõem o componente.

```dart
import 'package:flutter/material.dart';

class MeuWidgetStateless extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Meu Widget Stateless'),
      ),
      body: Center(
        child: Text('Olá, Flutter!'),
      ),
    );
  }
}

void main() {
  runApp(MaterialApp(
    home: MeuWidgetStateless(),
  ));
}
```

### Restrições de uso

1. **Imutabilidade**: Um `StatelessWidget` não pode alterar seu estado após ser criado. Para componentes que necessitam de atualizações dinâmicas, deve-se utilizar `StatefulWidget`.
2. **Redesenho**: Mesmo que o estado do widget não mude, ele pode ser redesenhado se for parte de uma árvore de widgets que é reconstruída.

### Quando utilizar?

Utilize `StatelessWidget` quando:

1. O widget não precisa manter um estado que muda.
2. O conteúdo do widget depende apenas das propriedades passadas e não de eventos dinâmicos.
3. Quer criar componentes de UI simples, como botões, textos estáticos, imagens, etc.

### Tabela com todas as propriedades

Aqui estão algumas propriedades e métodos comuns usados com `StatelessWidget`.

| Propriedade/Método | Descrição | Sintaxe de uso |
|--------------------|-----------|----------------|
| `key` | Chave usada para identificar o widget na árvore de widgets. | `const MeuWidgetStateless({Key? key}) : super(key: key);` |
| `context` | Fornece acesso ao contexto de construção do widget, necessário para acessar temas, navegação, etc. | `Widget build(BuildContext context)` |
| `build` | Método obrigatório que retorna a árvore de widgets para renderizar. | `Widget build(BuildContext context)` |

### Exemplo prático

Vamos criar um widget stateless mais complexo que inclui um botão e um texto estilizado:

```dart
import 'package:flutter/material.dart';

class SaudacaoWidget extends StatelessWidget {
  final String nome;

  const SaudacaoWidget({Key? key, required this.nome}) : super(key: key);

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: Text('Saudação'),
      ),
      body: Center(
        child: Column(
          mainAxisAlignment: MainAxisAlignment.center,
          children: <Widget>[
            Text(
              'Olá, $nome!',
              style: TextStyle(fontSize: 24),
            ),
            SizedBox(height: 20),
            ElevatedButton(
              onPressed: () {
                print('Botão pressionado!');
              },
              child: Text('Clique aqui'),
            ),
          ],
        ),
      ),
    );
  }
}

void main() {
  runApp(MaterialApp(
    home: SaudacaoWidget(nome: 'Luiz'),
  ));
}
```

### Informações adicionais

1. **Desempenho**: `StatelessWidget` é geralmente mais eficiente do que `StatefulWidget` porque não precisa gerenciar o estado.
2. **Reutilização**: São excelentes para criar componentes reutilizáveis e modulares.
3. **Testabilidade**: São mais fáceis de testar devido à sua natureza imutável.

Ao criar aplicativos com Flutter, entender a diferença entre `StatelessWidget` e `StatefulWidget` e quando usá-los é crucial para desenvolver uma UI eficiente e bem estruturada. Utilize `StatelessWidget` sempre que possível para componentes que não necessitam de mudanças dinâmicas e reserve `StatefulWidget` para quando o estado mutável é essencial.
