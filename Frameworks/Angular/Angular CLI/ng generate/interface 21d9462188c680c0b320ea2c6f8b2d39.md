# interface

**Domine o comando `ng generate interface` do Angular CLI**

---

## Introdução

O Angular CLI é uma ferramenta poderosa que agiliza o desenvolvimento ao gerar automaticamente arquivos e boilerplate para diferentes artefatos do seu projeto. Entre esses artefatos estão as **interfaces** do TypeScript, fundamentais para definir estruturas de dados de forma clara e garantir tipagem estática em toda a aplicação. O comando `ng generate interface` (ou `ng g i`) cria um arquivo de interface seguindo convenções do Angular, contribuindo para a organização e a mantenibilidade do código ([angular.dev](https://angular.dev/cli/generate/interface)).

## 1. Conceitos Fundamentais

- **Interface em TypeScript:** contrato que define a forma e tipos de um objeto, promovendo **type safety** e autocompletar no IDE.
- **Schematics do Angular CLI:** templates que geram e modificam código de forma reprodutível. O `ng generate interface` invoca o *schematic* `@schematics/angular:interface` para criar um arquivo de interface padronizado ([angular.dev](https://angular.dev/cli/generate/interface)).
- **Benefícios:** redução de erros de digitação em nomes de arquivos, consistência no naming convention (`.interface.ts`) e integração automática no fluxo de build do Angular.

---

## 2. Sintaxe e Uso

```bash
# Forma completa
ng generate interface <name> [type] [options]

# Abreviações
ng g i <name> [type] [options]

```

- `<name>` é obrigatório e determina o nome-base do arquivo (`<name>.interface.ts`).
- `[type]` é opcional e adiciona um sufixo ao filename (`<name>.<type>.ts`).
- `[options]` adapta o comportamento do gerador.
- Exemplo básico:
    
    ```bash
    ng g interface user
    # Gera src/app/user.interface.ts
    ``` :contentReference[oaicite:2]{index=2}
    
    ```
    

---

## 3. Restrições de Uso

- **Quando não usar:**
    - Para objetos dinâmicos cuja forma varia em runtime (prefira `type` ou `Record<…>`).
    - Em projetos não-Angular ou sem Angular CLI (interface manual).
    - Se a interface for muito específica de uma classe com lógica interna (prefira `class`).
- **Porque não aplicar:** o `ng generate interface` não vincula o arquivo a módulos Angular, portanto, não deve ser usado em lugar de `ng generate class` ou `ng generate service` quando se deseja uma implementação com métodos.

## 4. Argumentos

| Argumento | Descrição | Tipo |
| --- | --- | --- |
| `name` | Nome da interface. Usado para gerar `<name>.interface.ts`. | `string`([angular.dev](https://angular.dev/cli/generate/interface)) |
| `type` | Sufixo customizado no filename. Ex.: `data` → `<name>.data.ts`. | `string`([angular.dev](https://angular.dev/cli/generate/interface)) |

## 5. Opções

### 5.1. Schematic-specific

| Opção | Descrição | Tipo |
| --- | --- | --- |
| `--prefix` | Prefixo adicionado ao nome da interface (raro em interfaces, mais comum em componentes). | `string` ([angular.dev](https://angular.dev/cli/generate/interface)) |
| `--project` | Nome do projeto alvo (monorepos ou múltiplos apps). Define onde criar o arquivo. | `string` ([angular.dev](https://angular.dev/cli/generate/interface)) |

### 5.2. Global CLI

| Opção | Alias | Descrição | Tipo | Default |
| --- | --- | --- | --- | --- |
| `--defaults` | — | Desabilita prompts interativos, usa valores padrão. | `boolean` | `false` ([angular.dev](https://angular.dev/cli/generate)) |
| `--dry-run` | `-d` | Simula a execução sem escrever arquivos. | `boolean` | `false` ([angular.dev](https://angular.dev/cli/generate)) |
| `--force` | — | Sobrescreve arquivos existentes sem pedir confirmação. | `boolean` | `false` ([angular.dev](https://angular.dev/cli/generate)) |
| `--help` | — | Exibe ajuda do comando no console. | `boolean` | —       ([angular.dev](https://angular.dev/cli/generate)) |
| `--interactive` | — | Habilita/Desabilita prompts interativos. | `boolean` | `true`  ([angular.dev](https://angular.dev/cli/generate)) |

## 6. Melhores Práticas e Casos de Uso

1. **Models centralizados:** mantenha interfaces em `src/app/models` ou `src/app/shared/models` para facilitar imports.
2. **Naming Convention:** use suffixo `Model` ou `Dto` via `type` (ex.: `ng g i user --type=model`).
3. **Barrel files:** crie um `index.ts` no diretório de models para re-exportar interfaces, permitindo `import { User } from 'app/models'`.
4. **Consistência:** padronize uso de interfaces vs. classes em todo o time para evitar divergências.
5. **Versionamento:** altere interfaces de forma imutável; para adições, sempre acrescente novas propriedades opcionais.

## 7. Exemplos Completos

### Cenário: API de Usuários

1. **Geração da Interface**
    
    ```bash
    ng g interface user --type=model --project=my-app
    
    ```
    
    - Cria `src/app/user/user.model.ts` contendo:
        
        ```
        export interface UserModel {
          id: number;
          name: string;
          email: string;
        }
        
        ```
        
2. **Uso em Serviço de API**
    
    ```
    import { Injectable } from '@angular/core';
    import { HttpClient } from '@angular/common/http';
    import { Observable } from 'rxjs';
    import { UserModel } from './user.model';
    
    @Injectable({ providedIn: 'root' })
    export class UserService {
      private apiUrl = '/api/users';
    
      constructor(private http: HttpClient) {}
    
      getAll(): Observable<UserModel[]> {
        return this.http.get<UserModel[]>(this.apiUrl);
      }
    }
    
    ```
    
3. **Uso em Componente**
    
    ```
    import { Component, OnInit } from '@angular/core';
    import { UserService } from './user.service';
    import { UserModel } from './user.model';
    
    @Component({
      selector: 'app-user-list',
      template: `
        <ul>
          <li *ngFor="let user of users">{{ user.name }} ({{ user.email }})</li>
        </ul>
      `
    })
    export class UserListComponent implements OnInit {
      users: UserModel[] = [];
    
      constructor(private userService: UserService) {}
    
      ngOnInit() {
        this.userService.getAll().subscribe(data => this.users = data);
      }
    }
    
    ```

## 8. Tópicos Adicionais

- **Custom Schematics:** como estender o *schematic* `interface` para incluir templates personalizados.
- **Barrel Exports Automatizados:** utilizar `-export` em schematics que suportam barrela (não aplicável diretamente ao interface).
- **Comparativo com `ng generate class`:** quando optar por classes em vez de interfaces.
- **Integração com NgRx/Redux:** definir interfaces para estados e ações.
