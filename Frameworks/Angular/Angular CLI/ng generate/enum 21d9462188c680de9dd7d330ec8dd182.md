# enum

**Título:** Explorando o Comando `ng generate enum` no Angular CLI

---

**Introdução:**

O comando `ng generate enum` (ou sua forma abreviada `ng g e`) faz parte do conjunto de “schematics” do Angular CLI. Ele automatiza a criação de enums em TypeScript dentro de um projeto Angular, garantindo consistência na nomenclatura de arquivos e simplificando o fluxo de trabalho ao definir conjuntos de constantes nomeadas ([angular.dev](https://angular.dev/cli/generate/enum)).

### Conceitos Fundamentais

- **Enumerações (enums) em TypeScript:** estruturas que definem um conjunto de constantes nomeadas, melhorando a legibilidade e prevenindo erros de “magic strings” ([angular.dev](https://angular.dev/cli/generate/enum)).
- **Schematics do Angular CLI:** plugins que geram ou modificam código automaticamente, seguindo convenções do framework para padronizar projetos.

---

### Sintaxe e Uso

```bash
# Forma completa
ng generate enum <name> [options]

# Forma abreviada
ng g e <name> [options]

```

- `<name>` será transformado em `.enum.ts` (por exemplo, `user-status.enum.ts`).
- Caso o parâmetro `-type` seja fornecido, o sufixo do arquivo muda para `<name>.<type>.ts` ([angular.dev](https://angular.dev/cli/generate/enum)).

---

### Restrições de Uso

- **Projetos JavaScript puros:** enums são um recurso do TypeScript.
- **Casos que demandam “union types”:** em alguns cenários, tipos literais (e.g. `type Status = 'ACTIVE' | 'PENDING'`) podem ser mais adequados que enums.
- **Geração manual específica:** se for necessária lógica extra na criação do enum (decorators, comentários estruturados), talvez seja melhor criá-lo manualmente.

### Opções

### 1. Específicas ao schematic

| Opção | Tipo | Default | Descrição |
| --- | --- | --- | --- |
| `--project` | string | (do diretório) | Define em qual projeto (workspace) o enum será criado; útil em multi-projects. ([angular.dev](https://angular.dev/cli/generate/enum)) |
| `--type` | string | — | Adiciona um sufixo ao nome do arquivo: `<name>.<type>.ts` (e.g., `order.status.ts`). ([angular.dev](https://angular.dev/cli/generate/enum)) |

### Melhores Práticas e Casos de Uso

- **Suffix padrão `.enum.ts`:** facilite a identificação de enums no projeto.
- **Agrupar enums por domínio:** crie pastas como `shared/enums/` para reutilização.
- **Usar string enums:** ao interoperar com APIs externas, prefira enums de string para evitar reordenações numéricas.
- **Documentar cada valor:** adicione comentários JSDoc nos membros do enum para melhorar geração de documentação.

### Exemplos Completos

### 1. Gerando um enum de status de usuário

```bash
# Cria src/app/shared/user-status.status.ts no projeto padrão
ng g enum shared/user-status --type=status --flat

```

**Arquivo gerado:**

```tsx
// src/app/shared/user-status.status.ts
export enum UserStatus {
  Active   = 'ACTIVE',
  Inactive = 'INACTIVE',
  Pending  = 'PENDING',
}

```

### 2. Consumindo o enum em um componente

```tsx
// src/app/components/user-list/user-list.component.ts
import { Component } from '@angular/core';
import { UserStatus } from '../../shared/user-status.status';

@Component({
  selector: 'app-user-list',
  template: `
    <ul>
      <li *ngFor="let status of statuses">{{ status }}</li>
    </ul>
  `
})
export class UserListComponent {
  statuses = Object.values(UserStatus);
}

```
