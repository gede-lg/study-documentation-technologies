# class

**Blueprints em Ação: Desvendando o `ng generate class` do Angular CLI**

## 1. Introdução

O comando `ng generate class` (ou seu alias `ng g cl`) é um **schematic** do Angular CLI que cria, de forma padronizada, uma nova **classe** em TypeScript dentro do seu projeto, incluindo, por padrão, um arquivo de teste unitário correspondente. Essa funcionalidade acelera a scaffolding de código e garante consistência na estrutura dos arquivos. ([angular.dev](https://angular.dev/cli/generate/class))

## 3. Conceitos Fundamentais

- **Objetivo:** Gerar rapidamente a **estrutura básica** de uma classe TypeScript (`.ts`) e, opcionalmente, seu teste (`.spec.ts`).
- **Por quê classes?** Em TypeScript/Angular, classes são *blueprints* para objetos com **propriedades** e **métodos**, suportando encapsulamento e instanciamento. ([angular.dev](https://angular.dev/cli/generate/class))
- **Schematic:** É um “gerador” que automatiza criação e alterações de arquivos, parte do ecossistema de extensões do Angular CLI.

---

## 4. Sintaxe e Uso

```bash
# Forma completa
ng generate class <nome> [options]

# Forma abreviada
ng g cl <nome> [options]

```

- **`<nome>`**: nome da classe e do arquivo (ex.: `order => order.ts`). ([angular.dev](https://angular.dev/cli/generate/class))
- **Alias**: `cl` é o shorthand para `class`.
- **Pasta de destino**: defina caminhos com barras (e.g., `models/user`).

### Exemplos de invocação

```bash
# 1) Classe simples “order”
ng generate class order

# 2) Classe “user” em subpasta “models”, sem testes
ng g cl models/user --skip-tests

# 3) Classe com tipo customizado “dto”
ng g cl create-user --type dto

```

## 5. Restrições de Uso

- **Só tipagem?** Se precisa apenas de um contrato estrutural, prefira `ng generate interface`, pois interfaces não geram código JavaScript extra ([stackoverflow.com](https://stackoverflow.com/questions/58421035/angular-should-i-use-class-or-interface-for-model?utm_source=chatgpt.com)).
- **Lógica Angular:** Para componentes, serviços ou módulos com injeção de dependência, utilize os schematics específicos (`component` ou `service`), não `class`.
- **Demais cenários:** Evite classes triviais para as quais um *type alias* (e.g., `type ID = string`) seria suficiente, reduzindo código morto.

## 7. Opções

### 7.1. Schematic-specific (`ng generate class`)

| Opção | Tipo | Padrão | Descrição |
| --- | --- | --- | --- |
| `--project` | string | — | Nome do projeto no workspace onde a classe será gerada. Se omitido, inferido pela pasta atual. ([angular.dev](https://angular.dev/cli/generate/class)) |
| `--skip-tests` | boolean | false | Não gera o arquivo de teste `*.spec.ts`. ([angular.dev](https://angular.dev/cli/generate/class)) |
| `--type` | string | — | Sufixo customizado no nome do arquivo. Ex.: `--type helper` → `my-class.helper.ts`. ([angular.dev](https://angular.dev/cli/generate/class)) |

### 7.2. Globais do Angular CLI (`ng generate`)

| Opção | Alias | Tipo | Padrão | Descrição |
| --- | --- | --- | --- | --- |
| `--defaults` | — | boolean | false | Desativa prompts interativos e usa valores padrão. ([angular.dev](https://angular.dev/cli/generate)) |
| `--dry-run` | `-d` | boolean | false | Simula ações sem criar arquivos. ([angular.dev](https://angular.dev/cli/generate)) |
| `--force` | — | boolean | false | Sobrescreve arquivos existentes sem perguntar. ([angular.dev](https://angular.dev/cli/generate)) |
| `--help` | — | boolean | — | Exibe mensagem de ajuda no console. ([angular.dev](https://angular.dev/cli/generate)) |
| `--interactive` | — | boolean | true | Habilita ou desabilita prompts interativos. ([angular.dev](https://angular.dev/cli/generate)) |

> Observação: Dependendo da versão da CLI, há outras flags como --collection, --debug e --verbose.

## 8. Melhores Práticas e Casos de Uso

- **Organização por Domínio:** Crie pastas como `models/`, `dto/`, `utils/` para agrupar classes afins.
- **Nomenclatura Padronizada:** Use sufixos (`.model.ts`, `.dto.ts`, `.helper.ts`) para facilitar descoberta e importação.
- **Cobertura de Testes:** Deixe `-skip-tests` desativado para manter disciplina de testes.
- **Custom Schematics:** Em projetos grandes, desenvolva *schematics* próprios para padrões recorrentes, garantindo padronização.

## 9. Exemplos Completos

### 9.1. Modelo de Produto

```bash
ng g cl models/product --type model

```

**Cria**

- `src/app/models/product.model.ts`
    
    ```
    export class Product {
      constructor(
        public id: number,
        public name: string,
        public price: number
      ) {}
    }
    
    ```
    
- `src/app/models/product.model.spec.ts`

---

### 9.2. Helper de Formatação (sem testes)

```bash
ng g cl utils/formatter --type helper --skip-tests

```

**Cria**

- `src/app/utils/formatter.helper.ts`
    
    ```
    export class FormatterHelper {
      static formatCurrency(value: number): string {
        return `$${value.toFixed(2)}`;
      }
    }
    
    ```
    

---

### 9.3. Classe em Projeto Específico

```bash
ng g cl shared/logger --project dashboard --type service

```

**Cria**

- `projects/dashboard/src/app/shared/logger.service.ts`
