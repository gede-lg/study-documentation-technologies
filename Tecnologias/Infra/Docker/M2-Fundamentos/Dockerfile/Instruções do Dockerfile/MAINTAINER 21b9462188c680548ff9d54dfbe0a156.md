# MAINTAINER

# Instrução `MAINTAINER` em Dockerfile

## Introdução

A instrução `MAINTAINER` era utilizada para especificar o autor ou responsável pela imagem Docker, permitindo documentar o contato principal. Embora funcional, foi **deprecada** a partir do Docker 1.13 em favor de labels (`LABEL`) que seguem a especificação OCI de metadados.

## Conceitos Fundamentais

- **Autor da imagem:** Identifica quem criou ou mantém a imagem.
- **Depreciação:** Embora aceite, seu uso não deve ser encorajado em novos Dockerfiles. A comunidade recomenda `LABEL` para maior flexibilidade e interoperabilidade.
- **Documentação simples:** Fornece contato direto via endereço de e-mail.

---

## Sintaxe Detalhada e Uso Prático

```
MAINTAINER <Nome> <E-mail>

```

- `<Nome>`: Nome do responsável ou equipe.
- `<E-mail>`: Endereço de contato.

### Exemplo

```
MAINTAINER Juliana Gomes Miranda <juliana@exemplo.com>

```

---

## Cenários de Restrição ou Não Aplicação

- **Novos projetos:** Evite, pois a instrução é oficialmente deprecada.
- **Organizações que já padronizaram labels:** Prefira `LABEL maintainer=` para manter consistência com padrões OCI.
- **Imagens intermediárias em multi-stage builds:** Informações de manutenção podem não ser relevantes.

---

## Componentes Chave Associados

- **LABEL maintainer**: Substituto direto que inserta metadado compatível com OCI.
- **.dockerfilelint**: Ferramentas de linting que apontam o uso de MAINTAINER como alerta de depreciação.
- **Registries e metadata**: Muitos registries exibem “Author” extraído de labels em vez de MAINTAINER.

---

## Melhores Práticas e Padrões de Uso

1. **Substituir por LABEL:** Use `LABEL maintainer="Nome <email>"` em vez de `MAINTAINER`.
2. **Padronização de chaves:** Utilize chaves de namespace reverso (ex.: `org.opencontainers.image.authors`).
3. **Evitar informações sensíveis:** Nunca inclua senhas ou tokens.
4. **Atualização contínua:** Mantenha o contato atualizado em cada versão da imagem.

---

## Exemplo Prático Completo

```
FROM ubuntu:20.04

# Evitar esta instrução em novos Dockerfiles
MAINTAINER Juliana Gomes Miranda <juliana@exemplo.com>

# Recomenda-se usar:
LABEL maintainer="Juliana Gomes Miranda <juliana@exemplo.com>"
LABEL org.opencontainers.image.authors="Juliana Gomes Miranda"

RUN apt-get update && apt-get install -y python3

CMD ["python3", "-m", "http.server"]

```
