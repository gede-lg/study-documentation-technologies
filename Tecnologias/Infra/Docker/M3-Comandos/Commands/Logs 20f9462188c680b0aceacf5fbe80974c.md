# Logs

**1. Finalidade do `docker logs`**
O **`docker logs`** recupera a saída padrão (**stdout**) e de erro (**stderr**) gerada pelo processo principal de um contêiner. É a forma mais direta de obter **logs de aplicação** ou diagnosticar o comportamento de um contêiner em execução ou parado.
**2. Sintaxe básica**

`docker logs [OPTIONS] CONTAINER`

• **`CONTAINER`**: nome ou ID do contêiner cujos logs você deseja visualizar.
**3. Parâmetro**ParâmetroObrigatórioDescrição`CONTAINER`SimNome ou ID do contêiner alvo para exibir logs.
**4. Opções disponíveis**OpçãoTipoPadrãoDescrição`--details`boolean`false`Inclui campos extras que a aplicação possa ter adicionado às entradas de log.`-f, --follow`boolean`false`Segue (tail) o fluxo de logs em tempo real, imprimindo novas linhas conforme aparecem.`--since string`string—Mostra logs gerados **desde** um timestamp ou duração relativa (ex.: `2025-06-11T10:00:00Z` ou `2h`).`--until string`string—Mostra logs até um timestamp ou duração relativa (ex.: `2025-06-11T12:00:00Z` ou `30m`).`-n, --tail string`string`all`Exibe apenas as últimas **N** linhas (ex.: `-n 100`).`-t, --timestamps`boolean`false`Prefixa cada linha com seu timestamp (RFC3339).
**5. Exemplos de uso**
1. **Exibir todo o histórico de logs**

`docker logs my_container`

2. **Seguir logs em tempo real**

`docker logs -f my_container`

3. **Ver apenas as últimas 50 linhas**

`docker logs --tail 50 my_container`

4. **Incluir timestamps em cada linha**

`docker logs -t my_container`

5. **Logs de um intervalo específico**

`# Logs das últimas 2 horas
docker logs --since 2h my_container

# Logs até 30 minutos atrás
docker logs --until 30m my_container

# Logs de 1h a 30m atrás
docker logs --since 1h --until 30m my_container`

6. **Mostrar detalhes adicionais**

`docker logs --details -f my_container`
