# Rename

---

## 1. Finalidade do `docker rename`

O **`docker rename`** altera o **nome** de um contêiner existente, sem interrompê-lo ou recriá-lo. Útil para padronizar nomes, evitar conflitos ou tornar-os mais descritivos.

---

## 2. Sintaxe básica

```bash
docker rename CONTAINER NEW_NAME

```

- **`CONTAINER`**: nome **atual** ou ID do contêiner que você deseja renomear.
- **`NEW_NAME`**: novo nome que será atribuído ao contêiner.

---

## 3. Parâmetros

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `CONTAINER` | Sim | Nome (ou ID) atual do contêiner a ser renomeado. |
| `NEW_NAME` | Sim | Novo nome único para o contêiner. |

---

## 4. Restrições

- O **novo nome** não pode já existir em outro contêiner ativo ou parado.
- Não há opções adicionais: o comando é atômico e mantém todos os demais metadados e estado do contêiner.

---

## 5. Exemplos de uso

1. **Renomear por nome**
    
    ```bash
    docker rename meu_app_antigo meu_app_novo
    # Agora você pode usar 'docker ps --filter name=meu_app_novo' para encontrá-lo.
    
    ```
    
2. **Renomear usando ID abreviado**
    
    ```bash
    # Supondo que o ID (prefixo) do contêiner seja '8f3a1d2c'
    docker rename 8f3a1d2c web_back
    
    ```
    
3. **Fluxo típico para padronização**
    
    ```bash
    # Parar o contêiner antigo
    docker stop old_db
    
    # Renomear para identificar versão
    docker rename old_db db_v1
    
    # Iniciar novamente (se desejado)
    docker start db_v1
    
    ```

| Componente | Sintaxe | Descrição |

---

Use **`docker rename`** sempre que precisar ajustar nomes de contêineres para melhor organização ou clareza no seu ambiente.
