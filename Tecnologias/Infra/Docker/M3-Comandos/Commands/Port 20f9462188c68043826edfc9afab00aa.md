# Port

---

## 1. Finalidade do `docker port`

O **`docker port`** exibe os **mapeamentos de portas** de um contêiner em execução, indicando em qual porta e endereço do **host** cada porta **interna** do contêiner está disponível.

---

## 2. Sintaxe básica

```bash
docker port CONTAINER [PRIVATE_PORT[/PROTO]]

```

- **`CONTAINER`**: nome ou ID do contêiner.
- **`PRIVATE_PORT`** (opcional): porta interna do contêiner que você quer consultar.
- **`PROTO`** (opcional): protocolo (`tcp` ou `udp`), padrão `tcp`.

---

## 3. Parâmetros

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `CONTAINER` | Sim | Nome ou ID do contêiner cujos mapeamentos serão listados. |
| `PRIVATE_PORT[/PROTO]` | Não | Porta interna (ex.: `80` ou `80/tcp`) para filtrar a saída a esse mapeamento específico. |

---

## 4. Comportamento

- Sem **`PRIVATE_PORT`**, lista **todos** os mapeamentos de porta do contêiner.
- Com **`PRIVATE_PORT`**, retorna apenas o(s) mapeamento(s) da porta interna especificada.

---

## 5. Exemplos de uso

1. **Listar todos os mapeamentos de um contêiner**
    
    ```bash
    docker port web_app
    # Saída exemplificativa:
    # 80/tcp -> 0.0.0.0:8080
    # 443/tcp -> 0.0.0.0:8443
    
    ```
    
2. **Consultar mapeamento de uma porta específica**
    
    ```bash
    docker port web_app 80
    # Saída:
    # 0.0.0.0:8080
    
    ```
    
3. **Filtrar por protocolo UDP**
    
    ```bash
    docker port dns_server 53/udp
    # Saída:
    # 0.0.0.0:5353
    
    ```
    

---

## 6. Tabela de referência rápida

| Componente | Sintaxe | Descrição |
| --- | --- | --- |
| Comando base | `docker port CONTAINER [PORT[/PROTO]]` | Exibe mapeamentos de portas do contêiner. |
| Parâmetro | `CONTAINER` | Nome ou ID do contêiner. |
| Parâmetro | `PORT` | Porta interna a consultar (opcional). |
| Parâmetro | `PROTO` | Protocolo (`tcp` ou `udp`) para filtrar (opcional). |

---

Com isso, você consegue rapidamente verificar em quais portas e interfaces de rede do host seu contêiner está exposto. Se precisar de mais detalhes ou auxílio para outro comando Docker, é só avisar!
