# Pause

---

## 1. Finalidade do `docker pause`

O **`docker pause`** suspende todos os processos em execução dentro de um ou mais contêineres, utilizando o recurso de cgroups do Linux para enviar um **SIGSTOP** coletivo. É útil para:

- **Congelar** temporariamente workloads que consomem muitos recursos.
- Realizar **backups** de volumes sem atividade no contêiner.
- Testar comportamento de aplicações sob indisponibilidade momentânea.

---

## 2. Sintaxe básica

```bash
docker pause CONTAINER [CONTAINER...]

```

- **`CONTAINER`**: nome(s) ou ID(s) do(s) contêiner(es) a serem pausados. Você pode especificar vários de uma vez.

---

## 3. Comportamento

- **Pausa**: envia `SIGSTOP` a todos os processos do contêiner.
- **Mantém estado**: o contêiner permanece “up” mas todos os processos ficam congelados, sem consumir CPU.
- Para **retomar**, use `docker unpause`.

---

## 4. Exemplos de uso

1. **Pausar um único contêiner**
    
    ```bash
    docker pause web_server
    
    ```
    
2. **Pausar múltiplos contêineres**
    
    ```bash
    docker pause db_cache worker_api
    
    ```
    
3. **Fluxo típico de backup**
    
    ```bash
    docker pause data_service
    tar czf backup.tar.gz /var/lib/docker/volumes/mydata/_data
    docker unpause data_service
    
    ```
    

---

## 5. Tabela de referência rápida

| Comando base | `docker pause CONTAINER [CONTAINER...]` |
| --- | --- |
| Ação | Pausa (SIGSTOP) todos os processos do contêiner |
| Retomar | `docker unpause CONTAINER` |

---

Use **`docker pause`** sempre que precisar congelar ambientes de contêiner sem encerrá-los, garantindo um ponto estável para manutenção ou backup.
