# Unpause

---

## 1. Finalidade do `docker unpause`

O **`docker unpause`** **retoma** todos os processos que haviam sido suspensos (paused) dentro de um ou mais contêineres, liberando-os para continuarem a execução normal. É o complemento do comando `docker pause`.

---

## 2. Sintaxe básica

```bash
docker unpause CONTAINER [CONTAINER...]

```

- **`CONTAINER`**: nome(s) ou ID(s) do(s) contêiner(es) que você deseja **descongelar**. Pode especificar vários de uma só vez.

---

## 3. Parâmetros

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `CONTAINER` | Sim | Nome(s) ou ID(s) do(s) contêiner(es) a serem unpaused. |

---

## 4. Comportamento

- O Docker envia um **SIGCONT** aos processos do contêiner, retomando sua execução exatamente de onde haviam sido pausados.
- Volta automaticamente ao estado de “executando” do contêiner, mantendo todas as configurações de rede, volumes e metadados.

---

## 5. Exemplos de uso

1. **Retomar um único contêiner**
    
    ```bash
    docker unpause my_container
    
    ```
    
2. **Retomar múltiplos contêineres**
    
    ```bash
    docker unpause web_server db_cache worker
    
    ```
    
3. **Fluxo típico de manutenção**
    
    ```bash
    docker pause data_service
    # Execute tarefas de backup ou manutenção
    docker unpause data_service
    
    ```
    

---

## 6. Tabela de referência rápida

| Componente | Sintaxe | Descrição |
| --- | --- | --- |
| Comando base | `docker unpause CONTAINER...` | Retoma processos em contêiner(es) pausados. |
| Parâmetro | `CONTAINER` | Nome(s) ou ID(s) do(s) contêiner(es) alvo. |

---

Use **`docker unpause`** para voltar suas aplicações em contêiner ao funcionamento normal após uma suspensão temporária.
