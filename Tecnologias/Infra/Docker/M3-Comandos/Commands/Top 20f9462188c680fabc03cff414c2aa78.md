# Top

---

## 1. Finalidade do `docker top`

O **`docker top`** exibe os **processos ativos** dentro de um contêiner em execução, mostrando exatamente o que está rodando no namespace de processos daquele contêiner. É útil para:

- Diagnosticar aplicações que podem estar travadas ou consumindo muitos recursos.
- Verificar comandos e PIDs específicos sem entrar no contêiner.
- Integrar em scripts de monitoramento de processos de containers.

---

## 2. Sintaxe básica

```bash
docker top CONTAINER [ps OPTIONS]

```

- **`CONTAINER`**: nome ou ID do contêiner cujos processos você quer listar.
- **`[ps OPTIONS]`**: opções opcionais que você passaria ao comando `ps` do host, por exemplo `aux`, `ef`, `o pid,cmd`, etc.

---

## 3. Parâmetros

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `CONTAINER` | Sim | Nome ou ID do contêiner alvo. |
| `ps OPTIONS` | Não | Qualquer opção válida para o comando `ps`, para customizar colunas ou formato de saída. |

---

## 4. Comportamento

- Sem `ps OPTIONS`, o Docker usa um formato padrão semelhante a `ps aux`:
    
    ```
    UID        PID  PPID  C STIME TTY      STAT   TIME COMMAND
    root         1     0  0 10:00 ?        Ss     0:00 /bin/sh -c myapp
    root        10     1  0 10:00 ?        Ssl    0:01 /usr/local/bin/myapp
    
    ```
    
- Ao fornecer opções, por exemplo `ef`, você obtém a saída típica de `ps -ef`, incluindo informações de usuário, PPID, comando completo, etc.

---

## 5. Exemplos de uso

1. **Listar processos padrão do contêiner**
    
    ```bash
    docker top web_container
    
    ```
    
2. **Usar formato estendido (`ps aux`)**
    
    ```bash
    docker top web_container aux
    
    ```
    
3. **Mostrar colunas específicas (`PID` e `CMD`)**
    
    ```bash
    docker top web_container -o pid,cmd
    
    ```
    
4. **Ver processos com detalhes de usuário**
    
    ```bash
    docker top db_container -ef
    
    ```

| Componente | Sintaxe | Descrição |
| --- | --- | --- |

---

Use **`docker top`** sempre que precisar inspecionar rapidamente quais processos estão rodando dentro de um contêiner, ajustando a saída conforme suas necessidades através das opções do `ps`.
