# Start

---

## 1. Finalidade do `docker start`

O **`docker start`** inicia um ou mais contêineres **parados**, restaurando-os ao estado em que ficaram quando foram interrompidos (mantendo volumes, rede, variáveis e configurações originais).

---

## 2. Sintaxe básica

```bash
docker start [OPTIONS] CONTAINER [CONTAINER...]

```

- **`CONTAINER`**: nome(s) ou ID(s) de contêiner(es) que você deseja iniciar. Pode listar vários de uma vez.

---

## 3. Opções disponíveis

| Opção | Tipo | Padrão | Descrição |
| --- | --- | --- | --- |
| `-a, --attach` | boolean | `false` | Conecta seu terminal a **stdout**/**stderr** do contêiner logo após iniciar, exibindo a saída ao vivo. |
| `-i, --interactive` | boolean | `false` | Mantém o **stdin** do contêiner aberto, caso queira interagir com um processo que leia entrada padrão. |
| `--detach-keys string` | string | `ctrl-p,ctrl-q` | Sequência de teclas para desanexar do contêiner quando estiver em modo attach. |

---

## 4. Comportamento

1. **Arranque do contêiner**
    - Roda o contêiner a partir do seu último ponto de parada, recriando o namespace de processo, montagens e rede.
2. **Attach opcional**
    - Se usar `a`, imediatamente após o start você verá o fluxo de logs e poderá enviar sinais via teclado.
3. **Interação padrão**
    - Sem `a`/`i`, o contêiner inicia em background e você recebe apenas o ID de volta.

---

## 5. Exemplos de uso

1. **Iniciar um contêiner em background**
    
    ```bash
    docker start meu_container
    # Saída:
    # meu_container
    
    ```
    
2. **Iniciar múltiplos contêineres**
    
    ```bash
    docker start web1 db_cache worker
    
    ```
    
3. **Iniciar e anexar à saída**
    
    ```bash
    docker start --attach web_server
    # Agora você verá os logs de stdout/stderr no seu terminal.
    
    ```
    
4. **Iniciar com stdin aberto**
    
    ```bash
    docker start --interactive shell_container
    # Se o contêiner rodar um shell, você poderá digitar comandos.
    
    ```
    
5. **Personalizar teclas de detach**
    
    ```bash
    docker start -a --detach-keys="ctrl-x,ctrl-x" app
    # Pressione CTRL-X CTRL-X para sair do attach.
    
    ```
    

---

## 6. Tabela de referência rápida

| Componente | Sintaxe | Descrição |
| --- | --- | --- |
| Comando base | `docker start [OPTIONS] CONTAINER...` | Inicia contêiner(es) parados. |
| Parâmetro | `CONTAINER` | Nome(s) ou ID(s) do(s) contêiner(es) a iniciar. |
| Opção | `-a, --attach` | Anexa stdout/stderr após o start. |
| Opção | `-i, --interactive` | Mantém stdin aberto para interação. |
| Opção | `--detach-keys <seq>` | Sequência para desanexar no modo attach. |

---

Com isso, você pode retomar imediatamente qualquer contêiner parado e, se necessário, acompanhar sua saída ou interagir com ele diretamente. Caso queira ver outro comando, só avisar!
