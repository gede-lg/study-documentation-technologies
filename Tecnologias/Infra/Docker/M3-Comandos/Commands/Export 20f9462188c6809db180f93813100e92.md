# Export

---

## 1. Finalidade do `docker export`

O **`docker export`** gera um **snapshot** do **filesystem** de um contêiner, empacotando todo o seu sistema de arquivos em um arquivo TAR. Diferente de `docker commit`, ele **não** inclui o histórico de camadas Docker nem metadados de imagem—apenas o conteúdo do container em si. É útil para:

- Fazer backup dos dados de um contêiner.
- Extrair todo o filesystem para inspeção ou migração.
- Compartilhar o ambiente de um contêiner sem depender de imagens Docker.

---

## 2. Sintaxe básica

```bash
docker export [OPTIONS] CONTAINER

```

- **`CONTAINER`**: nome ou ID do contêiner cujo filesystem você deseja exportar.

---

## 3. Opções disponíveis

| Opção | Tipo | Padrão | Descrição |
| --- | --- | --- | --- |
| `-o, --output string` | string | stdout | Grava o conteúdo do TAR em um arquivo local. Ex.: `--output backup.tar`. |

Sem a opção `--output`, o TAR é enviado para **STDOUT**, permitindo uso com pipes, por exemplo:

```bash
docker export my_container | gzip > backup.tar.gz

```

---

## 4. Exemplos de uso

1. **Exportar para arquivo TAR**
    
    ```bash
    docker export --output meu_backup.tar my_container
    
    ```
    
    Gera `meu_backup.tar` contendo todo o filesystem do `my_container`.
    
2. **Exportar e comprimir via pipe**
    
    ```bash
    docker export my_container | gzip > backup.tar.gz
    
    ```
    
    Gera `backup.tar.gz` compactado.
    
3. **Inspecionar conteúdo sem salvar em disco**
    
    ```bash
    docker export my_container | tar t
    
    ```
    
    Lista os arquivos dentro do TAR enviado ao stdout.
    

---

## 5. Tabela de referência rápida

| Componente | Sintaxe | Descrição |
| --- | --- | --- |
| Comando base | `docker export [OPTIONS] CONTAINER` | Exporta o filesystem do contêiner como um TAR. |
| Parâmetro | `CONTAINER` | Nome ou ID do contêiner de origem. |
| Opção | `-o, --output <arquivo>.tar` | Salva o TAR em arquivo; padrão é enviar ao STDOUT. |

---

Com isso, você pode exportar todo o conteúdo de um contêiner Docker para um arquivo TAR ou manipulá-lo em tempo real via pipes. Se tiver mais dúvidas ou quiser restaurar esse TAR em outro host, é só perguntar!
