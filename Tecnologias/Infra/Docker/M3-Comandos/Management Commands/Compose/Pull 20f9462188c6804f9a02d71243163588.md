# Pull

**`docker compose pull`** permite baixar (pull) as imagens dos serviços definidos em um arquivo Compose, sem iniciar contêineres. A seguir, uma explicação estruturada de sua sintaxe, opções e finalidades.

---

## Sintaxe

```bash
docker compose pull [OPÇÕES] [SERVIÇO...]

```

- **`[OPÇÕES]`**: flags que modificam o comportamento do pull.
- **`[SERVIÇO...]`**: lista opcional de serviços (nomes conforme `compose.yaml`) cujas imagens devem ser baixadas; se omitido, todas as imagens de todos os serviços são consideradas. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/pull/))

---

## Descrição das Opções

- **`-dry-run`**
    - **Sintaxe**: `-dry-run`
    - **Finalidade**: simula a execução do comando, exibindo quais imagens seriam baixadas e em que ordem, **sem** realmente alterar o estado do seu ambiente Compose.
    - **Uso prático**: verificar precedência de dependências e políticas de pull antes de executar de fato. ([docs.docker.com](https://docs.docker.com/compose/releases/release-notes/?utm_source=chatgpt.com))
- **`-ignore-buildable`**
    - **Sintaxe**: `-ignore-buildable`
    - **Finalidade**: ignora serviços que possuem seção `build` (ou seja, que podem ser construídos localmente) e **não** tenta puxar suas imagens, evitando erro quando não há imagem remota disponível. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/pull/))
- **`-ignore-pull-failures`**
    - **Sintaxe**: `-ignore-pull-failures`
    - **Finalidade**: tenta baixar todas as imagens possíveis, mas **ignora** falhas em algumas imagens, continuando com o restante sem interromper todo o processo. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/pull/))
- **`-include-deps`**
    - **Sintaxe**: `-include-deps`
    - **Finalidade**: além de puxar a imagem do serviço solicitado, também baixa as imagens de **todos** os serviços dos quais ele depende (via `depends_on`). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/pull/))
- **`-policy string`**
    - **Sintaxe**: `-policy missing` ou `-policy always`
    - **Finalidade**: define a **política** de pull:
    - `"missing"` (padrão): baixa apenas se a imagem local **não existir**;
    - `"always"`: força o download mesmo que a imagem já exista localmente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/pull/))
- *`q`, **`-quiet`**
    - **Sintaxe**: `q` ou `-quiet`
    - **Finalidade**: suprime informações de progresso, exibindo somente mensagens essenciais (útil em scripts e logs). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/pull/))

---

## Tabela Resumo das Opções

| Opção | Sintaxe | Descrição resumida |
| --- | --- | --- |
| Dry run | `--dry-run` | Exibe as etapas do pull sem executá-las realmente |
| Ignorar buildable | `--ignore-buildable` | Não tenta baixar serviços que podem ser **buildados** localmente |
| Ignorar falhas de pull | `--ignore-pull-failures` | Continua o download das demais imagens mesmo se alguma falhar |
| Incluir dependências | `--include-deps` | Também puxa imagens de serviços dependentes |
| Política de pull | `--policy missing|always` | Controla quando puxar: somente imagens **ausentes** (`missing`) ou **sempre** (`always`) |
| Quiet | `-q`, `--quiet` | Modo silencioso: suprime a barra de progresso e outras mensagens verbosas |

> Fonte: Docker Docs e Release Notes (docs.docker.com, docs.docker.com)
> 

---

## Exemplo de Uso

```bash
# Simular pull de todos os serviços e suas dependências, sem alterações:
docker compose pull --dry-run --include-deps

# Baixar apenas imagens ausentes, ignorando serviços buildáveis:
docker compose pull --ignore-buildable

# Forçar pull de todas as imagens e suprimir progresso:
docker compose pull --policy always -q

```

Com essas informações, você pode controlar exatamente **quando** e **como** o `docker compose pull` buscará cada imagem do seu ambiente multi-container.
