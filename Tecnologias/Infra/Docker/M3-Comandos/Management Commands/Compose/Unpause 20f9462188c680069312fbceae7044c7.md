# Unpause

Olá Gedê!

Claro, aqui está uma explicação bem estruturada sobre o comando `docker compose unpause`, detalhando cada um de seus parâmetros.

O comando `docker compose unpause` é utilizado para **retomar a execução de todos os processos** que foram previamente pausados dentro de um ou mais contêineres de serviço. Quando você pausa um serviço com `docker compose pause`, o estado dos processos é congelado. O `unpause` descongela esse estado, permitindo que os processos continuem de onde pararam.

---

### **Sintaxe Geral**

A estrutura básica do comando é:

```bash
docker compose unpause [OPÇÕES] [SERVIÇOS...]

```

---

### **Parâmetros e Opções**

Vamos detalhar os componentes que você viu no `--help`.

### **`[SERVICE...]`**

Este é o parâmetro principal que indica **quais serviços** você deseja despausar.

- **Sintaxe**: Você pode listar um ou mais nomes de serviços definidos no seu arquivo `docker-compose.yml`. Os nomes devem ser separados por espaços.
- **Finalidade**: Permite direcionar a ação de `unpause` a contêineres específicos, em vez de todos os que estão em execução no projeto.
- **Uso**:
    - **Despausar um único serviço**: `docker compose unpause nome_do_servico`
    - **Despausar múltiplos serviços**: `docker compose unpause servico1 servico2`
    - **Despausar todos os serviços**: Se você não especificar nenhum nome de serviço, o comando será aplicado a **todos** os serviços definidos no seu `docker-compose.yml` que estiverem pausados.

### **`-dry-run`**

Esta é uma opção (ou *flag*) que modifica o comportamento do comando.

- **Sintaxe**: `-dry-run`
- **Finalidade**: Serve para **simular a execução** do comando. Ele mostrará quais comandos seriam executados para despausar os serviços, mas **não realizará a ação de fato**. É extremamente útil para verificar se você está despausando os serviços corretos antes de executar a ação real.
- **Uso**: `docker compose unpause --dry-run [SERVIÇO...]`

---

### **Tabela Resumo**

| Parâmetro / Opção | Sintaxe | Finalidade | Exemplo de Uso |
| --- | --- | --- | --- |
| **`[SERVICE...]`** | `[nome_do_serviço]` | Especifica o(s) serviço(s) alvo para despausar. Se omitido, aplica-se a todos os serviços. | `docker compose unpause db web` |
| **`--dry-run`** | `--dry-run` | Mostra o que seria executado sem realmente despausar os contêineres. | `docker compose unpause --dry-run` |
