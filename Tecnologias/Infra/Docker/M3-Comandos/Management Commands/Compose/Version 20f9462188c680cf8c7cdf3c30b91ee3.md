# Version

Olá Gedê! A.R.I.A. na área.

Com certeza! Abaixo está uma explicação detalhada sobre o comando `docker compose version` e todos os seus parâmetros.

O comando `docker compose version` é uma ferramenta simples e direta utilizada para exibir a versão do Docker Compose que está instalada em seu sistema. Isso é útil para verificar compatibilidade, relatar bugs ou simplesmente para saber qual versão você está utilizando.

---

### **Sintaxe Base**

A forma mais simples do comando, sem nenhum parâmetro adicional.

```bash
docker compose version

```

Ao executá-lo, ele exibe uma saída detalhada, geralmente no formato "pretty", que inclui a versão do Docker Compose e a versão do Docker Engine.

**Exemplo de Saída:**

```
Docker Compose version v2.27.0
Docker Engine version v26.1.4

```

---

### **Parâmetros (Options)**

Agora, vamos detalhar cada um dos parâmetros que podem modificar a saída do comando.

### **1. `-dry-run`**

- **Finalidade:** Este parâmetro simula a execução do comando sem realmente executá-lo. No contexto do `docker compose version`, seu efeito prático é nulo, pois o comando apenas lê informações de versão e não altera o estado do sistema. Ele existe por uma questão de consistência com outros comandos do Docker Compose (como `up` ou `down`), onde um "dry run" mostraria o que *seria* feito.
- **Sintaxe:**
    
    ```bash
    docker compose version --dry-run
    
    ```
    

### **2. `f, --format string`**

- **Finalidade:** Permite que você especifique o formato da saída. É muito útil para integrar a informação de versão em scripts ou outras ferramentas de automação.
- **Sintaxe:**
    
    ```bash
    docker compose version --format [FORMATO]
    
    ```
    
- **Valores Aceitos:**
    - `pretty`: (Padrão) Formato amigável para leitura humana.
    - `json`: Formato JSON (JavaScript Object Notation), ideal para ser processado por máquinas ou scripts.

**Exemplo com `json`:**

```bash
docker compose version --format json

```

**Saída em JSON:**

```json
{
  "version": "v2.27.0",
  "build": "a2b34b6",
  "engine": "Docker Engine",
  "engineVersion": "v26.1.4"
}

```

### **3. `-short`**

- **Finalidade:** Simplifica a saída para mostrar **apenas** o número da versão do Docker Compose. É o mais útil para scripts rápidos onde você só precisa do número da versão e nada mais.
- **Sintaxe:**
    
    ```bash
    docker compose version --short
    
    ```
    

**Exemplo de Saída:**

```
v2.27.0

```

### **Tabela Resumo**

Para facilitar a consulta, aqui está uma tabela que resume tudo.

| Parâmetro | Sintaxe Alternativa | Descrição | Exemplo de Uso |
| --- | --- | --- | --- |
| `--dry-run` | N/A | Simula a execução do comando. No caso de `version`, não tem efeito visível. | `docker compose version --dry-run` |
| `--format` | `-f` | Formata a saída. Aceita os valores `pretty` (padrão) ou `json`. | `docker compose version --format json` |
| `--short` | N/A | Exibe apenas o número da versão do Docker Compose, sem informações adicionais. | `docker compose version --short` |
