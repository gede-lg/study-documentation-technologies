# Images

## Visão Geral

O comando `docker compose images` é um subcomando do Docker Compose que **lista as imagens** usadas pelos contêineres criados a partir do seu arquivo Compose. Ele permite visualizar, de forma geral ou filtrada por serviço, quais imagens estão sendo referenciadas ou já baixadas para execução. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/images/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker compose images [OPTIONS] [SERVICE...]

```

- **SERVICE…** (opcional): nomes dos serviços definidos no `docker-compose.yml`; se omitido, lista imagens de todos os serviços. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/images/?utm_source=chatgpt.com))

---

## Opções

- `-dry-run`
    - **Sintaxe:** `-dry-run`
    - **Finalidade:** executa o comando em modo de simulação, mostrando o que ocorreria sem realmente alterar o estado do projeto ou interagir com o daemon. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/?utm_source=chatgpt.com))
- `-format string`
    - **Sintaxe:** `-format <formato>`
    - **Valores:** `table` | `json`
    - **Padrão:** `table`
    - **Finalidade:** define o formato de apresentação da lista de imagens — tabela legível no console ou JSON para processamento. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/images/?utm_source=chatgpt.com))
- `q`, `-quiet`
    - **Sintaxe:** `q` ou `-quiet`
    - **Finalidade:** exibe apenas os identificadores (IDs) das imagens, suprimindo colunas adicionais como nomes de serviço ou repositório. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/compose/images/?utm_source=chatgpt.com))

## Exemplos de Uso

> 1. Listar imagens de todos os serviços em formato de tabela (padrão):
> 

```bash
docker compose images

```

> 2. Listar apenas os IDs das imagens:
> 

```bash
docker compose images -q

```

> 3. Obter saída em JSON para script externo:
> 

```bash
docker compose images --format json

```

> 4. Simular a listagem sem interação real (dry run):
> 

```bash
docker compose images --dry-run

```
