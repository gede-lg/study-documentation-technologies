# Stats

**Visão Geral**

O comando `docker compose stats` exibe, em tempo real, estatísticas de uso de recursos (CPU, memória, I/O de rede e disco etc.) dos containers gerenciados por um projeto Docker Compose. Se nenhum serviço for especificado, mostra todas as containers em execução; caso contrário, filtra pelas containers de um serviço específico definido no seu arquivo Compose ([manpages.debian.org](https://manpages.debian.org/testing/docker-compose/docker-compose-stats.1.en.html)).

**Disponibilidade**

Introduzido na versão **2.24.0** da CLI do Docker Compose (publicada em 11 de janeiro de 2024) ([docs.docker.com](https://docs.docker.com/compose/releases/release-notes/)).

---

## Sintaxe

```
docker compose stats [OPÇÕES] [SERVIÇO]

```

- **SERVIÇO** (opcional): nome de um serviço definido no `docker-compose.yml`. Quando fornecido, apenas containers pertencentes a esse serviço terão suas estatísticas exibidas ([manpages.debian.org](https://manpages.debian.org/testing/docker-compose/docker-compose-stats.1.en.html)).

---

## Opções

| Opção | Sintaxe | Descrição |
| --- | --- | --- |
| **Mostrar todos** | `-a`, `--all` | Exibe *todas* as containers (inclusive as paradas). Por padrão exibe apenas as que estão em execução. |
| **Modo dry run** | `--dry-run` | Executa o comando em "simulação" (*dry run*) sem alterar o estado dos containers ou da pilha Compose. |
| **Ajuda** | `-h`, `--help` | Exibe a página de ajuda específica do subcomando `stats`. |
| **Formato de saída** | `--format string` | Formata a saída usando um template Go:• `table` – tabela com cabeçalhos (padrão)• `table TEMPLATE` – tabela customizada via Go template• `json` – JSON puro• `TEMPLATE` – Go template livre*Veja* [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/) para mais detalhes sobre templates. |
| **Snapshot único** | `--no-stream` | Desabilita o *streaming* contínuo de estatísticas e exibe apenas a primeira coleta. |
| **Sem truncamento** | `--no-trunc` | Não trunca valores longos na saída; mostra os strings completos das colunas. |

> Observação: as opções acima são herdadas de docker compose, exceto --dry-run, que vem de comandos pai na hierarquia do Compose (manpages.debian.org).
>

## Exemplos de Uso

```bash
# 1) Estatísticas em tempo real dos containers em execução
docker compose stats

# 2) Incluir containers parados
docker compose stats --all

# 3) Estatísticas apenas do serviço "web"
docker compose stats web

# 4) Obter um snapshot único, sem atualizar continuamente
docker compose stats --no-stream

# 5) Exibir resultado em JSON para consumo por scripts
docker compose stats --format json

```

Com essa estrutura, você tem à disposição:

1. **Visão geral** do propósito do comando.
2. **Informação de disponibilidade** na CLI.
3. **Sintaxe** clara, com parâmetro opcional de serviço.
4. **Tabela de opções** detalhando uso e finalidade.
5. **Exemplos práticos** para diferentes cenários.
