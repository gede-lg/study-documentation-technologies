# RM

## Visão Geral

O comando `docker manifest rm` remove **listas de manifestos** (manifest lists) do armazenamento local do cliente Docker. É uma funcionalidade **experimental**, usada para limpar definições de manifestos que você criou com `docker manifest create` ou importou, mas não deseja mais manter no cache local.

---

## Sintaxe

```bash
docker manifest rm MANIFEST_LIST [MANIFEST_LIST...]

```

- **MANIFEST_LIST**: nome (e tag) da lista de manifestos a ser removida.
- Você pode especificar múltiplas listas de uma vez, separadas por espaço.

---

## Detalhes

- **Experimental**: exigido que o Docker CLI esteja com recursos experimentais habilitados (`DOCKER_CLI_EXPERIMENTAL=enabled`).
- **Efeito**: exclui apenas a **configuração** local da lista de manifestos; não afeta imagens nos registries remotos.

---

## Exemplos de Uso

1. **Remover uma única lista de manifestos**
    
    ```bash
    docker manifest rm myrepo/myapp:multiarch
    
    ```
    
    Limpa do cliente a definição de `myrepo/myapp:multiarch`.
    
2. **Remover várias listas de uma vez**
    
    ```bash
    docker manifest rm app:latest app:staging app:test
    
    ```
    
    Remove as listas `app:latest`, `app:staging` e `app:test` em um único comando.
    

---

> Dica:
> 
> 
> Após remover localmente um manifest list, use `docker manifest inspect` ou tente `docker manifest push` novamente para recriá-lo ou sincronizá-lo com um registry remoto.
>
