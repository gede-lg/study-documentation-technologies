# Inspect

## Visão Geral

O comando `docker manifest inspect` permite **visualizar** em detalhes o manifesto de uma imagem ou de uma lista de manifestos (multi-plataforma), exibindo metadados como arquiteturas suportadas, camadas, digests e informações de plataforma. Esse recurso é **experimental** e requer que o Docker CLI esteja configurado para liberar funcionalidades experimentais.

---

## Sintaxe

```bash
docker manifest inspect [OPÇÕES] [MANIFEST_LIST] MANIFEST

```

- **MANIFEST_LIST** (opcional): nome ou referência de uma **lista de manifestos** (manifest list).
- **MANIFEST**: nome ou referência de uma **imagem** ou **lista de manifestos** a ser inspecionada.

> Se você passar apenas um único MANIFEST, o Docker exibirá o manifesto dessa imagem; se usar MANIFEST_LIST, exibirá o JSON que agrupa vários manifestos.
> 

---

## Opções

| Opção | Descrição |
| --- | --- |
| `--insecure` | Permite a comunicação com registries que não usam TLS válido (autoassinados ou sem certificado). |
| `-v, --verbose` | Exibe informações adicionais, incluindo a lista completa de camadas (`layers`) e detalhes de plataforma (`platform`). |

---

## Exemplo de Uso

1. **Inspecionar o manifesto de uma imagem única**
    
    ```bash
    docker manifest inspect nginx:latest
    
    ```
    
    Exibe o JSON do manifesto da imagem `nginx:latest`, mostrando digests e configurações padrão.
    
2. **Inspecionar uma lista de manifestos**
    
    ```bash
    docker manifest inspect myrepo/app:multiarch
    
    ```
    
    Retorna um JSON contendo um array `manifests`, cada item descrevendo uma plataforma (por exemplo, `amd64/linux`, `arm64/linux`).
    
3. **Exibir detalhes completos com `-verbose`**
    
    ```bash
    docker manifest inspect --verbose myrepo/app:multiarch
    
    ```
    
    Além dos metadados básicos, mostra todas as camadas e suas ordens, útil para depuração de compatibilidade.
    
4. **Inspecionar em registry inseguro**
    
    ```bash
    docker manifest inspect --insecure internal-registry.local:5000/myapp:dev
    
    ```
    
    Permite puxar e inspecionar o manifesto mesmo que o registry use certificados autoassinados.
    

---

> Dica:
> 
> 
> Combine com `docker manifest annotate` para ajustar plataformas e depois valide com `inspect` antes de dar `docker manifest push`. Isso garante que o manifest list final contenha exatamente os alvos de arquitetura desejados.
>
