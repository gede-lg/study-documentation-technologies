# Install

## Visão Geral

O comando `docker plugin install` baixa, instala e habilita um plugin Docker, solicitando ao usuário a concessão de permissões necessárias. Se o plugin não existir localmente, ele é buscado no registry (mínima versão de registry suportada: 2.3.0). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/plugin/install/?utm_source=chatgpt.com), [docs.docker.com](https://docs.docker.com/engine/extend/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker plugin install [OPÇÕES] PLUGIN [KEY=VALUE...]

```

- **PLUGIN**
    
    Nome (e opcionalmente tag) do plugin, por exemplo `fluent/fluentd` ou `rexray/s3fs`.
    
- **[KEY=VALUE...]**
    
    Parâmetros específicos de configuração exigidos pelo plugin (ex.: credenciais, endpoints).
    

---

## Opções

| Opção | Descrição |
| --- | --- |
| `--alias string` | Define um nome local alternativo para o plugin (ex.: `--alias rclone`). |
| `--disable` | Instala o plugin **sem** habilitá-lo imediatamente (`ENABLED=false`). |
| `--disable-content-trust` | Pula a verificação de assinatura do plugin (útil para registries não confiáveis). |
| `--grant-all-permissions` | Concede **automaticamente** todas as permissões solicitadas pelo plugin, sem prompt interativo. |

Todas as opções acima estão documentadas no help oficial do Docker CLI. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/plugin/install/?utm_source=chatgpt.com))

---

## Parâmetros de Configuração

Alguns plugins requerem parâmetros adicionais na forma `KEY=VALUE`, que podem incluir:

- Credenciais de acesso (ex.: `AWS_ACCESS_KEY_ID=…`, `AWS_SECRET_ACCESS_KEY=…`).
- Endpoints customizados (ex.: `SSH_USER=…`, `SSH_HOST=…`).
- Qualquer outra variável de ambiente ou configuração exigida pelo plugin.

Esses pares são repassados ao momento da instalação e ficam gravados na configuração do plugin. ([docs.docker.com](https://docs.docker.com/engine/extend/?utm_source=chatgpt.com))

## Exemplos de Uso

1. **Instalar e habilitar plugin padrão**
    
    ```bash
    docker plugin install rexray/s3fs
    
    ```
    
    Será exibido um resumo das permissões solicitadas, e após confirmação (`y`), o plugin é instalado e ativado. ([docs.docker.com](https://docs.docker.com/engine/extend/?utm_source=chatgpt.com))
    
2. **Instalar com alias**
    
    ```bash
    docker plugin install --alias rclone rclone/docker-volume-rclone
    
    ```
    
    Cria o plugin sob o nome local `rclone` em vez de usar sua full-reference.
    
3. **Instalar sem habilitar**
    
    ```bash
    docker plugin install --disable vieux/sshfs SSH_USER=labex SSH_HOST=localhost
    
    ```
    
    Instala o plugin `vieux/sshfs` configurado com `SSH_USER` e `SSH_HOST`, mas mantém seu estado como `ENABLED=false`. ([labex.io](https://labex.io/tutorials/docker-how-to-use-docker-plugin-install-command-to-install-and-manage-plugins-555191?utm_source=chatgpt.com))
    
4. **Conceder todas as permissões sem prompt**
    
    ```bash
    docker plugin install --grant-all-permissions fluent/fluentd
    
    ```
    
    Instala e habilita o plugin de logging Fluentd sem solicitar confirmação.
    
5. **Pular verificação de assinatura**
    
    ```bash
    docker plugin install --disable-content-trust secure-plugin:latest
    
    ```
    
    Usa o plugin mesmo que sua assinatura não seja verificada ou confiável.
    

---

> Dica:
> 
> 
> Após a instalação, verifique o status com `docker plugin ls` e, se necessário, desative ou reconfigure parâmetros adicionais usando `docker plugin disable` e `docker plugin inspect`.
