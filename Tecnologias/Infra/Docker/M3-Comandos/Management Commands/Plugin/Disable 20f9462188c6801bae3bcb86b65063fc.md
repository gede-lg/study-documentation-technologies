# Disable

## Visão Geral

O comando `docker plugin disable` desativa um plugin Docker que esteja habilitado, interrompendo sua operação sem removê-lo do sistema. O estado “DISABLED” mantém o plugin instalado, mas inativo, permitindo reativação posterior com `docker plugin enable`.

---

## Sintaxe

```bash
docker plugin disable [OPÇÕES] PLUGIN

```

- **PLUGIN**: nome (e opcionalmente tag) do plugin a desativar (ex.: `meussh/volume-plugin:latest`).

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | Força a desativação mesmo que containers estejam utilizando o plugin ativo. |

---

## Exemplos de Uso

1. **Desativar um plugin**
    
    ```bash
    docker plugin disable meussh/volume-plugin:latest
    
    ```
    
    Define o plugin como “DISABLED”, mas mantém sua instalação local.
    
2. **Forçar desativação**
    
    ```bash
    docker plugin disable --force traefik:latest
    
    ```
    
    Desativa o plugin mesmo que ele esteja em uso por containers, enquanto os desconecta automaticamente.
    
3. **Verificar estado antes e depois**
    
    ```bash
    docker plugin ls
    # Habilitado: ENABLED=true
    docker plugin disable meussh/volume-plugin:latest
    docker plugin ls
    # Agora: ENABLED=false
    
    ```
    

---

> Dica:
> 
> 
> Use `docker plugin inspect <PLUGIN>` para verificar dependências ou containers que estejam utilizando o plugin antes de desativar, garantindo um desligamento controlado.
