# Enable

## Visão Geral

O comando `docker plugin enable` ativa um plugin já instalado no host Docker, tornando-o funcional para containers que dependam de suas funcionalidades. Até então o plugin deve estar instalado e desativado (`ENABLED=false`). Após a habilitação, seu estado muda para `ENABLED=true`.

---

## Sintaxe

```bash
docker plugin enable [OPÇÕES] PLUGIN

```

- **PLUGIN**: nome (e opcionalmente tag) do plugin a habilitar (por exemplo, `meussh/volume-plugin:latest`).

---

## Opções

| Opção | Descrição |
| --- | --- |
| `--timeout int` | Tempo máximo (em segundos) que o cliente HTTP aguardará pela resposta do daemon (padrão 30s). |

---

## Exemplos de Uso

1. **Habilitar um plugin simples**
    
    ```bash
    docker plugin enable meussh/volume-plugin:latest
    
    ```
    
    Ativa imediatamente o plugin, permitindo que volumes sejam gerenciados por ele.
    
2. **Aumentar o timeout de comunicação**
    
    ```bash
    docker plugin enable --timeout 60 traefik:2.9
    
    ```
    
    Usa 60 segundos de timeout para evitar falha em conexões lentas ou sobrecarregadas ao daemon.
    
3. **Verificar antes e depois**
    
    ```bash
    docker plugin ls
    # Antes: ENABLED=false
    docker plugin enable meussh/volume-plugin:latest
    docker plugin ls
    # Depois: ENABLED=true
    
    ```
    

---

> Dica:
> 
> 
> Se a habilitação falhar por timeout ou erro de comunicação, verifique os logs do daemon (`journalctl -u docker.service` ou `docker plugin inspect <PLUGIN>`) e ajuste o `--timeout` conforme a necessidade.
>
