# RM

## Visão Geral

O comando `docker plugin rm` remove um ou mais plugins instalados localmente no Docker Engine. Por padrão, só é possível remover plugins que estejam desativados; para forçar a remoção de plugins ativos, utilize a opção `--force`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/plugin/rm/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker plugin rm [OPÇÕES] PLUGIN [PLUGIN...]

```

- **PLUGIN [PLUGIN…]**: um ou mais nomes (ou IDs) de plugins a serem removidos.

**Aliases:**

- `docker plugin remove`

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | Força a remoção de um plugin mesmo que esteja **habilitado** (`ENABLED=true`). Útil para limpar plugins em uso. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/plugin/rm/?utm_source=chatgpt.com)) |

---

## Exemplos de Uso

1. **Remover um plugin desativado**
    
    ```bash
    # Supondo que "meussh/volume-plugin" já esteja desativado:
    docker plugin rm meussh/volume-plugin:latest
    
    ```
    
2. **Remover múltiplos plugins de uma vez**
    
    ```bash
    docker plugin rm pluginA pluginB pluginC
    
    ```
    
3. **Forçar remoção de um plugin ativo**
    
    ```bash
    # Mesmo que o plugin esteja habilitado, ele será removido imediatamente:
    docker plugin rm --force traefik:latest
    
    ```
    

> Importante: o uso de --force não é recomendado em ambientes de produção, pois pode afetar containers que dependam do plugin removido.
