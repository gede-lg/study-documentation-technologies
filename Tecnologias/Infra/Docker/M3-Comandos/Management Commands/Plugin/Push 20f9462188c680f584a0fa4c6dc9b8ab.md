# Push

## Visão Geral

O comando `docker plugin push` envia um plugin previamente criado ou habilitado para um registry Docker, permitindo que ele seja instalado em outros hosts por meio de `docker plugin install`. Por padrão, o Docker tenta assinar (sign) o artefato; você pode desabilitar essa verificação se necessário.

---

## Sintaxe

```bash
docker plugin push [OPÇÕES] PLUGIN[:TAG]

```

- **PLUGIN[:TAG]**: nome (e opcionalmente tag) do plugin que você deseja enviar ao registry (ex.: `meussh/volume-plugin:latest`).

---

## Opções

| Opção | Descrição |
| --- | --- |
| `--disable-content-trust` | Não realiza a verificação ou assinatura do plugin durante o push (útil para registries sem suporte a assinatura). |

---

## Exemplos de Uso

1. **Push de um plugin com assinatura**
    
    ```bash
    docker plugin push meussh/volume-plugin:latest
    
    ```
    
    Envia o plugin ao registry e gera/valida sua assinatura digital.
    
2. **Push sem verificação de assinatura**
    
    ```bash
    docker plugin push --disable-content-trust meussh/volume-plugin:latest
    
    ```
    
    Envia o plugin mesmo que o registry não suporte ou não permita assinatura.
    
3. **Enviar outra tag**
    
    ```bash
    docker plugin push meussh/volume-plugin:v1.0.0
    
    ```
    
    Permite versionar e distribuir múltiplas versões do mesmo plugin.
    

---

> Dica:
> 
> - Após o push, verifique se o plugin está acessível no registry com `docker plugin install <registry>/meussh/volume-plugin:latest`.
> - Em ambientes corporativos onde a assinatura de conteúdo não é necessária, use sempre `-disable-content-trust` para evitar erros de assinatura.
