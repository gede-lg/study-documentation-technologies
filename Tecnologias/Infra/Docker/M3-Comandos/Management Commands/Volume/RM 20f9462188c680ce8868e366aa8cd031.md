# RM

A seguir, uma explicação completa e estruturada do comando **`docker volume rm`**, abrangendo sua finalidade, sintaxe, parâmetros, opções, fluxo de funcionamento e exemplos práticos.

---

## 1. O que faz este comando

O **`docker volume rm`** remove um ou mais **volumes** Docker do host.

- Não é possível remover um volume que esteja **montado** em algum contêiner em execução (a menos que use `-force`).
- Use-o para liberar espaço ou descartar dados persistentes que não são mais necessários.

---

## 2. Sintaxe geral

```bash
docker volume rm [OPTIONS] VOLUME [VOLUME...]

```

Você também pode usar o alias:

```bash
docker volume remove [OPTIONS] VOLUME [VOLUME...]

```

---

## 3. Parâmetros

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `VOLUME` | Sim | Nome do volume a ser removido. Pode especificar **vários** nomes separados por espaço. |

---

## 4. Opções disponíveis

| Opção | Tipo | Padrão | Finalidade |
| --- | --- | --- | --- |
| `-f`, `--force` | boolean | `false` | Força a remoção de volumes mesmo que estejam montados em contêineres (isto pode interromper containers que os usam). |

---

## 5. Fluxo de funcionamento

1. **Validação de existência**
    - Verifica se cada volume informado existe; em caso negativo, exibe erro.
2. **Checagem de uso**
    - Se `-force` não for usado, impede remoção de volumes montados.
3. **Remoção**
    - Invoca o driver de volume para excluir os dados do volume e atualiza o registro local.
4. **Retorno**
    - Para cada volume removido com sucesso, imprime o nome na saída padrão.

---

## 6. Exemplos de uso

1. **Remover um volume não em uso**
    
    ```bash
    docker volume rm meu_volume
    # Saída:
    # meu_volume
    
    ```
    
2. **Remover vários volumes de uma vez**
    
    ```bash
    docker volume rm dados_app cache_logs backups
    # Saída:
    # dados_app
    # cache_logs
    # backups
    
    ```
    
3. **Tentar remover um volume em uso (sem força)**
    
    ```bash
    docker volume rm shared_data
    # Erro:
    # Error response from daemon: remove shared_data: volume is in use - [container_id]
    
    ```
    
4. **Forçar remoção de um volume em uso**
    
    ```bash
    docker volume rm --force shared_data
    # Saída:
    # shared_data
    
    ```
