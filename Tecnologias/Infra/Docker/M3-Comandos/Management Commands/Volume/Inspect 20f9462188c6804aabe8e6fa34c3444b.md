# Inspect

A seguir, uma explicação completa e estruturada do comando **`docker volume inspect`**, abrangendo sua finalidade, sintaxe, parâmetros, opções e exemplos de uso.

---

## 1. O que faz este comando

O **`docker volume inspect`** exibe informações detalhadas sobre um ou mais volumes Docker existentes.

É útil para obter dados como:

- **Driver** usado (ex.: `local`, `nfs`, etc.);
- **Ponto de montagem** no host;
- **Labels** e **opções** específicas definidas na criação;
- **Data de criação** e **escopo** (local ou global).

---

## 2. Sintaxe geral

```bash
docker volume inspect [OPTIONS] VOLUME [VOLUME...]

```

- **`VOLUME`**: nome(s) do(s) volume(s) a ser(em) inspecionado(s).
- Você pode passar **vários volumes** de uma só vez, separados por espaço.

---

## 3. Parâmetros

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `VOLUME` | Sim | Nome do volume Docker a ser inspecionado (ex.: `meu_volume`). Pode repetir para múltiplos volumes. |

---

## 4. Opções disponíveis

| Opção | Tipo | Padrão | Finalidade |
| --- | --- | --- | --- |
| `-f`, `--format` | string | JSON | Formato de saída usando template Go: |
- **`json`**: imprime em JSON bruto (padrão).
- **`TEMPLATE`**: aplica template customizado para exibir apenas campos específicos. |

> Referência de templates Go:
> 
> 
> [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/)
> 

---

## 5. Fluxo de funcionamento

1. **Validação de parâmetros**
    - Verifica se cada nome de volume existe; caso não, retorna erro indicando volume não encontrado.
2. **Coleta de metadados**
    - Consulta o daemon Docker para recuperar as propriedades do(s) volume(s).
3. **Formatação da saída**
    - Se `-format` estiver presente, aplica o template especificado;
    - Caso contrário, retorna JSON completo com todos os campos.

---

## 6. Exemplos de uso

1. **Inspecionar um volume em JSON bruto**
    
    ```bash
    docker volume inspect meu_volume
    
    ```
    
    ```json
    [
      {
        "CreatedAt": "2025-06-10T20:45:30Z",
        "Driver": "local",
        "Labels": {
          "project": "backend"
        },
        "Mountpoint": "/var/lib/docker/volumes/meu_volume/_data",
        "Name": "meu_volume",
        "Options": {
          "o": "addr=192.168.0.10,rw",
          "device": ":/export/data",
          "type": "nfs"
        },
        "Scope": "local"
      }
    ]
    
    ```
    
2. **Inspecionar múltiplos volumes**
    
    ```bash
    docker volume inspect vol1 vol2
    
    ```
    
3. **Exibir apenas o ponto de montagem usando template**
    
    ```bash
    docker volume inspect --format '{{ .Mountpoint }}' meu_volume
    
    ```
    
    ```
    /var/lib/docker/volumes/meu_volume/_data
    
    ```
    
4. **Mostrar driver e labels em formato customizado**
    
    ```bash
    docker volume inspect --format 'Driver: {{ .Driver }}, Labels: {{ json .Labels }}' meu_volume
    
    ```
    
    ```
    Driver: local, Labels: {"project":"backend"}
    
    ```
