# LS

---

## 1. O que faz este comando

O **`docker volume ls`** lista todos os *volumes* Docker disponíveis no host, permitindo que você:

- Veja quais volumes existem
- Aplique filtros para exibir apenas volumes “pendentes” (*dangling*) ou pertencentes a um driver específico
- Formate a saída em tabela, JSON ou via *template*

---

## 2. Sintaxe geral

```bash
docker volume ls [OPTIONS]

```

Como alias, você também pode usar:

```bash
docker volume list [OPTIONS]

```

---

## 3. Parâmetros

Este comando **não** recebe parâmetros posicionais — somente as opções listadas a seguir.

---

## 4. Opções disponíveis

| Opção | Tipo | Padrão | Finalidade |
| --- | --- | --- | --- |
| `-f`, `--filter <filtro>` | string | — | Filtra volumes por condição. Ex.: `dangling=true`, `driver=local`. Veja abaixo exemplos de filtros comuns. |
| `--format <string>` | string | `table` | Formata a saída: |
- `table` (padrão): tabela com cabeçalhos
- `json`: JSON bruto
- `TEMPLATE`: Go template personalizado (ver [docs de formatação](https://docs.docker.com/go/formatting/)) |
    
    | `-q`, `--quiet`           | boolean  | `false`       | Exibe **apenas** os nomes dos volumes, um por linha, sem cabeçalhos nem colunas adicionais.                        |
    

---

### Exemplos de filtros (`-filter`)

- `dangling=true` — volumes órfãos (não usados por nenhum container)
- `driver=local` — volumes criados com o driver “local”
- `label=env=prod` — volumes com label `env=prod`

---

## 5. Fluxo de funcionamento

1. **Coleta de lista**
    - Docker consulta seu armazenamento interno de volumes.
2. **Aplicação de filtros**
    - Se `-filter` for usado, aplica cada expressão de filtro em série.
3. **Formatação**
    - Sem `-format`/`q`: exibe tabela com colunas `DRIVER` e `VOLUME NAME`.
    - Com `-format`: aplica o modelo escolhido.
    - Com `q`: exibe somente os nomes.

---

## 6. Exemplos de uso

1. **Listar todos os volumes em tabela (padrão)**
    
    ```bash
    docker volume ls
    # DRIVER    VOLUME NAME
    # local     meu_volume
    # local     7b9f1a3c6e1f3d9a2b4c5d6e7f8g9h0i
    
    ```
    
2. **Listar apenas volumes órfãos**
    
    ```bash
    docker volume ls --filter dangling=true
    # DRIVER    VOLUME NAME
    # local     7b9f1a3c6e1f3d9a2b4c5d6e7f8g9h0i
    
    ```
    
3. **Listar nomes somente (quiet)**
    
    ```bash
    docker volume ls -q
    # meu_volume
    # 7b9f1a3c6e1f3d9a2b4c5d6e7f8g9h0i
    
    ```
    
4. **Listar em JSON**
    
    ```bash
    docker volume ls --format json
    # [
    #   {
    #     "Driver": "local",
    #     "Name": "meu_volume"
    #   },
    #   {
    #     "Driver": "local",
    #     "Name": "7b9f1a3c6e1f3d9a2b4c5d6e7f8g9h0i"
    #   }
    # ]
    
    ```
    
5. **Listar com template personalizado**
    
    ```bash
    docker volume ls --format "{{.Name}}\t{{.Driver}}"
    # meu_volume    local
    # 7b9f1a3c6e1f3d9a2b4c5d6e7f8g9h0i    local
    
    ```

## 7. Tabela de referência rápida

| Componente | Sintaxe | Descrição |
| --- | --- | --- |
| Comando base | `docker volume ls [OPTIONS]` | Lista os volumes Docker no host. |
| Opção | `-f, --filter <filtro>` | Filtra volumes por condições (ex.: `dangling=true`). |
| Opção | `--format <string>` | Formata a saída (tabela, JSON ou Go template). |
| Opção | `-q, --quiet` | Exibe apenas os nomes dos volumes, um por linha. |
