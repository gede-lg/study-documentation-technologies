# Export

## Visão Geral

O comando `docker context export` permite **exportar** toda a configuração de um contexto Docker existente para um arquivo TAR ou para o fluxo de saída padrão (STDOUT). Esse arquivo pode ser compartilhado e posteriormente **importado** em outro host com `docker context import`, garantindo que as mesmas definições de endpoint e metadados sejam replicadas. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/export/))

---

## Sintaxe

```bash
docker context export [OPTIONS] CONTEXT [FILE|-]

```

- **CONTEXT**
    
    Nome do contexto a ser exportado.
    
- **FILE**
    
    Caminho do arquivo de saída (ex.: `meu-contexto.dockercontext`).
    
- 
    
    Use `-` em vez de **FILE** para enviar o TAR ao STDOUT. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/export/))
    

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTEXT` | Nome do contexto Docker que você deseja exportar (deve existir; veja `docker context ls`). |
| `FILE` | Arquivo de destino onde o TAR será gravado. Se omitido e sem `-`, o Docker usa `<CONTEXT>.dockercontext` por padrão. |
| `-` | Em vez de gravar em arquivo, envia o fluxo TAR para STDOUT, permitindo uso de pipes ou redirecionamentos em scripts. |

---

## Opções

Atualmente, não há opções específicas documentadas para `docker context export` além do placeholder `[OPTIONS]`. Todas as configurações (como `--kubeconfig`) são gerenciadas internamente pelo Docker CLI ou pela build específica do cliente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/export/))

---

## Exemplos de Uso

1. **Exportar para arquivo padrão**
    
    ```bash
    docker context export producao
    
    ```
    
    Gera o arquivo `producao.dockercontext` no diretório atual. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/export/))
    
2. **Exportar para arquivo customizado**
    
    ```bash
    docker context export dev-local ./configs/dev.dockercontext
    
    ```
    
    Grava a configuração do contexto `dev-local` em `./configs/dev.dockercontext`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/export/))
    
3. **Exportar via STDOUT e compactar**
    
    ```bash
    docker context export teste - | gzip > teste.dockercontext.tar.gz
    
    ```
    
    Envia o TAR para STDOUT, compacta com `gzip` e salva em um arquivo `.tar.gz`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/export/))
    
4. **Exportar e importar em outro host**
    
    ```bash
    # No host A
    docker context export staging > staging.dockercontext
    
    # Transferir staging.dockercontext para o host B e, nele:
    docker context import staging staging.dockercontext
    
    ```
    
    Mantém a mesma configuração sem precisar recriar manualmente. ([docs.docker.com](https://docs.docker.com/engine/manage-resources/contexts/?utm_source=chatgpt.com))
    

---

> Dica:
> 
> - Use `docker context ls` para listar e confirmar o nome exato do contexto antes de exportar.
> - Combine com controle de versão (por exemplo, Git) para rastrear mudanças na configuração de seus contextos.
