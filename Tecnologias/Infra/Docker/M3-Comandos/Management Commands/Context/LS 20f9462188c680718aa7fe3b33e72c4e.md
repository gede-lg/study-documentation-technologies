# LS

## Visão Geral

O comando `docker context ls` lista todos os contextos Docker configurados no cliente, indicando qual está ativo com um asterisco (`*`). Contextos agrupam configurações de endpoint e facilitam alternar entre ambientes (local, remoto, cloud). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker context ls [OPÇÕES]

```

([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))

---

## Aliases

- `docker context list`
- `docker context ps`
- `docker context ls` (mesma forma)
- `docker ps` (quando usado globalmente com `-context`) ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))

---

## Opções

| Opção | Padrão | Descrição |
| --- | --- | --- |
| `--format string` | `table` | Formata a saída usando um template Go:• `table`: tabela com cabeçalhos (padrão)• `table TEMPLATE`: tabela customizada• `json`: JSON bruto• `TEMPLATE`: Go template livre.Veja detalhes em [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/) ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com)) |
| `-q, --quiet` | `false` | Exibe apenas os **nomes** dos contextos, um por linha. |

## Exemplos de Uso

1. **Listar todos os contextos**
    
    ```bash
    $ docker context ls
    NAME                DESCRIPTION                               DOCKER ENDPOINT                      ORCHESTRATOR
    default *           Current DOCKER_HOST based configuration   unix:///var/run/docker.sock          swarm
    production          Production cluster                        tcp://prod.example.com:2376          swarm
    staging             Staging cluster                           tcp://stage.example.com:2376         swarm
    
    ```
    
    ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))
    
2. **Exibir somente nomes dos contextos**
    
    ```bash
    $ docker context ls --quiet
    default
    production
    staging
    
    ```
    
    ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))
    
3. **Formato JSON para processamento**
    
    ```bash
    $ docker context ls --format=json
    [
      {
        "Name": "default",
        "Current": true,
        "Description": "Current DOCKER_HOST based configuration",
        "Endpoints": { "docker": { "Host": "unix:///var/run/docker.sock" } },
        "Orchestrator": "swarm"
      },
      { ... }
    ]
    
    ```
    
    ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))
    
4. **Template customizada**
    
    ```bash
    $ docker context ls --format "Context: {{.Name}} ({{.Description}})"
    Context: default (Current DOCKER_HOST based configuration)
    Context: production (Production cluster)
    Context: staging (Staging cluster)
    
    ```
    
    ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/ls/?utm_source=chatgpt.com))
