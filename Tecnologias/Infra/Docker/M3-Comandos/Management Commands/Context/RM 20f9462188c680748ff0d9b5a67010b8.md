# RM

## Visão Geral

O comando `docker context rm` remove um ou mais **contexts** configurados no seu cliente Docker, permitindo limpar entradas que não são mais necessárias ou corrigir configurações antigas. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/rm/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker context rm [OPÇÕES] CONTEXT [CONTEXT...]

```

- **CONTEXT**
    
    Nome de um ou mais contexts já existentes a serem removidos. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/rm/?utm_source=chatgpt.com))
    

---

## Aliases

- `docker context remove`

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTEXT` | Nome de um context a ser removido. |
| `[CONTEXT...]` | Lista adicional de contexts; todos os informados serão removidos em sequência. |

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | Força a remoção de um context mesmo que ele esteja **ativo** no momento, sem alternar antes. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/rm/?utm_source=chatgpt.com), [manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/questing/en/man1/docker-context-rm.1.html?utm_source=chatgpt.com)) |

---

## Exemplos de Uso

1. **Remover um context inativo**
    
    ```bash
    docker context rm staging
    
    ```
    
    Remove o context `staging` apenas se ele não estiver em uso.
    
2. **Remover múltiplos contexts de uma só vez**
    
    ```bash
    docker context rm test dev temp
    
    ```
    
    Remove os contexts `test`, `dev` e `temp`, respeitando o estado de cada um.
    
3. **Forçar remoção de um context em uso**
    
    ```bash
    docker context rm -f default
    
    ```
    
    Mesmo que `default` seja o context ativo, ele será excluído sem trocar o contexto corrente.
    

---

## Dica

Antes de remover, sempre liste seus contexts para confirmar nomes e identificar qual está ativo:

```bash
docker context ls

```

Caso tente remover o contexto em uso sem `--force`, o Docker impedirá a ação para evitar interromper operações em curso.
