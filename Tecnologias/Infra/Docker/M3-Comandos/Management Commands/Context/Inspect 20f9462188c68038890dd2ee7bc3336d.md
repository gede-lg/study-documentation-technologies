# Inspect

## Visão Geral

O comando `docker context inspect` exibe informações detalhadas (em JSON) sobre um ou mais contextos Docker configurados no cliente, incluindo metadados, endpoints e caminhos de armazenamento. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/inspect/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker context inspect [OPÇÕES] [CONTEXT] [CONTEXT...]

```

- **CONTEXT**: nome(s) do(s) contexto(s) a serem inspecionados. Se omitido, retorna detalhes de todos os contextos. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/inspect/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --format string` | Formata a saída usando um template Go.• `json`: imprime JSON puro.• `TEMPLATE`: aplica o template Go definido.Consulte [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/inspect/?utm_source=chatgpt.com)) |

## Exemplos de Uso

1. **Inspeccionar todos os contextos**
    
    ```bash
    docker context inspect
    
    ```
    
    Retorna um array JSON com detalhes de cada contexto configurado localmente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/inspect/?utm_source=chatgpt.com))
    
2. **Inspecionar um contexto específico**
    
    ```bash
    docker context inspect my-context
    
    ```
    
    Exibe apenas o objeto JSON correspondente ao contexto `my-context`, com campos como `Name`, `Metadata`, `Endpoints` e `Storage`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/inspect/?utm_source=chatgpt.com))
    
3. **Formatar saída como JSON explícito**
    
    ```bash
    docker context inspect my-context --format json
    
    ```
    
    Garante que a saída seja JSON, útil para pipelines de automação ou parsing com `jq`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/inspect/?utm_source=chatgpt.com))
    
4. **Extrair apenas o endpoint Docker de um contexto**
    
    ```bash
    docker context inspect my-context --format '{{index .Endpoints "docker" "Host"}}'
    
    ```
    
    Utiliza um template Go para imprimir somente o valor de `Host` dentro de `Endpoints.docker`. ([labex.io](https://labex.io/tutorials/docker-how-to-use-docker-context-inspect-command-to-view-context-details-555132?utm_source=chatgpt.com))
