# Import

## Visão Geral

O comando `docker context import` carrega uma configuração de contexto previamente exportada (com `docker context export`) a partir de um arquivo TAR ou ZIP, ou de um fluxo via STDIN, permitindo recriar rapidamente o mesmo ambiente de endpoints Docker em outro host. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/import/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker context import CONTEXT FILE|-

```

- **CONTEXT**: nome que você deseja dar ao contexto importado (deve ser único entre os contextos existentes).
- **FILE**: caminho para o arquivo TAR/ZIP gerado pela exportação.
- : usa o STDIN como fonte (leia o tar do fluxo de entrada). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/import/?utm_source=chatgpt.com))

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTEXT` | Nome que será atribuído ao contexto importado. |
| `FILE` | Arquivo de contexto (tar ou zip) gerado por `docker context export`. |
| `-` | Hífen para indicar que o tar será fornecido pela entrada padrão (STDIN). |

---

## Comportamento

- Se **FILE** já existir, ele será lido e importado como novo contexto.
- Usar  permite pipeline em scripts: por exemplo, `docker context export prod | docker context import prod -`. ([labex.io](https://labex.io/tutorials/docker-how-to-use-docker-context-import-command-to-import-a-context-555131?utm_source=chatgpt.com))

---

## Exemplos de Uso

1. **Importar contexto de arquivo salvo**
    
    ```bash
    docker context import staging staging.dockercontext
    
    ```
    
    Cria o contexto `staging` usando o arquivo `staging.dockercontext`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/import/?utm_source=chatgpt.com))
    
2. **Importar diretamente via STDIN**
    
    ```bash
    docker context export prod | docker context import prod -
    
    ```
    
    Exporta o contexto `prod` e já o importa em sequência, sem gravar arquivo intermediário. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/import/?utm_source=chatgpt.com), [labex.io](https://labex.io/tutorials/docker-how-to-use-docker-context-import-command-to-import-a-context-555131?utm_source=chatgpt.com))
    
3. **Importar com nome diferente**
    
    ```bash
    docker context import dev-from-prod prod.dockercontext
    
    ```
    
    Usa o arquivo de produção para criar um contexto `dev-from-prod` local. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/context/import/?utm_source=chatgpt.com))
