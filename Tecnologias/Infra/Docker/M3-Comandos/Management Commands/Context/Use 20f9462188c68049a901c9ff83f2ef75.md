# Use

## Visão Geral

O comando `docker context use` define o **contexto ativo** para o cliente Docker, alterando o ambiente (endpoints, orquestrador, credenciais) que será usado em comandos subsequentes. É essencial para alternar rapidamente entre diferentes clusters ou hosts sem redefinir variáveis de ambiente manualmente.

---

## Sintaxe

```bash
docker context use CONTEXT

```

- **CONTEXT**: nome do contexto que você deseja ativar (visível em `docker context ls`).

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTEXT` | Nome do contexto Docker que você quer tornar ativo. Deve corresponder a um entry em `docker context ls`. |

---

## Exemplos de Uso

1. **Alternar para um contexto remoto de produção**
    
    ```bash
    docker context use production
    
    ```
    
    A partir desse momento, todos os comandos Docker apontarão para o endpoint e orquestrador configurados no contexto `production`.
    
2. **Voltar ao contexto padrão (local)**
    
    ```bash
    docker context use default
    
    ```
    
    Restaura o uso do daemon Docker local (por socket Unix) e o orquestrador padrão.
    
3. **Incluir em scripts de CI/CD**
    
    ```bash
    # Em um pipeline, ative o contexto correto antes de executar deploy:
    docker context use staging
    docker stack deploy -c docker-compose.yml my_app
    
    ```
    

---

> Dica:
> 
> 
> Execute `docker context ls` para ver todos os contextos disponíveis e identificar qual está ativo (marcado com `*`) antes de usar `docker context use`.
