# Stop

O comando `docker buildx stop` é utilizado para **parar uma instância de construtor (builder) do Buildx**. Parar um construtor significa encerrar os contêineres associados a ele, liberando os recursos que estavam sendo utilizados, mas sem remover a configuração do construtor.

---

### Sintaxe Geral

A estrutura básica do comando é:

```bash
docker buildx stop [NOME_DO_BUILDER] [OPÇÕES]

```

Vamos analisar cada um dos parâmetros e opções.

---

### Parâmetros e Opções

| Parâmetro/Opção | Sintaxe de Uso | Finalidade | Observações |
| --- | --- | --- | --- |
| **`[NAME]`** | `docker buildx stop meu_builder` | Especifica o nome do construtor (builder) que você deseja parar. | Este é um argumento posicional e opcional. Se você não especificar um nome, o comando tentará parar o construtor atualmente ativo ou o especificado pela flag `--builder`. |
| **`--builder`** | `--builder meu_builder` | Sobrescreve a instância de construtor configurada como padrão ou a atualmente em uso. | Útil em scripts ou quando você quer ter certeza de que está parando o construtor correto, independentemente de qual esteja ativo no seu contexto. O valor padrão é `"default"`. |
| **`-D`, `--debug`** | `-D` ou `--debug` | Habilita o modo de depuração (debug). | Exibe logs detalhados sobre o processo de parada do construtor. É extremamente útil para diagnosticar problemas caso o comando não funcione como esperado. |

---

### Exemplos Práticos

Vamos ver como usar o comando em situações reais.

### 1. Parando o Construtor Padrão

Se você não especificar nenhum nome, o Docker tentará parar o construtor que está em uso no momento. Se você nunca criou ou selecionou outro, será o construtor chamado `default`.

```bash
# Para o construtor que está ativo no contexto atual
docker buildx stop

```

### 2. Parando um Construtor Específico pelo Nome

Imagine que você criou um construtor chamado `meu-builder-rapido`. Para pará-lo, você informa o nome dele diretamente.

```bash
# Para o construtor chamado 'meu-builder-rapido'
docker buildx stop meu-builder-rapido

```

### 3. Usando a Flag `-builder`

Este método é mais explícito e recomendado para evitar erros, especialmente em automações.

```bash
# Garante que o construtor 'meu-builder-rapido' seja parado,
# mesmo que outro esteja ativo no momento.
docker buildx stop --builder meu-builder-rapido

```

### 4. Usando o Modo Debug

Se o comando `docker buildx stop meu-builder-rapido` estiver falhando, você pode usar a flag de debug para obter mais informações sobre o erro.

```bash
# Tenta parar o construtor e exibe logs detalhados da operação
docker buildx stop --debug meu-builder-rapido

```
