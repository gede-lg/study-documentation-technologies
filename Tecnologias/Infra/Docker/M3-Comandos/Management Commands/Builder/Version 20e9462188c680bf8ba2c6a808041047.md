# Version

Olá, Gedê! Como A.R.I.A, sua Assistente Rápida para Idiotas Atarefados, estou aqui para detalhar o comando `docker buildx version` para você.

O comando que você executou, `docker builder version --help`, na verdade, direciona para a ajuda do `docker buildx version`. O **Buildx** é um plugin da CLI do Docker que estende o comando `docker build` com todos os recursos do Moby BuildKit, permitindo compilações mais rápidas, gerenciamento de builders e compilações multi-plataforma.

O comando `docker buildx version` serve para um propósito simples e direto: **exibir a versão do plugin Buildx instalado e seus componentes**.

---

### Sintaxe e Estrutura do Comando

A estrutura básica do comando é:

```bash
docker buildx version [OPTIONS]

```

- **`docker`**: O executável principal do Docker.
- **`buildx`**: O plugin que gerencia as compilações avançadas.
- **`version`**: O subcomando específico para obter informações de versão.
- **`[OPTIONS]`**: Argumentos opcionais que modificam o comportamento do comando, como `-help` ou `-debug`.

---

### Parâmetros e Opções

Aqui está uma tabela detalhando os parâmetros que você pode usar com este comando, conforme a saída do `--help` que você enviou.

| Parâmetro/Opção | Sintaxe de Uso | Finalidade |
| --- | --- | --- |
| **(nenhum)** | `docker buildx version` | Executa a ação padrão: exibe as informações de versão do buildx e do buildkit. |
| **`--help`** | `docker buildx version --help` | Mostra uma mensagem de ajuda, descrevendo o comando, seu uso e as opções disponíveis. Foi o que você usou para gerar a saída. |
| **`-D`, `--debug`** | `docker buildx version -D` ou `docker buildx version --debug` | Ativa o modo de depuração (debug). Isso fará com que o comando exiba logs muito mais detalhados sobre sua execução, útil para diagnosticar problemas. |

---

### Nota Sobre Comandos Experimentais

A mensagem `Experimental commands and flags are hidden. Set BUILDX_EXPERIMENTAL=1 to show them.` indica que o Buildx possui funcionalidades que ainda estão em fase de testes.

- **O que isso significa?** Recursos experimentais não são considerados estáveis para produção e podem mudar ou ser removidos em versões futuras.
- **Como ativá-los?** Para visualizar e usar esses recursos, você precisa definir a variável de ambiente `BUILDX_EXPERIMENTAL` como `1` antes de executar o comando.

**Exemplo no Linux/macOS:**

```bash
export BUILDX_EXPERIMENTAL=1
docker buildx --help

```

**Exemplo no Windows (PowerShell):**

```powershell
$env:BUILDX_EXPERIMENTAL="1"
docker buildx --help

```

### Exemplo Prático

Se você executar o comando sem nenhuma opção, terá uma saída parecida com esta:

**Comando:**

```bash
gedelg@DESKTOP-H3T36NM:~$ docker buildx version

```

**Saída Típica:**

```
github.com/docker/buildx v0.14.1 81814a5a3a1d13a48e78809c06342898b7921200

```

Esta saída mostra a **versão exata do plugin `buildx`** que está instalada no seu sistema.
