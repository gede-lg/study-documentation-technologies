# RM

**Visão Geral**

O comando `docker buildx rm` (alias para `docker builder rm`) serve para **remover** instâncias de *builders* do BuildKit. Se você informar um ou mais nomes, ele remove essas instâncias; se não informar nomes, remove a instância atualmente configurada (por padrão, chamada “default”) ([docs.docker.com](https://docs.docker.com/reference/cli/docker/buildx/rm/?utm_source=chatgpt.com)).

---

## Sintaxe

```bash
docker buildx rm [OPÇÕES] [NOME...]

```

- **OPÇÕES**: flags que alteram o comportamento da remoção.
- **NOME...**: lista de instâncias de *builder* a serem removidas.

---

## Parâmetros e Flags

| Flag | Sintaxe | Finalidade | Padrão |
| --- | --- | --- | --- |
| `--all-inactive` | `--all-inactive` | Remove **todas** as instâncias que estão **inativas** (ou seja, não em execução). | — |
| `--builder string` | `--builder <nome>` | Substitui o *builder* configurado no contexto atual pela instância `<nome>`. | `default` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/buildx/rm/?utm_source=chatgpt.com)) |
| `-D, --debug` | `-D` **ou** `--debug` | Ativa logs em nível de **depuração**, úteis para diagnosticar problemas internos. | desativado ([github.com](https://github.com/sigoden/argc-completions/blob/main/completions/docker.sh?utm_source=chatgpt.com)) |
| `-f, --force` | `-f` **ou** `--force` | Não solicita confirmação antes de remover (útil em scripts). | solicita confirmação ([docs.docker.com](https://docs.docker.com/reference/cli/docker/buildx/rm/?utm_source=chatgpt.com)) |
| `--keep-daemon` | `--keep-daemon` | Mantém o **daemon** do BuildKit em execução, mesmo após a remoção da instância. | remove o daemon  ([docs.docker.com](https://docs.docker.com/reference/cli/docker/buildx/rm/?utm_source=chatgpt.com)) |
| `--keep-state` | `--keep-state` | Preserva o **estado** do BuildKit (cache, registros) mesmo depois de remover o *builder*. | descarta estado ([docs.docker.com](https://docs.docker.com/reference/cli/docker/buildx/rm/?utm_source=chatgpt.com)) |

> Observação:
> 
> - Instâncias “inativas” são aquelas não marcadas como `running` em `docker buildx ls`.
> - Se nenhum nome for informado, remove o *builder* atualmente selecionado (por padrão, “default”) ([docs.docker.com](https://docs.docker.com/reference/cli/docker/buildx/rm/?utm_source=chatgpt.com)).

---

## Exemplos de Uso

1. **Remover um *builder* específico**
    
    ```bash
    docker buildx rm meu-builder
    
    ```
    
    Remove a instância chamada `meu-builder`, solicitando confirmação.
    
2. **Forçar remoção sem confirmação**
    
    ```bash
    docker buildx rm meu-builder --force
    
    ```
    
    Remove `meu-builder` imediatamente, sem prompt.
    
3. **Remover todas as instâncias inativas**
    
    ```bash
    docker buildx rm --all-inactive
    
    ```
    
    Lista e remove todas as instâncias não em execução.
    
4. **Remover instância configurada apenas mantendo o daemon vivo**
    
    ```bash
    docker buildx rm --keep-daemon
    
    ```
    
    Remove a instância atual, mas mantém o serviço do BuildKit ativo.
