# RM

## Visão Geral

O comando `docker network rm` remove uma ou mais redes Docker do host. Ele falha se a rede estiver em uso (a menos que use `--force`) ou se a rede não existir (com `--force` não gera erro).

---

## Sintaxe

```bash
docker network rm [OPÇÕES] REDE [REDE...]

```

- **REDE [REDE…]**: nome(s) ou ID(s) da(s) rede(s) a serem removida(s).

---

## Aliases

- `docker network remove`

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | Não retorna erro se a rede não existir; também força a remoção mesmo que esteja em uso. |

---

## Exemplos de Uso

1. **Remover uma rede específica**
    
    ```bash
    docker network rm minha-rede
    
    ```
    
    Remove `minha-rede` se ela não estiver conectada a nenhum container.
    
2. **Remover múltiplas redes de uma só vez**
    
    ```bash
    docker network rm rede1 rede2 rede3
    
    ```
    
    Tenta remover `rede1`, `rede2` e `rede3` em sequência.
    
3. **Forçar remoção independentemente de uso ou existência**
    
    ```bash
    docker network rm --force rede-antiga
    
    ```
    
    Não importa se `rede-antiga` está em uso ou se sequer existe, o comando não abortará com erro.
    

---

> Dica:
> 
> 
> Antes de remover, use `docker network ls` para listar redes e identificar nomes/IDs. Se quiser limpar todas as redes não utilizadas, considere `docker network prune`.
>
