# LS

## Visão Geral

O comando `docker network ls` lista todas as redes gerenciadas pelo daemon Docker, exibindo para cada uma o ID, nome, driver e escopo. Por padrão, mostra apenas as redes ativas no host atual. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/ls/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker network ls [OPÇÕES]

```

**Aliases:**

- `docker network list` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/ls/?utm_source=chatgpt.com))

---

## Opções Principais

| Opção | Descrição |
| --- | --- |
| `-f, --filter filter` | Filtra a saída por condições no formato `chave=valor` (ex.: `driver=bridge`, `name=frontend`, `scope=swarm`, `type=custom`). ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com)) |
| `--format string` | Formata a saída usando um template Go:• `table`: tabela com cabeçalhos (padrão)• `table TEMPLATE`• `json`• `TEMPLATE`. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com)) |
| `-n, --no-trunc` | Não trunca campos como IDs ou nomes longos; mostra valores completos. ([labex.io](https://labex.io/tutorials/docker-how-to-fix-docker-network-ls-command-not-found-error-417656?utm_source=chatgpt.com)) |
| `-q, --quiet` | Exibe apenas os **IDs** das redes, um por linha. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com)) |

---

### Exemplos de Filtros (`-filter`)

- **Por driver:**
    
    ```bash
    docker network ls --filter driver=bridge
    
    ```
    
    Mostra só redes que usam o driver `bridge`. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/bionic/man1/docker-network-ls.1.html?utm_source=chatgpt.com))
    
- **Por nome:**
    
    ```bash
    docker network ls --filter name=frontend
    
    ```
    
    Exibe redes cujo nome contenha “frontend”. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com))
    
- **Por escopo:**
    
    ```bash
    docker network ls --filter scope=local
    
    ```
    
    Lista apenas redes de escopo `local`. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com))
    
- **Por tipo:**
    
    ```bash
    docker network ls --filter type=custom
    
    ```
    
    Filtra redes criadas pelo usuário (não as “builtin”). ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com))

## Exemplos de Uso

1. **Listar todas as redes**
    
    ```bash
    docker network ls
    
    ```
    
    Saída típica:
    
    ```
    NETWORK ID     NAME      DRIVER    SCOPE
    1befe23acd58   bridge    bridge    local
    726ead8f4e6b   host      host      local
    ef4896538cc7   none      null      local
    ``` :contentReference[oaicite:10]{index=10}
    
    ```
    
2. **Mostrar IDs completos (sem truncar)**
    
    ```bash
    docker network ls --no-trunc
    
    ```
    
    Revela o ID completo de cada rede. ([labex.io](https://labex.io/tutorials/docker-how-to-fix-docker-network-ls-command-not-found-error-417656?utm_source=chatgpt.com))
    
3. **Exibir só os IDs das redes**
    
    ```bash
    docker network ls -q
    
    ```
    
    Lista apenas valores como `1befe23acd58`. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com))
    
4. **Formatar com Go template**
    
    ```bash
    docker network ls --format "table {{.ID}}\t{{.Name}}\t{{.Driver}}"
    
    ```
    
    Customiza colunas para ID, nome e driver. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com))
    
5. **Excluir todas as redes definidas pelo usuário**
    
    ```bash
    docker network rm $(docker network ls -q --filter type=custom)
    
    ```
    
    Remove em lote redes “custom” usando o ID retornado por `-q --filter`. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_ls/?utm_source=chatgpt.com))

> Dica:
> 
> - Combine `docker network ls` com `docker network inspect <rede>` para investigar configurações detalhadas de IPAM, containers conectados e opções de driver.
> - Para limpar redes não utilizadas, use `docker network prune`.
