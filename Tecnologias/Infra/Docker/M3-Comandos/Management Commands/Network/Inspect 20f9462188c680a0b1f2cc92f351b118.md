# Inspect

## Visão Geral

O comando `docker network inspect` exibe informações detalhadas (em JSON) sobre uma ou mais redes Docker configuradas, incluindo suas sub-redes, gateways, containers conectados e opções de driver. É essencial para diagnosticar configurações de rede e entender como containers estão interligados. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_inspect/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker network inspect [OPÇÕES] REDE [REDE...]

```

- **REDE [REDE…]**: um ou mais nomes ou IDs de redes a serem inspecionadas.
- Se nenhum nome for informado, o comando retornará um erro solicitando pelo menos uma rede. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/inspect/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --format string` | Formata a saída usando um template Go:• `json`: imprime o JSON puro (default).• `TEMPLATE`: aplica um Go template.Consulte [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/inspect/?utm_source=chatgpt.com)) |
| `-v, --verbose` | Saída verbosa para diagnóstico, incluindo detalhes extras de configuração interna e estatísticas de driver.    ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/inspect/?utm_source=chatgpt.com)) |

## Exemplos de Uso

1. **Inspecionar uma rede específica**
    
    ```bash
    docker network inspect minha-rede
    
    ```
    
    Retorna um array JSON com objetos contendo campos como `Name`, `Id`, `Driver`, `IPAM`, `Containers`, etc. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_inspect/?utm_source=chatgpt.com))
    
2. **Inspecionar múltiplas redes de uma vez**
    
    ```bash
    docker network inspect frontend-net backend-net
    
    ```
    
    Produz dois objetos JSON, um para cada rede, facilitando comparações e auditorias.
    
3. **Formatar a saída para exibir apenas o driver**
    
    ```bash
    docker network inspect -f '{{.Driver}}' minha-rede
    
    ```
    
    Imprime somente o nome do driver (por ex., `bridge`, `overlay`).
    
4. **Obter saída verbosa para diagnóstico**
    
    ```bash
    docker network inspect --verbose minha-rede
    
    ```
    
    Inclui seções adicionais, como `Options` específicas do driver e detalhes de IPAM. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/inspect/?utm_source=chatgpt.com))
