# Create

## Visão Geral

O comando `docker network create` cria uma nova rede Docker, definindo seu driver, configurações de IPAM (subnet, gateway, IP range, aux-address) e outras opções, para que containers conectados possam se comunicar de acordo com as políticas e isolamento desejados. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker network create [OPÇÕES] NOME_DA_REDE

```

- **NOME_DA_REDE**: identificador único da rede a ser criada. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com))

---

## Opções

| Opção | Padrão | Descrição |
| --- | --- | --- |
| `-d, --driver string` | `bridge` | Driver que gerencia a rede (ex.: `bridge`, `overlay`, ou plugin customizado). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |
| `--attachable` | *false* | Permite que containers não orquestrados (manuais) se conectem a uma rede Swarm overlay. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/jammy/man1/docker-network-create.1.html?utm_source=chatgpt.com)) |
| `--internal` | *false* | Cria uma rede interna, sem acesso à rede externa ao host. ([medium.com](https://medium.com/%40nidhiashtikar/docker-network-47b35c6e0fc5?utm_source=chatgpt.com)) |
| `--ingress` | *false* | Cria uma rede Swarm “routing-mesh” para serviços, usada internamente pelo Swarm. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |
| `--subnet strings` | — | Sub-rede no formato CIDR para o IPAM (p. ex. `172.28.0.0/16`). ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_create/?utm_source=chatgpt.com)) |
| `--gateway strings` | — | Gateway IPv4 ou IPv6 da sub-rede mestre. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_create/?utm_source=chatgpt.com)) |
| `--ip-range strings` | — | Faixa de IPs dentro da sub-rede para alocação de containers. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_create/?utm_source=chatgpt.com)) |
| `--aux-address map` | — | Endereços IP auxiliares reservados, mapeados como `nome=IP` (útil para roteadores virtuais). ([forums.docker.com](https://forums.docker.com/t/what-is-the-docker-network-create-aux-address-option-used-for/6324?utm_source=chatgpt.com), [docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_create/?utm_source=chatgpt.com)) |
| `--ipv4` / `--ipv6` | `true`/`false` | Habilita ou desabilita atribuição de endereços IPv4 ou IPv6. ([docs.docker.com](https://docs.docker.com/engine/network/?utm_source=chatgpt.com)) |
| `--ipam-driver string` | `default` | Driver de IPAM (gerenciamento de endereços IP). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |
| `--ipam-opt map` | — | Opções específicas para o driver de IPAM, em formato chave-valor. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |
| `--config-from string` | — | Copia a configuração de IPAM de outra rede existente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |
| `--config-only` | *false* | Cria apenas a configuração de rede (sem provisionar um driver real imediatamente). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |
| `-o, --opt map` | — | Opções específicas do driver de rede (por ex., configurações de bridge via `com.docker.network.bridge.*`). ([test-dockerrr.readthedocs.io](https://test-dockerrr.readthedocs.io/en/latest/reference/commandline/network_create/?utm_source=chatgpt.com)) |
| `--aux-address` | — | Alias alternativo para `--aux-address map`. |
| `--label list` | — | Labels (metadados) para organizar e filtrar redes. |
| `--scope string` | — | Escopo da rede (`local`, `global`). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com)) |

## Exemplos de Uso

1. **Rede bridge padrão**
    
    ```bash
    docker network create minha-rede
    
    ```
    
    Cria `minha-rede` usando o driver `bridge` (isolamento no host). ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_create/?utm_source=chatgpt.com))
    
2. **Overlay ingress para Swarm**
    
    ```bash
    docker network create --driver overlay --ingress rede-ingress
    
    ```
    
    Gera uma rede Swarm para balanceamento de serviço. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/create/?utm_source=chatgpt.com))
    
3. **Rede interna somente containers**
    
    ```bash
    docker network create --internal rede-interna
    
    ```
    
    Containers comunicam-se apenas internamente. ([medium.com](https://medium.com/%40nidhiashtikar/docker-network-47b35c6e0fc5?utm_source=chatgpt.com))
    
4. **Rede com sub-rede, gateway, IP range e IP auxiliar**
    
    ```bash
    docker network create \
      -d bridge \
      --subnet=172.28.0.0/16 \
      --gateway=172.28.0.1 \
      --ip-range=172.28.5.0/24 \
      --aux-address="router=172.28.0.254" \
      rede-customizada
    
    ```
    
    Define mapeamentos avançados de IP para controle fino. ([docker-docs.uclv.cu](https://docker-docs.uclv.cu/engine/reference/commandline/network_create/?utm_source=chatgpt.com))
    
5. **Rede attachable para conexões manuais**
    
    ```bash
    docker network create --attachable overlay-debug
    
    ```
    
    Permite usar `docker network connect` em containers não gerenciados pelo Swarm. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/jammy/man1/docker-network-create.1.html?utm_source=chatgpt.com))
