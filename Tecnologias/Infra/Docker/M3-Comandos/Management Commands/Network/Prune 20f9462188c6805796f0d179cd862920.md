# Prune

## Visão Geral

O comando `docker network prune` remove **todas** as redes Docker não utilizadas, ou seja, aquelas que **não estão conectadas** a nenhum container. É uma maneira rápida de limpar regras de iptables, dispositivos de ponte e entradas de roteamento que ficam “órfãs” quando containers são removidos. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/prune/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker network prune [OPÇÕES]

```

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | **Não** solicita confirmação interativa; executa o prune imediatamente. |
| `--filter filter` | Limita quais redes serão removidas, usando expressões no formato `chave=valor` (ex.: `until=24h`). |
- **Filtros disponíveis**:
    - `until=<timestamp|duration>` — remove redes criadas **antes** do tempo especificado (por exemplo, `until=24h` remove redes com mais de 24 horas)
    - **Outros**: possivelmente `label`, `name`, etc., conforme documentação do Docker CLI ([docs.docker.com](https://docs.docker.com/engine/manage-resources/pruning/?utm_source=chatgpt.com))

---

## Exemplos de Uso

1. **Prune interativo (padrão)**
    
    ```bash
    $ docker network prune
    WARNING! This will remove all networks not used by at least one container.
    Are you sure you want to continue? [y/N] y
    Deleted Networks:
    bridge3abc2d4e
    custom-net-01
    Total reclaimed space: 0B
    
    ```
    
2. **Forçar remoção sem prompt**
    
    ```bash
    docker network prune --force
    
    ```
    
    Executa imediatamente, listando as redes removidas e liberando recursos sem pedir confirmação. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/prune/?utm_source=chatgpt.com))
    
3. **Prune com filtro por idade**
    
    ```bash
    docker network prune --filter "until=24h"
    
    ```
    
    Remove somente redes que **não** são usadas e foram criadas há **mais de 24 horas**. ([docs.docker.com](https://docs.docker.com/engine/manage-resources/pruning/?utm_source=chatgpt.com))
    
4. **Prune por label específica**
    
    ```bash
    docker network prune --filter "label=env=dev"
    
    ```
    
    Exclui apenas redes não utilizadas que tenham a label `env=dev`. ([docs.docker.com](https://docs.docker.com/engine/manage-resources/pruning/?utm_source=chatgpt.com))
    

---

> Dica:
> 
> - Antes de limpar, pode ser útil executar `docker network ls` para visualizar as redes atuais e entender quais serão afetadas.
> - Se precisar remover apenas redes personalizadas, combine `docker network ls -q --filter type=custom` com `docker network rm` para maior controle.

Com esses comandos, você mantém seu ambiente Docker enxuto, evitando o acúmulo de redes obsoletas.
