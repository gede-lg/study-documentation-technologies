# Disconnect

## Visão Geral

O comando `docker network disconnect` remove um container em execução de uma rede Docker específica, interrompendo sua capacidade de comunicar-se com outros containers naquele domínio de rede. O container **deve estar em execução** para que a desconexão seja bem-sucedida. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/disconnect/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker network disconnect [OPÇÕES] REDE CONTAINER

```

- **REDE**: nome ou ID da rede da qual o container será removido.
- **CONTAINER**: nome ou ID do container que será desconectado. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/network/disconnect/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | Força a desconexão mesmo que a operação possa interromper fluxos. |

---

## Exemplos de Uso

1. **Desconectar um container de uma rede**
    
    ```bash
    docker network disconnect minha-rede meu-container
    
    ```
    
    Remove `meu-container` da rede `minha-rede`, mantendo-o em execução mas sem acesso a essa rede.
    
2. **Forçar a desconexão**
    
    ```bash
    docker network disconnect --force interna rede-app1
    
    ```
    
    Garante que `rede-app1` seja removido da rede `interna`, mesmo se houver tráfego ativo ou bloqueios.
    

---

> Dica:
> 
> - Após desconectar, verifique o estado da rede com `docker network inspect <rede>` para confirmar que o container não aparece mais em `Containers`.
> - Se precisar reconectar, use `docker network connect <rede> <container>`.
