# Kill

## Visão Geral

O subcomando `docker container kill` (ou seu alias `docker kill`) força a parada imediata de um ou mais containers, enviando por padrão o sinal `SIGKILL` ao processo principal dentro de cada container. É útil quando o `docker stop` (que envia primeiro `SIGTERM` e aguarda um timeout) não consegue interromper corretamente o serviço. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))

---

## Sintaxe

```bash
docker container kill [OPÇÕES] CONTAINER [CONTAINER...]

```

- **CONTAINER [CONTAINER...]**: um ou mais IDs ou nomes de containers alvo. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))

---

## Aliases

- `docker container kill`
- `docker kill` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER [CONTAINER...]` | IDs ou nomes de um ou mais containers a serem finalizados. |

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-s, --signal string` | Sinal do sistema a ser enviado ao processo principal dentro do container.Formato: nome (`HUP`, `SIGINT`, etc.) ou número (`1`, `2`, etc.).Padrão: `SIGKILL`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/)) |

---

## Exemplos de Uso

1. **Enviar `SIGKILL` (padrão) a um container**
    
    ```bash
    docker container kill my_container
    
    ```
    
    Termina imediatamente o processo principal de `my_container`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))
    
2. **Enviar um sinal personalizado (`SIGHUP`)**
    
    ```bash
    docker container kill --signal=SIGHUP my_container
    
    ```
    
    Envia `SIGHUP` ao processo; dependendo da aplicação, este sinal pode não interromper o container. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))
    
3. **Finalizar múltiplos containers**
    
    ```bash
    docker container kill web db cache
    
    ```
    
    Envia `SIGKILL` simultaneamente para os containers `web`, `db` e `cache`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))
    
4. **Usar o alias curto**
    
    ```bash
    docker kill backend
    
    ```
    
    Idêntico a `docker container kill backend`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/kill/))
    

---

> Dica: Prefira usar docker stop para permitir shutdown gracioso (envia SIGTERM seguido de SIGKILL após timeout). Recorra ao docker kill quando o container não responder ao método padrão.
