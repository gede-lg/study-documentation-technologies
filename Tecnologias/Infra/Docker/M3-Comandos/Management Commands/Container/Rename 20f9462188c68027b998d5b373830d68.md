# Rename

## Visão Geral

O comando `docker container rename` (ou seu alias `docker rename`) permite alterar o nome de um container existente sem precisar recriá-lo, facilitando a identificação ou organização dos seus ambientes. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/rename/))

---

## Sintaxe

```bash
docker container rename CONTAINER NOVO_NOME

```

- **CONTAINER**: ID ou nome atual do container.
- **NOVO\_NOME**: novo nome que você deseja atribuir ao container. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/rename/))

---

## Aliases

- `docker container rename`
- `docker rename` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/rename/))

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER` | Identificador (nome ou ID) do container que será renomeado. |
| `NOVO_NOME` | Nome de substituição para o container; deve ser único no host Docker. |

---

## Exemplos de Uso

1. **Renomear um container pelo nome**
    
    ```bash
    docker container rename meu_app app_principal
    
    ```
    
    Altera o nome `meu_app` para `app_principal`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/rename/))
    
2. **Usar o alias curto**
    
    ```bash
    docker rename backend_api api_prod
    
    ```
    
    Executa a mesma operação usando o alias `docker rename`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/rename/))
    

---

> Dica: Após renomear um container, qualquer comando subsequente que faça referência ao nome antigo falhará—lembre-se de atualizar scripts, arquivos Compose ou referências em automações.
