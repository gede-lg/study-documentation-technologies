# Unpause

## Visão Geral

O comando `docker container unpause` (ou seu alias `docker unpause`) **descongela** todos os processos que estavam **pausados** em um ou mais containers, restaurando sua execução exatamente de onde parou. No Linux, isso é feito por meio do **cgroup freezer**. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/unpause/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container unpause CONTAINER [CONTAINER...]

```

- **CONTAINER [CONTAINER…]**
    
    Lista de nomes ou IDs de um ou mais containers que devem ser “despausados”.
    

Aliases equivalentes:

```bash
docker unpause CONTAINER [CONTAINER...]

```

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER` | Nome ou ID do container cujo(s) processo(s) você quer retomar. Pode ser múltiplos, separados por espaço. |

---

## Exemplos de Uso

1. **Retomar um único container**
    
    ```bash
    docker unpause my_container
    
    ```
    
    Despausa o container `my_container`. ([last9.io](https://last9.io/blog/pausing-docker-containers/?utm_source=chatgpt.com))
    
2. **Usando o alias completo**
    
    ```bash
    docker container unpause webapp
    
    ```
    
    Mesma operação, mas invocando o subcomando “container unpause”. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/unpause/?utm_source=chatgpt.com))
    
3. **Despausar múltiplos containers**
    
    ```bash
    docker unpause cache db worker
    
    ```
    
    Retoma simultaneamente os containers `cache`, `db` e `worker`. ([last9.io](https://last9.io/blog/pausing-docker-containers/?utm_source=chatgpt.com))
    

---

## Dica

Para verificar quais containers estão pausados antes de despausar, use:

```bash
docker ps --filter "status=paused"

```

Isso lista apenas os containers cujo status é **Paused**, facilitando a identificação daqueles que precisam ser “thaw”.
