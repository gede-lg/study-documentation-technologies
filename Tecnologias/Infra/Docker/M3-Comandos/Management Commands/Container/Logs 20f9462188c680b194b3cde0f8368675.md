# Logs

Olá Gedê! A.R.I.A. aqui para iluminar o `docker container logs`.

Este comando é o seu melhor amigo para depuração. Ele permite que você veja tudo o que um contêiner está imprimindo em sua saída padrão (`STDOUT`) e erro padrão (`STDERR`). Essencialmente, é como você "escuta" o que está acontecendo dentro de um contêiner em execução.

---

### **Sintaxe Geral**

```bash
docker container logs [OPÇÕES] <ID_ou_Nome_do_Container>

```

- **`[OPÇÕES]`**: Modificadores que filtram ou formatam a saída dos logs.
- **`<ID_ou_Nome_do_Container>`**: O identificador do contêiner cujos logs você deseja visualizar.

---

### **Parâmetros (Options)**

Vamos organizar as opções em uma tabela para facilitar a consulta.

| Parâmetro | Finalidade | Exemplo de Uso |
| --- | --- | --- |
| **`-f`, `--follow`** | **Segue a saída do log em tempo real**. Em vez de apenas mostrar os logs e sair, o comando permanece conectado, exibindo novas linhas assim que são geradas. É como um `tail -f` do Linux. | `docker logs -f meu_app` |
| **`-n`, `--tail string`** | Mostra um **número específico de linhas** a partir do final do log. O padrão é `"all"` (tudo). | `docker logs --tail 100 meu_app` |
| **`-t`, `--timestamps`** | Adiciona um **timestamp (data e hora)** no início de cada linha do log. | `docker logs -t meu_app` |
| **`--since string`** | Mostra os logs gerados **a partir de um determinado ponto no tempo**. Pode ser uma data/hora absoluta (formato RFC3339) ou relativa (ex: `10m` para 10 minutos, `2h` para 2 horas). | `docker logs --since 10m meu_app` |
| **`--until string`** | Mostra os logs gerados **até um determinado ponto no tempo**. Funciona da mesma forma que o `--since`. | `docker logs --until 2m meu_app` |
| **`--details`** | Mostra **detalhes extras** nos logs, como atributos e variáveis de ambiente associadas, caso o driver de log suporte. | `docker logs --details meu_app` |

---

### **Exemplos Práticos**

Vamos combinar essas opções para cenários comuns.

1. **Iniciar um contêiner de exemplo que gera logs:**
    
    ```bash
    # Este contêiner irá imprimir "hello world" a cada segundo
    docker run -d --name looper alpine /bin/sh -c "while true; do echo 'hello world'; sleep 1; done"
    
    ```
    
2. **Ver os logs em tempo real (o uso mais comum):**
Para acompanhar o que está acontecendo no seu aplicativo agora.
    
    ```bash
    docker logs -f looper
    
    ```
    
    *(Pressione `Ctrl+C` para parar de seguir)*
    
3. **Ver as últimas 5 linhas de log:**
Útil para uma verificação rápida do que aconteceu por último.
    
    ```bash
    docker logs --tail 5 looper
    
    ```
    
4. **Ver os logs dos últimos 2 minutos com timestamps:**
Perfeito para investigar um problema que acabou de ocorrer.
    
    ```bash
    docker logs -t --since 2m looper
    
    ```
    
5. **Combinando tudo: seguir as últimas 10 linhas em tempo real com timestamp:**
Uma ótima combinação para começar a depurar um serviço.
    
    ```bash
    docker logs --tail 10 -tf looper
    
    ```
    
    *(Primeiro ele mostrará as últimas 10 linhas e depois continuará exibindo as novas em tempo real, todas com timestamp).*
    

Dominar o `docker logs` e suas flags vai economizar muito do seu tempo na hora de encontrar e resolver problemas em seus contêineres, Gedê. Se tiver mais alguma dúvida, é só perguntar!
