# Top

## Visão Geral

O comando `docker container top` (ou `docker top`) exibe os processos em execução dentro de um container, apresentando-os tal como o `ps` faria no host. É útil para monitorar e depurar o que está rodando no interior do container. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/top/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container top CONTAINER [ps OPTIONS]

```

- **CONTAINER**: nome ou ID do container alvo.
- **[ps OPTIONS]**: quaisquer flags suportadas pelo utilitário `ps` do Linux, como `ef`, `aux`, ou `o PID,CMD`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/top/?utm_source=chatgpt.com))

---

## Aliases

- `docker container top`
- `docker top` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/top/?utm_source=chatgpt.com))

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER` | Nome ou ID do container cujos processos você deseja listar. |
| `ps OPTIONS` | Opcional: qualquer opção válida do comando `ps` (por exemplo, `-ef`, `aux`, `-o PID,USER,%CPU,COMMAND`). |

> Observação:
> 
> 
> Todas as informações são coletadas do ponto de vista do host, não do namespace interno do container. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/focal/man1/docker-container-top.1.html?utm_source=chatgpt.com))
>

## Exemplos de Uso

1. **Listar processos padrão**
    
    ```bash
    docker container top my_container
    
    ```
    
    Exibe colunas padrão do `ps` (UID, PID, PPID, C, STIME, TTY, TIME, CMD). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/top/?utm_source=chatgpt.com))
    
2. **Ver detalhes completos com `ps -ef`**
    
    ```bash
    docker container top my_container -ef
    
    ```
    
    Lista todos os processos em formato estendido, incluindo PPID, C, STIME, TTY, TIME, CMD completo. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/focal/man1/docker-container-top.1.html?utm_source=chatgpt.com))
    
3. **Formato estilo BSD com `aux`**
    
    ```bash
    docker top my_container aux
    
    ```
    
    Mostra processos no formato `ps aux`, útil para UX semelhante a sistemas BSD. ([devopsschool.com](https://www.devopsschool.com/blog/docker-top/?utm_source=chatgpt.com))
    
4. **Selecionar colunas específicas**
    
    ```bash
    docker container top my_container -o pid,cmd,%cpu,%mem
    
    ```
    
    Exibe apenas PID, comando, %CPU e %MEM para foco em desempenho. ([devopsschool.com](https://www.devopsschool.com/blog/docker-top/?utm_source=chatgpt.com))
    

---

> Dica:
> 
> 
> Combine `docker container top` com opções de formatação do `ps` para obter exatamente as colunas necessárias ao seu diagnóstico, como identificação de processos que consomem mais CPU ou memória.
