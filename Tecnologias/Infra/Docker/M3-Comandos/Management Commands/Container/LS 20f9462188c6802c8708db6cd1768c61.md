# LS

## Visão Geral

O comando `docker container ls` (ou seus aliases `docker ps`, `docker container list`) lista containers Docker na máquina local. Por padrão, exibe apenas os containers em execução; com opções, você pode ver também os parados, filtrar por critérios, mudar o formato da saída, etc. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/ls/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container ls [OPÇÕES]

```

Ou usando os aliases equivalentes:

```bash
docker container list [OPÇÕES]
docker container ps   [OPÇÕES]
docker ps             [OPÇÕES]

```

## Opções Principais

| Opção | Padrão | Descrição |
| --- | --- | --- |
| `-a`, `--all` | `false` | Mostra **todos** os containers (rodando e parados). Por padrão, somente os que estão em execução aparecem. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/ls/?utm_source=chatgpt.com)) |
| `-f`, `--filter filter` | — | Filtra a saída de acordo com condições (status, name, label, etc.). Consulte a tabela de filtros abaixo. ([man.archlinux.org](https://man.archlinux.org/man/docker-container-ls.1.en?utm_source=chatgpt.com)) |
| `--format string` | `table` | Formata a saída usando um template Go: |
- `table` (padrão): exibe em formato de tabela com cabeçalhos
- `table TEMPLATE`: tabela customizada
- `json`: JSON bruto
- `TEMPLATE`: template Go puro ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/ls/?utm_source=chatgpt.com)) |
\| `n`, `-last int` | `1` | Mostra os últimos **n** containers criados (todos os estados). Use `-last 1` para o container mais recente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/ls/?utm_source=chatgpt.com)) |
\| `l`, `-latest` | `false` | Exibe apenas o **último** container criado (independente do estado). |
\| `-no-trunc` | `false` | Não trunca (encurta) informações como IDs ou nomes longos na saída. |
\| `q`, `-quiet` | `false` | Mostra **apenas** os IDs dos containers, um por linha. |
\| `s`, `-size` | `false` | Exibe também o tamanho total de cada container (camada de escrita + camadas de imagem). |

---

### Tabela de Filtros (`-filter`)

Você pode combinar vários filtros no formato `--filter key=value`. Alguns exemplos de chaves:

| Chave | Valores possíveis | Descrição |
| --- | --- | --- |
| `status` | `created`<br>`restarting`<br>`running`<br>`paused`<br>`exited`<br>`dead` | Filtra por estado do container. |
| `name` | `<nome_do_container>` | Container cujo nome contenha o valor especificado. |
| `label` | `<chave>=<valor>` | Containers com a label Docker correspondente. |
| `ancestor` | `<imagem>[:<tag>]` | Containers derivados de uma determinada imagem. |
| `before` | `<container>` | Containers criados **antes** do especificado. |
| `since` | `<container>` | Containers criados **depois** do especificado. |

*Consulte a documentação completa para outras chaves de filtro.* ([man.archlinux.org](https://man.archlinux.org/man/docker-container-ls.1.en?utm_source=chatgpt.com))

## Exemplos de Uso

1. **Listar apenas os containers em execução**
    
    ```bash
    docker container ls
    
    ```
    
2. **Incluir containers parados na listagem**
    
    ```bash
    docker container ls --all
    
    ```
    
3. **Exibir somente IDs dos containers**
    
    ```bash
    docker container ls --quiet
    
    ```
    
4. **Mostrar último container criado**
    
    ```bash
    docker container ls --latest
    
    ```
    
5. **Listar os 5 últimos containers, incluindo os parados**
    
    ```bash
    docker container ls --last 5 --all
    
    ```
    
6. **Filtrar containers em estado “exited”**
    
    ```bash
    docker container ls --filter status=exited
    
    ```
    
7. **Formatar saída como JSON**
    
    ```bash
    docker ps --format=json
    
    ```
    
8. **Tabela customizada (ID, NAMES, STATUS)**
    
    ```bash
    docker container ls --format "table {{.ID}}\\t{{.Names}}\\t{{.Status}}"
    
    ```
