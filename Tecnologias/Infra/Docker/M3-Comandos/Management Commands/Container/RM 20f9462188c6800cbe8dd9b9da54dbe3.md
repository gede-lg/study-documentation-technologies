# RM

## Visão Geral

O comando `docker container rm` (ou seu alias `docker rm`) remove um ou mais containers do host Docker. Por padrão, só remove containers parados; para remover containers em execução, é necessário forçar a operação. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html), [docs.docker.com](https://docs.docker.com/reference/cli/docker/container/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container rm [OPÇÕES] CONTAINER [CONTAINER...]

```

- **CONTAINER [CONTAINER…]**: nomes ou IDs de um ou mais containers a serem removidos. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html))

---

## Aliases

- `docker container rm`
- `docker rm` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --force` | Força a remoção de containers em execução, enviando `SIGKILL` ao processo principal. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html)) |
| `-l, --link` | Remove um link específico (legado do recurso de links do Docker). ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html)) |
| `-v, --volumes` | Remove volumes anônimos associados ao container. Volumes nomeados não serão removidos automaticamente. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html)) |

---

## Exemplos de Uso

1. **Remover um container parado**
    
    ```bash
    docker container rm meu_container
    
    ```
    
    Remove `meu_container` somente se estiver parado. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html))
    
2. **Forçar remoção de um container em execução**
    
    ```bash
    docker container rm --force app_rodando
    
    ```
    
    Envia `SIGKILL` e remove o container imediatamente. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html))
    
3. **Remover múltiplos containers de uma vez**
    
    ```bash
    docker rm db_cache web_frontend jobs_runner
    
    ```
    
    Remove `db_cache`, `web_frontend` e `jobs_runner`, desde que não estejam em execução (ou use `-f`). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/?utm_source=chatgpt.com))
    
4. **Remover volumes anônimos junto com o container**
    
    ```bash
    docker container rm -v redis_data
    
    ```
    
    Remove o container `redis_data` e quaisquer volumes anônimos que ele tenha criado. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html))
    
5. **Remover todos os containers parados**
    
    ```bash
    docker container rm $(docker container ls -aq --filter status=exited)
    
    ```
    
    Utiliza `docker container ls -aq --filter status=exited` para listar IDs de todos os containers que saíram e passa ao `docker container rm`. ([manpages.ubuntu.com](https://manpages.ubuntu.com/manpages/lunar/man1/docker-container-rm.1.html))
    

---

> Dica:
> 
> - Sempre verifique o estado dos containers com `docker container ls -a` antes de removê-los.
> - Para limpar **todos** os containers parados em um único comando sem precisar listar, use `docker container prune`.
