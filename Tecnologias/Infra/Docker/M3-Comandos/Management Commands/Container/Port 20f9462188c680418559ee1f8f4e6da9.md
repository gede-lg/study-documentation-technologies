# Port

## Visão Geral

O comando `docker container port` (ou `docker port`) lista as portas publicadas de um container em execução, ou informa o mapeamento de uma porta específica do container para o host. É útil para descobrir em quais portas externas você pode acessar serviços que rodam dentro de containers. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/port/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container port CONTAINER [PRIVATE_PORT[/PROTO]]

```

- **CONTAINER**: nome ou ID do container alvo.
- **PRIVATE\_PORT\[/PROTO]** (opcional): porta interna do container, com protocolo opcional (`tcp` ou `udp`). Se omitido, o comando retorna todos os mapeamentos. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/port/?utm_source=chatgpt.com))

---

## Aliases

- `docker container port`
- `docker port` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/port/?utm_source=chatgpt.com))

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER` | Nome ou ID do container cujas portas você deseja inspecionar. |
| `PRIVATE_PORT[/PROTO]` | Porta interna do container que você quer consultar. Pode ser apenas o número da porta (ex.: `8080`) ou incluindo protocolo (`8080/tcp`, `53/udp`). |

