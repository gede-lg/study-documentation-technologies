# Export

## Visão Geral

O comando `docker container export` (ou `docker export`) exporta o sistema de arquivos de um container em execução ou parado como um arquivo TAR. É útil para criar um snapshot da camada de escrita de um container, permitindo backup, análise offline ou transferência de um estado específico para outra máquina. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/export/?utm_source=chatgpt.com), [phoenixnap.com](https://phoenixnap.com/kb/docker-export?utm_source=chatgpt.com))

> Importante: Não inclui o conteúdo de volumes montados no container; se um volume cobre um diretório, será exportado apenas o conteúdo subjacente da imagem, não o do volume (docs.docker.com, reddit.com).
> 

---

## Sintaxe

```bash
# Forma completa
docker container export [OPÇÕES] CONTAINER

# Forma curta (alias)
docker export [OPÇÕES] CONTAINER

```

- **CONTAINER**
Nome ou ID do container alvo. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/export/?utm_source=chatgpt.com))

## Aliases

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER` | Nome ou ID do container cujo sistema de arquivos será exportado. |

---

## Opções

| Opção | Padrão | Descrição |
| --- | --- | --- |
| `-o, --output string` | STDOUT | Grava o TAR diretamente em arquivo especificado, em vez de enviá-lo para a saída padrão. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/export/?utm_source=chatgpt.com)) |

## Exemplos de Uso

1. **Exportar para arquivo via redirecionamento**
    
    ```bash
    docker container export webapp > webapp-fs.tar
    
    ```
    
    Exporta o filesystem do container `webapp` para `webapp-fs.tar`. ([phoenixnap.com](https://phoenixnap.com/kb/docker-export?utm_source=chatgpt.com))
    
2. **Exportar diretamente com opção `-output`**
    
    ```bash
    docker container export --output webapp-fs.tar webapp
    
    ```
    
    Mesmo resultado do exemplo anterior, mas usando a flag dedicada. ([phoenixnap.com](https://phoenixnap.com/kb/docker-export?utm_source=chatgpt.com))
    
3. **Gerar um TAR compactado com gzip**
    
    ```bash
    docker container export db01 | gzip > db01-fs.tar.gz
    
    ```
    
    Envia o TAR ao STDOUT e o canaliza para o `gzip`, criando um arquivo compactado. ([phoenixnap.com](https://phoenixnap.com/kb/docker-export?utm_source=chatgpt.com))
    
4. **Importar o snapshot em outra máquina**
Após transferir o TAR, crie uma imagem com:
    
    ```bash
    docker import webapp-fs.tar webapp-snapshot:latest
    
    ```
    
    Isso gera uma nova imagem a partir do filesystem exportado. ([phoenixnap.com](https://phoenixnap.com/kb/docker-export?utm_source=chatgpt.com))
    

---

> Dica: Para incluir dados persistidos em volumes, use docker cp ou soluções de backup de volumes específicas, pois o export não captura esses dados.
