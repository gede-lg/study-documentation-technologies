# Pause

Olá Gedê!

O comando `docker container pause` "congela" todos os processos em execução dentro de um ou mais contêineres.

O estado do contêiner, incluindo o uso de memória, é preservado, mas seus processos param de consumir CPU. Para retomar a execução, você usa o comando `docker unpause`.

---

### **Sintaxe**

A sintaxe é bem direta, não possui opções adicionais:

```bash
docker container pause <ID_ou_Nome_do_Container_1> [ID_ou_Nome_do_Container_2...]

```

- **`<ID_ou_Nome_do_Container...>`**: O identificador (ID ou nome) de um ou mais contêineres que você deseja pausar.

---

### **Diferença Crucial: `pause` vs. `stop`**

É muito importante não confundir `pause` com `stop`:

- **`docker pause`**: Suspende (congela) o contêiner. O estado é mantido na memória e ele pode ser retomado instantaneamente com `docker unpause` exatamente de onde parou.
- **`docker stop`**: Envia um sinal de parada (SIGTERM) para o processo principal do contêiner, encerrando-o. O contêiner para de rodar completamente. Para executá-lo novamente, é preciso usar `docker start`, que reinicia o processo do zero.

---

### **Exemplo Prático**

Vamos ver o ciclo completo:

1. **Inicie um contêiner:**
    
    ```bash
    docker run -d --name meu-teste alpine /bin/sh -c "while true; do echo 'Rodando...'; sleep 1; done"
    
    ```
    
2. **Pause o contêiner:**
    
    ```bash
    docker pause meu-teste
    
    ```
    
    - Se você verificar com `docker ps`, verá o status `Up ... (Paused)`. O contêiner para de gerar os logs "Rodando...".
3. **Retome a execução do contêiner:**
    
    ```bash
    docker unpause meu-teste
    
    ```
    
    - O status em `docker ps` volta a ser apenas `Up ...`, e os logs voltam a aparecer imediatamente.

Este comando é útil quando você precisa liberar recursos da CPU temporariamente sem perder o estado de execução de uma aplicação.
