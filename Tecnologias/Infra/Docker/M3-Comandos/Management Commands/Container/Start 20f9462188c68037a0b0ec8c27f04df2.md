# Start

## Visão Geral

O comando `docker container start` inicia um ou mais containers que estejam parados, reaproveitando seu estado e configurações pré-existentes, sem criar novos containers. É a forma recomendada para “reviver” containers desligados sem recriá-los do zero. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com), [docs.docker.com](https://docs.docker.com/reference/cli/docker/container/run/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container start [OPÇÕES] CONTAINER [CONTAINER...]

```

- **CONTAINER \[CONTAINER…]**: nome(s) ou ID(s) dos containers a iniciar. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))

---

## Aliases

- `docker container start`
- `docker start` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))

---

## Descrição Detalhada

- **Reinício de containers parados**: ao contrário de `docker run`, que cria um novo container, o `start` preserva todas as alterações já feitas no container existente e simplesmente o coloca em execução.
- **Estados compatíveis**: só atua sobre containers no estado “exited” (parados). Containers em outros estados (running, restarting) são ignorados.
- **Fluxos de trabalho**: útil em scripts de CI/CD ou rotinas de manutenção que precisam garantir que um serviço seja reiniciado após uma interrupção planejada. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com), [docs.docker.com](https://docs.docker.com/reference/cli/docker/container/run/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-a, --attach` | Anexa **STDOUT** e **STDERR** do container à sessão atual e encaminha sinais (ex.: `SIGINT`). |
| `-i, --interactive` | Anexa **STDIN** do terminal ao container, permitindo interação direta se o processo suportar entrada. |
| `--detach-keys string` | Substitui a sequência de teclas usada para desanexar (`ctrl-p,ctrl-q` por padrão). |

> Todas as opções acima estão disponíveis conforme o help do Docker CLI. (docs.docker.com)
>

## Exemplos de Uso

1. **Iniciar um único container**
    
    ```bash
    docker container start webapp
    
    ```
    
    Inicia `webapp`, reaproveitando seu filesystem e configurações. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))
    
2. **Iniciar múltiplos containers**
    
    ```bash
    docker container start web db cache
    
    ```
    
    Coloca em execução os containers `web`, `db` e `cache` simultaneamente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))
    
3. **Iniciar e visualizar logs em tempo real**
    
    ```bash
    docker container start --attach api
    
    ```
    
    Inicia `api` e exibe sua saída stdout/stderr imediatamente. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))
    
4. **Iniciar em modo interativo**
    
    ```bash
    docker container start -i worker
    
    ```
    
    Anexa stdin, permitindo enviar comandos ao `worker` se ele aceitar entrada. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))
    
5. **Customizar sequência de desanexar**
    
    ```bash
    docker container start --attach --detach-keys="ctrl-\\\\,ctrl-." my_app
    
    ```
    
    Usa `CTRL+\\` e depois `CTRL+.` para sair da sessão sem parar `my_app`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/start/?utm_source=chatgpt.com))
