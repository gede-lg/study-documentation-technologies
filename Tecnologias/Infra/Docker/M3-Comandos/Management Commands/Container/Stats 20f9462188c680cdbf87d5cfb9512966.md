# Stats

## Visão Geral

O comando `docker container stats` (ou `docker stats`) exibe, em tempo real, métricas de consumo de recursos (CPU, memória, I/O etc.) de um ou mais containers em execução. É ideal para monitoramento rápido de desempenho e identificação de containers que estiverem consumindo recursos excessivos.

---

## Sintaxe

```bash
docker container stats [OPÇÕES] [CONTAINER...]

```

- **CONTAINER...** (opcional)
    
    Lista de nomes ou IDs de containers a monitorar. Se omitido, exibe apenas os containers em execução.
    

---

## Aliases

- `docker container stats`
- `docker stats`

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-a`, `--all` | Mostrar **todos** os containers (incluindo os parados). Por padrão, exibe apenas os que estão em execução. |
| `--format string` | Formatar a saída usando um template Go: |
- `table` (padrão): tabela com cabeçalhos
- `table TEMPLATE`: tabela customizada
- `json`: JSON bruto
- `TEMPLATE`: template Go puro
    
    (Mais em [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/)) |
    
    | `--no-stream`       | Desativa o streaming contínuo, exibindo somente a primeira métrica capturada e encerrando.                            |
    
    | `--no-trunc`        | Não trunca (encurta) campos longos como nomes de containers ou IDs.                                                   |
    

---

## Colunas de Saída

Ao executar sem `--format`, você verá algo como:

| CONTAINER | CPU % | MEM USAGE / LIMIT | MEM % | NET I/O | BLOCK I/O | PIDS |
| --- | --- | --- | --- | --- | --- | --- |
| my_app | 0.15% | 24.5MiB / 1GiB | 2.39% | 1.2kB / 3.4kB | 0B / 0B | 12 |
| db_service | 5.67% | 512MiB / 2GiB | 25.00% | 15MB / 7.3MB | 1.2GB / 640MB | 85 |
- **CPU %**: percentual de uso da CPU pelo container.
- **MEM USAGE / LIMIT**: memória consumida versus limite configurado.
- **MEM %**: percentual de memória em uso.
- **NET I/O**: tráfego de rede recebido/enviado.
- **BLOCK I/O**: leitura/gravação em disco.
- **PIDS**: número de processos/threads ativos no container.

---

## Exemplos de Uso

1. **Monitorar todos os containers em execução**
    
    ```bash
    docker stats
    
    ```
    
2. **Incluir containers parados na listagem**
    
    ```bash
    docker stats --all
    
    ```
    
3. **Obter apenas uma “fotografia” rápida (sem streaming)**
    
    ```bash
    docker stats --no-stream
    
    ```
    
4. **Exibir em JSON**
    
    ```bash
    docker stats --format=json
    
    ```
    
5. **Monitorar containers específicos**
    
    ```bash
    docker stats webapp redis cache
    
    ```
    
6. **Tabela customizada: container e uso de CPU**
    
    ```bash
    docker stats --format "table {{.Container}}\t{{.CPUPerc}}"
    
    ```
