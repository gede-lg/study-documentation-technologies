# Stop

## Visão Geral

O comando `docker container stop` (ou `docker stop`) envia inicialmente um **SIGTERM** ao processo principal de um ou mais containers em execução, aguardando um período de graça (por padrão, 10 s em Linux e 30 s em Windows) para que o container encerre de forma ordenada; caso ainda esteja ativo após esse tempo, o Docker envia um **SIGKILL** para forçar a parada. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/stop/?utm_source=chatgpt.com), [phoenixnap.com](https://phoenixnap.com/kb/stop-docker-containers?utm_source=chatgpt.com))

---

## Sintaxe e Aliases

```bash
docker container stop [OPÇÕES] CONTAINER [CONTAINER...]

```

- **CONTAINER [CONTAINER…]**: nome(s) ou ID(s) de um ou mais containers em execução.
- Se omitido o nome, não há ação.

Aliases:

- `docker stop`
- `docker container stop` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/stop/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-s, --signal string` | Define o sinal a ser enviado em vez do padrão **SIGTERM** (ex.: `SIGINT`, `SIGQUIT`, ou número como `1`, `2`, etc.). |
| `-t, --timeout int` | Tempo (em segundos) que o Docker aguardará após o **SIGTERM** antes de forçar o **SIGKILL** (use `-1` para aguardar indefinidamente). |

> Padrões:
> 
> - **Sinal:** `SIGTERM`
> - **Timeout:** 10 s (Linux) ou 30 s (Windows), configurável no momento da criação do container via `-stop-timeout`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/stop/?utm_source=chatgpt.com))

## Exemplos de Uso

1. **Parada padrão (10 s de graça)**
    
    ```bash
    docker container stop webapp
    
    ```
    
    Envia `SIGTERM` e aguarda 10 s antes de `SIGKILL`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/stop/?utm_source=chatgpt.com))
    
2. **Parar múltiplos containers de uma vez**
    
    ```bash
    docker stop api db cache
    
    ```
    
    Tenta parar `api`, `db` e `cache` sequencialmente. ([upwork.com](https://www.upwork.com/resources/how-to-stop-a-docker-container?utm_source=chatgpt.com))
    
3. **Enviar outro sinal (por exemplo, SIGHUP)**
    
    ```bash
    docker stop --signal=SIGHUP worker
    
    ```
    
    Usa `SIGHUP` ao invés de `SIGTERM`. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/stop/?utm_source=chatgpt.com))
    
4. **Aumentar o período de graça para 30 segundos**
    
    ```bash
    docker container stop --time=30 backend
    
    ```
    
    Garante até 30 s para encerramento gracioso antes do `SIGKILL`. ([phoenixnap.com](https://phoenixnap.com/kb/stop-docker-containers?utm_source=chatgpt.com))
    
5. **Aguardar indefinidamente**
    
    ```bash
    docker stop --timeout=-1 my_container
    
    ```
    
    Nunca aplica timeout; espera até o processo dentro do container encerrar sozinho. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/stop/?utm_source=chatgpt.com))
    

---

> Dica:
> 
> - Para terminar **imediatamente** sem período de graça, use `docker kill` (envia `SIGKILL` diretamente) ou `docker stop -t 0`.
> - Confirme sempre o status dos containers com `docker ps` antes de pará-los para evitar impactos indesejados.
