# Diff

## Visão Geral

O comando `docker container diff` (ou `docker diff`) inspeciona as alterações no sistema de arquivos de um container desde sua criação, listando arquivos ou diretórios que foram adicionados, modificados ou removidos. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/diff/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker container diff [OPÇÕES] CONTAINER

```

- **CONTAINER**
ID completo, ID abreviado ou nome do container a ser inspecionado. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/diff/?utm_source=chatgpt.com))

---

## Aliases

- `docker container diff`
- `docker diff` ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/diff/?utm_source=chatgpt.com))

---

## Parâmetros

| Parâmetro | Descrição |
| --- | --- |
| `CONTAINER` | Identificador (nome ou ID) do container cujas alterações serão mostradas. |

---

## Tipos de Mudança

O Docker categoriza cada mudança com um símbolo:

| Símbolo | Descrição |
| --- | --- |
| `A` | Arquivo ou diretório **A**dicionado |
| `C` | Arquivo ou diretório **C**ommodificado |
| `D` | Arquivo ou diretório **D**eletado |

Esses estados representam, respectivamente, adições, modificações e remoções no layer de escrita do container. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/diff/?utm_source=chatgpt.com))

---

## Exemplos de Uso

1. **Inspecionar mudanças em um container nomeado**
    
    ```bash
    docker container diff webapp
    
    ```
    
    Saída típica:
    
    ```
    A /app/new_file.txt
    C /usr/share/nginx/html/index.html
    D /usr/share/nginx/html/old_page.html
    
    ```
    
    Aqui, um arquivo foi adicionado, outro modificado e um terceiro removido. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/container/diff/?utm_source=chatgpt.com))
    
2. **Usar alias curto e ID abreviado**
    
    ```bash
    docker diff 1fdfd1f54c1b
    
    ```
    
3. **Interpretar o resultado**
    - Combine `docker container diff` com `docker exec` para inspecionar diretamente dentro do container os arquivos listados.
    - Para acompanhar logs históricos, utilize também `docker logs --timestamps`.

---

> Dica:
Use esse comando para depuração e auditoria de containers, garantindo que você saiba exatamente o que mudou em relação à imagem base.
