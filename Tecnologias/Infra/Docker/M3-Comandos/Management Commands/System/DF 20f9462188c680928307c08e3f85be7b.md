# DF

## Visão Geral

O comando `docker system df` exibe um resumo do uso de espaço em disco pelo daemon Docker, mostrando quanto está ocupado por imagens, containers, volumes locais e build cache, além de quanto desse espaço é recuperável (reclaimable). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/df/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker system df [OPÇÕES]

```

- Sem parâmetros, mostra o **resumo** de todos os tipos de dados.
- As opções permitem ajustes de formato e nível de detalhe. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/df/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `--format string` | Formata a saída usando um template Go (veja [https://docs.docker.com/go/formatting/](https://docs.docker.com/go/formatting/)): |
|  | • `table` (padrão): tabela com cabeçalhos |
|  | • `table TEMPLATE`: tabela personalizada |
|  | • `json`: JSON bruto |
|  | • `TEMPLATE`: template Go livre |
| `-v, --verbose` | Exibe **detalhes** adicionais, listando cada item individual (imagens, containers e volumes). |

---

## Colunas do Resumo

Ao rodar sem `-v`, a saída apresenta:

| TYPE | TOTAL | ACTIVE | SIZE | RECLAIMABLE |
| --- | --- | --- | --- | --- |
| **Images** | número de imagens | quantas em uso | espaço total ocupado | espaço que pode ser recuperado ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/df/?utm_source=chatgpt.com)) |
| **Containers** | total de containers | em execução | espaço ocupado | espaço que pode ser recuperado ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/df/?utm_source=chatgpt.com)) |
| **Local Volumes** | total de volumes | em uso | espaço total | espaço recuperável |
| **Build Cache** | total de caches | em uso | espaço total | espaço recuperável |
- **SIZE**: soma dos tamanhos de todos os itens daquele tipo.
- **RECLAIMABLE**: portion of SIZE que está associado a itens inativos ou dangling, e que pode ser liberado via `docker system prune`. ([stackoverflow.com](https://stackoverflow.com/questions/50659233/what-is-the-reclaimable-space-displayed-in-docker-system-df?utm_source=chatgpt.com))

## Exemplos de Uso

1. **Resumo padrão**
    
    ```bash
    docker system df
    
    ```
    
    Exibe algo como:
    
    ```
    TYPE                TOTAL   ACTIVE   SIZE      RECLAIMABLE
    Images              5       2        2.5GB     1.7GB (68%)
    Containers          3       1        300MB     200MB (66%)
    Local Volumes       4       2        100MB     50MB (50%)
    Build Cache         0       0        0B        0B
    ``` :contentReference[oaicite:5]{index=5}
    
    ```
    
2. **Detalhamento por item**
    
    ```bash
    docker system df --verbose
    
    ```
    
    Lista cada imagem (com REPOSITORY, TAG, SIZE), cada container (ID, IMAGE, SIZE) e cada volume (NAME, SIZE), permitindo identificar exatamente onde está o consumo maior. ([labex.io](https://labex.io/tutorials/docker-how-to-use-docker-system-df-command-to-check-disk-usage-555247?utm_source=chatgpt.com))
    
3. **Saída em JSON**
    
    ```bash
    docker system df --format=json
    
    ```
    
    Gera um objeto JSON programaticamente consumível, útil em scripts de automação:
    
    ```json
    [
      {
        "Type": "Images",
        "TotalCount": 5,
        "ActiveCount": 2,
        "Size": "2.5GB",
        "Reclaimable": "1.7GB"
      },
      …
    ]
    ``` :contentReference[oaicite:7]{index=7}
    
    ```
    
4. **Saída em tabela customizada**
    
    ```bash
    docker system df --format "table {{.Type}}\t{{.Size}}\t{{.Reclaimable}}"
    
    ```
    
    Foca apenas em coluna de tipo, tamanho e espaço recuperável. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/df/?utm_source=chatgpt.com))
    

---

> Dicas Práticas:
> 
> - Use `docker system prune` (ou `docker system prune --volumes`) **apos** revisar o `RECLAIMABLE` para liberar espaço.
> - Agende `docker system df --format=json` em scripts de monitoramento para alertas de disco cheio.
> - Lembre-se: logs de containers e arquivos fora do escopo de imagens/containers/volumes não são contabilizados aqui; revise `/var/lib/docker/containers/*/*.log` se notar discrepâncias. ([code-maven.com](https://code-maven.com/docker-uses-a-lot-of-diskspace?utm_source=chatgpt.com))
