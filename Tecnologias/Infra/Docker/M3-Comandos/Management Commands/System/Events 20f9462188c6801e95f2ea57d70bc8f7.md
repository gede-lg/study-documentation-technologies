# Events

## Visão Geral

O comando `docker system events` (alias `docker events`) fornece um fluxo em tempo real de eventos gerados pelo daemon Docker, tais como criação, início, parada e remoção de containers, imagens, volumes, redes e outros objetos. Esses eventos permitem monitorar o comportamento do Docker e reagir automaticamente a mudanças de estado. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/events/?utm_source=chatgpt.com))

---

## Sintaxe

```bash
docker system events [OPÇÕES]

```

Sem parâmetros, inicia o streaming contínuo de eventos até ser interrompido (Ctrl +C). ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/events/?utm_source=chatgpt.com))

---

## Opções

| Opção | Descrição |
| --- | --- |
| `-f, --filter filter` | Filtra eventos por critérios `chave=valor`, como `type=container`, `event=start`, ou `container=<ID>`. |
| `--format string` | Formata a saída usando template Go:• `table` (padrão)• `table TEMPLATE`• `json`• `TEMPLATE`. |
| `--since string` | Exibe apenas eventos gerados **após** o timestamp ou expressão de duração (ex.: `2025-06-10T10:00:00` ou `1h`). |
| `--until string` | Continua recebendo eventos **até** o timestamp ou duração especificada. |

Todas as opções acima são suportadas conforme o CLI do Docker. ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/events/?utm_source=chatgpt.com))

## Exemplos de Uso

1. **Fluxo completo de eventos**
    
    ```bash
    docker events
    
    ```
    
    Mostra todos os eventos Docker em tempo real, sem filtros. ([labex.io](https://labex.io/tutorials/docker-how-to-monitor-docker-system-events-411575?utm_source=chatgpt.com))
    
2. **Filtrar apenas eventos de containers**
    
    ```bash
    docker system events --filter type=container
    
    ```
    
    Exibe somente eventos relacionados a containers (create, start, stop, die etc.). ([devopsschool.com](https://www.devopsschool.com/blog/docker-commands-tutorial-docker-events/?utm_source=chatgpt.com))
    
3. **Eventos de parada das últimas 30 minutos**
    
    ```bash
    docker events --filter 'event=stop' --since 30m
    
    ```
    
    Recebe apenas eventos de parada ocorridos na última meia hora. ([community.splunk.com](https://community.splunk.com/t5/Getting-Data-In/How-to-get-docker-system-events-not-container-logs-into-Splunk/td-p/656098?utm_source=chatgpt.com))
    
4. **Formato JSON para ingestão em sistemas de log**
    
    ```bash
    docker system events --format=json
    
    ```
    
    Emite cada evento como objeto JSON, facilitando parsing por ferramentas de monitoramento. ([medium.com](https://medium.com/%40calebackom/docker-events-explained-your-window-into-container-activity-76a2f501b11f?utm_source=chatgpt.com))
