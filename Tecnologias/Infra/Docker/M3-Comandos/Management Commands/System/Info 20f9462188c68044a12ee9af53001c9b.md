# Info

## Visão Geral

O comando `docker system info` exibe informações abrangentes sobre o ambiente Docker em execução, incluindo versão do kernel, quantidade de containers (ativos e totais), imagens únicas, driver de armazenamento e detalhes de espaço alocado para dados e metadados do Docker ([docs.docker.com](https://docs.docker.com/reference/cli/docker/system/info/?utm_source=chatgpt.com)).

---

## Sintaxe

```bash
docker system info [OPÇÕES]

```

Alias:

```bash
docker info
``` :contentReference[oaicite:1]{index=1}

---

## Opções

| Opção                   | Descrição                                                                                                                                                                                                                      |
|-------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `-f, --format string`   | Formata a saída usando um template Go. As opções incluem:
  - `table` (padrão): tabela com cabeçalhos
  - `table TEMPLATE`: tabela personalizada via Go template
  - `json`: imprime JSON bruto
  - `TEMPLATE`: aplica qualquer Go template definido
  Consulte a documentação de formatação para detalhes sobre as variáveis disponíveis. :contentReference[oaicite:2]{index=2} |
| `-v, --verbose`         | Exibe informações **detalhadas**, listando componentes individuais (imagens, containers, volumes, caches) e, dependendo do driver de armazenamento, mostra pool name, data file, metadata file e uso de espaço em cada um. :contentReference[oaicite:3]{index=3} |

## Exemplos de Uso

1. **Resumo padrão**
   ```bash
   docker system info

```

Exibe o panorama geral do Docker: número de containers, imagens únicas, driver de armazenamento e estatísticas de espaço.

1. **Saída em JSON**
    
    ```bash
    docker system info --format json
    
    ```
    
    Gera um objeto JSON, útil para ingestão em scripts de automação ou monitoramento.
    
2. **Tabela customizada**
    
    ```bash
    docker system info --format "table {{.Driver}}\t{{.Containers}}\t{{.Images}}"
    
    ```
    
    Exibe apenas colunas de driver, total de containers e imagens.
    
3. **Modo verboso**
    
    ```bash
    docker system info --verbose
    
    ```
    
    Inclui detalhes de cada componente individual e informações aprofundadas sobre o driver de armazenamento (pool name, data file, metadata file, espaço usado, etc.).
