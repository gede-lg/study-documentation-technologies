# PS

---

## O que faz o comando

O **`docker ps`** lista os **containers** no host Docker, permitindo que você veja quais estão em execução, qual sua configuração e estado. Por padrão, exibe apenas os containers **ativos**; com opções, você pode incluir parados, filtrar, ver em formatos personalizados etc.

---

## Sintaxe básica

```bash
docker ps [OPTIONS]

```

- **`docker ps`**: comando para listar containers.
- **`[OPTIONS]`**: flags que controlam quais containers aparecem e como a informação é exibida.

---

## Opções disponíveis

### Seleção de containers

| Opção | Descrição |
| --- | --- |
| `-a`, `--all` | Exibe **todos** os containers (ativos e parados). |
| `-n`, `--last N` | Mostra apenas os **N** últimos containers criados (qualquer estado). |
| `-l`, `--latest` | Exibe somente o **container mais recente** criado (qualquer estado). |

### Filtragem

| Opção | Descrição |
| --- | --- |
| `-f`, `--filter EXP` | Filtra a saída com base em condições (ex.: `status=exited`, `ancestor=nginx`, `name=web`). |

### Formato de saída

| Opção | Descrição |
| --- | --- |
| `--format STRING` | Usa um *Go template* para customizar a saída. Exemplos: |
- `table {{.ID}}\t{{.Image}}`
- `json` |
    
    | `--no-trunc`                  | Não trunca campos longos (como IDs e nomes) — mostra valor completo.                                            |
    
    | `-q`, `--quiet`               | Exibe **apenas** os IDs dos containers, um por linha.                                                            |
    
    | `-s`, `--size`                | Inclui também o **tamanho** de disco utilizado por cada container (`RW` + volumes montados).                    |
    

---

## Exemplos de uso

1. **Listar apenas containers em execução**
    
    ```bash
    docker ps
    
    ```
    
2. **Listar todos os containers (ativos e parados)**
    
    ```bash
    docker ps -a
    
    ```
    
3. **Listar últimos 3 containers criados**
    
    ```bash
    docker ps -n 3
    
    ```
    
4. **Filtrar por imagem “nginx” e mostrar sem cabeçalhos**
    
    ```bash
    docker ps --filter ancestor=nginx -q
    
    ```
    
5. **Exibir detalhes completos, sem truncar IDs**
    
    ```bash
    docker ps --no-trunc
    
    ```
    
6. **Mostrar tamanho de cada container**
    
    ```bash
    docker ps -s
    
    ```
    
7. **Customizar saída em JSON**
    
    ```bash
    docker ps --format json
    
    ```
