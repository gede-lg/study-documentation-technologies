# Run

---

## Sintaxe Básica

```bash
docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

```

- **`docker run`**: comando para criar e iniciar um novo contêiner a partir de uma imagem.
- **`[OPTIONS]`**: flags que configuram comportamentos de rede, volumes, recursos, etc.
- **`IMAGE`**: nome (e opcionalmente tag) da imagem Docker que será usada (por ex.: `nginx:latest`).
- **`[COMMAND] [ARG...]`**: (opcional) comando e argumentos que sobrescrevem o `ENTRYPOINT`/`CMD` definido na imagem.

---

## Finalidade do `docker run`

1. **Criação do contêiner**
    
    Baixa a imagem do registry (se necessário) e a instância como um contêiner isolado.
    
2. **Configuração de recursos**
    
    Aplica limites de CPU, memória, políticas de reinício, rede, volumes, variáveis de ambiente, etc.
    
3. **Execução do processo**
    
    Inicia o processo principal dentro do contêiner (definido pelo `ENTRYPOINT`/`CMD` da imagem ou pelo comando informado).
    
4. **Gerenciamento de ciclo de vida**
    
    Permite rodar containers em foreground, background (`-d`), temporários (`--rm`), interativos (`-it`), etc.
    

Com esses pontos, você tem a visão inicial do comando antes de mergulhar nas opções detalhadas. Qualquer dúvida, me avise!
