# Info

---

## 1. Finalidade do `docker info`

O **`docker info`** exibe uma visão geral **do ambiente Docker** instalado no host, incluindo informações do daemon, contêineres, imagens, volumes, redes, plugins, configuração de storage, cgroup, versões, etc. É essencial para diagnóstico e monitoramento do sistema.

---

## 2. Sintaxe básica

```bash
docker info [OPTIONS]

```

Como alias, também pode ser invocado por:

```bash
docker system info

```

---

## 3. Parâmetros

Este comando **não** aceita argumentos posicionais — todas as configurações são feitas via opções.

---

## 4. Opções disponíveis

| Opção | Tipo | Padrão | Descrição |
| --- | --- | --- | --- |
| `-f, --format string` | string | `plain` | Formata a saída usando um template Go ou JSON. |
|  |  |  | - `json` → imprime toda a informação em JSON bruto. |
|  |  |  | - `TEMPLATE` → customiza via Go template (ex.: `--format '{{json .Driver}}'`). |

---

## 5. Exemplo de uso

1. **Exibir todas as informações em formato padrão**
    
    ```bash
    docker info
    
    ```
    
    Exemplo de saída (resumida):
    
    ```
    Client:
     Context:    default
     Debug Mode: false
    
    Server:
     Containers: 12
      Running: 5
      Paused: 0
      Stopped: 7
     Images: 34
     Storage Driver: overlay2
      Backing Filesystem: extfs
      Supports d_type: true
     ...
    
    ```
    
2. **Saída em JSON**
    
    ```bash
    docker info --format json
    # {
    #   "ID": "ABCD:1234:...",
    #   "Containers": 12,
    #   "Images": 34,
    #   "Driver": "overlay2",
    #   ...
    # }
    
    ```
    
3. **Extrair apenas o driver de storage**
    
    ```bash
    docker info --format '{{.Driver}}'
    # overlay2
    
    ```
    
4. **Mostrar apenas o número de contêineres em execução**
    
    ```bash
    docker info --format '{{.ContainersRunning}}'
    # 5
    
    ```
    

---

## 6. Tabela de referência rápida

| Componente | Sintaxe | Descrição |
| --- | --- | --- |
| Comando base | `docker info [OPTIONS]` | Exibe informações completas do sistema Docker. |
| Opção | `-f, --format <template | json>` |

---

Com isso, você tem tudo para usar o **`docker info`**: desde obter um panorama completo do seu ambiente Docker até extrair valores específicos em scripts ou monitoramento. Se precisar de ajuda com algum template ou campo específico, é só avisar!
