# Images

---

## 1. Finalidade do `docker images`

O **`docker images`** lista as **imagens** Docker armazenadas localmente. Ele permite ver quais imagens e tags você tem, checar digests, filtrar por repositório ou tag e inspecionar intermediários gerados durante builds.

---

## 2. Sintaxe básica

```bash
docker images [OPTIONS] [REPOSITORY[:TAG]]

```

- **`docker images`**: comando para listar imagens locais.
- **`[REPOSITORY[:TAG]]`**: (opcional) filtra apenas imagens daquele repositório e, se fornecida, daquela tag.

---

## 3. Parâmetro

| Parâmetro | Obrigatório | Descrição |
| --- | --- | --- |
| `REPOSITORY[:TAG]` | Não | Repositório (e opcionalmente tag) para restringir a listagem. |

---

## 4. Opções disponíveis

| Opção | Tipo | Padrão | Finalidade |
| --- | --- | --- | --- |
| `-a, --all` | Boolean | `false` | Exibe **todas** as imagens, incluindo intermediárias (geradas em etapas de build). |
| `--digests` | Boolean | `false` | Mostra a coluna **DIGEST** (sha256) de cada imagem, se disponível. |
| `-f, --filter filter` | String | — | Filtra a saída por condições (ex.: `dangling=true`, `reference=nginx:*`, `before=repo:tag`). |
| `--format string` | String | `table` | Formata a saída com *Go templates* ou em `json`. |
| `--no-trunc` | Boolean | `false` | Não trunca campos longos (IDs, nomes, digests), mostrando valores completos. |
| `-q, --quiet` | Boolean | `false` | Exibe apenas os **IDs** das imagens, um por linha. |
| `--tree` *(experimental)* | Boolean | `false` | Exibe imagens multi-plataforma em forma de **árvore** de manifestos. |

---

### Exemplos de filtros (`-filter`)

- `dangling=true` — imagens órfãs (sem tag).
- `label=stage=prod` — imagens com label específico.
- `reference=nginx:*` — todas as tags do repositório `nginx`.
- `before=repo:tag` — imagens criadas antes da mencionada.
- `since=repo:tag` — imagens criadas após a mencionada.

---

## 5. Exemplos de uso

1. **Listar imagens não-intermediárias**
    
    ```bash
    docker images
    
    ```
    
2. **Listar todas, incluindo intermediárias**
    
    ```bash
    docker images -a
    
    ```
    
3. **Mostrar digests junto das imagens**
    
    ```bash
    docker images --digests
    
    ```
    
4. **Listar apenas IDs**
    
    ```bash
    docker images -q
    
    ```
    
5. **Filtrar imagens órfãs**
    
    ```bash
    docker images --filter dangling=true
    
    ```
    
6. **Não truncar IDs**
    
    ```bash
    docker images --no-trunc
    
    ```
    
7. **Filtrar por repositório e tag**
    
    ```bash
    docker images redis:6.2*
    
    ```
    
8. **Saída em JSON**
    
    ```bash
    docker images --format json
    
    ```
    
9. **Exibir árvore de manifestos (experimental)**
    
    ```bash
    docker images --tree
    
    ```
