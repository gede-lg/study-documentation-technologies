## PreparedStatement: Dominando a Interação com Bancos de Dados em Java

### 1\. Introdução

No desenvolvimento Java, especialmente em aplicações que interagem com bancos de dados, a segurança, performance e manutenibilidade do código são pilares fundamentais. O `PreparedStatement`, parte integrante da API JDBC (Java Database Connectivity), é uma classe que endereça diretamente esses pilares, oferecendo uma maneira eficiente e segura de executar comandos SQL parametrizados.

A relevância do `PreparedStatement` no contexto do desenvolvimento backend em Java é imensa. Ele é a espinha dorsal para qualquer aplicação que precise persistir ou recuperar dados de forma confiável, garantindo que as consultas sejam otimizadas e protegidas contra vulnerabilidades comuns, como a injeção de SQL.

**Definição e Conceitos Fundamentais:**

`PreparedStatement` é uma subclasse de `Statement` que representa uma instrução SQL pré-compilada. Em vez de enviar a consulta SQL bruta ao banco de dados a cada execução, o `PreparedStatement` permite que a consulta seja enviada apenas uma vez, com "placeholders" (`?`) para os valores variáveis. Esses placeholders são preenchidos programaticamente antes de cada execução.

**Para que servem:**

- **Segurança (Prevenção de Injeção de SQL):** Ao lidar com dados inseridos pelo usuário, o `PreparedStatement` trata esses dados como valores literais, e não como parte do código SQL. Isso impede que caracteres maliciosos alterem a lógica da consulta.
- **Performance:** A pré-compilação da consulta no banco de dados economiza o tempo de análise e otimização da consulta a cada execução. Isso é especialmente benéfico para consultas que são executadas várias vezes com diferentes parâmetros.
- **Reusabilidade:** Uma vez que um `PreparedStatement` é criado, ele pode ser reutilizado para executar a mesma consulta diversas vezes, apenas alterando os valores dos parâmetros.
- **Clareza e Manutenibilidade do Código:** A sintaxe com placeholders torna o código mais limpo e fácil de entender, pois os valores são atribuídos explicitamente.

### Sintaxe e Estrutura

A criação de um `PreparedStatement` geralmente segue um padrão: primeiro, obtém-se uma conexão com o banco de dados; em seguida, a consulta SQL é preparada usando o método `prepareStatement()` da interface `Connection`.

**Exemplos de declaração e utilização:**

```java
import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.SQLException;
import java.sql.ResultSet;

public class PreparedStatementExemplo {

    private static final String URL = "jdbc:mysql://localhost:3306/meubanco";
    private static final String USER = "root";
    private static final String PASSWORD = "senha";

    public static void main(String[] args) {
        Connection connection = null;
        PreparedStatement preparedStatement = null;
        ResultSet resultSet = null;

        try {
            // 1. Obter uma conexão
            connection = DriverManager.getConnection(URL, USER, PASSWORD);

            // 2. Preparar a consulta SQL com placeholders
            String sql = "INSERT INTO usuarios (nome, email, idade) VALUES (?, ?, ?)";
            preparedStatement = connection.prepareStatement(sql);

            // 3. Definir os valores para os placeholders
            preparedStatement.setString(1, "Gedê");
            preparedStatement.setString(2, "gede@example.com");
            preparedStatement.setInt(3, 23);

            // 4. Executar a atualização
            int linhasAfetadas = preparedStatement.executeUpdate();
            System.out.println("Linhas inseridas: " + linhasAfetadas);

            // Exemplo de SELECT
            String selectSql = "SELECT nome, email, idade FROM usuarios WHERE idade > ?";
            preparedStatement = connection.prepareStatement(selectSql);
            preparedStatement.setInt(1, 20);
            resultSet = preparedStatement.executeQuery();

            while (resultSet.next()) {
                String nome = resultSet.getString("nome");
                String email = resultSet.getString("email");
                int idade = resultSet.getInt("idade");
                System.out.println("Nome: " + nome + ", Email: " + email + ", Idade: " + idade);
            }

        } catch (SQLException e) {
            System.err.println("Erro ao interagir com o banco de dados: " + e.getMessage());
            e.printStackTrace();
        } finally {
            // 5. Fechar os recursos (em ordem inversa de abertura)
            try {
                if (resultSet != null) resultSet.close();
                if (preparedStatement != null) preparedStatement.close();
                if (connection != null) connection.close();
            } catch (SQLException e) {
                System.err.println("Erro ao fechar recursos: " + e.getMessage());
            }
        }
    }
}

```

### Componentes Principais: Métodos `setX()` e `execute()`

A interface `PreparedStatement` estende `Statement` e adiciona métodos para definir os valores dos parâmetros e para executar a consulta.

**Métodos `setX()` (Setters):**

Esses métodos são usados para atribuir valores aos placeholders (`?`) na consulta SQL. O `X` no nome do método corresponde ao tipo de dado que está sendo definido (ex: `setString`, `setInt`, `setDate`, `setDouble`, etc.). O primeiro argumento desses métodos é o **índice do parâmetro** (começando em `1`, não em `0`), e o segundo argumento é o **valor** a ser atribuído.

| --- | --- |

| `setAsciiStream(int parameterIndex, InputStream x, int length)` | Define o parâmetro no índice especificado para o fluxo de entrada ASCII especificado. Pode ser usado para armazenar grandes quantidades de dados de caracteres ASCII. |
| `setBigDecimal(int parameterIndex, BigDecimal x)` | Define o parâmetro no índice especificado para o objeto `java.math.BigDecimal`. |

**Métodos de Execução:**

Após definir todos os parâmetros, um dos métodos a seguir é usado para executar a consulta.

- `boolean execute()`: Usado para executar **qualquer tipo de comando SQL**. Retorna `true` se o primeiro resultado for um `ResultSet` (como em `SELECT`), e `false` se for um contador de atualização (como em `INSERT`, `UPDATE`, `DELETE`) ou se não houver resultados. É geralmente usado para comandos DDL (Data Definition Language) ou quando o tipo de resultado é desconhecido.
- `int executeUpdate()`: Usado para executar comandos SQL que **modificam o banco de dados** (DML - Data Manipulation Language), como `INSERT`, `UPDATE` ou `DELETE`. Retorna o número de linhas afetadas pela operação. Também é usado para comandos DDL (como `CREATE TABLE`, `DROP TABLE`).
- `ResultSet executeQuery()`: Usado para executar comandos SQL que **retornam um conjunto de resultados**, como `SELECT`. Retorna um objeto `ResultSet`, que pode ser iterado para acessar os dados.

**Outros Métodos Importantes:**

- `void addBatch()`: Adiciona o conjunto atual de parâmetros para o batch de comandos. Usado para executar múltiplas operações de `INSERT`, `UPDATE` ou `DELETE` de forma mais eficiente (ver seção de exemplos).
- `int[] executeBatch()`: Executa todos os comandos no batch atual. Retorna um array de `int`s, onde cada elemento indica o número de linhas afetadas por cada comando no batch.
- `void clearParameters()`: Limpa os valores atuais dos parâmetros definidos para este `PreparedStatement`. Isso permite que o `PreparedStatement` seja reutilizado com um novo conjunto de parâmetros sem a necessidade de criar uma nova instância.
- `Connection getConnection()`: Recupera o objeto `Connection` que criou este `PreparedStatement`.
- `ResultSetMetaData getMetaData()`: Recupera um objeto `ResultSetMetaData` que contém informações sobre as colunas do `ResultSet` que seria retornado se este `PreparedStatement` fosse executado. (Útil para `SELECT` statements).
- `ParameterMetaData getParameterMetaData()`: Recupera um objeto `ParameterMetaData` que contém informações sobre os parâmetros deste `PreparedStatement`.
- `void close()`: Fecha o `PreparedStatement` e libera seus recursos. É crucial chamar este método para evitar vazamentos de recursos. (Preferencialmente usado com `try-with-resources`).

**Interação entre eles:**

A interação é sequencial e lógica:

1. **Criação:** Uma conexão é obtida e o `PreparedStatement` é criado com a consulta SQL pré-definida.
2. **Definição de Parâmetros:** Os métodos `setX()` são chamados para preencher os placeholders.
3. **Execução:** Um dos métodos `execute()`, `executeUpdate()` ou `executeQuery()` é chamado para enviar a consulta ao banco de dados.
4. **Processamento de Resultados (se aplicável):** Se for um `SELECT`, o `ResultSet` é processado.
5. **Fechamento:** Os recursos são fechados para liberar a conexão e o `PreparedStatement`.

### Restrições de Uso

Embora poderoso, o `PreparedStatement` tem algumas considerações:

- **Consultas Estáticas com Parâmetros Dinâmicos:** Ele é ideal para consultas onde a estrutura SQL é fixa, mas os valores são variáveis. Não pode ser usado para construir dinamicamente o nome de tabelas ou colunas (para isso, seria necessário concatenar strings, o que reintroduz riscos de injeção de SQL se não for validado cuidadosamente).
- **Performance vs. Complexidade:** Para consultas que são executadas apenas uma vez, a sobrecarga de pré-compilação pode não trazer ganhos significativos em comparação com um `Statement` simples. No entanto, o ganho de segurança ainda justifica seu uso.
- **Gerenciamento de Recursos:** É vital garantir que os objetos `Connection`, `PreparedStatement` e `ResultSet` sejam fechados corretamente após o uso para evitar vazamento de recursos e problemas de desempenho. A instrução `try-with-resources` do Java 7+ é a forma recomendada de fazer isso.

### Inserção de Dados

```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;
import java.sql.Date; // Para java.sql.Date
import java.time.LocalDate; // Para java.time.LocalDate

public class InsercaoPreparedStatement {

    // Simulação de uma conexão com o banco de dados
    private Connection connection;

    public InsercaoPreparedStatement(Connection connection) {
        this.connection = connection;
    }

    /**
     * Insere um novo usuário no banco de dados.
     * @param nome O nome do usuário.
     * @param email O email do usuário.
     * @param idade A idade do usuário.
     * @param dataNascimento A data de nascimento do usuário.
     * @return true se a inserção foi bem-sucedida, false caso contrário.
     */
    public boolean inserirUsuario(String nome, String email, int idade, LocalDate dataNascimento) {
        String sql = "INSERT INTO usuarios (nome, email, idade, data_nascimento) VALUES (?, ?, ?, ?)";

        // Usando try-with-resources para garantir o fechamento automático do PreparedStatement
        try (PreparedStatement preparedStatement = connection.prepareStatement(sql)) {
            preparedStatement.setString(1, nome); // Define o primeiro placeholder (nome)
            preparedStatement.setString(2, email); // Define o segundo placeholder (email)
            preparedStatement.setInt(3, idade); // Define o terceiro placeholder (idade)
            preparedStatement.setDate(4, Date.valueOf(dataNascimento)); // Converte LocalDate para java.sql.Date

            int linhasAfetadas = preparedStatement.executeUpdate();
            return linhasAfetadas > 0; // Retorna true se pelo menos uma linha foi inserida
        } catch (SQLException e) {
            System.err.println("Erro ao inserir usuário: " + e.getMessage());
            e.printStackTrace();
            return false;
        }
    }

    public static void main(String[] args) {
        // Exemplo de como obter uma conexão (em uma aplicação real, usaria um pool de conexões)
        try (Connection conn = ConexaoBanco.getConnection()) { // Supondo uma classe ConexaoBanco que fornece a conexão
            InsercaoPreparedStatement insercao = new InsercaoPreparedStatement(conn);

            // Inserindo dados de Gedê
            LocalDate dataNascimentoGede = LocalDate.of(2001, 9, 19);
            if (insercao.inserirUsuario("Luiz Gustavo Gomes Damasceno", "luiz.gustavo@example.com", 23, dataNascimentoGede)) {
                System.out.println("Usuário Gedê inserido com sucesso!");
            } else {
                System.out.println("Falha ao inserir usuário Gedê.");
            }

            // Inserindo dados da Ju
            LocalDate dataNascimentoJu = LocalDate.of(2000, 5, 23); // Data de nascimento fictícia para Ju, já que não foi fornecida
            if (insercao.inserirUsuario("Juliana Gomes Miranda", "ju@example.com", 24, dataNascimentoJu)) {
                System.out.println("Usuária Ju inserida com sucesso!");
            } else {
                System.out.println("Falha ao inserir usuária Ju.");
            }

        } catch (SQLException e) {
            System.err.println("Erro ao obter conexão: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

// Classe de exemplo para obter conexão (deve ser substituída por sua implementação real)
class ConexaoBanco {
    private static final String URL = "jdbc:mysql://localhost:3306/seubanco";
    private static final String USER = "seu_usuario";
    private static final String PASSWORD = "sua_senha";

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(URL, USER, PASSWORD);
    }
}
```

### Atualização de Dados

```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;

public class AtualizacaoPreparedStatement {

    private Connection connection;

    public AtualizacaoPreparedStatement(Connection connection) {
        this.connection = connection;
    }

    /**
     * Atualiza o email de um usuário com base no ID.
     * @param id O ID do usuário a ser atualizado.
     * @param novoEmail O novo email do usuário.
     * @return true se a atualização foi bem-sucedida, false caso contrário.
     */
    public boolean atualizarEmailUsuario(int id, String novoEmail) {
        String sql = "UPDATE usuarios SET email = ? WHERE id = ?";

        try (PreparedStatement preparedStatement = connection.prepareStatement(sql)) {
            preparedStatement.setString(1, novoEmail); // Define o novo email
            preparedStatement.setInt(2, id); // Define o ID do usuário

            int linhasAfetadas = preparedStatement.executeUpdate();
            return linhasAfetadas > 0;
        } catch (SQLException e) {
            System.err.println("Erro ao atualizar email do usuário: " + e.getMessage());
            e.printStackTrace();
            return false;
        }
    }

    public static void main(String[] args) {
        try (Connection conn = ConexaoBanco.getConnection()) {
            AtualizacaoPreparedStatement atualizacao = new AtualizacaoPreparedStatement(conn);

            // Assumindo que o usuário Gedê tenha ID 1 (pode variar dependendo do seu banco)
            // Você poderia buscar o ID pelo nome/email antes de atualizar
            if (atualizacao.atualizarEmailUsuario(1, "novo.email.gede@example.com")) {
                System.out.println("Email do usuário Gedê atualizado com sucesso!");
            } else {
                System.out.println("Falha ao atualizar email do usuário Gedê.");
            }

        } catch (SQLException e) {
            System.err.println("Erro ao obter conexão: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
```

### Seleção de Dados

```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

public class SelecaoPreparedStatement {

    private Connection connection;

    public SelecaoPreparedStatement(Connection connection) {
        this.connection = connection;
    }

    /**
     * Busca usuários com idade maior que um valor especificado.
     * @param idadeMinima A idade mínima para a busca.
     */
    public void buscarUsuariosPorIdadeMinima(int idadeMinima) {
        String sql = "SELECT id, nome, email, idade FROM usuarios WHERE idade > ?";

        // Usando try-with-resources para PreparedStatement e ResultSet
        try (PreparedStatement preparedStatement = connection.prepareStatement(sql)) {
            preparedStatement.setInt(1, idadeMinima); // Define o parâmetro de idade mínima

            try (ResultSet resultSet = preparedStatement.executeQuery()) {
                System.out.println("\\nUsuários com mais de " + idadeMinima + " anos:");
                while (resultSet.next()) {
                    int id = resultSet.getInt("id");
                    String nome = resultSet.getString("nome");
                    String email = resultSet.getString("email");
                    int idade = resultSet.getInt("idade");
                    System.out.printf("ID: %d, Nome: %s, Email: %s, Idade: %d%n", id, nome, email, idade);
                }
            }
        } catch (SQLException e) {
            System.err.println("Erro ao buscar usuários: " + e.getMessage());
            e.printStackTrace();
        }
    }

    /**
     * Busca um usuário pelo email.
     * @param email O email do usuário a ser buscado.
     * @return Um array de String com id, nome, email e idade do usuário, ou null se não encontrado.
     */
    public String[] buscarUsuarioPorEmail(String email) {
        String sql = "SELECT id, nome, email, idade FROM usuarios WHERE email = ?";
        String[] userData = null;

        try (PreparedStatement preparedStatement = connection.prepareStatement(sql)) {
            preparedStatement.setString(1, email);

            try (ResultSet resultSet = preparedStatement.executeQuery()) {
                if (resultSet.next()) { // Se encontrou um resultado
                    userData = new String[4];
                    userData[0] = String.valueOf(resultSet.getInt("id"));
                    userData[1] = resultSet.getString("nome");
                    userData[2] = resultSet.getString("email");
                    userData[3] = String.valueOf(resultSet.getInt("idade"));
                }
            }
        } catch (SQLException e) {
            System.err.println("Erro ao buscar usuário por email: " + e.getMessage());
            e.printStackTrace();
        }
        return userData;
    }

    public static void main(String[] args) {
        try (Connection conn = ConexaoBanco.getConnection()) {
            SelecaoPreparedStatement selecao = new SelecaoPreparedStatement(conn);

            selecao.buscarUsuariosPorIdadeMinima(22);

            String[] gededata = selecao.buscarUsuarioPorEmail("gede@example.com"); // Ou o novo email se você atualizou
            if (gededata != null) {
                System.out.printf("\\nDetalhes do Gedê: ID: %s, Nome: %s, Email: %s, Idade: %s%n",
                                  gededata[0], gededata[1], gededata[2], gededata[3]);
            } else {
                System.out.println("\\nUsuário Gedê não encontrado.");
            }

        } catch (SQLException e) {
            System.err.println("Erro ao obter conexão: " + e.getMessage());
            e.printStackTrace();
        }
    }
}
```

### Deleção de Dados

```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;

public class DelecaoPreparedStatement {

    private Connection connection;

    public DelecaoPreparedStatement(Connection connection) {
        this.connection = connection;
    }

    /**
     * Deleta um usuário com base no ID.
     * @param id O ID do usuário a ser deletado.
     * @return true se a deleção foi bem-sucedida, false caso contrário.
     */
    public boolean deletarUsuario(int id) {
        String sql = "DELETE FROM usuarios WHERE id = ?";

        try (PreparedStatement preparedStatement = connection.prepareStatement(sql)) {
            preparedStatement.setInt(1, id); // Define o ID do usuário

            int linhasAfetadas = preparedStatement.executeUpdate();
            return linhasAfetadas > 0;
        } catch (SQLException e) {
            System.err.println("Erro ao deletar usuário: " + e.getMessage());
            e.printStackTrace();
            return false;
        }
    }

    public static void main(String[] args) {
        try (Connection conn = ConexaoBanco.getConnection()) {
            DelecaoPreparedStatement delecao = new DelecaoPreparedStatement(conn);

            // Exemplo: Deletar o usuário com ID 2 (pode ser a Ju, dependendo da ordem de inserção)
            if (delecao.deletarUsuario(2)) {
                System.out.println("Usuário com ID 2 deletado com sucesso!");
            } else {
```

### Batch Updates (Atualizações em Lote)

Para inserções ou atualizações em massa, o `PreparedStatement` pode ser combinado com o processamento em lote para melhorar significativamente a performance, reduzindo o número de viagens de ida e volta ao banco de dados.

```java
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.SQLException;
import java.time.LocalDate;
import java.sql.Date;

public class BatchUpdatePreparedStatement {

    private Connection connection;

    public BatchUpdatePreparedStatement(Connection connection) {
        this.connection = connection;
    }

    /**
     * Insere múltiplos usuários usando batch update.
     * @param usuarios Uma lista de arrays, onde cada array contém {nome, email, idade, dataNascimento}.
     * @return Um array de int representando o número de linhas afetadas por cada operação no batch.
     */
    public int[] inserirMultiplosUsuarios(String[][] usuarios) {
        String sql = "INSERT INTO usuarios (nome, email, idade, data_nascimento) VALUES (?, ?, ?, ?)";
        int[] resultados = {};

        try (PreparedStatement preparedStatement = connection.prepareStatement(sql)) {
            // Desativa o auto-commit para gerenciar a transação manualmente
            connection.setAutoCommit(false);

            for (String[] usuario : usuarios) {
                preparedStatement.setString(1, usuario[0]);
                preparedStatement.setString(2, usuario[1]);
                preparedStatement.setInt(3, Integer.parseInt(usuario[2]));
                preparedStatement.setDate(4, Date.valueOf(LocalDate.parse(usuario[3]))); // Assume "YYYY-MM-DD"
                preparedStatement.addBatch(); // Adiciona o comando ao batch
            }

            resultados = preparedStatement.executeBatch(); // Executa todos os comandos no batch
            connection.commit(); // Confirma a transação
            System.out.println("Batch executado com sucesso. Resultados: " + java.util.Arrays.toString(resultados));

        } catch (SQLException e) {
            System.err.println("Erro ao inserir múltiplos usuários em batch: " + e.getMessage());
            e.printStackTrace();
            try {
                if (connection != null) {
                    connection.rollback(); // Desfaz a transação em caso de erro
                    System.out.println("Transação desfeita.");
                }
            } catch (SQLException rollbackEx) {
                System.err.println("Erro ao desfazer transação: " + rollbackEx.getMessage());
            }
        } finally {
            try {
                if (connection != null) {
                    connection.setAutoCommit(true); // Reativa o auto-commit
                }
            } catch (SQLException e) {
                System.err.println("Erro ao reativar auto-commit: " + e.getMessage());
            }
        }
        return resultados;
    }

    public static void main(String[] args) {
        try (Connection conn = ConexaoBanco.getConnection()) {
            BatchUpdatePreparedStatement batchInsercao = new BatchUpdatePreparedStatement(conn);

            String[][] novosUsuarios = {
                {"Maria", "maria@example.com", "30", "1994-01-15"},
                {"João", "joao@example.com", "28", "1996-07-20"},
                {"Ana", "ana@example.com", "25", "1999-03-10"}
            };

            batchInsercao.inserirMultiplosUsuarios(novosUsuarios);

        } catch (SQLException e) {
            System.err.println("Erro ao obter conexão: " + e.getMessage());
            e.printStackTrace();
        }
    }
}

```

### 5\. Informações Adicionais

### Gerenciamento de Recursos (Try-with-resources)

Desde o Java 7, a instrução `try-with-resources` é a maneira mais robusta e segura de garantir que recursos que implementam `java.lang.AutoCloseable` (como `Connection`, `PreparedStatement` e `ResultSet`) sejam fechados automaticamente, mesmo que ocorram exceções.

**Código Antigo (Comum, mas propenso a erros):**

```java
Connection conn = null;
PreparedStatement ps = null;
ResultSet rs = null;
try {
    conn = DriverManager.getConnection(URL, USER, PASSWORD);
    ps = conn.prepareStatement("SELECT * FROM tabela");
    rs = ps.executeQuery();
    // Processamento
} catch (SQLException e) {
    // Tratamento de exceção
} finally {
    try {
        if (rs != null) rs.close();
        if (ps != null) ps.close();
        if (conn != null) conn.close();
    } catch (SQLException e) {
        // Log ou tratamento de erro ao fechar
    }
}

```

**Código Recomendado (Try-with-resources):**

```java
try (Connection conn = DriverManager.getConnection(URL, USER, PASSWORD);
     PreparedStatement ps = conn.prepareStatement("SELECT * FROM tabela");
     ResultSet rs = ps.executeQuery()) {
    // Processamento
} catch (SQLException e) {
    // Tratamento de exceção
}
// Recursos são fechados automaticamente aqui

```

### Tratamento de Exceções

Todas as operações JDBC podem lançar `SQLException`. É crucial capturá-las e tratá-las adequadamente. Isso pode incluir:

- **Logar a exceção:** Registrar os detalhes do erro para depuração.
- **Mensagens amigáveis:** Apresentar mensagens de erro compreensíveis ao usuário (mas sem expor detalhes internos do banco de dados).
- **Rollback de transações:** Em operações que envolvem múltiplas modificações (como um batch update), é fundamental realizar um rollback em caso de falha para manter a consistência dos dados.

### Conexões de Banco de Dados e `DataSource`

Em aplicações corporativas (especialmente web e de grande escala), o uso direto de `DriverManager.getConnection()` para cada requisição ao banco de dados é ineficiente e pode levar a problemas de desempenho e exaustão de recursos. A prática recomendada é utilizar um **Pool de Conexões**, que gerencia um conjunto de conexões abertas e prontas para uso.

Bibliotecas populares para pools de conexão incluem:

- **HikariCP:** Conhecido por sua alta performance e baixo overhead.
- **Apache DBCP (Database Connection Pool):** Uma opção robusta e madura.
- **c3p0:** Outra alternativa popular.

A interface `javax.sql.DataSource` é a forma padrão de abstrair os detalhes da conexão com o banco de dados. Ao invés de `DriverManager.getConnection()`, você obteria uma conexão de um `DataSource` configurado, que por sua vez, gerencia o pool de conexões.

**Exemplo Básico com `DataSource` (requer configuração prévia do pool):**

```java
import javax.sql.DataSource; // Importe esta interface
import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

// Supondo que você tenha um DataSource configurado e injetado (via Spring, JNDI, etc.)
public class ExemploComDataSource {

    private DataSource dataSource; // O DataSource é injetado/configurado

    public ExemploComDataSource(DataSource dataSource) {
        this.dataSource = dataSource;
    }

    public void buscarProdutosPorPreco(double precoMaximo) {
        String sql = "SELECT id, nome, preco FROM produtos WHERE preco <= ?";

        try (Connection conn = dataSource.getConnection(); // Obtém conexão do pool
             PreparedStatement ps = conn.prepareStatement(sql)) {

            ps.setDouble(1, precoMaximo);

            try (ResultSet rs = ps.executeQuery()) {
                while (rs.next()) {
                    System.out.println("Produto: " + rs.getString("nome") + ", Preço: " + rs.getDouble("preco"));
                }
            }
        } catch (SQLException e) {
            System.err.println("Erro ao buscar produtos: " + e.getMessage());
        }
    }

    // No main ou em um framework, você configuraria o DataSource
    public static void main(String[] args) {
        // Exemplo de configuração simples (HikariCP é mais avançado e comum em produção)
        // Isso é apenas para ilustrar, em uma aplicação real, seria configurado via framework
        org.h2.jdbcx.JdbcDataSource h2DataSource = new org.h2.jdbcx.JdbcDataSource(); // Exemplo com H2 para simplicidade
        h2DataSource.setURL("jdbc:h2:mem:testdb;DB_CLOSE_DELAY=-1"); // Banco de dados em memória
        h2DataSource.setUser("sa");
        h2DataSource.setPassword("");

        // Crie algumas tabelas e insira dados para o exemplo funcionar
        try (Connection conn = h2DataSource.getConnection();
             java.sql.Statement stmt = conn.createStatement()) {
            stmt.execute("CREATE TABLE produtos (id INT PRIMARY KEY AUTO_INCREMENT, nome VARCHAR(255), preco DOUBLE)");
            stmt.execute("INSERT INTO produtos (nome, preco) VALUES ('Notebook', 1200.00)");
            stmt.execute("INSERT INTO produtos (nome, preco) VALUES ('Mouse', 25.50)");
            stmt.execute("INSERT INTO produtos (nome, preco) VALUES ('Teclado', 75.00)");
        } catch (SQLException e) {
            e.printStackTrace();
        }

        ExemploComDataSource app = new ExemploComDataSource(h2DataSource);
        app.buscarProdutosPorPreco(100.00);
    }
}

```
