# Anotações JPA

---

### **Anotações de Mapeamento Básico**

Essas anotações são usadas para mapear classes Java para tabelas de banco de dados e seus atributos para colunas.

- **`@Entity`**: Indica que uma classe Java é uma entidade e será mapeada para uma tabela no banco de dados.
- **`@Table`**: Especifica o nome da tabela no banco de dados para a qual a entidade será mapeada. É opcional se o nome da tabela for o mesmo nome da classe.
- **`@Id`**: Marca um campo como a **chave primária** da entidade.
- **`@GeneratedValue`**: Configura a estratégia de geração de valores para a chave primária (e.g., `IDENTITY`, `AUTO`, `SEQUENCE`, `TABLE`).
- **`@Column`**: Mapeia um campo da entidade para uma coluna no banco de dados. Permite configurar propriedades como nome da coluna, nulidade (`nullable`), tamanho (`length`), precisão (`precision`), escala (`scale`), etc.
- **`@Transient`**: Indica que um campo da entidade **não deve ser persistido** no banco de dados.
- **`@Basic`**: Especifica o tipo de *fetch* (e.g., `LAZY`, `EAGER`) para um atributo. É o padrão para atributos simples.

---

### **Anotações de Mapeamento de Relacionamentos**

Essas anotações são usadas para definir os relacionamentos entre entidades (1:1, 1:N, N:1, N:N).

- **`@OneToOne`**: Mapeia um relacionamento **um-para-um**.
- **`@OneToMany`**: Mapeia um relacionamento **um-para-muitos**.
- **`@ManyToOne`**: Mapeia um relacionamento **muitos-para-um**.
- **`@ManyToMany`**: Mapeia um relacionamento **muitos-para-muitos**.
- **`@JoinColumn`**: Usada em relacionamentos para especificar a **coluna de junção** (chave estrangeira).
- **`@JoinTable`**: Usada em relacionamentos muitos-para-muitos para especificar a **tabela de junção**.

---

### **Anotações de Herança**

Usadas para definir estratégias de herança entre entidades.

- **`@Inheritance`**: Define a estratégia de herança para uma hierarquia de entidades (e.g., `SINGLE_TABLE`, `JOINED`, `TABLE_PER_CLASS`).
- **`@DiscriminatorColumn`**: Usada com a estratégia `SINGLE_TABLE` para definir a coluna que diferencia os tipos de entidades.
- **`@DiscriminatorValue`**: Usada com a estratégia `SINGLE_TABLE` para definir o valor da coluna discriminatória para uma subclasse específica.

---

### **Outras Anotações Úteis**

- **`@Embedded`**: Indica que uma classe é **incorporável** e seus atributos devem ser mapeados como colunas na tabela da entidade que a contém.
- **`@Embeddable`**: Marca uma classe como sendo uma classe incorporável (para ser usada com `@Embedded`).
- **`@Enumerated`**: Define como um `enum` deve ser persistido (e.g., como `ORDINAL` ou `STRING`).
- **`@Temporal`**: Especifica o tipo de mapeamento para campos de data e hora (e.g., `DATE`, `TIME`, `TIMESTAMP`).
- **`@Version`**: Usada para **controle de concorrência otimista**. O JPA incrementa automaticamente o valor desta coluna em cada atualização.
- **`@MappedSuperclass`**: Indica que uma classe não é uma entidade, mas suas propriedades persistentes devem ser herdadas por suas subclasses de entidade.

---

### **Anotações para Consultas e Gráficos de Entidades**

- **`@NamedQuery`**: Define uma **consulta JPQL nomeada** que pode ser reutilizada.
- **`@NamedNativeQuery`**: Define uma **consulta SQL nativa nomeada**.
- **`@NamedEntityGraph`**: Define um **gráfico de entidade** para especificar o *fetching* de dados relacionados de forma mais eficiente.
- **`@EntityGraph`**: Usada em métodos de repositório para aplicar um `@NamedEntityGraph`.

---

### **Anotações para Listeners de Ciclo de Vida**

- **`@PrePersist`**: Um método anotado com isso será executado **antes** que uma entidade seja persistida (salva pela primeira vez).
- **`@PostPersist`**: Um método anotado com isso será executado **depois** que uma entidade for persistida.
- **`@PreUpdate`**: Um método anotado com isso será executado **antes** que uma entidade existente seja atualizada.
- **`@PostUpdate`**: Um método anotado com isso será executado **depois** que uma entidade existente for atualizada.
- **`@PreRemove`**: Um método anotado com isso será executado **antes** que uma entidade seja removida.
- **`@PostRemove`**: Um método anotado com isso será executado **depois** que uma entidade for removida.
- **`@PostLoad`**: Um método anotado com isso será executado **depois** que uma entidade for carregada do banco de dados.

---

### **Anotações Adicionais (Casos Específicos)**

Essas são anotações para cenários mais avançados ou menos comuns no dia a dia.

- **`@ElementCollection`**: Usada para mapear **coleções de tipos básicos ou incorporáveis** (não entidades). Por exemplo, uma lista de strings ou uma lista de endereços incorporáveis.
- **`@MapKey`**: Para mapear a **chave de um `Map`** persistido.
- **`@MapKeyColumn`**: Especifica a coluna para a chave de um `Map`.
- **`@OrderColumn`**: Usada para preservar a **ordem de uma lista persistida** em um relacionamento.
- **`@AttributeOverride`**: Para sobrescrever o mapeamento de atributos de uma classe `@Embeddable` ou `@MappedSuperclass` em uma entidade específica.
- **`@AttributeOverrides`**: Usada para agrupar várias anotações `@AttributeOverride`.
- **`@AssociationOverride`**: Para sobrescrever mapeamentos de associações (relacionamentos).
- **`@AssociationOverrides`**: Usada para agrupar várias anotações `@AssociationOverride`.
- **`@Convert`**: Para definir e aplicar **conversores personalizados** entre tipos Java e tipos de banco de dados.
- **`@Converter`**: Usada para especificar que a implementação do atributo atual `AttributeConverter` pode ser usada como um conversor de atributo básico JPA.
- **`@OptimisticLocking`**: Mais comum em implementações como o Hibernate para controlar o bloqueio otimista **sem o `@Version`**.
- **`@SecondaryTable`**: Para mapear uma entidade para **múltiplas tabelas**.
- **`@SecondaryTables`**: Usada para agrupar várias anotações `@SecondaryTable`.
