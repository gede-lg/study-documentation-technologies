# Introdução

TypeScript é um superconjunto do JavaScript que adiciona tipagem estática e outros recursos avançados à linguagem. Criado pela Microsoft, ele tem se tornado cada vez mais relevante no desenvolvimento moderno devido à sua capacidade de melhorar a produtividade e reduzir erros em aplicações escaláveis. Enquanto o JavaScript é amplamente utilizado e conhecido por sua flexibilidade, essa mesma característica pode levar a erros difíceis de depurar. O TypeScript foi projetado para mitigar esses problemas, tornando o código mais previsível e robusto.

## Definição e Conceitos Fundamentais

### O que é TypeScript?

TypeScript é uma linguagem de programação que transpila para JavaScript, permitindo que desenvolvedores usem tipagem estática, recursos de POO (programação orientada a objetos) e ferramentas aprimoradas de desenvolvimento.

### Diferenças entre JavaScript e TypeScript

| Característica | JavaScript | TypeScript |
| --- | --- | --- |
| **Tipagem** | Dinâmica | Estática |
| **Compilação** | Não requer compilação | Requer compilação para JS |
| **Ferramentas** | Menos suporte para IDEs | Suporte aprimorado para IDEs |
| **Classes/Interfaces** | Suporte limitado (ES6+) | Suporte completo |

## Sintaxe e Estrutura

### Declaração de Tipos

```tsx
let nome: string = "Luiz"; // Exemplo de tipagem explícita
let idade = 22; // Inferência de tipo

```

### Interfaces e Classes

### Interfaces

```tsx
interface Usuario {
    nome: string;
    idade: number;
    ativo?: boolean; // Propriedade opcional
}

const user: Usuario = { nome: "Luiz", idade: 22 };

```

### Classes

```tsx
class Pessoa {
    private nome: string;
    constructor(nome: string) {
        this.nome = nome;
    }
    apresentar(): string {
        return `Meu nome é ${this.nome}`;
    }
}

const pessoa = new Pessoa("Juliana");
console.log(pessoa.apresentar());

```

### Generics

```tsx
function retornarElemento<T>(item: T): T {
    return item;
}

console.log(retornarElemento<string>("Teste"));

```

## Componentes Principais

### Tipos Primitivos e Complexos

- Primitivos: `string`, `number`, `boolean`, `null`, `undefined`.
- Complexos: `array`, `tuple`, `enum`, `any`, `unknown`, `never`.

### Módulos e Namespaces

- **Módulos**: Permitem dividir o código em arquivos menores e reutilizáveis.

```tsx
export class Carro {
    marca: string;
    constructor(marca: string) {
        this.marca = marca;
    }
}

import { Carro } from "./carro";

```

### Suporte a ECMAScript Moderno

TypeScript suporta recursos como `async/await`, `destructuring` e `spread/rest operators`, mantendo compatibilidade com navegadores antigos através de configurações no `tsconfig.json`.

## Uso Avançado

### Configuração do `tsconfig.json`

```json
{
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src"
  }
}

```

### Integração com Frameworks

- **Angular**: Desenvolvido com TypeScript nativamente.
- **React**: Usa `@types` para tipagem em componentes.

### Casos de Uso Reais

- Aplicações corporativas complexas.
- Ambientes com grandes equipes de desenvolvimento.

## Exemplos de Código Otimizados

### Uso Básico

```tsx
const soma = (a: number, b: number): number => a + b;
console.log(soma(5, 10));

```

### Uso Avançado com Generics e Interfaces

```tsx
interface Repositorio<T> {
    buscarTodos(): T[];
    salvar(item: T): void;
}

class UsuarioRepositorio implements Repositorio<Usuario> {
    private usuarios: Usuario[] = [];

    buscarTodos(): Usuario[] {
        return this.usuarios;
    }

    salvar(usuario: Usuario): void {
        this.usuarios.push(usuario);
    }
}

```

## Informações Adicionais

- **Melhores Práticas**:
    - Use tipagem estrita sempre que possível.
    - Refatore códigos JavaScript gradualmente para TypeScript.
- **Benefícios na Equipe**:
    - Código mais legível e manutenível.
    - Redução de bugs em ambientes complexos.
