# Herança e Abstração

## 1. Introdução

A **herança** e a **abstração** são dois pilares fundamentais da programação orientada a objetos (POO). Em TypeScript, esses conceitos permitem a criação de estruturas de código mais organizadas, reutilizáveis e escaláveis.

- **Herança:** Permite que classes compartilhem atributos e comportamentos comuns, estabelecendo uma relação hierárquica entre superclasses e subclasses.
- **Abstração:** Permite definir classes e métodos que servem como “modelo” para outras classes, forçando a implementação de comportamentos essenciais sem expor detalhes internos.

Esses conceitos são importantes porque promovem a reutilização de código, melhoram a manutenção e facilitam a compreensão de sistemas complexos.

## 3. Conteúdo Detalhado

### Definição e Conceitos Fundamentais

### Herança

- **Conceito Básico:** Herança é um mecanismo pelo qual uma classe (subclasse) herda atributos e métodos de outra classe (superclasse).
- **Conceito Avançado:** Permite polimorfismo, onde uma subclasse pode sobrescrever métodos da superclasse, proporcionando implementações específicas.

### Abstração

- **Conceito Básico:** Abstração consiste em definir uma interface ou modelo simplificado que oculta os detalhes de implementação.
- **Conceito Avançado:** Em TypeScript, isso é realizado através de classes abstratas e métodos abstratos, que servem como contratos para subclasses implementarem funcionalidades específicas.

### Sintaxe e Estrutura

### Classes Abstratas e Métodos Abstratos

- **Definição de Classe Abstrata:**
    
    Uma classe abstrata não pode ser instanciada diretamente e pode conter métodos abstratos que devem ser implementados pelas subclasses.
    
- **Sintaxe:**
    
    ```tsx
    abstract class Animal {
        // Método abstrato (não possui implementação)
        abstract fazerSom(): void;
    
        // Método comum a todas as subclasses
        mover(): void {
            console.log("O animal está se movendo.");
        }
    }
    
    ```
    
- **Características:**
    - Não pode ser instanciada.
    - Pode conter tanto métodos abstratos quanto métodos com implementação.
    - Força as subclasses a implementarem os métodos abstratos.

### Subclasses e Superclasses

- **Herança Simples:**
Uma subclasse estende uma superclasse e herda seus métodos e propriedades.
- **Sintaxe:**
    
    ```tsx
    class Cachorro extends Animal {
        // Implementação do método abstrato
        fazerSom(): void {
            console.log("Au Au!");
        }
    }
    
    ```
    
- **Características:**
    - A subclasse pode sobrescrever métodos da superclasse.
    - Pode adicionar novos métodos e propriedades específicos.

### Componentes Principais

- **Classes Abstratas:** Definem um modelo para outras classes.
    - **Métodos Abstratos:** São declarados sem implementação e devem ser implementados pelas subclasses.
    - **Métodos Concretos:** Têm uma implementação padrão que pode ser herdada ou sobrescrita.
- **Subclasses:** Estendem classes abstratas ou outras classes e implementam métodos abstratos conforme necessário.
    - **Herança:** Facilita a reutilização de código.
    - **Polimorfismo:** Permite que diferentes classes sejam tratadas de forma uniforme.

### Uso Avançado

- **Polimorfismo Avançado:**
    
    Ao definir métodos abstratos em classes abstratas, você garante que todas as subclasses possuam implementações desses métodos, permitindo a utilização de objetos de subclasses de forma intercambiável.
    
- **Padrões de Projeto:**
    
    Herança e abstração são frequentemente utilizadas em padrões de design, como o Template Method, onde a estrutura de um algoritmo é definida em uma classe abstrata, e as subclasses preenchem os detalhes.
    
- **Interfaces vs. Classes Abstratas:**
    
    Enquanto as interfaces definem contratos sem implementação, as classes abstratas podem fornecer uma implementação padrão, sendo úteis quando há uma lógica comum entre as subclasses.
    

## 4. Exemplos de Código Otimizados

### Exemplo Básico: Herança e Implementação de Métodos Abstratos

```tsx
// Classe abstrata com método abstrato e método concreto
abstract class Veiculo {
    // Método abstrato: deve ser implementado pelas subclasses
    abstract mover(): void;

    // Método concreto com implementação comum
    ligar(): void {
        console.log("O veículo está ligado.");
    }
}

// Subclasse que implementa o método abstrato
class Carro extends Veiculo {
    mover(): void {
        console.log("O carro está se movendo.");
    }
}

// Uso do código
const meuCarro = new Carro();
meuCarro.ligar(); // Saída: O veículo está ligado.
meuCarro.mover(); // Saída: O carro está se movendo.

```

### Exemplo Avançado: Polimorfismo com Classes Abstratas

```tsx
// Classe abstrata Animal
abstract class Animal {
    abstract fazerSom(): void;

    mover(): void {
        console.log("O animal está se movendo.");
    }
}

// Subclasses que implementam o método abstrato
class Gato extends Animal {
    fazerSom(): void {
        console.log("Miau!");
    }
}

class Cachorro extends Animal {
    fazerSom(): void {
        console.log("Au Au!");
    }
}

// Função que utiliza polimorfismo para chamar o método fazerSom
function executarSom(animal: Animal): void {
    animal.fazerSom();
}

// Instanciando objetos das subclasses
const gato = new Gato();
const cachorro = new Cachorro();

executarSom(gato);      // Saída: Miau!
executarSom(cachorro);  // Saída: Au Au!

```

## 5. Informações Adicionais

- **Boas Práticas:**
    - Use classes abstratas quando existir lógica compartilhada entre subclasses.
    - Evite hierarquias de herança muito profundas para manter o código legível.
    - Prefira a composição sobre a herança quando apropriado.
- **Nuances:**
    - **Métodos Concretos vs. Abstratos:** Métodos concretos podem ser herdados ou sobrescritos, permitindo flexibilidade.
    - **Interfaces:** São ideais para definir contratos sem implementação, enquanto classes abstratas são úteis quando há comportamentos comuns a serem compartilhados.
- **Limitações:**
    - TypeScript não permite múltipla herança de classes, mas suporta a implementação de múltiplas interfaces.
