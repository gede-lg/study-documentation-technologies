# Classes

# Introdução

As **classes** são um dos pilares da programação orientada a objetos (POO) e desempenham um papel fundamental na organização e modularização do código em aplicações modernas. Em TypeScript, as classes possibilitam a criação de estruturas reutilizáveis que encapsulam dados (propriedades) e comportamentos (métodos). Este conceito é essencial para o desenvolvimento de aplicações escaláveis e de fácil manutenção, permitindo aos desenvolvedores modelar entidades do mundo real de forma intuitiva e robusta.

# Conteúdo Detalhado

## Definição e Conceitos Fundamentais

### O que é uma classe?

Uma **classe** é um molde (ou blueprint) que define a forma e o comportamento de objetos. Em TypeScript, ela permite:

- **Encapsular dados e comportamentos**: Propriedades (variáveis) e métodos (funções) que operam sobre esses dados.
- **Modelar entidades reais**: Permite representar conceitos do mundo real (por exemplo, `Carro`, `Usuário`, `Produto`) de maneira organizada.

### Conceitos Básicos vs. Avançados

- **Conceitos Básicos:**
    - **Declaração de classe:** Como definir uma classe usando a palavra-chave `class`.
    - **Propriedades e métodos:** Atributos e funções que pertencem à classe.
    - **Construtor:** Função especial usada para inicializar novos objetos.
    - **Modificadores de acesso:** `public`, `private` e `protected`, que definem a visibilidade de propriedades e métodos.
- **Conceitos Avançados:**
    - **Herança:** Permite que uma classe herde propriedades e métodos de outra.
    - **Polimorfismo:** Permite que métodos se comportem de forma diferente com base no objeto que os invoca.
    - **Classes abstratas e interfaces:** Definem contratos e estruturas para classes derivadas, ajudando a criar arquiteturas mais robustas.
    - **Encapsulamento e abstração:** Princípios fundamentais que ajudam na organização do código e na proteção dos dados internos.

## Sintaxe e Estrutura

### Estrutura Básica de uma Classe em TypeScript

Em TypeScript, a estrutura básica de uma classe inclui:

- **Declaração da classe:** Definida com a palavra-chave `class`.
- **Construtor:** Método especial que é chamado quando um objeto é instanciado.
- **Propriedades:** Variáveis associadas à classe.
- **Métodos:** Funções que definem o comportamento dos objetos da classe.

### Exemplo Simples de Declaração e Utilização

```tsx
// Declaração de uma classe 'Pessoa'
class Pessoa {
  // Propriedades
  public nome: string;
  public idade: number;

  // Construtor para inicializar as propriedades
  constructor(nome: string, idade: number) {
    this.nome = nome;
    this.idade = idade;
  }

  // Método para exibir informações da pessoa
  public apresentar(): string {
    return `Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`;
  }
}

// Utilizando a classe Pessoa
const pessoa1 = new Pessoa("João", 30);
console.log(pessoa1.apresentar());

```

Neste exemplo, a classe `Pessoa` encapsula dados (nome e idade) e comportamentos (o método `apresentar`) de uma pessoa.

## Componentes Principais

### Propriedades

- **Definição:** Variáveis que armazenam o estado de um objeto.
- **Exemplo:**
    
    ```tsx
    class Produto {
      public nome: string;
      public preco: number;
    
      constructor(nome: string, preco: number) {
        this.nome = nome;
        this.preco = preco;
      }
    }
    
    ```
    

### Métodos

- **Definição:** Funções que descrevem os comportamentos da classe.
- **Exemplo:**
    
    ```tsx
    class Calculadora {
      // Método para somar dois números
      public somar(a: number, b: number): number {
        return a + b;
      }
    }
    
    const calc = new Calculadora();
    console.log(calc.somar(10, 5)); // Saída: 15
    
    ```
    

### Construtores

- **Definição:** Método especial usado para inicializar os objetos.
- **Exemplo:** Já demonstrado na classe `Pessoa`.

### Modificadores de Acesso

- **public:** Propriedade ou método acessível de qualquer lugar.
- **private:** Acessível somente dentro da classe onde foi definido.
- **protected:** Acessível na classe onde foi definido e em classes derivadas.
- **Exemplo com modificadores de acesso:**
    
    ```tsx
    class ContaBancaria {
      private saldo: number;
    
      constructor(saldoInicial: number) {
        this.saldo = saldoInicial;
      }
    
      // Método público para consultar o saldo
      public getSaldo(): number {
        return this.saldo;
      }
    
      // Método para depositar valor
      public depositar(valor: number): void {
        if (valor > 0) {
          this.saldo += valor;
        }
      }
    }
    
    const conta = new ContaBancaria(100);
    conta.depositar(50);
    console.log(conta.getSaldo()); // Saída: 150
    
    ```
    

### Interação entre Componentes

- **Propriedades** armazenam os dados.
- **Construtor** inicializa os dados.
- **Métodos** operam sobre os dados e permitem interações com os objetos.
- **Modificadores de acesso** garantem o encapsulamento, protegendo os dados e expondo apenas o que for necessário.

## Uso Avançado

### Herança e Polimorfismo

- **Herança:** Permite que uma classe (subclasse) herde propriedades e métodos de outra (superclasse).
    
    ```tsx
    class Animal {
      public nome: string;
    
      constructor(nome: string) {
        this.nome = nome;
      }
    
      public emitirSom(): void {
        console.log("Som genérico");
      }
    }
    
    class Cachorro extends Animal {
      constructor(nome: string) {
        super(nome);
      }
    
      // Polimorfismo: redefinindo o método emitirSom
      public emitirSom(): void {
        console.log("Latido");
      }
    }
    
    const cachorro = new Cachorro("Rex");
    cachorro.emitirSom(); // Saída: Latido
    
    ```
    

### Interfaces e Classes Abstratas

- **Interfaces:** Definem um contrato para as classes implementarem.
    
    ```tsx
    interface IVeiculo {
      acelerar(): void;
      frear(): void;
    }
    
    class Carro implements IVeiculo {
      public acelerar(): void {
        console.log("Carro acelerando...");
      }
    
      public frear(): void {
        console.log("Carro freando...");
      }
    }
    
    const meuCarro = new Carro();
    meuCarro.acelerar();
    
    ```
    
- **Classes Abstratas:** Não podem ser instanciadas e servem como base para outras classes.
    
    ```tsx
    abstract class Forma {
      abstract calcularArea(): number;
    }
    
    class Retangulo extends Forma {
      constructor(private largura: number, private altura: number) {
        super();
      }
    
      public calcularArea(): number {
        return this.largura * this.altura;
      }
    }
    
    const retangulo = new Retangulo(5, 10);
    console.log(retangulo.calcularArea()); // Saída: 50
    
    ```
    

### Padrões de Projeto

- **Singleton:** Garante que uma classe tenha somente uma instância.
- **Factory:** Cria objetos sem expor a lógica de criação.
- **Decorator:** Adiciona funcionalidades a objetos dinamicamente.

Cada padrão utiliza as classes para criar soluções elegantes e escaláveis para problemas comuns em design de software.

# Exemplos de Código Otimizados

### Exemplo 1: Classe Básica com Métodos e Propriedades

```tsx
// Exemplo de uma classe 'Livro'
class Livro {
  // Propriedades com modificador de acesso
  private titulo: string;
  private autor: string;
  private paginas: number;

  // Construtor para inicializar os atributos
  constructor(titulo: string, autor: string, paginas: number) {
    this.titulo = titulo;
    this.autor = autor;
    this.paginas = paginas;
  }

  // Método para exibir detalhes do livro
  public exibirDetalhes(): void {
    console.log(`Título: ${this.titulo}`);
    console.log(`Autor: ${this.autor}`);
    console.log(`Número de páginas: ${this.paginas}`);
  }
}

const meuLivro = new Livro("Dom Quixote", "Miguel de Cervantes", 863);
meuLivro.exibirDetalhes();

```

### Exemplo 2: Uso de Herança e Polimorfismo

```tsx
// Classe base 'Veiculo'
class Veiculo {
  protected modelo: string;

  constructor(modelo: string) {
    this.modelo = modelo;
  }

  public mover(): void {
    console.log(`${this.modelo} está se movendo.`);
  }
}

// Subclasse 'Bicicleta' que herda de 'Veiculo'
class Bicicleta extends Veiculo {
  constructor(modelo: string) {
    super(modelo);
  }

  // Sobrescrevendo o método mover
  public mover(): void {
    console.log(`${this.modelo} está pedalando.`);
  }
}

const bicicleta = new Bicicleta("Caloi");
bicicleta.mover(); // Saída: Caloi está pedalando.

```

# Informações Adicionais

- **Nuances do TypeScript:**
    
    TypeScript é um superset do JavaScript, permitindo o uso de tipagem estática, o que melhora a confiabilidade e a manutenção do código. Ao utilizar classes, o uso de interfaces e tipos torna o código mais previsível e menos propenso a erros.
    
- **Boas Práticas:**
    - Utilize os modificadores de acesso para encapsular dados e proteger a integridade do estado do objeto.
    - Prefira a composição sobre a herança quando apropriado, para evitar hierarquias excessivamente complexas.
    - Escreva comentários claros e mantenha o código bem documentado para facilitar a manutenção e a escalabilidade.
- **Ferramentas e Ecossistema:**
    - Ferramentas como o Visual Studio Code e extensões de lint podem ajudar a manter o padrão de código e identificar problemas precocemente.
    - O uso de testes unitários é altamente recomendado para garantir o funcionamento correto das classes e de seus métodos.
