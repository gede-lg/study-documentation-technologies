# Interfaces

## 1. Introdução

As **interfaces** são um dos pilares fundamentais da programação em TypeScript, oferecendo uma maneira robusta de definir contratos para objetos e classes. Elas permitem que os desenvolvedores especifiquem quais propriedades e métodos um objeto deve possuir, promovendo maior consistência, legibilidade e manutenção do código. A combinação de interfaces com o recurso `extends` possibilita a criação de hierarquias e a reutilização de definições, tornando o desenvolvimento de aplicações mais escalável e modular.

## 3. Conteúdo Detalhado

### Definição e Conceitos Fundamentais

- **Interface:**
    
    Em TypeScript, uma interface define a estrutura que um objeto deve seguir. Ela declara quais propriedades e métodos um objeto possui, funcionando como um contrato que assegura a conformidade do objeto com uma determinada forma.
    
- **Implementação de Interfaces:**
    
    Uma classe pode implementar uma ou mais interfaces, o que obriga a classe a fornecer implementações para os membros definidos na interface. Isso garante que a classe cumpra o contrato estabelecido.
    
- **Combinação com `extends`:**
    
    Assim como classes, interfaces podem estender outras interfaces. Essa funcionalidade permite a criação de interfaces mais complexas e a reutilização de definições, facilitando a manutenção e a escalabilidade do código.
    
- **Conceitos Básicos vs. Avançados:**
    - **Básico:** Definição de propriedades e métodos obrigatórios, implementação de interfaces em classes.
    - **Avançado:** Herança entre interfaces com `extends`, interfaces que descrevem funções, interseção de tipos com interfaces e a aplicação em cenários complexos (por exemplo, definição de contratos para bibliotecas ou APIs).

### Sintaxe e Estrutura

### Declaração de Interface

A sintaxe básica para declarar uma interface em TypeScript é:

```tsx
interface Pessoa {
  nome: string;
  idade: number;
  saudacao(): void;
}

```

### Implementação em Classes

Uma classe pode implementar uma interface utilizando a palavra-chave `implements`:

```tsx
class Funcionario implements Pessoa {
  nome: string;
  idade: number;

  constructor(nome: string, idade: number) {
    this.nome = nome;
    this.idade = idade;
  }

  saudacao(): void {
    console.log(`Olá, meu nome é ${this.nome} e tenho ${this.idade} anos.`);
  }
}

```

### Extensão de Interfaces com `extends`

Interfaces podem estender outras interfaces para compor novas estruturas:

```tsx
interface Funcionario extends Pessoa {
  cargo: string;
  salario: number;
}

const funcionario: Funcionario = {
  nome: "João",
  idade: 30,
  cargo: "Desenvolvedor",
  salario: 5000,
  saudacao: function () {
    console.log(`Olá, meu nome é ${this.nome} e trabalho como ${this.cargo}.`);
  },
};

funcionario.saudacao();

```

### Componentes Principais

- **Propriedades:**
    
    São os atributos definidos na interface que descrevem o formato do objeto. Cada propriedade pode ter um tipo primitivo ou composto e pode ser obrigatória ou opcional (usando `?`).
    
    ```tsx
    interface Produto {
      id: number;
      nome: string;
      preco?: number; // propriedade opcional
    }
    
    ```
    
- **Métodos:**
    
    As interfaces podem declarar métodos que as classes implementadoras devem definir.
    
    ```tsx
    interface Operacao {
      calcular(a: number, b: number): number;
    }
    
    ```
    
- **Interação entre Componentes:**
    
    Ao combinar interfaces com `extends`, é possível criar estruturas hierárquicas. Interfaces podem herdar membros de outras, garantindo que todos os membros sejam implementados conforme o contrato definido. Essa abordagem promove a reutilização de código e a padronização das estruturas dos dados.
    

### Uso Avançado

- **Interfaces para Funções:**
    
    Interfaces podem descrever a assinatura de funções, permitindo tipar funções que obedecem a um formato específico.
    
    ```tsx
    interface Soma {
      (a: number, b: number): number;
    }
    
    const somar: Soma = (x, y) => x + y;
    console.log(somar(3, 4)); // 7
    
    ```
    
- **Interseção de Interfaces:**
    
    Combinar interfaces utilizando interseção de tipos (`&`) pode ser útil para compor contratos complexos.
    
    ```tsx
    interface A {
      propA: string;
    }
    
    interface B {
      propB: number;
    }
    
    type AB = A & B;
    
    const exemplo: AB = {
      propA: "Exemplo",
      propB: 42,
    };
    
    ```
    
- **Interfaces Genéricas:**
    
    Ao utilizar generics, as interfaces podem se tornar mais flexíveis e reutilizáveis em diferentes contextos.
    
    ```tsx
    interface Reposta<T> {
      sucesso: boolean;
      dados: T;
    }
    
    const respostaString: Reposta<string> = {
      sucesso: true,
      dados: "Operação concluída",
    };
    
    const respostaNumero: Reposta<number> = {
      sucesso: true,
      dados: 200,
    };
    
    ```
    
- **Combinação com Classes Abstratas:**
    
    Em cenários mais avançados, interfaces podem ser combinadas com classes abstratas para definir contratos e comportamentos que as classes concretas devem implementar.
    

## 4. Exemplos de Código Otimizados

### Exemplo Básico: Definindo e Implementando uma Interface

```tsx
// Definição da interface Pessoa
interface Pessoa {
  nome: string;
  idade: number;
  saudacao(): void;
}

// Implementação da interface em uma classe
class Estudante implements Pessoa {
  nome: string;
  idade: number;
  curso: string;

  constructor(nome: string, idade: number, curso: string) {
    this.nome = nome;
    this.idade = idade;
    this.curso = curso;
  }

  saudacao(): void {
    console.log(`Olá, meu nome é ${this.nome} e estudo ${this.curso}.`);
  }
}

const aluno = new Estudante("Ana", 22, "Engenharia");
aluno.saudacao(); // Saída: Olá, meu nome é Ana e estudo Engenharia.

```

### Exemplo Avançado: Extensão e Interfaces Genéricas

```tsx
// Interface base Pessoa
interface Pessoa {
  nome: string;
  idade: number;
  saudacao(): void;
}

// Extensão da interface Pessoa para definir um Funcionário
interface Funcionario extends Pessoa {
  cargo: string;
  salario: number;
}

// Interface genérica para respostas de uma API
interface RespostaAPI<T> {
  sucesso: boolean;
  dados: T;
  mensagem?: string;
}

// Implementação da interface Funcionario em uma classe
class Gerente implements Funcionario {
  nome: string;
  idade: number;
  cargo: string;
  salario: number;

  constructor(nome: string, idade: number, salario: number) {
    this.nome = nome;
    this.idade = idade;
    this.cargo = "Gerente";
    this.salario = salario;
  }

  saudacao(): void {
    console.log(`Olá, eu sou ${this.nome}, ${this.cargo} da empresa.`);
  }
}

const gerente = new Gerente("Carlos", 40, 8000);
gerente.saudacao();

// Utilizando a interface genérica RespostaAPI para uma resposta de funcionário
const respostaFuncionario: RespostaAPI<Funcionario> = {
  sucesso: true,
  dados: gerente,
  mensagem: "Funcionário encontrado com sucesso.",
};

console.log(respostaFuncionario);

```

## 5. Informações Adicionais

- **Interfaces x Tipos (Type Aliases):**
    
    Embora interfaces e type aliases possam ser usados para definir formas de objetos, as interfaces oferecem vantagens em termos de extensão e declaração aberta, o que permite a adição de novos membros mesmo após sua definição inicial.
    
- **Declaração Mergulhada (Declaration Merging):**
    
    Em TypeScript, interfaces com o mesmo nome podem se fundir, o que permite a extensão de definições já existentes de maneira incremental. Esse recurso é particularmente útil em contextos de bibliotecas e frameworks.
    
- **Melhores Práticas:**
    - Utilize interfaces para definir contratos públicos de classes e objetos.
    - Prefira interfaces a type aliases para contratos de objetos, especialmente quando a extensão ou a declaração aberta são necessárias.
    - Documente as interfaces para facilitar a compreensão e manutenção do código.
