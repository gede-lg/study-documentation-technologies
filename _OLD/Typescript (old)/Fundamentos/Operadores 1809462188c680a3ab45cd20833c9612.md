# Operadores

## 1. Introdução

Os operadores são elementos fundamentais em qualquer linguagem de programação, incluindo TypeScript. Eles permitem realizar operações sobre valores e variáveis, manipulando dados de diversas formas. Em TypeScript, os operadores não apenas suportam operações básicas, como aritméticas e lógicas, mas também oferecem funcionalidades avançadas que aproveitam os recursos de tipagem estática da linguagem. Compreender os diferentes tipos de operadores e suas aplicações é crucial para escrever código eficiente, legível e seguro.

### Definição e Conceitos Fundamentais

**Tema Principal:** *Operadores*

**Subtemas:**

- Operadores de Atribuição

- Operadores Relacionais
- Operadores Aritméticos
- Operadores Lógicos
- Operadores de Acesso
- Operadores de Nulidade
- Encadeamento Opcional
- Operador Spread

**Conceitos Básicos vs. Avançados:**

- *Conceitos Básicos:* Operadores aritméticos, de atribuição simples, lógicos e relacionais que são amplamente utilizados em operações diárias de programação.
- *Conceitos Avançados:* Operadores de desestruturação, encadeamento opcional, spread e operadores de nulidade que permitem manipulações mais complexas e seguras de dados, aproveitando as funcionalidades avançadas do TypeScript.

### Sintaxe e Estrutura

Os operadores em TypeScript seguem uma sintaxe específica que permite realizar operações sobre operandos (valores ou variáveis). A estrutura básica varia conforme o tipo de operador.

**Exemplo de Operador Aritmético:**

```tsx
let a: number = 10;
let b: number = 5;
let soma: number = a + b; // 15

```

**Exemplo de Operador Ternário:**

```tsx
let idade: number = 20;
let status: string = idade >= 18 ? 'Adulto' : 'Menor';

```

### Componentes Principais

### Operadores de Atribuição

- **Operadores Compostos (`+=`, `=`, `=`, `/=`, `%=`):**
Permitem realizar uma operação e atribuição simultaneamente.
    
    ```tsx
    let x: number = 10;
    x += 5; // x = 15
    x *= 2; // x = 30
    
    ```
    
- **Operadores de Desestruturação com Objetos e Arrays:**
Facilitam a extração de valores de objetos e arrays em variáveis distintas.
    
    ```tsx
    const pessoa = { nome: 'Ana', idade: 25 };
    const { nome, idade } = pessoa;
    
    const numeros = [1, 2, 3];
    const [primeiro, segundo] = numeros;
    
    ```
    

### Operadores Condicionais

- **Operador Ternário (`condition ? expr1 : expr2`):**
Simplifica expressões condicionais.
    
    ```tsx
    let acesso: string = (idade >= 18) ? 'Permitido' : 'Negado';
    
    ```
    
- **Short-Circuiting com Operadores Lógicos:**
Executa operações com base na avaliação lógica.
    
    ```tsx
    let nomeUsuario: string | null = obterNome();
    let saudacao = nomeUsuario || 'Convidado';
    
    ```
    

### Operadores Relacionais

- **Igualdade e Identidade (`==`, `===`, `!=`, `!==`):**
Comparam valores com e sem verificação de tipo.
    
    ```tsx
    5 == '5'; // true
    5 === '5'; // false
    
    ```
    
- **Maior, Menor, Maior ou Igual e Menor ou Igual (`<`, `>`, `<=`, `>=`):**
Comparam valores numéricos ou de strings.
    
    ```tsx
    10 > 5; // true
    'a' < 'b'; // true
    
    ```
    

### Operadores Aritméticos

- **Soma, Subtração, Multiplicação, Divisão, Módulo:**
    
    ```tsx
    let soma = 10 + 5; // 15
    let modulo = 10 % 3; // 1
    
    ```
    
- **Incremento (`++`) e Decremento (`-`):**
    
    ```tsx
    let contador = 0;
    contador++; // 1
    contador--; // 0
    
    ```
    

### Operadores Lógicos

- **AND (`&&`), OR (`||`) e NOT (`!`):**
    
    ```tsx
    let resultado = (true && false); // false
    resultado = (true || false); // true
    resultado = !true; // false
    
    ```
    

### Operadores de Acesso

- **Acesso a Propriedades (`obj.property` e `obj['property']`):**
    
    ```tsx
    const carro = { marca: 'Toyota', modelo: 'Corolla' };
    console.log(carro.marca); // 'Toyota'
    console.log(carro['modelo']); // 'Corolla'
    
    ```
    
- **Acesso Dinâmico com `keyof`:**
    
    ```tsx
    function acessarPropriedade<T, K extends keyof T>(obj: T, chave: K): T[K] {
      return obj[chave];
    }
    
    const pessoa = { nome: 'João', idade: 30 };
    const nome = acessarPropriedade(pessoa, 'nome'); // 'João'
    
    ```
    

### Operadores de Nulidade

- **Operador de Coalescência Nula (`??`):**
Retorna o operando à direita se o da esquerda for `null` ou `undefined`.
    
    ```tsx
    let valor: string | null = null;
    let resultado = valor ?? 'Valor padrão'; // 'Valor padrão'
    
    ```
    
- **Non-null Assertion (`!`):**
Informa ao compilador que uma expressão não é `null` ou `undefined`.
    
    ```tsx
    function imprimirComprimento(texto?: string) {
      console.log(texto!.length);
    }
    
    ```
    

### Encadeamento Opcional

- **Como Evitar Erros de Acesso a Propriedades Indefinidas ou Nulas (`obj?.property`):**
    
    ```tsx
    const usuario = { perfil: { nome: 'Maria' } };
    console.log(usuario.perfil?.nome); // 'Maria'
    
    const usuarioSemPerfil = { };
    console.log(usuarioSemPerfil.perfil?.nome); // undefined
    
    ```
    

### Operador Spread

- **Uso do Spread (`...`) para Espalhar Elementos de Arrays e Objetos:**
    
    ```tsx
    const array1 = [1, 2, 3];
    const array2 = [...array1, 4, 5]; // [1, 2, 3, 4, 5]
    
    const obj1 = { a: 1, b: 2 };
    const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }
    
    ```
    
- **Combinação de Objetos e Arrays com Spread:**
    
    ```tsx
    const objA = { x: 10 };
    const objB = { y: 20 };
    const combinado = { ...objA, ...objB }; // { x: 10, y: 20 }
    
    const arrA = [1, 2];
    const arrB = [3, 4];
    const combinadoArr = [...arrA, ...arrB]; // [1, 2, 3, 4]
    
    ```
    

### Uso Avançado

Os operadores em TypeScript podem ser combinados para criar expressões sofisticadas que aproveitam a tipagem estática e os recursos de segurança da linguagem.

**Desestruturação Avançada com Operadores:**

```tsx
interface Produto {
  nome: string;
  preco: number;
  tags?: string[];
}

const produto: Produto = { nome: 'Laptop', preco: 1500, tags: ['eletrônico', 'computador'] };
const { nome, preco, tags = [] } = produto;

const [primeiraTag, ...outrasTags] = tags;

```

**Encadeamento Opcional com Operadores de Nulidade:**

```tsx
interface Endereco {
  rua: string;
  cidade: string;
}

interface Usuario {
  nome: string;
  endereco?: Endereco;
}

const usuario: Usuario = { nome: 'Carlos' };
const cidade = usuario.endereco?.cidade ?? 'Cidade não informada';

```

**Uso do Operador Spread para Atualização Imutável de Objetos:**

```tsx
const estado = {
  usuario: { nome: 'Ana', idade: 25 },
  autenticado: false
};

const novoEstado = {
  ...estado,
  autenticado: true,
  usuario: { ...estado.usuario, idade: 26 }
};

```

## 4. Exemplos de Código Otimizados

### Operadores de Atribuição Compostos

```tsx
let total: number = 100;

// Incrementando o total em 50
total += 50; // total = 150

// Multiplicando o total por 2
total *= 2; // total = 300

```

### Desestruturação com Objetos

```tsx
const livro = {
  titulo: '1984',
  autor: 'George Orwell',
  ano: 1949
};

// Extraindo propriedades
const { titulo, autor } = livro;
console.log(titulo); // '1984'
console.log(autor); // 'George Orwell'

```

### Operador Ternário e Short-Circuiting

```tsx
const idade: number = 17;

// Usando operador ternário
const podeVotar: string = idade >= 18 ? 'Sim' : 'Não';
console.log(podeVotar); // 'Não'

// Usando short-circuiting
const nome: string | null = null;
const saudacao: string = nome || 'Visitante';
console.log(saudacao); // 'Visitante'

```

### Encadeamento Opcional e Operador de Coalescência Nula

```tsx
interface Perfil {
  nome: string;
  redesSociais?: {
    twitter?: string;
    facebook?: string;
  };
}

const perfil: Perfil = { nome: 'Luiza' };

// Encadeamento opcional
const twitter = perfil.redesSociais?.twitter ?? 'Não possui Twitter';
console.log(twitter); // 'Não possui Twitter'

```

### Operador Spread para Combinação de Objetos

```tsx
const configuracaoBase = {
  host: 'localhost',
  porta: 8080
};

const configuracaoDesenvolvimento = {
  ...configuracaoBase,
  modo: 'desenvolvimento'
};

console.log(configuracaoDesenvolvimento);
// { host: 'localhost', porta: 8080, modo: 'desenvolvimento' }

```

## 5. Informações Adicionais

- **Prioridade dos Operadores:** Entender a precedência dos operadores é essencial para evitar resultados inesperados em expressões complexas. Por exemplo, multiplicação tem precedência maior que a adição.
    
    ```tsx
    const resultado = 2 + 3 * 4; // 14, não 20
    
    ```
    
- **Operadores Bitwise:** Além dos operadores mencionados, TypeScript também suporta operadores bitwise (`&`, `|`, `^`, `~`, `<<`, `>>`, `>>>`), que operam em nível de bits, úteis em aplicações de baixo nível e otimizações.
- **Overloading de Operadores:** Embora TypeScript não suporte diretamente o overloading de operadores, é possível simular comportamentos similares através de funções e métodos personalizados.
- **Boas Práticas:**
    - **Clareza vs. Concisão:** Evite expressões excessivamente complexas que comprometam a legibilidade do código.
    - **Uso Consistente de Operadores:** Mantenha um padrão no uso de operadores para facilitar a manutenção e compreensão do código.
    - **Validação de Tipos:** Aproveite a tipagem estática do TypeScript para garantir que os operadores sejam utilizados com os tipos corretos, prevenindo erros em tempo de compilação.

### Notas Adicionais

- **`keyof`**: É um operador TypeScript que retorna um tipo representando as chaves de um objeto. Útil para acessar propriedades de forma segura e dinâmica.
- **Non-null Assertion `!`**: Deve ser usado com cautela, pois força o compilador a tratar a expressão como não nula ou indefinida, podendo levar a erros em tempo de execução se a suposição estiver incorreta.
- **Operador Spread `...`**: Além de espalhar elementos, também é usado para criar cópias superficiais de arrays e objetos, facilitando operações imutáveis.
