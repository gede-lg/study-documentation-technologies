# Objetos

Uma abordagem detalhada sobre *Type Aliases*, *Interfaces* e a atribuição de tipos opcionais (`?`) para a definição e utilização de objetos em TypeScript.

---

## 1. Introdução

TypeScript é uma linguagem que adiciona tipagem estática ao JavaScript, permitindo a detecção de erros em tempo de compilação e melhorando a manutenção e escalabilidade dos projetos. A definição de objetos é um dos pilares dessa linguagem, onde **Type Aliases** e **Interfaces** são ferramentas poderosas para definir a forma e os tipos dos objetos utilizados em uma aplicação. Além disso, a utilização de tipos opcionais (através do `?`) permite uma maior flexibilidade na definição dos campos dos objetos.

## 3. Conteúdo Detalhado

### Definição e Conceitos Fundamentais

- **Objetos em TypeScript:**
    
    Em TypeScript, objetos são estruturas que agrupam propriedades e métodos, definindo modelos para dados complexos. A tipagem desses objetos ajuda a garantir que os dados manipulados estejam de acordo com a estrutura esperada.
    
- **Type Aliases:**
    
    Um *Type Alias* permite criar um novo nome para um tipo, que pode ser um objeto, união, interseção, etc. São úteis para definir tipos que serão reutilizados em diferentes partes da aplicação.
    
    ```tsx
    type Pessoa = {
      nome: string;
      idade: number;
      email?: string; // campo opcional
    };
    
    ```
    
- **Interfaces:**
    
    As *Interfaces* definem a estrutura de um objeto, servindo como um contrato que as classes ou objetos devem seguir. Elas podem ser estendidas e implementadas, oferecendo uma forma robusta de definir modelos de dados.
    
    ```tsx
    interface IProduto {
      id: number;
      nome: string;
      preco: number;
      descricao?: string; // propriedade opcional
    }
    
    ```
    
- **Atribuição de Tipos Opcionais (`?`):**
    
    O sinal de interrogação (`?`) após o nome de uma propriedade indica que ela é opcional. Isso significa que o objeto pode ou não incluir essa propriedade, sem que isso gere erros de compilação.
    
    ```tsx
    interface IUsuario {
      id: number;
      username: string;
      telefone?: string; // propriedade opcional
    }
    
    ```
    

### Sintaxe e Estrutura

- **Type Aliases:**
    
    A declaração de um *Type Alias* utiliza a palavra-chave `type` seguida de um nome e da definição do objeto.
    
    ```tsx
    type Endereco = {
      rua: string;
      cidade: string;
      cep?: string; // opcional
    };
    
    ```
    
- **Interfaces:**
    
    As *Interfaces* são definidas com a palavra-chave `interface` e seguem uma estrutura similar à dos *Type Aliases*, porém com suporte a herança e implementação.
    
    ```tsx
    interface IEndereco {
      rua: string;
      cidade: string;
      cep?: string;
    }
    
    ```
    
- **Propriedades Opcionais:**
    
    A inclusão do `?` ao final do nome da propriedade indica que ela não é obrigatória.
    
    ```tsx
    interface IContato {
      nome: string;
      email?: string;
    }
    
    ```
    

### Componentes Principais

- **Funções e Métodos Relacionados:**
    
    Tanto *Interfaces* quanto *Type Aliases* podem definir funções, embora *Interfaces* permitam a declaração de métodos que podem ser implementados por classes.
    
    ```tsx
    interface ICalculadora {
      soma(a: number, b: number): number;
      subtrai?(a: number, b: number): number; // método opcional
    }
    
    ```
    
- **Elementos e Propriedades:**
    
    Os elementos principais são as propriedades definidas nos objetos, que podem ser obrigatórias ou opcionais.
    
    - **Obrigatórias:** Devem ser fornecidas quando o objeto é instanciado.
    - **Opcionais:** Não precisam ser definidas, o que confere flexibilidade para diferentes cenários de uso.
- **Interação entre Type Aliases e Interfaces:**
    
    Embora ambos possam definir a forma de um objeto, **Interfaces** oferecem vantagens em termos de extensão e implementação, enquanto **Type Aliases** são mais versáteis, podendo representar uniões e interseções.
    
    ```tsx
    // Exemplo de interface estendendo outra interface
    interface IPessoa {
      nome: string;
      idade: number;
    }
    
    interface IFuncionario extends IPessoa {
      cargo: string;
      salario?: number; // opcional
    }
    
    ```
    

### Uso Avançado

- **Herança e Extensão:**
    
    Interfaces permitem a herança múltipla, facilitando a criação de modelos mais complexos e reutilizáveis.
    
    ```tsx
    interface IEndereco {
      rua: string;
      cidade: string;
      cep?: string;
    }
    
    interface IEmpresa extends IEndereco {
      nome: string;
      cnpj: string;
    }
    
    ```
    
- **Unindo Tipos com Type Aliases:**
    
    *Type Aliases* possibilitam a criação de tipos complexos através de uniões (`|`) e interseções (`&`), que podem combinar diferentes formas de objetos.
    
    ```tsx
    type Animal = {
      nome: string;
      idade: number;
    };
    
    type Pet = Animal & {
      dono: string;
      vacinado?: boolean;
    };
    
    ```
    
- **Diferenças Práticas e Casos de Uso:**
    - **Interfaces:** São ideais para definir contratos de objetos que serão utilizados e implementados em diferentes partes de uma aplicação, especialmente em arquiteturas orientadas a objetos.
    - **Type Aliases:** São preferíveis quando se necessita definir tipos mais flexíveis, como uniões ou interseções, ou quando o tipo não será implementado, mas apenas utilizado para tipagem.
- **Utilizando Propriedades Opcionais em Funções:**
    
    Ao definir funções que recebem objetos, as propriedades opcionais garantem que a função possa ser chamada mesmo que nem todas as informações estejam presentes.
    
    ```tsx
    function exibirUsuario(usuario: { id: number; nome: string; email?: string }) {
      console.log(`ID: ${usuario.id}, Nome: ${usuario.nome}`);
      if (usuario.email) {
        console.log(`Email: ${usuario.email}`);
      }
    }
    
    ```

## 4. Exemplos de Código Otimizados

### Exemplo 1: Definindo um Objeto com Type Alias e Propriedades Opcionais

```tsx
// Definindo o tipo 'Pessoa' com uma propriedade opcional
type Pessoa = {
  nome: string;
  idade: number;
  email?: string; // opcional
};

// Função que recebe um objeto do tipo Pessoa
function mostrarPessoa(pessoa: Pessoa): void {
  console.log(`Nome: ${pessoa.nome}`);
  console.log(`Idade: ${pessoa.idade}`);
  // Verifica se o email foi informado
  if (pessoa.email) {
    console.log(`Email: ${pessoa.email}`);
  }
}

// Exemplo de uso
const pessoa1: Pessoa = { nome: "João", idade: 30 };
const pessoa2: Pessoa = { nome: "Maria", idade: 25, email: "maria@example.com" };

mostrarPessoa(pessoa1);
mostrarPessoa(pessoa2);

```

### Exemplo 2: Utilizando Interface com Métodos e Propriedades Opcionais

```tsx
// Definindo a interface ICalculadora
interface ICalculadora {
  soma(a: number, b: number): number;
  subtrai?(a: number, b: number): number; // método opcional
}

// Implementação da interface em uma classe
class Calculadora implements ICalculadora {
  soma(a: number, b: number): number {
    return a + b;
  }

  // Implementação opcional do método subtrai
  subtrai(a: number, b: number): number {
    return a - b;
  }
}

// Uso da classe
const calc: ICalculadora = new Calculadora();
console.log(calc.soma(5, 3));        // Saída: 8
console.log(calc.subtrai ? calc.subtrai(5, 3) : "Método não implementado"); // Saída: 2

```

### Exemplo 3: Interfaces com Extensão

```tsx
// Interface básica para pessoa
interface IPessoa {
  nome: string;
  idade: number;
}

// Extensão da interface IPessoa para incluir informações de funcionário
interface IFuncionario extends IPessoa {
  cargo: string;
  salario?: number; // opcional
}

const funcionario: IFuncionario = {
  nome: "Carlos",
  idade: 40,
  cargo: "Gerente"
};

console.log(`Funcionário: ${funcionario.nome}, Cargo: ${funcionario.cargo}`);

```

## 5. Informações Adicionais

- **Quando Utilizar Type Aliases vs. Interfaces:**
    - **Type Aliases:** São mais flexíveis e podem representar não apenas objetos, mas também uniões e interseções.
    - **Interfaces:** São recomendadas para definição de contratos que podem ser estendidos e implementados, especialmente em cenários com classes e herança.
- **Propriedades Opcionais:**
    
    O uso de `?` facilita o trabalho com dados que podem não estar sempre presentes, evitando erros e permitindo a criação de funções e métodos mais robustos.
    
- **Boas Práticas:**
    - Sempre documente suas interfaces e tipos para melhorar a legibilidade do código.
    - Utilize interfaces para definir contratos de objetos e classes, e *Type Aliases* para composições mais complexas.
    - Combine tipagem estática com testes unitários para garantir a integridade do código.
