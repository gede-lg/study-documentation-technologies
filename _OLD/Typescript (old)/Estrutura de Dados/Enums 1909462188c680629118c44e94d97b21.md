# Enums

## 1. Introdução

Enums (ou "enumerações") são um recurso poderoso do TypeScript que permite definir um conjunto de constantes nomeadas. Eles são amplamente utilizados para melhorar a legibilidade do código e garantir a consistência dos valores utilizados ao longo de uma aplicação. Em TypeScript, os enums podem ser numéricos ou baseados em strings, oferecendo flexibilidade e segurança em tempo de compilação. Além disso, o TypeScript permite a utilização de **enum reverso**, que possibilita a obtenção do nome do enum a partir de seu valor, ampliando as possibilidades de uso e depuração.

## 3. Conteúdo Detalhado

### Definição e Conceitos Fundamentais

### O que é um Enum?

Um **enum** é uma estrutura que permite definir um conjunto de constantes nomeadas. Eles ajudam a reduzir o risco de erros, já que ao utilizar nomes ao invés de valores literais espalhados pelo código, é possível manter um controle melhor dos valores válidos.

### Enums Numéricos vs. Enums Baseados em Strings

- **Enums Numéricos**:
    
    São os enums padrão em TypeScript. Cada membro do enum recebe um valor numérico automaticamente, a partir de `0` ou do valor definido para o primeiro membro.
    
    **Exemplo:**
    
    ```tsx
    enum Direcao {
        Norte,  // 0
        Sul,    // 1
        Leste,  // 2
        Oeste   // 3
    }
    
    ```
    
- **Enums Baseados em Strings**:
    
    Cada membro do enum é explicitamente associado a uma string. Esse tipo de enum é útil para quando os valores precisam ser legíveis ou quando há necessidade de integração com sistemas que esperam valores literais específicos.
    
    **Exemplo:**
    
    ```tsx
    enum Status {
        Ativo = "ATIVO",
        Inativo = "INATIVO",
        Pendente = "PENDENTE"
    }
    
    ```
    

### Enum Reverso no TypeScript

Em enums numéricos, o TypeScript gera um mapeamento reverso, ou seja, dado um valor numérico, é possível recuperar a chave correspondente do enum. Essa funcionalidade não está disponível para enums baseados em strings.

**Exemplo de Enum Reverso:**

```tsx
enum Dias {
    Segunda,
    Terca,
    Quarta
}

console.log(Dias[1]); // Saída: "Terca"

```

### Sintaxe e Estrutura

### Enum Numérico

```tsx
enum Cores {
    Vermelho,  // 0
    Verde,     // 1
    Azul       // 2
}

```

- **Inicialização automática**: O primeiro membro recebe o valor `0` e os demais incrementam sequencialmente.
- **Inicialização manual**: É possível definir um valor inicial e os demais serão incrementados a partir dele.
    
    ```tsx
    enum Niveis {
        Baixo = 1,  // 1
        Medio,      // 2
        Alto        // 3
    }
    
    ```
    

### Enum Baseado em Strings

```tsx
enum Resposta {
    Sim = "SIM",
    Nao = "NAO"
}

```

- **Flexibilidade**: Cada membro deve ser explicitamente atribuído a uma string.
- **Sem mapeamento reverso**: Ao utilizar strings, o mapeamento reverso não é gerado automaticamente pelo TypeScript.

### Componentes Principais

### Membros do Enum

- **Chave (Nome)**: Representa o nome descritivo do valor.
- **Valor**: Pode ser um número ou uma string, dependendo do tipo de enum.

### Métodos e Propriedades Associadas

- **Mapeamento direto**: Permite acessar o valor associado à chave.
    
    ```tsx
    let cor: Cores = Cores.Vermelho;
    console.log(cor); // Saída: 0 (no caso de enums numéricos)
    
    ```
    
- **Mapeamento reverso (apenas para enums numéricos)**: Permite acessar a chave a partir de um valor.
    
    ```tsx
    console.log(Cores[0]); // Saída: "Vermelho"
    
    ```
    

### Interação Entre Componentes

- **Comparação e atribuição**: Os enums podem ser utilizados para comparar valores e atribuí-los a variáveis, garantindo que somente os valores definidos sejam utilizados.
- **Validação em tempo de compilação**: O TypeScript verifica se os valores atribuídos a variáveis do tipo enum pertencem ao conjunto definido, reduzindo a possibilidade de erros.

### Uso Avançado

### Casos de Uso Complexos

- **Integração com APIs**: Enums baseados em strings podem ser úteis para comunicação com APIs que requerem valores específicos.
- **Configurações e estados**: Em aplicações complexas, enums podem representar estados (por exemplo, status de uma requisição ou fases de um processo) de maneira clara e consistente.
- **Flags (bandeiras)**: Embora não sejam enums no sentido tradicional, os enums podem ser utilizados para representar flags usando operações bitwise.
    
    ```tsx
    enum Permissoes {
        Leitura = 1 << 0,  // 1
        Escrita = 1 << 1,  // 2
        Execucao = 1 << 2  // 4
    }
    
    let permissoesUsuario = Permissoes.Leitura | Permissoes.Escrita;
    // Verifica se o usuário possui a permissão de escrita
    let podeEscrever = (permissoesUsuario & Permissoes.Escrita) === Permissoes.Escrita;
    console.log(podeEscrever); // Saída: true
    
    ```
    

### Enums Computados e Heterogêneos

- **Enums Computados**: Permitem que os valores dos membros sejam o resultado de expressões computadas.
    
    ```tsx
    enum Calculos {
        A = 10,
        B = A * 2,  // B = 20
        C = Math.random() * 100  // Valor computado dinamicamente (não recomendado para enums, pois perde o mapeamento reverso)
    }
    
    ```
    
- **Enums Heterogêneos**: Misturam strings e números, mas seu uso é desencorajado devido à inconsistência e complexidade na manutenção.
    
    ```tsx
    enum Misturado {
        Numero = 1,
        Texto = "texto"
    }
    
    ```

## 4. Exemplos de Código Otimizados

### Exemplo 1: Enum Numérico com Mapeamento Reverso

```tsx
enum DiasDaSemana {
    Domingo,   // 0
    Segunda,   // 1
    Terca,     // 2
    Quarta,    // 3
    Quinta,    // 4
    Sexta,     // 5
    Sabado     // 6
}

// Utilização direta
let hoje: DiasDaSemana = DiasDaSemana.Quarta;
console.log(hoje); // Saída: 3

// Mapeamento reverso
console.log(DiasDaSemana[hoje]); // Saída: "Quarta"

```

### Exemplo 2: Enum Baseado em Strings

```tsx
enum StatusPedido {
    Pendente = "PENDENTE",
    Processando = "PROCESSANDO",
    Concluido = "CONCLUIDO",
    Cancelado = "CANCELADO"
}

function atualizarStatus(status: StatusPedido) {
    console.log(`O status do pedido foi atualizado para: ${status}`);
}

atualizarStatus(StatusPedido.Processando); // Saída: O status do pedido foi atualizado para: PROCESSANDO

```

### Exemplo 3: Uso de Flags com Enums

```tsx
enum Permissoes {
    Leitura = 1 << 0,   // 1
    Escrita = 1 << 1,   // 2
    Execucao = 1 << 2   // 4
}

let permissoesUsuario = Permissoes.Leitura | Permissoes.Escrita;

if ((permissoesUsuario & Permissoes.Escrita) === Permissoes.Escrita) {
    console.log("O usuário tem permissão de escrita.");
} else {
    console.log("O usuário não tem permissão de escrita.");
}

```

## 5. Informações Adicionais

- **Performance e Compilação**:
    
    Enums são resolvidos em tempo de compilação, o que pode ajudar na performance e na detecção precoce de erros. No entanto, é importante utilizá-los de maneira consciente para evitar código excessivamente complexo.
    
- **Boas Práticas**:
    - Utilize enums para representar conjuntos de valores fixos e conhecidos.
    - Prefira enums baseados em strings quando os valores precisam ser legíveis ou quando a integração com outros sistemas exige valores específicos.
    - Evite misturar tipos (heterogêneos) para manter a clareza e a consistência do código.
- **Limitações**:
    
    O mapeamento reverso só funciona com enums numéricos. Se for necessário utilizar a funcionalidade de mapeamento reverso, opte por enums numéricos.
    
- **Compatibilidade**:
    
    Enums são uma feature exclusiva do TypeScript e não possuem um equivalente direto em JavaScript. Durante a transpilação, o TypeScript gera código JavaScript correspondente, mas é importante compreender essa diferença ao trabalhar com ambos os ambientes.
